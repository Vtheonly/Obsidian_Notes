# 5. Review of Linear and Logistic Regression

## Linear Regression (Predicting Values)

Linear regression tries to fit a straight line through a cloud of data points.

- **Hypothesis Function:** $h(x) = \theta_0 + \theta_1 x_1 + \theta_2 x_2 ... \theta_n x_n$
  - Vectorized form: $h(x) = \theta^T x$
- **Cost Function:** MSE (Mean Squared Error).
- **Goal:** Find the values of $\theta$ (weights/slope and bias/intercept) that minimize the MSE.

### Feature Scaling

To make Gradient Descent converge faster, we must scale our features so they are on a similar playing field.

1.  **Min-Max Normalization:** Scales data strictly between 0 and 1.
    $$ x*{scaled} = \frac{x - x*{min}}{x*{max} - x*{min}} $$
2.  **Standardization (Z-score):** Centers data around a mean of 0 with a standard deviation of 1.
    $$ x\_{scaled} = \frac{x - \mu}{\sigma} $$

## Logistic Regression (Predicting Classes)

Despite the name "regression," this is a **Classification** algorithm. It outputs a probability between 0 and 1.

- **The Sigmoid Function:** We wrap the linear equation inside a Sigmoid function to squash the output between 0 and 1.
  $$ g(z) = \frac{1}{1 + e^{-z}} $$
    where $z = \theta^T x$.

### The Decision Boundary

The parameters $\theta$ define a line (or hyperplane) that separates the classes.

- If $h(x) \ge 0.5$ (which means $z \ge 0$), predict Class 1.
- If $h(x) < 0.5$ (which means $z < 0$), predict Class 0.

> [!WARNING] Why MSE fails for Logistic Regression
> If you plug the sigmoid function into the Mean Squared Error cost function, the resulting mathematical shape is **non-convex** (wavy with multiple local minima). Gradient Descent would get stuck. This is why we _must_ use **Binary Cross-Entropy (Log Loss)** for logistic regression, which mathematically guarantees a perfect, convex, single-minimum bowl.

### The Beautiful Math Coincidence

When you calculate the partial derivative of the Cost Function for Linear Regression, and the partial derivative of the Cost Function for Logistic Regression, the final resulting formulas look **exactly the same**:
$$ \frac{\partial J(\theta)}{\partial \theta*j} = \frac{1}{m} \sum*{i=1}^{m} (h*\theta(x^{(i)}) - y^{(i)}) x_j^{(i)} $$
**Why?** Because the derivative of the Log Loss perfectly cancels out the complex derivative of the Sigmoid function, leaving this highly elegant, simple update rule. \*Note: Even though the formula looks the same, $h*\theta(x)$ is entirely different (linear vs. sigmoid).\*
