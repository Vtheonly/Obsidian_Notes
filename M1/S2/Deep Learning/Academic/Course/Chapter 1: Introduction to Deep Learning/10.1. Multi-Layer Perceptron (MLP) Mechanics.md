# 10.1. Multi-Layer Perceptron (MLP) Mechanics

An MLP (or ANN - Artificial Neural Network) is a feedforward network capable of treating non-linearly separable data via dense connections.

### Network Architecture

- **Input Layer:** A 1D array representing features. _No computation happens here._
- **Hidden Layers:** Characterized by hyperparameters (number of layers, number of neurons). Each neuron applies $W^T X + b$ followed by an activation function.
- **Output Layer:** Returns the prediction. Size depends on the task (1 for binary, $N$ for multi-class).

### Training the MLP (The 4 Stages)

1.  **Forward Pass:** Data flows through the network to generate a prediction (Actual Outputs vs Desired Outputs).
2.  **Loss Calculation:** Evaluate the error using a Loss Function (MSE, BCE, CCE).
3.  **Backward Pass (Backpropagation of Errors):** The core algorithm of Deep Learning. It utilizes the **Chain Rule of Calculus** to calculate the gradient (partial derivative) of the Loss Function with respect to _every single weight and bias_ in the network.
4.  **Update Parameters:** Using SGD or Adam on mini-batches, the weights are shifted slightly to decrease the error.

### Implementation Guidelines (Mise en œuvre)

**1. Data Preparation**

- **Missing Values:** Must be imputed or deleted.
- **Categorical Variables:** Must be numericized using **One-Hot Encoding**.
- **Scaling:** Crucial for convergence. Use Standardization (Z-score) usually paired with ReLU, or Min-Max paired with Sigmoid/Tanh.

**2. Data Structuring**

- **Train Set:** Used to update the weights.
- **Validation Set:** Used to evaluate hyperparameter choices (number of layers, learning rate) and prevent overfitting during training.
- **Test Set:** Kept strictly isolated. Used only once at the very end to evaluate final real-world performance.
- _Data Augmentation:_ Rotating, flipping, or zooming images artificially creates more data to improve model generalization.

**3. Architectural Strategy**

- **Incremental Approach:** Always start with a very simple model (1 hidden layer). Only add complexity if the model fails to capture the data relationships.
- **Weight Initialization:** _Never initialize weights to zero_ (this causes symmetrical calculations where neurons learn the exact same things). Use **Xavier (Glorot)** initialization for Tanh/Sigmoid, or **He** initialization for ReLU.

**4. Preventing Overfitting (Surapprentissage)**
When a model memorizes the training data but fails on new data.

- **Regularization (L1/L2):** Adds a penalty to the loss function for having overly large weights.
- **Dropout:** Randomly deactivates a percentage of neurons during training. This prevents the network from becoming overly reliant on any specific feature or neuron.
- **Early Stopping (Arrêt précoce):** Continuously monitor the Validation Loss. If the Training Loss continues to drop, but the Validation Loss suddenly starts to rise, the model is beginning to overfit. Stop training immediately.
