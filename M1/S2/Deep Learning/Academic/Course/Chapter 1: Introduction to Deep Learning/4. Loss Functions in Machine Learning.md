# 4. Loss Functions in Machine Learning

The Loss Function (or Cost Function) is the mathematical formula that calculates how far off our predictions ($\hat{y}$) are from the actual ground truth ($y$).

## 1. Regression Loss Functions

Used when predicting continuous numbers (e.g., house prices, temperature).

**A. Mean Squared Error (MSE)**
Calculates the average of the squared differences between predicted and actual values.
$$ MSE = \frac{1}{n} \sum\_{i=1}^{n} (y_i - \hat{y}\_i)^2 $$

- **Pros:** Punishes large errors heavily (because of the square).
- **Cons:** Very sensitive to outliers.

**B. Mean Absolute Error (MAE)**
Calculates the average of the absolute differences.
$$ MAE = \frac{1}{n} \sum\_{i=1}^{n} |y_i - \hat{y}\_i| $$

- **Pros:** Robust to outliers.

**C. Huber Loss**
A hybrid of MSE and MAE. It acts like MSE for small errors and MAE for large errors (outliers), controlled by a threshold parameter $\delta$.

## 2. Classification Loss Functions

Used when predicting discrete categories (e.g., Cat vs. Dog).

### Why use Logarithms in Classification?

In classification, our model outputs a probability between 0 and 1.

- If $y=1$ (True) and the model predicts $\hat{y} = 0.99$, the log of 0.99 is close to 0 (very low loss).
- If $y=1$ (True) and the model predicts $\hat{y} = 0.01$, the log of 0.01 approaches $-\infty$. We add a negative sign to the formula to make this a massive positive penalty (high loss).

**A. Binary Cross-Entropy (BCE) / Log Loss**
Used for Binary Classification (2 classes).
$$ BCE = -\frac{1}{n} \sum\_{i=1}^{n} [y_i \log(\hat{y}_i) + (1 - y_i) \log(1 - \hat{y}_i)] $$

**B. Categorical Cross-Entropy (CCE)**
Used for Multi-class classification (3+ classes).
$$ CCE = -\frac{1}{n} \sum*{i=1}^{n} \sum*{j=1}^{C} y*{i,j} \log(\hat{y}*{i,j}) $$

**C. Hinge Loss**
Used primarily for Support Vector Machines (SVMs) and some neural networks to force a strict, clear margin of separation between classes.
$$ Hinge = \sum \max(0, 1 - y_i \cdot \hat{y}\_i) $$
*(Where $y \in \{-1, 1\}$)\*
