# 1.5 Image File Formats and Compression

Image formats are containers that define how the pixel data is organized and compressed.

## 1. The Necessity of Compression
As calculated in Note 1.3, raw images are huge. To store them efficiently, we need compression.
*   **Data Redundancy:** Images contain repeated data. (e.g., A blue sky is thousands of consecutive blue pixels).
*   **Irrelevancy:** Images contain details the human eye cannot see.

## 2. Lossless vs. Lossy Compression

### A. Lossless Compression
*   **Principle:** Compresses data without losing a single bit of information. The reconstructed image is mathematically identical to the original.
*   **Techniques:**
    *   *Run Length Encoding (RLE):* Instead of storing "White, White, White, White, White", store "5 Whites".
    *   *Huffman Coding / LZW:* Assign shorter binary codes to frequently occurring patterns.
*   **Formats:** **PNG**, **TIFF**, **GIF**, **BMP** (usually uncompressed, but supports RLE).
*   **Best For:** Medical imaging, technical drawings, logos, text, archiving.

### B. Lossy Compression
*   **Principle:** Discards information that is "least important" to human perception to achieve massive file size reduction (often 10:1 or more).
*   **Technique:**
    *   Uses the Discrete Cosine Transform (DCT) to separate low frequencies (general color/shape) from high frequencies (fine noise/texture).
    *   It quantizes (rounds off) the high frequencies, effectively deleting fine detail.
*   **Formats:** **JPEG**, **WebP**, **HEIF**.
*   **Best For:** Photographs, web images.
*   **Artifacts:** Blockiness, "ringing" around text, smudged details.

## 3. Specific Formats Overview

| Format | Compression | Transparency? | Use Case |
| :--- | :--- | :--- | :--- |
| **JPEG** | Lossy | No | Photos, Web. Small size, variable quality. |
| **PNG** | Lossless | Yes (Alpha) | Web graphics, logos, screenshots. |
| **GIF** | Lossless* | Yes (1-bit) | Simple animations. Limited to 256 colors only. |
| **TIFF** | Lossless/None | Yes | Print, Archival, Professional Photography. |
| **DICOM** | N/A | N/A | **Medical Standard.** Contains image data + metadata (Patient ID, Scan type). |

*Note on DICOM:* It is not just an image format; it is a protocol. A DICOM file wraps the raw pixel data (often 16-bit grayscale for X-rays) with headers containing critical medical information.
