# 8. Exercise 1. Identifying Applications and Roles

This exercise tests your ability to map real-world scenarios to the correct data domain and identify the correct professional roles for specific tasks.

## Part 1: Identify Applications

**Task:** Decide which domain (Data Science, ML, AI, Big Data, BI) fits each scenario best.

1.  **Scenario:** A dashboard shows last year's sales trends.
    *   **Answer:** **Business Intelligence (BI)**
    *   **Reasoning:** It focuses on *descriptive* analysis of historical data ("last year") to report on what happened.

2.  **Scenario:** An app recommends products based on user behavior.
    *   **Answer:** **Machine Learning (ML)**
    *   **Reasoning:** The system is "learning" user preferences from data to make predictions (recommendations). This is a Recommender System.

3.  **Scenario:** Storing and processing terabytes of sensor data from IoT devices.
    *   **Answer:** **Big Data**
    *   **Reasoning:** The keywords are "terabytes" (Volume) and "sensor data" (Velocity/Variety). This is an infrastructure/storage challenge.

4.  **Scenario:** An AI agent plays chess and learns strategies over time.
    *   **Answer:** **Artificial Intelligence (AI) / Reinforcement Learning**
    *   **Reasoning:** The agent is making autonomous decisions to achieve a goal (winning) and simulating intelligent behavior.

5.  **Scenario:** A company wants to predict customer churn using historical data.
    *   **Answer:** **Data Science (Predictive Analytics)**
    *   **Reasoning:** This involves the full lifecycle: cleaning data, analyzing it, and building a model to predict a future outcome (churn).

## Part 2: Roles in a Data Science Project

**Scenario:** A hospital wants to predict heart disease risk.

**Task:** Assign the following tasks to the correct role (Data Analyst, Data Scientist, Data Engineer, ML Engineer, Business Analyst).

### Task A: "Collect patient records from multiple hospital databases and prepare them for use."
*   **Role:** **Data Engineer**
*   **Reasoning:** This involves building pipelines (ETL), connecting to databases, and handling data infrastructure/architecture.

### Task B: "Build a predictive model using patient data to estimate heart disease risk."
*   **Role:** **Data Scientist**
*   **Reasoning:** This is the core modeling task—selecting features, choosing algorithms, and training the model.

### Task C: "Create a dashboard to show current statistics about patient risk factors."
*   **Role:** **Data Analyst**
*   **Reasoning:** This involves descriptive statistics and visualization to report on the *current state* of the data.

### Task D: "Deploy the predictive model so doctors can access it via a web application."
*   **Role:** **Machine Learning (ML) Engineer**
*   **Reasoning:** This involves taking the theoretical model and putting it into **production** (deployment, scalability, API integration).

### Task E: "Define the business question: 'How can we reduce the number of undetected high-risk patients?'"
*   **Role:** **Business Analyst**
*   **Reasoning:** This role bridges the gap between hospital stakeholders and the tech team, translating business needs into data requirements.

## Critical Thinking Question
**Why can't a Data Engineer and an ML Engineer simply replace each other?**

*   **Answer:** 
    *   A **Data Engineer** is focused on the **flow and storage** of data (SQL, Pipelines, Warehousing). They build the roads.
    *   An **ML Engineer** is focused on the **deployment and optimization** of models (Software Engineering, APIs, Latency). They build the cars that drive on the roads.
    *   While both code, their daily tools and objectives are fundamentally different (Database architecture vs. Model serialization and serving).
