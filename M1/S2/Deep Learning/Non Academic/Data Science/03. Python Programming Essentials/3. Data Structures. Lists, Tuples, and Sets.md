# 3. Data Structures. Lists, Tuples, and Sets

Data structures are containers that organize and store data. Choosing the right one is essential for memory efficiency and performance.

## 3.1 Lists

**Definition:** Ordered, mutable (changeable) collection of items. Allows duplicates.
*   **Syntax:** `my_list = [1, "two", 3.0]`

### Indexing and Slicing
Python uses 0-based indexing.
*   `my_list[0]` -> First item.
*   `my_list[-1]` -> Last item.
*   `my_list[start:stop:step]` -> Slicing.

```python
nums = [0, 1, 2, 3, 4, 5]
print(nums[1:4])    # [1, 2, 3] (Stop is exclusive)
print(nums[::2])    # [0, 2, 4] (Every second item)
print(nums[::-1])   # [5, 4, 3, 2, 1, 0] (Reverses list)
```

### Key Methods
*   `.append(x)`: Adds x to the end.
*   `.pop(i)`: Removes and returns item at index i.
*   `.sort()`: Sorts the list in place.

## 3.2 Tuples

**Definition:** Ordered, **immutable** collection. Once created, cannot be changed.
*   **Syntax:** `my_tuple = (1, 2, 3)`
*   **Why use them?**
    1.  **Safety:** Ensures data isn't accidentally modified.
    2.  **Performance:** Slightly faster than lists.
    3.  **Keys:** Can be used as keys in dictionaries (lists cannot).

### Unpacking
Tuples allow for variable assignment in a single line.
```python
coordinates = (4, 5)
x, y = coordinates  # x=4, y=5
```

## 3.3 Sets

**Definition:** Unordered, mutable collection of **unique** elements. No duplicates allowed.
*   **Syntax:** `my_set = {1, 2, 3}`

### Use Cases
1.  **Deduplication:** Removing duplicates from a list.
    ```python
    raw_data = [1, 2, 2, 3, 3, 3]
    unique_data = list(set(raw_data)) # [1, 2, 3]
    ```
2.  **Membership Testing:** Checking `if x in my_set` is $O(1)$ (instant), whereas checking a list is $O(n)$ (slower as list grows).

### Set Operations
*   **Union (`|`):** All elements from both sets.
*   **Intersection (`&`):** Only elements present in both.
*   **Difference (`-`):** Elements in A but not in B.
