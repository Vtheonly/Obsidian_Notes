# 6. Project Structure and Best Practices

Organizing a Data Science project correctly is crucial for reproducibility and collaboration. If you dump all your files into one folder, you will eventually lose track of which dataset is "clean" and which notebook is the "final" version.

## 6.1 Standard Directory Structure

A widely accepted standard (adapted from "Cookiecutter Data Science") looks like this:

```text
MyProject/
│
├── data/
│   ├── raw/            # Original, immutable data dump. NEVER edit this.
│   ├── processed/      # Cleaned data ready for modeling.
│   └── external/       # Data from third-party sources.
│
├── notebooks/          # Jupyter notebooks for exploration.
│   ├── 01_cleaning.ipynb
│   └── 02_analysis.ipynb
│
├── src/                # Source code (Python scripts).
│   ├── __init__.py     # Makes src a Python module.
│   ├── data_prep.py    # Functions to clean data.
│   └── visualization.py # Functions to plot graphs.
│
├── models/             # Trained models (pickles, h5 files).
│
├── env/                # Virtual Environment (Ignored by Git).
├── .gitignore          # Files to ignore.
├── README.md           # Documentation for the project.
└── requirements.txt    # List of dependencies.
```

## 6.2 Key Rules

### Rule 1: Data is Immutable
Never manually edit a raw CSV file in Excel. Always write a script to clean it and save the result as a *new* file in `data/processed/`. This ensures you can always trace back to the source.

### Rule 2: Notebooks are for Exploration, Scripts are for Production
Notebooks are great for visualizing data. However, if you copy-paste the same function into 5 different notebooks, you are creating technical debt.
*   **Step 1:** Write code in a Notebook.
*   **Step 2:** Once it works, move the function to `src/my_script.py`.
*   **Step 3:** Import the function back into the Notebook (`from src.my_script import my_func`).

### Rule 3: Document Everything
Your `README.md` should explain:
1.  What the project does.
2.  How to install dependencies.
3.  How to run the code.

## 6.3 Naming Conventions
*   **Files:** `snake_case.py`, `lower_case_with_underscores.csv`.
*   **Notebooks:** Number them to show order: `01_load_data.ipynb`, `02_clean_data.ipynb`.
*   **Variables:** Descriptive names (`patient_age` is better than `x`).
