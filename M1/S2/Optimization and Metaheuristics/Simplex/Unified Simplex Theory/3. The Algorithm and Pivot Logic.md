
# 3. The Algorithm and Pivot Logic

This note details the iterative process. This is the mechanical engine of the method.

## **3.1. The Algorithm Flowchart**

```mermaid
graph TD
    A[Start: Initial Tableau] --> B{Check Cj - Zj Row};
    B -- All values <= 0 --> C[STOP: Optimal Solution Found];
    B -- Positive value exists --> D[Identify Entering Variable];
    D --> E[Identify Pivot Column];
    E --> F[Calculate Ratios (RHS / Pivot Col)];
    F --> G{Check Ratios};
    G -- All Ratios <= 0 or Undefined --> H[STOP: Unbounded Solution];
    G -- Positive Ratios exist --> I[Identify Leaving Variable (Min Positive Ratio)];
    I --> J[Identify Pivot Element];
    J --> K[Perform Gaussian Elimination (Pivoting)];
    K --> L[New Tableau];
    L --> B;
```

## **3.2. Detailed Steps**

### **Step 1: Check Optimality (The Stop Condition)**
Look at the **$C_j - Z_j$** row (Net Evaluation Row).
*   **Maximization:** If all values are **$\le 0$**, you are done. The current solution is optimal.
*   **Reasoning:** Positive numbers mean "If I add this variable, profit goes up." If there are no positive numbers, profit cannot go up anymore.

### **Step 2: Determine the Entering Variable (The Pivot Column)**
If there are positive values in $C_j - Z_j$:
*   Select the variable with the **Largest Positive Value**.
*   *Tie-breaker:* Pick the leftmost one (arbitrary, but standard for consistency).
*   **Why?** This variable offers the steepest gradient (fastest increase) in profit per unit.
*   This column is now the **Pivot Column** (or Key Column).

### **Step 3: Determine the Leaving Variable (The Pivot Row)**
We need to remove one current Basic variable to make room for the entering one.
1.  Take the **$b$ (Quantity)** column.
2.  Divide it by the corresponding value in the **Pivot Column**.
3.  $Ratio = \frac{\text{RHS}}{\text{Pivot Column Value}}$
4.  **Crucial Rule:** Ignore negative values and division by zero. Only calculate for **strictly positive** denominators.
5.  Select the row with the **Minimum Non-Negative Ratio**.
6.  **Why?** This constraint is the "bottleneck." It is the first resource to run out as we increase the entering variable.
7.  This row is the **Pivot Row**.

### **Step 4: The Pivot Operation (Gaussian Elimination)**
The intersection of the Pivot Column and Pivot Row is the **Pivot Element**. We must turn this element into **1** and all other numbers in that column to **0**.

**A. Create New Pivot Row:**
$$ \text{New Pivot Row} = \frac{\text{Old Pivot Row}}{\text{Pivot Element}} $$

**B. Update All Other Rows:**
For every other row $i$:
$$ \text{New Row}_i = \text{Old Row}_i - (\text{Value in Pivot Col for Row}_i \times \text{New Pivot Row}) $$

> [!NOTE] **The Rectangle Method (from File 1 & 4)**
> Some sources use a "Rectangle Rule" to calculate new values for non-pivot rows manually:
> $$ \text{New Val} = \text{Old Val} - \frac{(\text{Pivot Col Val} \times \text{Pivot Row Val})}{\text{Pivot Element}} $$
> This is mathematically identical to the row operation formula above.

### **Step 5: Update the Basis**
In the "Basis" column and "$C_B$" column, replace the Leaving Variable with the Entering Variable (and its corresponding cost coefficient).
