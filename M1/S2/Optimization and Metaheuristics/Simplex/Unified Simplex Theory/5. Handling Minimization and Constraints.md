
# 5. Handling Minimization and Constraints

The Standard Form assumes **Maximization** and **$\le$ constraints**. Here is how to handle the rest.

## **5.1. Minimization Problems**
You have two choices.

**Choice 1: The Mirror Method (Easy)**
1.  Convert the Minimization problem into a Maximization problem.
2.  $\text{Min } Z = \dots$ is equivalent to $\text{Max } (-Z)$.
3.  Multiply the entire Objective Function by $-1$.
4.  Solve as a normal Maximization problem using the Standard rules.
5.  At the end, multiply the final $Z$ value by $-1$ to get the true cost.

**Choice 2: The Direct Method (Harder to remember)**
1.  Keep the objective function as is.
2.  **Entering Variable Rule Reversal:** Select the variable with the **Most Negative** $C_j - Z_j$ (or most positive if using Variation B).
3.  **Stop Condition Reversal:** Stop when all $C_j - Z_j \ge 0$.

> [!TIP] Recommendation
> Use **Choice 1**. It prevents you from having to memorize two sets of rules. Always Maximize.

## **5.2. Constraint Types**

| Constraint Type | Action Required | Variables Added |
| :--- | :--- | :--- |
| $\le$ (Less than) | Add **Slack** ($s_i$) | $+ s_i$ |
| $\ge$ (Greater than) | Subtract **Surplus** ($e_i$) AND Add **Artificial** ($a_i$) | $- e_i + a_i$ |
| $=$ (Equal to) | Add **Artificial** ($a_i$) | $+ a_i$ |

### **Why Artificial Variables?**
Standard Simplex requires an initial identity matrix (a "1" in each row) to start.
*   $\le$ constraints provide Slack variables ($+s$) which act as this identity.
*   $\ge$ constraints subtract Surplus ($-e$), which is $-1$, not $+1$.
*   We add Artificial Variables ($a_i$) purely to create that $+1$ to start the math. They represent "fake" resources.

### **Getting rid of Artificial Variables (Big M Method)**
Since $a_i$ is fake, it cannot be in the final solution.
1.  Assign a penalty of **$-M$** (a huge number) to the artificial variable in the Objective Function.
2.  This forces the Simplex algorithm to remove $a_i$ from the basis as fast as possible because it is "expensive" to keep.
