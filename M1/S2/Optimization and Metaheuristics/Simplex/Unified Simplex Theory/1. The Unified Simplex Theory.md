
# 1. The Unified Simplex Theory

## **1.1. The Fundamental Concept**
The Simplex Method is an algebraic procedure that solves Linear Programming (LP) problems by moving from one **Basic Feasible Solution (BFS)** to an adjacent, better BFS until the optimal solution is found.

Geometrically (as explained in *6_A Geometric Interpretation*), the set of all feasible solutions forms a **Polytope** (a polygon in N-dimensions).
1. The **Optimal Solution** always lies on a **Vertex** (corner point) of this polytope.
2. The Simplex method starts at the **Origin** (usually).
3. It slides along the **Edges** of the polytope to the next vertex that improves the objective function.
4. When no neighboring vertex offers a better value, the algorithm stops.

## **1.2. The Core Vocabulary**
To eliminate confusion, we must define the standard terminology used in these notes.

*   **Decision Variables ($x_j$):** The variables we want to solve for (e.g., number of chairs to build).
*   **Slack Variables ($s_i$):** Added to turn $\le$ inequalities into equalities. They represent unused resources.
*   **Surplus Variables ($e_i$):** Subtracted to turn $\ge$ inequalities into equalities.
*   **Artificial Variables ($a_i$):** Fake variables added to force a mathematical solution when the origin is not feasible (used in Big M / Two-Phase).
*   **Basic Variables (Basis):** [[Qs2]] The variables currently "in the mix." In a system with $m$ constraints and $n$ variables, there are always $m$ basic variables. They have non-zero values.
*   **Non-Basic Variables:** The variables set to **zero** in the current iteration.
*   **$C_j$:** The coefficient of a variable in the original Objective Function (Profit/Cost per unit).
*   **$Z_j$:** The **Opportunity Cost** (or marginal cost). It represents the value of resources consumed if we produce one unit of this variable.
*   **$C_j - Z_j$ (Net Evaluation Row):** The **Net Profit**. It tells us: "If I bring one unit of this variable into the solution, how much will my total profit increase?"

> [!TIP] **The Golden Rule of Simplex**
> The entire algorithm is just a fancy way of asking two questions repeatedly:
> 1. **Who enters?** (Which non-basic variable will improve the result the fastest?)
> 2. **Who leaves?** (Which current basic variable runs out of resources first?)
