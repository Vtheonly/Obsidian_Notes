
# 4. The Rosetta Stone of Simplex Variations

**This is the most important note for you.** This explains why different sources look different and how to map them to the canonical form above.

There are essentially two main ways Simplex is taught. They are mathematically identical but visually distinct.

## **Variation A: The "Coefficients" Method (Standard)**
*   **Used by:** File 1, File 3, File 5.
*   **Structure:** Explicitly calculates $Z_j$ and $C_j - Z_j$.
*   **Logic:** Economic interpretation. $C_j$ is revenue, $Z_j$ is opportunity cost.
*   **Optimality:** Stop when max positive value in $C_j - Z_j$ is gone.

## **Variation B: The "Equation" Method (Row 0 Method)**
*   **Used by:** File 2, File 6.
*   **Structure:** Rewrites the objective function as an equation equal to zero.
    *   $Z = 3x_1 + 5x_2 \rightarrow Z - 3x_1 - 5x_2 = 0$.
*   **The Difference:** The coefficients in the top row (Row 0) are **Negated**.
    *   Instead of seeing $3$ and $5$, you see $-3$ and $-5$.
*   **Pivot Rule Change:**
    *   **Entering Variable:** Look for the **Most Negative** number in Row 0 (instead of most positive).
    *   **Optimality:** Stop when all numbers in Row 0 are **$\ge 0$**.
*   **Why use this?** It allows you to treat the objective function row exactly like a constraint row during Gaussian elimination. You don't need a separate $Z_j$ calculation step; you just do row operations on everything.

| Feature | Variation A (Coefficients) | Variation B (Row 0 / Equation) |
| :--- | :--- | :--- |
| **Objective Row** | Listing $C_j$ (Positive) | $Z - \dots = 0$ (Negative) |
| **Calculation** | Compute $C_j - Z_j$ manually | Row Ops included in algorithm |
| **Entering Rule** | Largest Positive | Most Negative |
| **Stop Rule** | All $\le 0$ | All $\ge 0$ |
| **Meaning** | Same Result | Same Result |

> [!WARNING] **Don't Mix Them!**
> If you start a problem using Variation A, do not switch to "Most Negative" logic midway. Stick to one convention. Variation A is generally better for understanding the *economics* (profit margins). Variation B is faster for *computation* (matrix math).
