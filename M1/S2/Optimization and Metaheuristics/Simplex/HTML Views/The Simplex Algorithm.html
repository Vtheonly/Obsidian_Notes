Part 1

Linear Programming Forms

Before we can use the Simplex Algorithm, we need to understand how LP problems are structured and how to convert them into the required format.

General Form

The most flexible formulation. Allows any combination of:

  • Min or Max objective
  • =, ≤, ≥ constraints
  • Free or bounded variables
min/max cTx
s.t. Ax ≤/=/≥ b
x ≥ 0 or free

Canonical Form

Used for duality theory and specific problems:

  • Minimize only
  • All ≥ constraints
  • All x ≥ 0
min cTx
s.t. Ax ≥ b
x ≥ 0

Standard Form

Required

The Simplex Algorithm requires this exact format:

  • Minimize only
  • All = constraints
  • All x ≥ 0
min cTx
s.t. Ax = b
x ≥ 0

Step 1: Identify Issues

Look at the original problem. We see two issues that violate Standard Form: 1) The objective is "maximize" instead of "minimize". 2) The constraints use "≤" instead of "=".

Quick Reference: Transformation Rules

Maximization → Minimization

Multiply the objective by -1:

max z = cTx → min (-z) = -cTx

≤ Inequality → Equality

Add a slack variable s ≥ 0:

ax ≤ b → ax + s = b

≥ Inequality → Equality

Subtract a surplus variable s ≥ 0:

ax ≥ b → ax - s = b

Free Variable → Nonnegative

Split into difference of two variables:

x (free) → x = x⁺ - x⁻, where x⁺,x⁻ ≥ 0

Important: The RHS vector b must be non-negative. If any bi < 0, multiply that entire equation by -1.

Part 2

Basic Feasible Solutions

The algebraic representation of "corners" in the feasible region. Understanding BFS is key to understanding why Simplex works.

📐

Basis (B)

A collection of m linearly independent columns of A that form an m×m nonsingular matrix B.

🎯

Basic Variables (xB)

Variables corresponding to basis columns. We solve for these: xB = B⁻¹b

🔒

Non-Basic Variables (xN)

The remaining n-m variables. We set these equal to zero to find a corner.

Theorem 2.1: Existence

If a feasible solution exists, a basic feasible solution exists.

→ We only need to search the corners, not the entire region!

Theorem 2.2: Boundedness

If the LP has a finite optimum, the feasible region can be bounded to a large hypercube without changing the optimal value.

Degeneracy: When Things Get Tricky

A BFS is degenerate if it has more than n-m zeros (i.e., a basic variable equals 0).

Algebraic Meaning:

One of the basic variables turns out to be zero.

Geometric Meaning:

More constraints pass through the vertex than necessary (over-defined point).

⚠️ Degeneracy can cause the Simplex algorithm to stall or cycle!

Part 5

Understanding the Tableau

The tableau is the bookkeeping system of Simplex. Let's understand each part and why the reduced costs tell us everything.

Anatomy of a Tableau

Non-basic columns Basic columns (Identity) RHS
Row 0 c̄₁ c̄₂ ... ... 0 0 -z
Row 1 a₁₁ a₁₂ ... ... 1 0 b₁
Row 2 a₂₁ a₂₂ ... ... 0 1 b₂

Reduced Cost (c̄ⱼ)

The reduced cost tells you how much the objective changes if you introduce one unit of variable xⱼ:

c̄ⱼ = cⱼ - Σ c_B(i) · aᵢⱼ

If c̄ⱼ < 0 and we're minimizing, bringing in xⱼ will decrease the cost!

Optimality Criterion

A BFS is optimal if all reduced costs are non-negative:

c̄ⱼ ≥ 0 for all j

This means no variable can improve the objective. We're done!

The Ratio Test: Why It Works

When we choose a column j to enter the basis, we need to determine:

  1. How much can we increase xⱼ? (called θ)
  2. Which basic variable will hit zero first and leave?

The Formula

θ₀ = min { bᵢ / aᵢⱼ : aᵢⱼ > 0 }

Why only positive aᵢⱼ?

If aᵢⱼ ≤ 0, increasing xⱼ doesn't decrease the basic variable x_B(i). It either stays the same (aᵢⱼ = 0) or increases (aᵢⱼ < 0). We only care about variables that might go negative!

// Example: Finding θ₀

Entering column: x₁

Column values: [1, 2]

RHS values: [4, 6]

// Calculate ratios:

Row 1: 4/1 = 4

Row 2: 6/2 = 3 ← minimum!

θ₀ = 3

Row 2 (s₂) leaves the basis

Bland's Anticycling Rule

In degenerate cases (θ₀ = 0), the algorithm might cycle forever through the same bases. Bland's Rule guarantees termination.

1 Entering Variable

Among all columns with c̄ⱼ < 0, choose the one with the smallest index j.

2 Leaving Variable

If there's a tie in the ratio test, choose the row with the smallest basic variable index.

Simple Rule: When in doubt, always pick the variable with the lowest subscript!

Part 6

The Two-Phase Method

What if we can't find an obvious starting BFS? The two-phase method creates one for us!

The Starting Point Problem

Easy Case: Ax ≤ b

When all constraints are ≤ inequalities with b ≥ 0, slack variables form an identity matrix!

x₁ + x₂ + s₁ = 4
2x₁ + x₂ + s₂ = 6

Initial BFS: (0, 0, 4, 6) with basis {s₁, s₂}

Hard Case: Ax = b or Ax ≥ b

With equalities or ≥ constraints, there's no obvious identity submatrix!

x₁ + 2x₂ = 5
3x₁ + x₂ = 4

How do we find a starting feasible basis?

Original Problem

Minimize z = 2x₁ + 3x₂

Subject to:

x₁ + 2x₂ = 5

3x₁ + x₂ = 4

x₁, x₂ ≥ 0

Problem: We have equality constraints but no obvious identity submatrix for a starting basis. We need to find a feasible solution first!

Phase I: Add Artificial Variables

Minimize ξ = a₁ + a₂

Subject to:

x₁ + 2x₂ + a₁ = 5

3x₁ + x₂ + a₂ = 4

All variables ≥ 0

We add artificial variables a₁ and a₂ to create an identity matrix. The new objective is to minimize the sum of artificial variables. If we can drive them to zero, the original problem is feasible!

Phase I: Initial Tableau

ξ -4 -3 0 0 9
a₁ 1 2 1 0 5
a₂ 3 1 0 1 4

Start with artificial variables as the basis: {a₁, a₂}. Current ξ = 9 (sum of artificial variables). We need to pivot to reduce ξ to 0.

Phase I: After Pivoting

After several pivot operations...

Basis x₁ x₂ a₁ a₂ RHS
ξ 0 0 0.6 0.8 0
x₂ 0 1 0.6 -0.2 2.2
x₁ 1 0 -0.2 0.4 0.6

Success! ξ = 0, meaning all artificial variables are zero. We found a feasible solution: x₁ = 0.6, x₂ = 2.2. Now we can proceed to Phase II!

Phase II: Restore Original Objective

Basis x₁ x₂ RHS
z 0 0 7.8
x₂ 0 1 2.2
x₁ 1 0 0.6

Drop artificial columns and replace Row 0 with the original objective z = 2x₁ + 3x₂. We must compute the reduced costs for the current basis.

Phase II: Optimal Solution!

🏆
Optimal Solution Found
x₁* = 0.6
x₂* = 2.2
💰
Optimal Value
z* = 2(0.6) + 3(2.2) = 7.8

Summary: Phase I found a feasible starting point, Phase II optimized from there. The two-phase method guarantees we can solve any LP (or determine it's infeasible)!

If min ξ = 0

All artificial variables are zero. The remaining basis consists of original variables. The original problem is feasible. Proceed to Phase II with this starting BFS.

If min ξ > 0

We can't eliminate all artificial variables. Some constraints can't be satisfied simultaneously. The original problem is infeasible. No solution exists!

Phase I finds feasibility, Phase II optimizes. Handles any LP!

Key Insight

The Simplex Algorithm works because LP is a convex problem. Any local optimum is automatically a global optimum. By checking only the vertices (finitely many), we're guaranteed to find the best solution.
