🔑 The Golden Rule of Simplex

The entire algorithm is just a fancy way of asking two questions repeatedly:

1. Who Enters?

Which non-basic variable will improve the result the fastest?

2. Who Leaves?

Which current basic variable runs out of resources first?

Core Vocabulary

Master these terms first. Click each card to expand the explanation.

xⱼ

Decision Variables

The variables we want to solve for

Example: If you're optimizing production, x₁ might be "number of chairs" and x₂ might be "number of tables."

Max Z = 25x₁ + 15x₂
sᵢ

Slack Variables

Added to ≤ inequalities

Slack represents unused resources. If you have 100 hours available and use 70, the slack is 30.

2x₁ + 3x₂ ≤ 100
→ 2x₁ + 3x₂ + s₁ = 100
eᵢ

Surplus Variables

Subtracted from ≥ inequalities

Surplus represents excess over minimum. If you need at least 50 units and produce 75, the surplus is 25.

x₁ + x₂ ≥ 50
→ x₁ + x₂ - e₁ = 50
aᵢ

Artificial Variables

Fake variables for starting the algorithm

Added to ≥ and = constraints to create an initial identity matrix. Must be driven out of the solution using Big M method.

Objective: Z - Maᵢ
(M = very large penalty)
B

Basic Variables

Variables "in the mix" with values ≠ 0

In a system with m constraints, exactly m variables are basic. They form a column of the identity matrix in the tableau.

💡 Initially, slack variables are basic (origin point)
N

Non-Basic Variables

Variables set to zero

These are "waiting on the sidelines." The algorithm decides which one to bring into the basis to improve the solution.

✓ The entering variable comes from this set

Key Calculation Values

Cⱼ
Objective Coefficient

Profit/cost per unit of variable j in the original objective function. Never changes.

Zⱼ
Opportunity Cost

Value of resources consumed to produce one unit. Calculated as Σ(Cᵦ × column values).

Cⱼ - Zⱼ
Net Evaluation

Net profit from adding one unit. Positive = room for improvement.

How to Calculate Zⱼ

The Zⱼ value for each column is calculated by multiplying the Cᵦ column by the corresponding column values and summing:

For x₁ column:
Zⱼ = (0 × 2) + (0 × 1) = 0
For x₂ column:
Zⱼ = (0 × 1) + (0 × 2) = 0

Since all basic variables are slack (Cᵦ = 0), all Zⱼ = 0 initially. This changes as we bring decision variables into the basis!

💡 Interpretation

Zⱼ tells you: "How much profit am I giving up to produce one more unit of this variable?"

The Rosetta Stone

Different textbooks use different notations. They're all the same algorithm! Toggle between methods to see how they compare.

Method A: Coefficients (Standard)

Explicitly calculates Zⱼ and Cⱼ-Zⱼ. Economic interpretation: Cⱼ is revenue, Zⱼ is opportunity cost.

Cⱼ 3 5 0 0
Cᵦ Basis x₁ x₂ s₁ s₂ RHS
0 s₁ 1 0 1 0 4
0 s₂ 0 2 0 1 12
Zⱼ 0 0 0 0 0
Cⱼ-Zⱼ 3 5 ← 0 0
Entering Rule:

Pick the largest positive value in Cⱼ-Zⱼ row

Stop When:

All values in Cⱼ-Zⱼ are ≤ 0

Method B: Row 0 / Equation

Rewrites objective as equation: Z - 3x₁ - 5x₂ = 0. Coefficients are negated.

Basis x₁ x₂ s₁ s₂ RHS
Z -3 -5 ← 0 0 0
s₁ 1 0 1 0 4
s₂ 0 2 0 1 12
Entering Rule:

Pick the most negative value in Row 0

Stop When:

All values in Row 0 are ≥ 0

Quick Comparison

Feature Method A (Coefficients) Method B (Row 0)
Objective Row Cⱼ listed positive Z - ... = 0 (negative)

Entering Rule Largest Positive Most Negative
Stop Rule All ≤ 0 All ≥ 0
Result ✓ Same ✓ Same
⚠️
Don't Mix Them!

If you start a problem using Method A, do not switch to "Most Negative" logic midway. Stick to one convention throughout.

Special Cases

Sometimes the algorithm encounters unusual situations. Learn to recognize and handle them.

∞

Unbounded Solution

Symptom: All entries in the pivot column are ≤ 0 when looking for the leaving variable.

Meaning: The feasible region is open in the direction of improvement. Profit can grow infinitely.
≡

Multiple Optimal Solutions

Symptom: At optimum, a non-basic variable has Cⱼ-Zⱼ = 0.

Meaning: The objective function is parallel to a constraint. Any point on that edge is optimal.
⟳

Degeneracy

Symptom: Tie in the minimum ratio test.

Ratio calculation:
Row 1: 12/3 = 4
Row 2: 8/2 = 4 ← Tie!
Solution: Pick either (use Bland's rule to prevent cycling: choose the one with smallest index).
∅

Infeasible Solution

Symptom: At "optimum," an artificial variable is still in the basis with value > 0.

Meaning: The constraints contradict each other. No solution satisfies all of them.

Constraint Conversion Reference

Constraint Type Action Variables Added Example
≤ (Less than) Add Slack + sᵢ 2x + 3y ≤ 10 → 2x + 3y + s₁ = 10
≥ (Greater than) Subtract Surplus + Add Artificial - eᵢ + aᵢ x + y ≥ 5 → x + y - e₁ + a₁ = 5
= (Equal to) Add Artificial only + aᵢ x + 2y = 8 → x + 2y + a₁ = 8
