**Origin:** Defined by Edgar F. Codd (1970).
**Core Concept:** Data is represented as **Relations** (Tables).

## Key Properties
1.  **Atomic Values:** Every cell must contain a single value (e.g., no arrays in a single cell).
2.  **Uniqueness:** No two rows can be identical (enforced via Primary Keys).
3.  **Order Irrelevance:** The order of rows and columns does not matter to the logic of the data.

## SQL Language Categories
*   **DDL (Data Definition Language):** `CREATE`, `ALTER`, `DROP` (Defines structure).
*   **DML (Data Manipulation Language):** `SELECT`, `INSERT`, `UPDATE`, `DELETE` (Manipulates data).
*   **DCL (Data Control Language):** `GRANT`, `REVOKE` (Permissions).
