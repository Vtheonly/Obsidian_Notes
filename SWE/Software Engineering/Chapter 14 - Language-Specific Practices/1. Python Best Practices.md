# 1. Python Best Practices

> **Tags:** #python #best-practices #tooling #conventions

This note covers Python-specific practices for testing, debugging, refactoring, and tooling. It assumes you know Python syntax and focuses on the conventions that professional Python developers follow.

---

## 14.1.1 Project Structure

A well-organized Python project:

```text
my-project/
  src/
      mypackage/
          __init__.py
          main.py
          models.py
          services/
              __init__.py
              auth.py
  tests/
      conftest.py
      test_models.py
      services/
          test_auth.py
  pyproject.toml
  README.md
  .gitignore
```

The `src/` layout prevents accidental imports from the working directory and is the recommended structure.

## 14.1.2 pyproject.toml (Modern Python Configuration)

`pyproject.toml` is the standard configuration file for Python projects, replacing `setup.py`, `setup.cfg`, and `requirements.txt`.

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "mypackage"
version = "1.0.0"
description = "A sample Python package"
readme = "README.md"
requires-python = ">=3.11"
license = {text = "MIT"}
authors = [{name = "Your Name", email = "you@example.com"}]
dependencies = [
    "fastapi>=0.100.0",
    "pydantic>=2.0.0",
    "sqlalchemy>=2.0.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0",
    "pytest-cov>=4.0",
    "ruff>=0.4.0",
    "mypy>=1.5.0",
]

[project.scripts]
mycli = "mypackage.cli:main"

[tool.ruff]
line-length = 88
target-version = "py311"

[tool.ruff.lint]
select = ["E", "W", "F", "I", "B", "UP", "SIM"]

[tool.mypy]
strict = true

[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-v --cov=mypackage"
```

## 14.1.3 Dependency Management

### pip + venv (Basic)

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -e .  # install the package in development mode
```

### Poetry (Recommended)

```bash
poetry init
poetry add fastapi pydantic
poetry add --group dev pytest ruff mypy
poetry install
poetry run pytest
```

Poetry manages the virtual environment, dependencies, and lockfile together.

### uv (Fastest)

```bash
uv init
uv add fastapi pydantic
uv add --dev pytest ruff mypy
uv run pytest
```

`uv` is a Rust-based Python package manager that is 10-100x faster than pip and Poetry.

## 14.1.4 Type Hints and MyPy

Python type hints are optional but strongly recommended. They catch bugs and improve IDE support.

```python
from typing import Optional, List, Dict, Any
from pathlib import Path

# Function with type hints
def greet(name: str, times: int = 1) -> str:
    return f"Hello, {name}! " * times

# Optional and None
def find_user(user_id: int) -> Optional[User]:
    return users.get(user_id)

# Collections (Python 3.9+: use built-in types)
def process_items(items: list[str]) -> dict[str, int]:
    return {item: len(item) for item in items}

# Path instead of str for file paths
def read_config(path: Path) -> dict[str, Any]:
    return json.loads(path.read_text())

# Callable
from typing import Callable
def apply(func: Callable[[int], int], value: int) -> int:
    return func(value)
```

Run mypy to check:

```bash
mypy --strict mypackage/
```

## 14.1.5 Testing with pytest

```python
# tests/test_auth.py
import pytest
from mypackage.services.auth import authenticate, InvalidCredentials

class TestAuthenticate:
    def test_valid_credentials(self):
        result = authenticate("alice", "password123")
        assert result.is_valid is True
        assert result.user.email == "alice@example.com"
    
    def test_invalid_password(self):
        with pytest.raises(InvalidCredentials):
            authenticate("alice", "wrong")
    
    @pytest.mark.parametrize("username,password,expected", [
        ("alice", "password123", True),
        ("bob", "password456", True),
        ("alice", "wrong", False),
        ("nonexistent", "password", False),
    ])
    def test_various_credentials(self, username, password, expected):
        if expected:
            result = authenticate(username, password)
            assert result.is_valid
        else:
            with pytest.raises(InvalidCredentials):
                authenticate(username, password)
```

### Fixtures

```python
# tests/conftest.py
import pytest
from mypackage.db import Database
from mypackage.models import User

@pytest.fixture
def db():
    '''In-memory database for testing.'''
    db = Database("sqlite:///:memory:")
    db.create_tables()
    yield db
    db.close()

@pytest.fixture
def user(db):
    '''A test user.'''
    user = User(name="Alice", email="alice@example.com")
    db.save(user)
    return user
```

## 14.1.6 Common Python Mistakes

### Mutable Default Arguments

```python
# BAD: the default list is shared across calls
def add_item(item, items=[]):
    items.append(item)
    return items

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2] — not [2]!

# GOOD: use None as default
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

### Bare Except

```python
# BAD: catches everything, including KeyboardInterrupt
try:
    do_something()
except:
    pass

# GOOD: catch specific exceptions
try:
    do_something()
except (ValueError, KeyError) as e:
    logger.error("Failed: %s", e)
```

### Using `is` for Value Comparison

```python
# BAD: is checks identity, not equality
if x is 5:  # may not work for large integers
    ...

# GOOD: use == for value comparison
if x == 5:
    ...
```

### Not Using Context Managers

```python
# BAD: file may not be closed if an exception occurs
f = open("file.txt")
content = f.read()
f.close()

# GOOD: context manager ensures file is closed
with open("file.txt") as f:
    content = f.read()
```

### String Formatting

```python
# Old style (avoid)
name = "Alice"
greeting = "Hello, %s" % name

# str.format (better)
greeting = "Hello, {}".format(name)

# f-strings (best, Python 3.6+)
greeting = f"Hello, {name}!"
```

## 14.1.8 Key Takeaways

- Use the `src/` layout for project structure.
- `pyproject.toml` is the standard config file (replaces setup.py, requirements.txt).
- Use Poetry or uv for dependency management.
- Add type hints and check with mypy (`--strict`).
- Test with pytest; use fixtures and parametrization.
- Avoid mutable default arguments, bare except, and `is` for value comparison.
- Use f-strings, context managers, and specific exception types.
- Tools: ruff (lint+format), mypy (types), pytest (tests), poetry/uv (deps).
