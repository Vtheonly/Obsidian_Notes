## 14.3.2 Use Modern Java Features

Java 17+ (LTS) and Java 21+ (LTS) have many features that make Java more concise:

### Records (Java 16+)

```java
// Old: verbose class with getters, equals, hashCode, toString
public class User {
    private final long id;
    private final String name;
    private final String email;
    
    public User(long id, String name, String email) {
        this.id = id;
        this.name = name;
        this.email = email;
    }
    
    public long getId() { return id; }
    public String getName() { return name; }
    public String getEmail() { return email; }
    
    // equals, hashCode, toString...
}

// New: record (one line)
public record User(long id, String name, String email) {}
```

### Pattern Matching (Java 21+)

```java
// Old: instanceof + cast
if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.length());
}

// New: pattern matching
if (obj instanceof String s) {
    System.out.println(s.length());
}

// Switch pattern matching
String format(Object obj) {
    return switch (obj) {
        case Integer i -> String.format("int %d", i);
        case Long l    -> String.format("long %d", l);
        case Double d  -> String.format("double %f", d);
        case String s  -> String.format("String %s", s);
        default        -> obj.toString();
    };
}
```

### var (Java 10+)

```java
// Old
Map<String, List<User>> usersByCity = new HashMap<>();

// New: var (type inferred)
var usersByCity = new HashMap<String, List<User>>();
```

Use `var` when the type is obvious from the right side. Do not use it when the type adds clarity.

### Stream API

```java
// Old: imperative
List<String> names = new ArrayList<>();
for (User user : users) {
    if (user.isActive()) {
        names.add(user.getName());
    }
}

// New: Stream API
List<String> names = users.stream()
    .filter(User::isActive)
    .map(User::getName)
    .toList();
```

### Optional

```java
// BAD: returning null
public User findUser(long id) {
    return users.getOrDefault(id, null);  // caller must null-check
}

// GOOD: returning Optional
public Optional<User> findUser(long id) {
    return Optional.ofNullable(users.get(id));
}

// Usage
findUser(1)
    .map(User::getName)
    .ifPresent(name -> System.out.println("Found: " + name));
```

## 14.3.3 Dependency Injection

Use dependency injection (Spring, Guice, or manual) to make code testable:

```java
// BAD: hard dependency
public class UserService {
    private final MySQLDatabase db = new MySQLDatabase();  // hard to test
    
    public User getUser(long id) {
        return db.findById(id);
    }
}

// GOOD: dependency injection
public class UserService {
    private final UserRepository repository;
    
    public UserService(UserRepository repository) {
        this.repository = repository;
    }
    
    public User getUser(long id) {
        return repository.findById(id)
            .orElseThrow(() -> new UserNotFoundException(id));
    }
}
```

## 14.3.4 Testing with JUnit 5

```java
import org.junit.jupiter.api.*;
import static org.junit.jupiter.api.Assertions.*;
import static org.mockito.Mockito.*;

class UserServiceTest {
    
    private UserRepository repository;
    private UserService service;
    
    @BeforeEach
    void setUp() {
        repository = mock(UserRepository.class);
        service = new UserService(repository);
    }
    
    @Test
    @DisplayName("getUser returns user when found")
    void getUserReturnsUserWhenFound() {
        User alice = new User(1, "Alice", "alice@example.com");
        when(repository.findById(1L)).thenReturn(Optional.of(alice));
        
        User result = service.getUser(1);
        
        assertEquals(alice, result);
        verify(repository).findById(1L);
    }
    
    @Test
    @DisplayName("getUser throws when not found")
    void getUserThrowsWhenNotFound() {
        when(repository.findById(1L)).thenReturn(Optional.empty());
        
        assertThrows(UserNotFoundException.class, () -> service.getUser(1));
    }
    
    @ParameterizedTest
    @CsvSource({"1, Alice", "2, Bob", "3, Charlie"})
    void getUserReturnsCorrectName(long id, String expectedName) {
        when(repository.findById(id))
            .thenReturn(Optional.of(new User(id, expectedName, "")));
        
        assertEquals(expectedName, service.getUser(id).name());
    }
}
```

## 14.3.5 Common Java Mistakes

### Not Using Try-With-Resources

```java
// BAD: resource may not be closed
BufferedReader reader = new BufferedReader(new FileReader("file.txt"));
String line = reader.readLine();
reader.close();  // not called if exception

// GOOD: try-with-resources
try (BufferedReader reader = new BufferedReader(new FileReader("file.txt"))) {
    String line = reader.readLine();
}  // reader is automatically closed
```

### Ignoring Generics

```java
// BAD: raw type
List list = new ArrayList();
list.add("hello");
list.add(42);  // no error, but will crash at runtime
String s = (String) list.get(1);  // ClassCastException

// GOOD: generics
List<String> list = new ArrayList<>();
list.add("hello");
// list.add(42);  // compile error
String s = list.get(0);  // no cast needed
```

### Swallowing Exceptions

```java
// BAD: exception is swallowed
try {
    riskyOperation();
} catch (Exception e) {
    // do nothing
}

// GOOD: log or rethrow
try {
    riskyOperation();
} catch (Exception e) {
    logger.error("Risky operation failed", e);
    throw new RuntimeException("Operation failed", e);
}
```

## 14.3.7 Key Takeaways

- Use modern Java features: records, pattern matching, var, Stream API, Optional.
- Use dependency injection for testable code.
- Test with JUnit 5 and Mockito; use parameterized tests.
- Use try-with-resources for AutoCloseable resources.
- Use generics, never raw types.
- Do not swallow exceptions — log and rethrow.
- Tools: Maven/Gradle (build), JUnit 5 + Mockito (tests), Checkstyle/SpotBugs (analysis), Spring Boot (web).
