### Introduction

The **Strategy Pattern** is often the first pattern taught because it addresses one of the most fundamental principles of good object-oriented design: **Composition over Inheritance**.

It provides a way to manage behavior that changes across different objects of the same type without relying on rigid class hierarchies or massive `if/else` statements.

### The Official Definition

> **The Strategy Pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. Strategy lets the algorithm vary independently from clients that use it.**

Let's break this definition down, as it contains three distinct parts:

#### 1. "Defines a family of algorithms"

In this context, an "algorithm" isn't just a complex mathematical formula. It is simply **behavior**.

- A specific way of flying (e.g., Jet propulsion vs. Flapping wings).
- A specific way of quacking (e.g., Squeak vs. Loud Quack).
- A specific way of sorting a list (e.g., Bubble Sort vs. Quick Sort).

The pattern suggests gathering all these variations and treating them as a related set (a family).

#### 2. "Encapsulates each one and makes them interchangeable"

Instead of writing the code for "Flying" inside the `Duck` class, you extract it into its own class.

- We create a specific class just for `JetFlying`.
- We create a specific class just for `NoFlying`.
- Because they implement a common interface (e.g., `IFlyBehavior`), the main object (the Duck) doesn't care which one it has. It can swap a `JetFlying` behavior for a `SimpleFlying` behavior without breaking.

#### 3. "Lets the algorithm vary independently from clients that use it"

This is the concept of **Decoupling**.

- **The Client:** The object using the behavior (e.g., the `Duck`).
- **The Algorithm:** The behavior itself (e.g., `Quacking`).

If you need to change how `JetFlying` works (maybe optimize the fuel consumption logic), you change code in the `JetFlying` class. You do **not** have to touch the `Duck` class. The client is isolated from the changes in the behavior.
