# 3. Lambda Triggers and Events

> [!info] Chapter Context
> Lambda functions are triggered by events. This note covers the major trigger sources (API Gateway, S3, DynamoDB Streams, SQS, SNS, EventBridge, CloudWatch Events/alarms) and how to configure them.

Related: [[2. Lambda Fundamentals]] | [[4. API Gateway]] | [[5. Step Functions]]

---

## 1. Trigger Sources

Lambda can be triggered by:

- **API Gateway** — HTTP requests.
- **S3** — Object creation/deletion.
- **DynamoDB Streams** — Table changes.
- **SQS** — New messages in a queue.
- **SNS** — New messages in a topic.
- **EventBridge** — Events from AWS services, custom events, scheduled.
- **Kinesis Data Streams** — Real-time data.
- **CloudWatch Logs** — Log subscription filters.
- **CloudWatch Alarms** — Alarm state changes.
- **CodeCommit** — Repository events.
- **SES** — Email reception.
- **Cognito** — User pool events.
- **Alexa** — Voice skills.
- **Manual / CLI** — Direct invocation.

## 2. S3 Triggers

When an object is created in S3, S3 can invoke a Lambda function.

```bash
aws lambda add-permission \
  --function-name my-func \
  --statement-id s3-invoke \
  --action lambda:InvokeFunction \
  --principal s3.amazonaws.com \
  --source-arn arn:aws:s3:::my-bucket \
  --source-account 123456789012

aws s3api put-bucket-notification-configuration \
  --bucket my-bucket \
  --notification-configuration '{
    "LambdaFunctionConfigurations": [{
      "LambdaFunctionArn": "arn:aws:lambda:us-east-1:123456789012:function:my-func",
      "Events": ["s3:ObjectCreated:*"],
      "Filter": {"Prefix": "uploads/", "Suffix": ".jpg"}
    }]
  }'
```

The event:

```json
{
  "Records": [{
    "eventName": "ObjectCreated:Put",
    "s3": {
      "bucket": {"name": "my-bucket"},
      "object": {"key": "uploads/photo.jpg", "size": 123456}
    }
  }]
}
```

Common use case: when a user uploads an image, generate a thumbnail.

> [!warning] S3 -> Lambda Can Invoke Once Per Event
> S3 may invoke Lambda multiple times for the same event (at-least-once). Make your function idempotent.

## 3. DynamoDB Streams Triggers

When an item in DynamoDB changes (insert, update, delete), the change is captured in DynamoDB Streams. Lambda can consume the stream.

```bash
aws lambda create-event-source-mapping \
  --function-name my-func \
  --event-source-arn arn:aws:dynamodb:us-east-1:123456789012:table/my-table/stream/2024-01-15T00:00:00.000 \
  --starting-position LATEST \
  --batch-size 100
```

The event:

```json
{
  "Records": [{
    "eventName": "INSERT",
    "dynamodb": {
      "Keys": {"id": {"S": "1"}},
      "NewImage": {"id": {"S": "1"}, "name": {"S": "Alice"}}
    }
  }]
}
```

Common use cases:

- Maintain an aggregation (e.g., count of items per partition key).
- Replicate to another table or service.
- Trigger notifications.

## 4. SQS Triggers

Lambda polls an SQS queue and invokes the function with batches of messages.

```bash
aws lambda create-event-source-mapping \
  --function-name my-func \
  --event-source-arn arn:aws:sqs:us-east-1:123456789012:my-queue \
  --batch-size 10 \
  --maximum-batching-window-in-seconds 5
```

The event:

```json
{
  "Records": [{
    "messageId": "abc123",
    "body": "{\"order_id\": 123}",
    "receiptHandle": "...",
    "attributes": {"ApproximateReceiveCount": "1"}
  }]
}
```

If the function succeeds, Lambda deletes the messages from the queue. If the function fails, the messages become visible again after the visibility timeout.

### 4.1 Batch Failure Reporting

For partial failures (some messages in a batch fail), use `ReportBatchItemFailures`:

```bash
aws lambda create-event-source-mapping \
  --function-name my-func \
  --event-source-arn arn:aws:sqs:us-east-1:123456789012:my-queue \
  --function-response-types ['ReportBatchItemFailures']
```

```python
def lambda_handler(event, context):
    batch_item_failures = []
    for record in event['Records']:
        try:
            process(record)
        except Exception:
            batch_item_failures.append({'itemIdentifier': record['messageId']})
    return {'batchItemFailures': batch_item_failures}
```

Only the failed messages are returned to the queue; successful ones are deleted.

## 5. SNS Triggers

SNS can invoke Lambda directly:

```bash
aws sns subscribe --topic-arn arn:aws:sns:us-east-1:123456789012:my-topic \
  --protocol lambda \
  --notification-endpoint arn:aws:lambda:us-east-1:123456789012:function:my-func
```

The event:

```json
{
  "Records": [{
    "EventSource": "aws:sns",
    "Sns": {
      "TopicArn": "arn:aws:sns:us-east-1:123456789012:my-topic",
      "Subject": "Alert",
      "Message": "{\"event\":\"alert\"}"
    }
  }]
}
```

## 6. EventBridge Triggers

EventBridge routes events from AWS services, custom applications, and scheduled events to Lambda.

```bash
aws events put-rule \
  --name daily-report \
  --schedule-expression "rate(1 day)"

aws lambda add-permission \
  --function-name my-func \
  --statement-id eventbridge-invoke \
  --action lambda:InvokeFunction \
  --principal events.amazonaws.com \
  --source-arn arn:aws:events:us-east-1:123456789012:rule/daily-report

aws events put-targets \
  --rule daily-report \
  --targets '[{"Id": "1", "Arn": "arn:aws:lambda:us-east-1:123456789012:function:my-func"}]'
```

### 6.1 AWS Service Events

Many AWS services emit events to the **default event bus**:

- EC2 instance state changes.
- S3 object creation (via CloudTrail).
- RDS instance events.
- Health events.
- API calls via CloudTrail.

Create a rule matching specific events:

```bash
aws events put-rule \
  --name ec2-stopped \
  --event-pattern '{
    "source": ["aws.ec2"],
    "detail-type": ["EC2 Instance State-change Notification"],
    "detail": {"state": ["stopped"]}
  }'
```

### 6.2 Scheduled Events

Use `rate()` or `cron()` expressions:

- `rate(5 minutes)` — every 5 minutes.
- `rate(1 day)` — daily.
- `cron(0 2 * * ? *)` — daily at 2 AM UTC.

## 7. Kinesis Triggers

Lambda can consume Kinesis Data Streams:

```bash
aws lambda create-event-source-mapping \
  --function-name my-func \
  --event-source-arn arn:aws:kinesis:us-east-1:123456789012:stream/my-stream \
  --starting-position LATEST \
  --batch-size 100
```

Similar to DynamoDB Streams, but for general-purpose streaming data (logs, clickstream).

## 8. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting to Add Permission for the Trigger
> S3, SNS, and EventBridge need `lambda:InvokeFunction` permission to call your function. Use `aws lambda add-permission` (or the Console's "Add trigger" flow, which does it for you).

> [!warning] Mistake 2 — Not Handling At-Least-Once Delivery
> Lambda may be invoked multiple times for the same event. Make your function idempotent.

> [!warning] Mistake 3 — Forgetting Batch Failure Reporting for SQS
> Without `ReportBatchItemFailures`, a single bad message in a batch causes the whole batch to be retried, blocking the queue.

> [!warning] Mistake 4 — Setting the Visibility Timeout Too Short for SQS Triggers
> Lambda's timeout for SQS triggers should be at least 6x the queue's visibility timeout. Otherwise, the function may run longer than the visibility timeout, causing duplicate processing.

> [!warning] Mistake 5 — Forgetting to Filter S3 Events
> Without a filter (prefix/suffix), S3 invokes Lambda for EVERY object created in the bucket. Use a prefix like `uploads/` to limit.

> [!warning] Mistake 6 — Not Using Reserved Concurrency for Critical Functions
> Without reserved concurrency, your function may be throttled if other functions consume all account concurrency.

## 9. Summary Checklist

- [ ] Lambda triggers: API Gateway, S3, DynamoDB Streams, SQS, SNS, EventBridge, Kinesis, CloudWatch, and more.
- [ ] S3 trigger: configure on the bucket; event has Records[].s3.bucket/object.
- [ ] DynamoDB Streams: event-source-mapping; event has Records[].dynamodb.NewImage/OldImage.
- [ ] SQS: event-source-mapping; Lambda deletes successful messages. Use ReportBatchItemFailures for partial failures.
- [ ] SNS: subscribe Lambda to the topic; event has Records[].Sns.Message.
- [ ] EventBridge: rule + target. Supports AWS service events, custom events, scheduled (cron/rate).
- [ ] Add `lambda:InvokeFunction` permission for S3/SNS/EventBridge triggers.
- [ ] All triggers are at-least-once; make functions idempotent.
