# 3. AWS Service Reference

> [!info] Chapter Context
> A comprehensive reference of AWS services, organized by category, with a one-paragraph description of each.

## 1. Compute

| Service | Description |
| :--- | :--- |
| **EC2** | Virtual machines. Pick an AMI (OS image), instance type (CPU/RAM), and run. The most flexible compute option. |
| **Lambda** | Serverless functions. Upload code; AWS runs it on events. Pay per invocation. 15-minute max duration. |
| **ECS** | Container orchestration (Docker). Run containers on EC2 or Fargate. |
| **EKS** | Managed Kubernetes. AWS manages the control plane; you manage (or use Fargate for) worker nodes. |
| **Fargate** | Serverless containers. Run ECS/EKS tasks without managing EC2 instances. Pay per task-second. |
| **Elastic Beanstalk** | PaaS. Upload code; AWS handles capacity, load balancing, scaling. |
| **Lightsail** | Simplified VPS-like service. For simple websites/apps. |
| **Batch** | Run batch computing workloads. Manages queues, compute environments, job definitions. |
| **Outposts** | Run AWS infrastructure in your own data center. |

---

## 2. Storage

| Service | Description |
| :--- | :--- |
| **S3** | Object storage. Infinite scale, 11 nines durability, 5 TB max object. The most-used AWS storage service. |
| **EBS** | Block storage for EC2. Like a virtual hard drive. Single-AZ. |
| **EFS** | Shared file storage (NFS). Multi-AZ; accessible from many EC2 instances. |
| **FSx** | Managed Windows / Lustre / NetApp / OpenZFS file systems. |
| **Glacier** | Cold archive storage. Cheap, slow retrieval (minutes to hours). |
| **Glacier Deep Archive** | Long-term archive. Cheapest, slowest (hours to retrieve). |
| **Storage Gateway** | On-premises storage backed by S3. File, volume, or tape interface. |
| **Backup** | Centralized backup management across AWS services. |

---

## 3. Databases

| Service | Description |
| :--- | :--- |
| **RDS** | Managed relational databases (MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Aurora). |
| **Aurora** | AWS's proprietary SQL. MySQL/PostgreSQL-compatible. Separated compute/storage, fast failover. |
| **Aurora Serverless** | Aurora with auto-scaling compute. Good for intermittent workloads. |
| **DynamoDB** | Managed NoSQL (key-value/document). Single-digit-ms latency at any scale. |
| **ElastiCache** | Managed Redis or Memcached. For caching, sessions, leaderboards. |
| **Redshift** | Data warehouse. Petabyte-scale analytics. |
| **Neptune** | Managed graph database. |
| **DocumentDB** | Managed MongoDB-compatible database. |
| **Timestream** | Time-series database. For IoT, metrics. |
| **QLDB** | Quantum ledger database. Immutable, verifiable. |

---

## 4. Networking

| Service | Description |
| :--- | :--- |
| **VPC** | Virtual Private Cloud. Your private network in AWS. |
| **Subnet** | A range of IPs in a VPC, in a specific AZ. |
| **Route 53** | DNS service + domain registrar. |
| **CloudFront** | CDN. Caches content at edge locations worldwide. |
| **ALB** | Application Load Balancer. L7 (HTTP/HTTPS). |
| **NLB** | Network Load Balancer. L4 (TCP/UDP). |
| **GWLB** | Gateway Load Balancer. L3. For third-party appliances. |
| **Direct Connect** | Dedicated network connection from your data center to AWS. |
| **Transit Gateway** | Connect multiple VPCs and on-premises networks. |
| **VPN** | Site-to-site or client VPN. |
| **VPC Peering** | Connect two VPCs. |
| **VPC Endpoints** | Access AWS services without traversing the internet. Gateway (free) for S3/DynamoDB; Interface (paid) for others. |

---

## 5. Security and Identity

| Service | Description |
| :--- | :--- |
| **IAM** | Identity and Access Management. Users, groups, roles, policies. |
| **IAM Identity Center** | AWS SSO. Federated login to multiple AWS accounts. |
| **Cognito** | User identity for web/mobile apps. Sign-up, sign-in, OAuth. |
| **KMS** | Key Management Service. Manage encryption keys. |
| **CloudHSM** | Hardware security modules for cryptographic operations. |
| **Secrets Manager** | Store and rotate secrets (database credentials, API keys). |
| **Parameter Store** | Store config and secrets (part of Systems Manager). |
| **WAF** | Web Application Firewall. Filter HTTP traffic. |
| **Shield** | DDoS protection. Standard (free) and Advanced (paid). |
| **GuardDuty** | Threat detection. ML-based analysis of CloudTrail, VPC Flow Logs, DNS logs. |
| **Macie** | Discover and protect sensitive data in S3. |
| **Inspector** | Automated vulnerability scanning of EC2 and ECR. |
| **Certificate Manager (ACM)** | Provision and manage TLS certificates. Free for use with AWS services. |
| **Directory Service** | Managed Active Directory. |

## 11. Common Patterns (One-Liners)

| Pattern | Services |
| :--- | :--- |
| Static website | S3 + CloudFront + Route 53 + ACM |
| Serverless API | API Gateway + Lambda + DynamoDB |
| Web app with auth | Cognito + API Gateway + Lambda + RDS |
| Image processing | S3 + Lambda + S3 (thumbnails) |
| Event-driven fan-out | SNS + SQS + Lambda |
| Multi-AZ HA | ALB + Auto Scaling + RDS Multi-AZ |
| Multi-region DR | Route 53 failover + cross-region replication |
| Real-time dashboard | Kinesis + Lambda + DynamoDB + API Gateway WebSocket |
| Long-running workflow | Step Functions + Lambda + SQS |
| Scheduled task | EventBridge cron + Lambda |
