## 1. Project Overview

- User submits a long URL.
- The service returns a short URL (e.g., `https://short.url/abc123`).
- Visiting the short URL redirects to the long URL.
- Track click counts.

## 2. DynamoDB Table

```bash
aws dynamodb create-table --table-name urls \
  --attribute-definitions AttributeName=short_code,AttributeType=S \
  --key-schema AttributeName=short_code,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST
```

Item:

```json
{
  "short_code": {"S": "abc123"},
  "long_url": {"S": "https://example.com/very/long/path"},
  "created_at": {"S": "2024-01-15T10:30:00Z"},
  "click_count": {"N": "0"}
}
```

The `short_code` is the partition key. Lookups are O(1).

## 3. The Create Lambda

```python
import boto3
import json
import os
import secrets
from datetime import datetime

dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table(os.environ['TABLE_NAME'])

def lambda_handler(event, context):
    body = json.loads(event['body'])
    long_url = body['url']
    
    # Generate a short code
    short_code = secrets.token_urlsafe(6)[:6]
    
    # Store in DynamoDB
    table.put_item(Item={
        'short_code': short_code,
        'long_url': long_url,
        'created_at': datetime.utcnow().isoformat(),
        'click_count': 0
    })
    
    return {
        'statusCode': 200,
        'body': json.dumps({
            'short_url': f'https://{event["headers"]["Host"]}/{short_code}',
            'short_code': short_code
        })
    }
```

### 3.1 Generating the Short Code

Options:

- `secrets.token_urlsafe(6)` — Random, may have collisions (rare with 6 chars).
- Hash the long URL (deterministic, but reveals the URL).
- Counter-based (atomic counter in DynamoDB).

For a real service, use a counter to guarantee uniqueness.

## 4. The Redirect Lambda

```python
import boto3
import os

dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table(os.environ['TABLE_NAME'])

def lambda_handler(event, context):
    short_code = event['pathParameters']['code']
    
    response = table.get_item(Key={'short_code': short_code})
    if 'Item' not in response:
        return {'statusCode': 404, 'body': 'Not found'}
    
    long_url = response['Item']['long_url']
    
    # Increment click count (fire-and-forget)
    table.update_item(
        Key={'short_code': short_code},
        UpdateExpression='ADD click_count :inc',
        ExpressionAttributeValues={':inc': 1}
    )
    
    return {
        'statusCode': 301,
        'headers': {'Location': long_url}
    }
```

## 8. Common Student Mistakes

> [!warning] Mistake 1 — Using a Hash of the URL as the Short Code
#  Hashes are longer than 6 chars and reveal the original URL. Use random generation.

> [!warning] Mistake 2 — Forgetting 301 Redirect
#  Use 301 (permanent) so browsers cache the redirect. 302 (temporary) means every visit hits your service.

> [!warning] Mistake 3 — Synchronous Click Count Increment
#  Incrementing on every redirect slows down the redirect. Consider using DynamoDB Streams + Lambda to batch increments.

> [!warning] Mistake 4 — No Collision Handling
#  If using random codes, handle collisions (retry with a new code).

> [!warning] Mistake 5 — Forgetting the Host Header
#  The Lambda needs to construct the short URL with the right host. Use `event['headers']['Host']` (or hardcode the domain).
