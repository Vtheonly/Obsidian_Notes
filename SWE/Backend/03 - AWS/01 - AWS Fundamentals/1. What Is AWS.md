## 1. What AWS Is

AWS is Amazon's cloud computing platform. Launched in 2006 with S3 (Simple Storage Service) and EC2 (Elastic Compute Cloud), it has grown into the dominant cloud provider, with about 30-32% market share (larger than Microsoft Azure and Google Cloud combined).

AWS offers over 200 services across categories including compute, storage, databases, networking, analytics, machine learning, security, and developer tools. Most services are managed — AWS handles the infrastructure, scaling, patching, and backups; you configure and use them.

### 1.1 The "Primitive" Services

The foundational AWS services (the ones everything else builds on) are:

- **EC2 (Elastic Compute Cloud)** — Virtual machines.
- **S3 (Simple Storage Service)** — Object storage.
- **VPC (Virtual Private Cloud)** — Private networks.
- **IAM (Identity and Access Management)** — Users, roles, permissions.
- **CloudWatch** — Monitoring and logging.

Once you understand these, the rest of AWS becomes much easier to learn.

### 1.2 How AWS Services Are Built

AWS services are themselves distributed systems running on AWS infrastructure. S3, for example, is a massive object store built on thousands of servers across multiple datacenters. When you `aws s3 cp file.txt s3://my-bucket/`, you are calling an HTTP API that routes to one of these servers.

This is why understanding distributed systems ([[00 - Foundations/2. Distributed Systems Basics]]) matters — every AWS service is one.

## 2. AWS History and Scale

| Year | Milestone |
| :--- | :--- |
| 2002 | AWS launched (initial offering: queue, storage). |
| 2006 | S3 and EC2 launched — the modern cloud is born. |
| 2009 | RDS (managed databases), VPC (private networks). |
| 2012 | DynamoDB (NoSQL), Glacier (cold storage). |

## 3. AWS Service Categories

### 3.1 Compute

- **EC2** — Virtual machines. You pick the size, OS, and run anything.
- **Lambda** — Serverless functions. You upload code; AWS runs it on events.
- **ECS** — Container orchestration (Docker). Like a managed Kubernetes for Docker.
- **EKS** — Managed Kubernetes.
- **Fargate** — Serverless containers (run ECS/EKS tasks without managing EC2 instances).
- **Beanstalk** — PaaS; upload code, AWS handles capacity, load balancing, scaling.

### 3.2 Storage

- **S3** — Object storage. Unlimited size, pay per GB stored and per request.
- **EBS (Elastic Block Store)** — Block storage attached to EC2 instances (like a virtual hard drive).
- **EFS (Elastic File System)** — Shared file storage (NFS) accessible from multiple EC2 instances.
- **FSx** — Managed Windows / Lustre / NetApp file systems.
- **Glacier** — Cold storage for archives (cheap, slow retrieval).
- **Storage Gateway** — On-premises storage backed by S3.

### 3.3 Databases

- **RDS** — Managed relational databases (MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Aurora).
- **Aurora** — AWS's proprietary high-performance MySQL/PostgreSQL-compatible database.
- **DynamoDB** — Managed NoSQL (key-value) database. Single-digit-millisecond latency at any scale.
- **ElastiCache** — Managed Redis and Memcached.
- **Neptune** — Graph database.
- **DocumentDB** — Managed MongoDB-compatible database.

### 3.4 Networking

- **VPC** — Virtual Private Cloud. Your private network in AWS.
- **Route 53** — DNS.
- **CloudFront** — CDN (Content Delivery Network).
- **ELB (Elastic Load Balancing)** — ALB, NLB, GWLB, CLB.
- **Direct Connect** — Dedicated network connection from your datacenter to AWS.
- **Transit Gateway** — Connect multiple VPCs and on-premises networks.

### 3.5 Security and Identity

- **IAM** — Users, groups, roles, policies.
- **Cognito** — User identity for web/mobile apps (sign-up, sign-in, OAuth).
- **KMS** — Key Management Service (encryption keys).
- **Secrets Manager** — Store and rotate secrets.
- **WAF** — Web Application Firewall.
- **Shield** — DDoS protection.
- **GuardDuty** — Threat detection.

### 3.6 Analytics

- **Athena** — Query S3 with SQL (serverless).
- **Redshift** — Data warehouse.
- **EMR** — Managed Hadoop/Spark.
- **Kinesis** — Real-time data streaming.
- **Glue** — ETL (extract, transform, load).
- **QuickSight** — BI dashboards.

### 3.7 Machine Learning

- **SageMaker** — Build, train, deploy ML models.
- **Comprehend** — NLP (text analysis).
- **Rekognition** — Image and video analysis.
- **Polly** — Text-to-speech.
- **Transcribe** — Speech-to-text.
- **Bedrock** — Foundation models (LLMs) as a service.

### 3.8 Developer Tools

- **CodeCommit** — Git hosting (like GitHub).
- **CodeBuild** — CI builds.
- **CodeDeploy** — Deployment automation.
- **CodePipeline** — CI/CD pipelines.
- **Cloud9** — Cloud IDE.
- **CloudFormation** — Infrastructure as Code.
- **CDK** — Infrastructure as Code in TypeScript/Python.

## 4. How You Interact with AWS

### 4.1 The AWS Management Console

A web GUI at `https://console.aws.amazon.com`. Good for exploration and one-off tasks. Not good for repeatable operations or automation.

### 4.2 The AWS CLI

A command-line tool that calls AWS APIs. Good for scripting and quick operations.

```bash
aws s3 ls
aws ec2 describe-instances
aws lambda invoke --function-name my-func output.json
```

Covered in detail in [[02 - AWS CLI and SDKs/1. The AWS CLI]].

### 4.3 SDKs

AWS provides SDKs for many languages: Python (Boto3), JavaScript, Java, Go, .NET, Ruby, C++. SDKs handle authentication, retries, and serialization.

```python
import boto3
s3 = boto3.client('s3')
s3.upload_file('file.txt', 'my-bucket', 'file.txt')
```

### 4.4 Infrastructure as Code

For production deployments, you describe your infrastructure as code (Terraform, CloudFormation, CDK) and apply it. This makes infrastructure reproducible, versionable, and auditable. Covered in [[14 - Infrastructure as Code/1. IaC Fundamentals]].

## 5. The AWS Free Tier

AWS offers a free tier to help you learn. There are three categories:

1. **12 months free** — Starts when you create your account. Examples: 750 hours/month of t2.micro EC2, 5 GB S3, 750 hours RDS.
2. **Always free** — Does not expire. Examples: 1 million Lambda requests/month, 25 GB DynamoDB.
3. **Trials** — Short-term trials of premium services (e.g., 30 days of GuardDuty).

> [!warning] Free Tier Is Not Unlimited
> The free tier has limits. Exceed them, and you pay. Set up a billing alarm on day one to alert you when your bill exceeds $1, $10, $50, etc.

## 6. Common Student Mistakes

> [!warning] Mistake 1 — Picking Services Before Understanding Requirements
> "Should I use Lambda or EC2?" depends on your workload. Understand the tradeoffs (covered in later chapters) before committing.

> [!warning] Mistake 2 — Leaving Resources Running
> Cloud is pay-as-you-go. A forgotten EC2 instance costs money 24/7. Use Auto Scaling Groups, lifecycle policies, and billing alarms.

> [!warning] Mistake 3 — Using the Root Account for Everything
> The root account has unlimited power. Lock it down (MFA, no access keys) and use IAM users or roles for daily work.

> [!warning] Mistake 4 — Ignoring the Free Tier
> Track your free tier usage. Many students are surprised by a $50 bill from a forgotten RDS instance.

> [!warning] Mistake 5 — Hardcoding Region
> AWS services are region-specific. `aws s3 ls` in `us-east-1` does not show buckets in `eu-west-1` (S3 is global, but most services are regional). Always know which region you are operating in.
