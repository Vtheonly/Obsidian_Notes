# 5. AWS Pricing and Cost Management

> [!info] Chapter Context
> Cloud is pay-as-you-go, which means costs can spiral out of control if you are not careful. This note covers AWS pricing models, the Free Tier, billing alarms, and strategies to keep costs under control.

## 1. AWS Pricing Models

AWS charges based on different models depending on the service:

### 1.1 Pay-as-You-Go (On-demand)

You pay per hour (or per second) of usage. No upfront commitment, no long-term contract. Examples: EC2 on-demand, S3 storage, Lambda invocations.

- **Pros:** Flexible, no commitment, easy to start.
- **Cons:** Most expensive per-unit pricing.

### 1.2 Reserved Instances (RI)

You commit to a specific instance type in a specific region for 1 or 3 years. In exchange, you get a discount of 30-72% vs. on-demand.

- **Standard RI** — Specific instance type, OS, AZ. Maximum discount.
- **Convertible RI** — Can change instance type, OS, AZ during the term. Smaller discount.
- **Scheduled RI** — Reserve for specific time windows (e.g., every weekday 9am-5pm).

### 1.3 Savings Plans

A more flexible alternative to RIs. You commit to a specific $/hour spend for 1 or 3 years. Applies to any instance family, size, OS, AZ, region.

- **Compute Savings Plans** — Apply to EC2, Fargate, Lambda.
- **EC2 Instance Savings Plans** — Apply to a specific instance family in a specific region.

### 1.4 Spot Instances

Spare EC2 capacity that AWS sells at up to 90% discount. The catch: AWS can reclaim the instance with 2 minutes' notice.

- **Use case** — Batch processing, distributed workloads, stateless web servers, CI/CD.
- **Do NOT use for** — Databases, long-running single-instance jobs that cannot be checkpointed.

### 1.5 Per-Request Pricing

Some services charge per request, not per hour:

- **Lambda** — $0.20 per million requests + compute time.
- **DynamoDB** — Per read/write capacity unit (or on-demand per request).
- **S3** — Per GB stored + per request (PUT, GET, etc.).
- **CloudWatch** — Per metric, per log ingested, per alarm.

### 1.6 Data Transfer Pricing

Data transfer costs are the #1 surprise for AWS newcomers:

- **Inbound data transfer is free.** (Uploading to S3, pulling data into EC2.)
- **Outbound data transfer to the internet is $0.09/GB.** (First 100 GB/month is free.)
- **Data transfer between AZs is $0.01/GB each way.**
- **Data transfer between regions is $0.02/GB.**
- **Data transfer within an AZ is free** (for most services).

> [!warning] Data Transfer Is the Hidden Cost
> Many architectures accidentally incur cross-AZ or cross-region data transfer costs. A web app that reads from a database in another AZ pays $0.01/GB each direction. For high-traffic apps, this can be hundreds of dollars per month.

## 2. The Free Tier

AWS offers three free tier categories:

### 2.1 12-Months Free

Starts when you create your first resource. Examples:

- 750 hours/month of t2.micro or t3.micro EC2 (enough to run one instance 24/7).
- 5 GB of S3 standard storage + 20,000 GET / 2,000 PUT requests.
- 750 hours/month of RDS (db.t2.micro or db.t3.micro).
- 25 GB of DynamoDB.
- 1 million Lambda requests/month + 400,000 GB-seconds of compute.

### 2.2 Always Free

Does not expire. Examples:

- 25 GB DynamoDB (for tables created after 2024).
- 1 million Lambda requests/month.
- 10 GB CloudWatch Logs.
- 5 GB S3 Glacier (long-term retrieval).

### 2.3 Trials

Short-term trials (7-30 days) of premium services. Examples:

- GuardDuty — 30-day trial.
- Inspector — 15-day trial.
- Detective — 30-day trial.

> [!tip] Set Up Billing Alarms on Day One
> The Free Tier is a great way to learn, but exceeding it costs money. Set up AWS Budgets to alert you at $1, $10, $50, and $100.

## 3. Cost Management Tools

### 3.1 AWS Billing Console

The Billing Console (`https://console.aws.amazon.com/billing`) shows:

- **Bills** — Monthly invoices.
- **Cost Explorer** — Visualize spending by service, region, tag.
- **Budgets** — Set spending thresholds and alerts.
- **Cost and Usage Report** — Detailed CSV of all usage.
- **Cost Categories** — Group spending by team, project, environment.

### 3.2 AWS Budgets

Create a budget and get notified when you exceed it:

```bash
aws budgets create-budget \
  --account-id 123456789012 \
  --budget 'BudgetName=MonthlyBudget,BudgetLimit={Amount=50,Unit=USD},BudgetType=COST,TimeUnit=MONTHLY' \
  --notifications-with-subscribers '[{"Notification":{"NotificationType=ACTUAL,ComparisonOperator=GREATER_THAN,Threshold=80},"Subscribers":[{"SubscriptionType=EMAIL,Address=alice@example.com}]}]'
```

### 3.3 Cost Allocation Tags

Tag resources with metadata (e.g., `Team: Backend`, `Environment: Production`). Cost Explorer can group spending by tag, showing which team or environment is consuming the budget.

### 3.4 AWS Cost Anomaly Detection

ML-based detection of unusual spending patterns. Free for the first 10,000 anomaly evaluations per month.

## 4. Cost Optimization Strategies

### 4.1 Right-Size Your Resources

Most EC2 instances are over-provisioned. Use AWS Compute Optimizer (ML-based recommendations) or CloudWatch metrics to identify under-utilized instances.

### 4.2 Use Reserved Instances or Savings Plans

For steady-state workloads (running 24/7/365), commit to RIs or Savings Plans for 30-72% savings.

### 4.3 Use Spot Instances for Flexible Workloads

Batch jobs, CI/CD, and other interruptible workloads can run on Spot for 90% off.

### 4.4 Auto Scale

Don't run 10 instances 24/7 if your traffic peaks at 10 and averages 2. Use Auto Scaling Groups to scale with demand.

### 4.5 Use Serverless Where Possible

Lambda, DynamoDB, and Fargate scale to zero — you pay only for actual usage. For low-traffic apps, serverless can be much cheaper than always-on EC2.

### 4.6 Lifecycle S3 Objects

Move old S3 objects to cheaper tiers automatically:

- Standard (first 30 days) → Standard-IA (30-90 days) → Glacier (90-365 days) → Glacier Deep Archive (365+ days).
- Each tier is cheaper but slower to retrieve.

### 4.7 Delete Unused Resources

EBS volumes attached to stopped instances, unattached EIPs, old snapshots, idle load balancers — these all cost money. Audit periodically.

```bash
# Find unattached EBS volumes
aws ec2 describe-volumes --filters Name=status,Values=available

# Find unattached EIPs
aws ec2 describe-addresses --query 'Addresses[?InstanceId==`null`]'
```

## 5. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting to Stop EC2 Instances
> EC2 instances cost money 24/7 they are running. Stop them when not in use. Use Auto Scaling or scheduled scaling.

> [!warning] Mistake 2 — Leaving RDS Running
> RDS instances are expensive (often $15-100+/month). Stop them when not in use (note: they auto-start after 7 days).

> [!warning] Mistake 3 — Forgetting Cross-AZ Data Transfer Costs
> Cross-AZ data transfer is $0.01/GB each way. For high-traffic multi-AZ apps, this adds up. Keep related services in the same AZ where possible.

> [!warning] Mistake 4 — Not Using Spot for Batch Jobs
> Batch processing, CI/CD, and other interruptible workloads should run on Spot. 90% savings.

> [!warning] Mistake 5 — Forgetting to Delete Old Snapshots
> EBS snapshots accumulate. Old snapshots of deleted volumes should be cleaned up. Use lifecycle policies.

> [!warning] Mistake 6 — No Billing Alarms
> Without alarms, you might not notice a runaway resource until the bill arrives. Set up AWS Budgets on day one.

> [!warning] Mistake 7 — Using On-Demand for Steady-State Workloads
> If your database runs 24/7/365, on-demand pricing is the most expensive option. Use Reserved Instances or Savings Plans.

- [ ] Pricing models: on-demand (flexible, expensive), RI (commit, save 30-72%), Savings Plans (commit $/hour, flexible), Spot (up to 90% off, can be reclaimed).
- [ ] Free Tier: 12-months free, always free, trials. Set up billing alarms.
- [ ] Data transfer: inbound free, outbound $0.09/GB, cross-AZ $0.01/GB, cross-region $0.02/GB.
- [ ] Tools: Billing Console, Cost Explorer, Budgets, Cost Allocation Tags, Cost Anomaly Detection.
- [ ] Optimization: right-size, use RIs/Savings Plans, use Spot, auto-scale, use serverless, lifecycle S3 objects, delete unused resources.
- [ ] Set up AWS Budgets on day one to avoid surprises.
