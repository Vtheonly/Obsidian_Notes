## 1. Installing Terraform

```bash
# macOS
brew install terraform

# Linux
curl -fsSL https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform

# Verify
terraform version
```

## 2. A Minimal Example

Create a directory with these files:

```hcl
# versions.tf
terraform {
  required_version = ">= 1.5"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
```

```hcl
# providers.tf
provider "aws" {
  region = "us-east-1"
}
```

```hcl
# main.tf
resource "aws_s3_bucket" "my_bucket" {
  bucket = "my-unique-bucket-name-12345"

  tags = {
    Name        = "My bucket"
    Environment = "Dev"
  }
}
```

```hcl
# outputs.tf
output "bucket_arn" {
  value = aws_s3_bucket.my_bucket.arn
}
```

Run:

```bash
terraform init      # download providers
terraform plan      # preview changes
terraform apply     # apply changes (prompts for confirmation)
terraform destroy   # tear down everything
```

## 3. Providers

A provider is a plugin that lets Terraform talk to a specific API (AWS, GCP, Azure, GitHub, etc.).

```hcl
provider "aws" {
  region = "us-east-1"
}

# Multiple AWS regions
provider "aws" {
  alias  = "west"
  region = "us-west-2"
}

resource "aws_s3_bucket" "west_bucket" {
  provider = aws.west
  bucket   = "my-west-bucket"
}
```

## 4. Resources

A resource is a piece of infrastructure (an S3 bucket, an EC2 instance, a Lambda function).

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c7217cdde317cfec"
  instance_type = "t3.micro"

  tags = {
    Name = "WebServer"
  }
}
```

The resource type is `aws_instance`. The local name is `web`. Reference it as `aws_instance.web`.

### 4.1 Resource Attributes

Resources have attributes you can read:

```hcl
output "instance_public_ip" {
  value = aws_instance.web.public_ip
}
```

### 4.2 Dependencies

Terraform auto-detects dependencies from references:

```hcl
resource "aws_eip" "ip" {
  instance = aws_instance.web.id   # implicit dependency
}
```

For explicit dependencies:

```hcl
resource "aws_instance" "web" {
  # ...
  depends_on = [aws_iam_role_policy.my_policy]
}
```

## 5. Variables

Variables let you parameterize your config.

```hcl
# variables.tf
variable "instance_count" {
  description = "Number of instances"
  type        = number
  default     = 1
}

variable "environment" {
  description = "Environment name"
  type        = string
  validation {
    condition     = contains(["dev", "staging", "prod"], var.environment)
    error_message = "Environment must be dev, staging, or prod."
  }
}

variable "tags" {
  description = "Tags to apply"
  type        = map(string)
  default     = {}
}
```

Use:

```hcl
resource "aws_instance" "web" {
  count         = var.instance_count
  instance_type = "t3.micro"
  tags          = merge(var.tags, { Name = "web-${count.index}" })
}
```

Set on the command line:

```bash
terraform apply -var="instance_count=3" -var="environment=prod"
```

Or in `terraform.tfvars`:

```hcl
# terraform.tfvars
instance_count = 3
environment    = "prod"
tags = {
  Team = "Backend"
  Owner = "Alice"
}
```

## 6. Outputs

Outputs are values you want to see after applying:

```hcl
output "instance_ips" {
  value       = aws_instance.web[*].public_ip
  description = "Public IPs of the web instances"
}

output "bucket_name" {
  value = aws_s3_bucket.my_bucket.id
  sensitive = false   # default; set true for secrets
}
```

```bash
terraform output                  # all outputs
terraform output instance_ips     # one output
terraform output -raw bucket_name # raw value (no quotes)
```

## 7. Data Sources

Data sources read existing resources (not created by Terraform).

```hcl
# Get the latest Amazon Linux 2 AMI
data "aws_ami" "amazon_linux" {
  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["al2023-ami-*-x86_64"]
  }
}

resource "aws_instance" "web" {
  ami           = data.aws_ami.amazon_linux.id
  instance_type = "t3.micro"
}
```

## 8. The `for_each` and `count` Meta-Arguments

### 8.1 `count`

```hcl
resource "aws_instance" "web" {
  count         = 3
  ami           = data.aws_ami.amazon_linux.id
  instance_type = "t3.micro"
  tags          = { Name = "web-${count.index}" }
}
# Creates aws_instance.web[0], aws_instance.web[1], aws_instance.web[2]
```

### 8.2 `for_each`

```hcl
resource "aws_s3_bucket" "buckets" {
  for_each = toset(["logs", "data", "backups"])
  bucket   = "my-${each.key}-bucket"
}
# Creates aws_s3_bucket.buckets["logs"], etc.
```

`for_each` is preferred for maps/sets; `count` for integer counts.

## 9. The `locals` Block

Compute values once and reuse:

```hcl
locals {
  common_tags = {
    Project     = "MyApp"
    Environment = var.environment
    ManagedBy   = "Terraform"
  }
  bucket_name = "myapp-${var.environment}-${data.aws_caller_identity.current.account_id}"
}

resource "aws_s3_bucket" "my_bucket" {
  bucket = local.bucket_name
  tags   = local.common_tags
}
```

## 10. State Management

### 10.1 Remote State on S3

```hcl
terraform {
  backend "s3" {
    bucket         = "my-terraform-state"
    key            = "myapp/terraform.tfstate"
    region         = "us-east-1"
    dynamodb_table = "terraform-locks"
    encrypt        = true
  }
}
```

### 10.2 Importing Existing Resources

If you have an S3 bucket created outside Terraform, import it:

```bash
terraform import aws_s3_bucket.my_bucket my-existing-bucket-name
```

Now Terraform tracks the bucket. Add the resource to your config so the state matches.

### 10.3 State Commands

```bash
terraform state list                    # list resources in state
terraform state show aws_s3_bucket.my_bucket   # show one resource
terraform state mv aws_s3_bucket.old aws_s3_bucket.new   # rename
terraform state rm aws_s3_bucket.my_bucket   # remove from state (without destroying)
```

## 11. Terraform with LocalStack

```hcl
provider "aws" {
  region                      = "us-east-1"
  access_key                  = "test"
  secret_key                  = "test"
  s3_use_path_style           = true
  skip_credentials_validation = true
  skip_metadata_api_check     = true
  skip_requesting_account_id  = true

  endpoints {
    s3       = "http://localhost:4566"
    dynamodb = "http://localhost:4566"
    lambda   = "http://localhost:4566"
    # ... add all services you use
  }
}
```

Or use `tflocal` (a wrapper that auto-configures):

```bash
pip install terraform-local
tflocal init
tflocal apply
```

## 12. Common Student Mistakes

> [!warning] Mistake 1 — Manual Changes Outside Terraform
> Manual changes cause state drift. Always use Terraform (or `terraform import` existing resources).

> [!warning] Mistake 2 — Local State for Team Projects
> Use remote state (S3 + DynamoDB) for any team project.

> [!warning] Mistake 3 — Committing `.tfstate` to Git
> State contains secrets. Add to `.gitignore`.

> [!warning] Mistake 4 — Forgetting `terraform init` After Adding a Provider
> Each new provider requires `terraform init` to download the plugin.

> [!warning] Mistake 5 — Hardcoding AMI IDs
> AMI IDs change over time and differ per region. Use a `data "aws_ami"` source to look up the latest.

> [!warning] Mistake 6 — Not Pinning Provider Versions
> Use `version = "~> 5.0"` to avoid breaking changes from major version bumps.

> [!warning] Mistake 7 — Applying Directly to Production
> Always run `plan` first. Review the plan. Use a CI/CD pipeline that requires approval for production.

## 13. Summary Checklist

- [ ] Install Terraform; create `.tf` files; run `init`, `plan`, `apply`.
- [ ] Providers (aws, gcp, etc.) are plugins.
- [ ] Resources (`resource "aws_s3_bucket" "name"`) define infrastructure.
- [ ] Variables parameterize; outputs expose values; data sources read existing resources.
- [ ] `count` for integer counts; `for_each` for maps/sets.
- [ ] `locals` for computed values.
- [ ] Remote state (S3 + DynamoDB) for teams; never commit state to Git.
- [ ] `terraform import` to bring existing resources under Terraform management.
- [ ] Pin provider versions; use `data "aws_ami"` instead of hardcoding AMIs.
- [ ] Always `plan` before `apply`.
