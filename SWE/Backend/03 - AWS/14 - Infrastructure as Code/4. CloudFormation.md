# 4. CloudFormation

> [!info] Chapter Context
> AWS CloudFormation is the native AWS IaC service. While Terraform and CDK are more popular, CloudFormation underlies CDK and is used by AWS services internally. This note covers CloudFormation templates, stacks, drift detection, and StackSets.

Related: [[3. AWS CDK]] | [[2. Terraform Fundamentals]] | [[1. IaC Fundamentals]]

---

## 1. CloudFormation Concepts

- **Template** — A JSON or YAML file describing resources.
- **Stack** — A deployed instance of a template. CloudFormation creates, updates, and deletes resources as a unit.
- **StackSet** — A stack deployed across multiple accounts/regions.
- **Change set** — A preview of changes before applying.
- **Drift detection** — Detects manual changes outside CloudFormation.

---

## 2. A Minimal Template

```yaml
# template.yaml
AWSTemplateFormatVersion: '2010-09-09'
Description: My first CloudFormation template

Parameters:
  BucketName:
    Type: String
    Description: Name for the S3 bucket
    Default: my-bucket

Resources:
  MyBucket:
    Type: AWS::S3::Bucket
    Properties:
      BucketName: !Ref BucketName
      VersioningConfiguration:
        Status: Enabled
      Tags:
        - Key: Environment
          Value: !Ref Environment

Outputs:
  BucketArn:
    Description: ARN of the created bucket
    Value: !GetAtt MyBucket.Arn

Mappings:
  RegionMap:
    us-east-1:
      AMI: ami-0c7217cdde317cfec
    us-west-2:
      AMI: ami-0c7217cdde317cfec

Conditions:
  IsProd: !Equals [!Ref Environment, prod]

Parameters:
  Environment:
    Type: String
    AllowedValues: [dev, staging, prod]
    Default: dev
```

---

## 3. Template Sections

- **AWSTemplateFormatVersion** — Always `2010-09-09`.
- **Description** — Free text.
- **Parameters** — Input values (prompt at deploy time).
- **Mappings** — Lookup tables.
- **Conditions** — Conditional resource creation.
- **Resources** — The actual AWS resources to create. **Required.**
- **Outputs** — Values to display after deployment.
- **Metadata** — Optional metadata.

## 4. Deploying a Stack

### 4.1 With the CLI

```bash
# Validate
aws cloudformation validate-template --template-body file://template.yaml

# Create
aws cloudformation create-stack \
  --stack-name my-stack \
  --template-body file://template.yaml \
  --parameters ParameterKey=BucketName,ParameterValue=my-bucket-12345 \
  --capabilities CAPABILITY_NAMED_IAM

# Update
aws cloudformation update-stack \
  --stack-name my-stack \
  --template-body file://template.yaml \
  --parameters ParameterKey=BucketName,ParameterValue=my-bucket-12345

# Delete
aws cloudformation delete-stack --stack-name my-stack

# List stacks
aws cloudformation describe-stacks

# Watch stack events
aws cloudformation describe-stack-events --stack-name my-stack
```

### 4.2 With a Change Set

Preview changes before applying:

```bash
aws cloudformation create-change-set \
  --stack-name my-stack \
  --change-set-name my-change \
  --template-body file://template.yaml

aws cloudformation describe-change-set \
  --stack-name my-stack \
  --change-set-name my-change

# If approved:
aws cloudformation execute-change-set \
  --stack-name my-stack \
  --change-set-name my-change
```

## 5. Intrinsic Functions

CloudFormation has built-in functions (prefixed with `!`):

- `!Ref` — Reference a parameter or resource.
- `!GetAtt` — Get an attribute of a resource.
- `!Sub` — String substitution.
- `!Join` — Join strings.
- `!Select` — Pick from a list.
- `!FindInMap` — Look up in a Mapping.
- `!If` — Conditional.
- `!Equals`, `!Not`, `!And`, `!Or` — Logic.
- `!Base64` — Base64-encode.
- `!ImportValue` — Import an exported output from another stack.

```yaml
# Examples
BucketName: !Ref BucketNameParameter
BucketArn: !GetAtt MyBucket.Arn
Description: !Sub "Bucket ${BucketName} in ${AWS::Region}"
AllBuckets: !Join ["-", ["my", "bucket", "name"]]
```

## 6. Cross-Stack References

Export a value from one stack, import in another:

```yaml
# Stack A
Outputs:
  VpcId:
    Value: !Ref MyVPC
    Export:
      Name: my-vpc-id
```

```yaml
# Stack B
Resources:
  MySubnet:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !ImportValue my-vpc-id
      CidrBlock: 10.0.1.0/24
```

> [!warning] Cross-Stack Exports Cannot Be Deleted While Imported
> If stack B imports `my-vpc-id`, you cannot delete the export from stack A until you remove the import from stack B.

## 7. Nested Stacks

A stack can contain other stacks:

```yaml
Resources:
  VpcStack:
    Type: AWS::CloudFormation::Stack
    Properties:
      TemplateURL: https://s3.amazonaws.com/my-bucket/vpc.yaml
      Parameters:
        CidrBlock: 10.0.0.0/16

  DbStack:
    Type: AWS::CloudFormation::Stack
    Properties:
      TemplateURL: https://s3.amazonaws.com/my-bucket/db.yaml
      Parameters:
        VpcId: !GetAtt VpcStack.Outputs.VpcId
```

Use nested stacks to break large templates into manageable pieces.

## 8. Drift Detection

Drift = changes made outside CloudFormation (via Console or CLI).

```bash
aws cloudformation detect-stack-drift --stack-name my-stack
# Returns a detection ID

aws cloudformation describe-stack-drift-detection-status \
  --stack-drift-detection-id abc123
```

Drift detection shows which resources have drifted from the template. Use it periodically to catch manual changes.

---

## 9. StackSets

Deploy a stack across multiple accounts and/or regions in one operation.

```bash
aws cloudformation create-stack-set \
  --stack-set-name my-stack-set \
  --template-body file://template.yaml \
  --capabilities CAPABILITY_NAMED_IAM

aws cloudformation create-stack-instances \
  --stack-set-name my-stack-set \
  --accounts 123456789012 210987654321 \
  --regions us-east-1 eu-west-1
```

Use StackSets with AWS Organizations for enterprise-wide deployments (e.g., a baseline IAM role in every account).

## 10. CloudFormation vs. Terraform

| Feature | CloudFormation | Terraform |
| :--- | :--- | :--- |
| Cloud | AWS only | Multi-cloud |
| Language | YAML/JSON | HCL |
| State | Managed by AWS | You manage |
| New AWS services | Same-day support | Days/weeks lag |
| Cost | Free | Free (Terraform) / paid (Terraform Cloud) |
| Drift detection | Built-in | Manual (terrafom plan) |
| Multi-account | StackSets | Manual (or Terraform Cloud) |

Use CloudFormation when:
- You're AWS-only and want native support.
- You need same-day support for new AWS services.
- You want StackSets for multi-account.
- Your organization mandates it.

Use Terraform when:
- You're multi-cloud.
- You want a simpler DSL.
- You want flexibility (e.g., custom providers).

## 11. Common Student Mistakes

> [!warning] Mistake 1 — Manual Changes Outside CloudFormation
> Manual changes cause drift. Use CloudFormation (or `cdk deploy`) for changes.

> [!warning] Mistake 2 — Forgetting `--capabilities CAPABILITY_NAMED_IAM`
> Templates that create IAM resources require this flag. Without it, the deployment fails.

> [!warning] Mistake 3 — Deleting an Export While Imported
> Cannot delete an exported output while another stack imports it. Remove imports first.

> [!warning] Mistake 4 — Hardcoding ARNs/IDs
> Use `!Ref` and `!GetAtt` instead of hardcoding.

> [!warning] Mistake 5 — Not Using Change Sets for Production
> Change sets let you preview before applying. Use them for production updates.

> [!warning] Mistake 6 — Templates Too Large
> CloudFormation templates have a 1 MB limit (460 KB for direct upload). Use nested stacks or upload to S3 first.
