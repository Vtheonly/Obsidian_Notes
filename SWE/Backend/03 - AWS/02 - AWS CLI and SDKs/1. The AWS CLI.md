# 1. The AWS CLI

> [!info] Chapter Context
> The AWS CLI is your primary tool for interacting with AWS from the command line. This note covers installation, configuration, profiles, common commands, and the conventions that apply across all AWS services.

Related: [[01 - AWS Fundamentals/1. What Is AWS]] | [[2. Boto3 and AWS SDKs]] | [[3. Credentials and Profiles]] | [[06 - LocalStack/1. Installing LocalStack]]

## 1. Why the CLI Matters

The AWS Management Console is fine for one-off tasks, but for automation, scripting, and Infrastructure as Code, you need the CLI. The CLI:

- Calls AWS APIs directly via HTTP.
- Is scriptable (use in shell scripts, CI/CD pipelines).
- Supports all AWS services (the Console sometimes lags).
- Outputs JSON (by default), text, or table format.
- Is the foundation for tools like `aws-local` (for LocalStack).

## 2. Installation

### 2.1 Linux

```bash
# Official installer (recommended)
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Or via apt (Ubuntu)
sudo apt update && sudo apt install -y awscli
```

### 2.2 macOS

```bash
# Via Homebrew
brew install awscli

# Or official installer
curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
sudo installer -pkg AWSCLIV2.pkg -target /
```

### 2.3 Windows

Download the official MSI installer from <https://aws.amazon.com/cli/>, or:

```powershell
# Via winget
winget install Amazon.AWSCLI

# Via Chocolatey
choco install awscli
```

### 2.4 Verify

```bash
aws --version
# aws-cli/2.15.0 Python/3.11 Linux/6.5.0 source/x86_64.ubuntu.22
```

## 3. Configuration: `aws configure`

```bash
aws configure
```

Prompts for:

- **AWS Access Key ID** — From IAM (or LocalStack's `test` for local development).
- **AWS Secret Access Key** — From IAM.
- **Default region name** — e.g., `us-east-1`.
- **Default output format** — `json`, `text`, `table`, or `yaml` (default: `json`).

This creates two files:

- `~/.aws/credentials` — Access key and secret (for the default profile).
- `~/.aws/config` — Region, output format, and other settings.

### 3.1 The `~/.aws/credentials` File

```ini
[default]
aws_access_key_id = AKIAIOSFODNN7EXAMPLE
aws_secret_access_key = wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

[production]
aws_access_key_id = AKIAPRODACCESSEXAMPLE
aws_secret_access_key = prodsecretkeyexample
```

### 3.2 The `~/.aws/config` File

```ini
[default]
region = us-east-1
output = json

[profile production]
region = us-west-2
output = table

[profile localstack]
region = us-east-1
endpoint_url = http://localhost:4566
```

### 3.3 Using Profiles

```bash
# Use the default profile
aws s3 ls

# Use a named profile
aws s3 ls --profile production

# Set the profile for the current shell
export AWS_PROFILE=production
aws s3 ls
```

## 4. Common CLI Conventions

### 4.1 The Command Structure

```bash
aws <service> <operation> [--parameters]
```

Examples:

```bash
aws s3 ls                                   # list buckets
aws s3 cp file.txt s3://my-bucket/          # upload a file
aws ec2 describe-instances                  # list EC2 instances
aws lambda list-functions                   # list Lambda functions
aws dynamodb list-tables                    # list DynamoDB tables
```

### 4.2 Output Formats

```bash
aws s3 ls --output json                     # JSON (default)
aws s3 ls --output text                     # tab-separated text
aws s3 ls --output table                    # ASCII table
aws s3 ls --output yaml                     # YAML
```

### 4.3 Querying with `--query`

JMESPath query language to filter and transform output:

```bash
# Get just the bucket names
aws s3api list-buckets --query 'Buckets[*].Name'

# Get instance IDs and their state
aws ec2 describe-instances \
  --query 'Reservations[*].Instances[*].[InstanceId,State.Name]' \
  --output text

# Filter running instances
aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running" \
  --query 'Reservations[*].Instances[*].InstanceId'
```

### 4.4 Pagination

APIs that return lists paginate. The CLI handles this automatically with `--no-paginate` (default) or `--paginate`. For custom page size:

```bash
aws s3api list-objects --bucket my-bucket --page-size 100
```

### 4.5 Waiters

Some commands have a `wait` subcommand that blocks until a condition is met:

```bash
aws ec2 wait instance-running --instance-ids i-1234567890
aws s3 wait bucket-exists --bucket my-bucket
```

## 5. Common Service Commands

### 5.1 S3 (high-level)

```bash
aws s3 ls                                   # list buckets
aws s3 ls s3://my-bucket/                   # list objects in a bucket
aws s3 mb s3://my-new-bucket                # make a bucket
aws s3 rb s3://my-bucket                    # remove a bucket
aws s3 cp file.txt s3://my-bucket/          # upload
aws s3 cp s3://my-bucket/file.txt .         # download
aws s3 sync ./local-dir s3://my-bucket/     # sync directories
aws s3 presign s3://my-bucket/file.txt      # generate a presigned URL
aws s3 rm s3://my-bucket/file.txt           # delete an object
```

### 5.2 EC2

```bash
aws ec2 run-instances \
  --image-id ami-0c7217cdde317cfec \
  --instance-type t2.micro \
  --key-name my-key \
  --security-group-ids sg-12345 \
  --subnet-id subnet-12345

aws ec2 describe-instances
aws ec2 start-instances --instance-ids i-12345
aws ec2 stop-instances --instance-ids i-12345
aws ec2 terminate-instances --instance-ids i-12345
```

### 5.3 IAM

```bash
aws iam list-users
aws iam create-user --user-name alice
aws iam create-access-key --user-name alice
aws iam attach-user-policy --user-name alice --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess
```

### 5.4 DynamoDB

```bash
aws dynamodb list-tables
aws dynamodb create-table --table-name Users \
  --attribute-definitions AttributeName=id,AttributeType=N \
  --key-schema AttributeName=id,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST

aws dynamodb put-item --table-name Users --item '{"id":{"N":"1"},"name":{"S":"Alice"}}'
aws dynamodb get-item --table-name Users --key '{"id":{"N":"1"}}'
aws dynamodb scan --table-name Users
```

### 5.5 Lambda

```bash
aws lambda list-functions
aws lambda create-function --function-name my-func \
  --runtime python3.11 --handler index.handler \
  --role arn:aws:iam::123456789012:role/lambda-role \
  --zip-file fileb://function.zip

aws lambda invoke --function-name my-func output.json
aws lambda logs my-func  # (uses CloudWatch Logs)
```

## 6. Using the CLI with LocalStack

LocalStack simulates AWS APIs locally. To point the CLI at LocalStack, set the endpoint URL:

```bash
# One-off
aws --endpoint-url=http://localhost:4566 s3 ls

# For a profile (recommended)
aws configure set region us-east-1 --profile localstack
aws configure set endpoint_url http://localhost:4566 --profile localstack

# Now use the profile
aws s3 ls --profile localstack
```

Or install `awslocal` (a wrapper that adds the endpoint URL automatically):

```bash
pip install awscli-local
awslocal s3 ls
```

Covered in detail in [[06 - LocalStack/1. Installing LocalStack]].

## 7. Shell Completion

The CLI supports shell completion for commands and resource names:

```bash
# Bash
echo "complete -C '/usr/local/bin/aws_completer' aws" >> ~/.bashrc
source ~/.bashrc

# Zsh
echo "source /usr/local/bin/aws_zsh_completer.sh" >> ~/.zshrc

# Fish
echo "complete --command aws --no-files --arguments '(begin; set --local --export COMP_SHELL fish; set --local --export COMP_LINE (commandline); aws; end)'" >> ~/.config/fish/config.fish
```

After enabling, type `aws s3` and press Tab to see subcommands.

## 8. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting to Set the Region
> Without a region, many commands fail with "Could not connect to the endpoint URL." Set a default region with `aws configure` or `AWS_DEFAULT_REGION`.

> [!warning] Mistake 2 — Using the Wrong Profile
> If you have multiple profiles, `aws s3 ls` uses the default. Use `--profile production` or `AWS_PROFILE=production`.

> [!warning] Mistake 3 — Hardcoding Access Keys in Scripts
> Use IAM roles (on EC2/ECS/Lambda) or environment variables. Never hardcode access keys in scripts that you commit to Git.

> [!warning] Mistake 4 — Forgetting That S3 Bucket Names Are Global
> S3 bucket names are globally unique. If `aws s3 mb s3://my-bucket` fails with "Bucket already exists," someone else owns that name. Use a unique name (e.g., `my-bucket-12345`).

> [!warning] Mistake 5 — Using `aws s3` vs `aws s3api`
> `aws s3` is the high-level interface (simpler, fewer options). `aws s3api` is the low-level interface (more options, matches the API exactly). Use `aws s3` for common tasks; `aws s3api` for advanced operations.

> [!warning] Mistake 6 — Forgetting `fileb://` for Binary Files
> When uploading a binary file (e.g., a Lambda zip), use `fileb://path` instead of `file://path`. `file://` reads as text and corrupts binary data.

## 9. Summary Checklist

- [ ] Install with `curl ... && sudo ./aws/install` (Linux), `brew install awscli` (Mac), or MSI (Windows).
- [ ] `aws configure` creates `~/.aws/credentials` and `~/.aws/config`.
- [ ] Profiles let you switch between AWS accounts/environments.
- [ ] Command structure: `aws <service> <operation> [--parameters]`.
- [ ] Output formats: `json` (default), `text`, `table`, `yaml`.
- [ ] `--query` uses JMESPath to filter output.
- [ ] High-level (`aws s3`) vs low-level (`aws s3api`) commands.
- [ ] For LocalStack, use `--endpoint-url=http://localhost:4566` or the `localstack` profile.
- [ ] Enable shell completion for productivity.
- [ ] Never hardcode access keys; use roles or env vars.
