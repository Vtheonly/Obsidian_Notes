# 2. Boto3 and AWS SDKs

> [!info] Chapter Context
> While the CLI is for shell scripts, SDKs let you call AWS from application code. Boto3 is the Python SDK and is the most popular. This note covers Boto3 patterns that apply to all AWS SDKs.

## 1. Why SDKs

The CLI is great for shell scripts, but for application code (a Python web app calling S3, a Node.js service querying DynamoDB), you need a programmatic interface. AWS provides SDKs for many languages:

- **Python** — `boto3`
- **JavaScript/TypeScript** — `aws-sdk` (v2) or `@aws-sdk/client-*` (v3, modular)
- **Java** — `aws-sdk-java`
- **Go** — `aws-sdk-go-v2`
- **.NET** — `AWSSDK`
- **Ruby** — `aws-sdk-ruby`
- **PHP** — `aws-sdk-php`
- **Rust** — `aws-sdk-rust`

All SDKs share concepts: clients, resources, credential resolution, retries, pagination. We use Boto3 (Python) for examples because it is the most readable.

## 3. Clients and Resources

Boto3 has two APIs:

### 3.1 Client (Low-Level)

Maps 1:1 to the AWS API. Returns dicts.

```python
import boto3

s3 = boto3.client('s3')
response = s3.list_buckets()
for bucket in response['Buckets']:
    print(bucket['Name'])
```

### 3.2 Resource (High-Level, Object-Oriented)

Wraps the client with a more Pythonic interface.

```python
import boto3

s3 = boto3.resource('s3')
for bucket in s3.buckets.all():
    print(bucket.name)

bucket = s3.Bucket('my-bucket')
bucket.upload_file('local.txt', 'remote.txt')
```

> [!tip] Use Resources for Simplicity, Clients for Control
> Resources are easier to read and write. Clients give you full control (every API operation, every parameter). Some services only have clients (e.g., Step Functions, EventBridge). When in doubt, use the client.

## 4. Credential Resolution

Boto3 searches for credentials in this order:

1. **Parameters** — `boto3.client('s3', aws_access_key_id=..., aws_secret_access_key=...)`.
2. **Environment variables** — `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`.
3. **Shared credentials file** — `~/.aws/credentials`.
4. **AWS config file** — `~/.aws/config`.
5. **Assume Role provider** — From `~/.aws/config`'s `role_arn` and `source_profile`.
6. **Boto2 config** — Legacy.
7. **IAM role** — On EC2 instances, ECS tasks, Lambda functions (via the metadata service).

You rarely set credentials explicitly. On your laptop, you use the shared credentials file. On EC2, you attach an IAM role to the instance and Boto3 picks it up automatically.

### 4.1 With LocalStack

```python
import boto3
import os

# Option 1: Set endpoint_url in code
s3 = boto3.client('s3', endpoint_url='http://localhost:4566')

# Option 2: Set environment variable (affects all clients)
os.environ['AWS_ENDPOINT_URL'] = 'http://localhost:4566'
s3 = boto3.client('s3')

# Option 3: Use a profile with endpoint_url set in ~/.aws/config
session = boto3.Session(profile_name='localstack')
s3 = session.client('s3')
```

For LocalStack, the access key and secret can be anything (e.g., `test`).

## 5. Sessions

A `Session` is a container for configuration (credentials, region). Use sessions when working with multiple profiles or regions:

```python
import boto3

# Default session (uses environment / config)
default_s3 = boto3.client('s3')

# Named profile session
prod_session = boto3.Session(profile_name='production')
prod_s3 = prod_session.client('s3')

# Explicit credentials
explicit_session = boto3.Session(
    aws_access_key_id='AKIA...',
    aws_secret_access_key='...',
    region_name='us-west-2'
)
explicit_s3 = explicit_session.client('s3')
```

## 6. Common Patterns

### 6.1 S3 Upload and Download

```python
import boto3

s3 = boto3.client('s3')

# Upload a file
s3.upload_file('local.txt', 'my-bucket', 'remote.txt')

# Download a file
s3.download_file('my-bucket', 'remote.txt', 'local.txt')

# Upload from a file-like object
with open('local.txt', 'rb') as f:
    s3.upload_fileobj(f, 'my-bucket', 'remote.txt')

# Generate a presigned URL (download)
url = s3.generate_presigned_url(
    'get_object',
    Params={'Bucket': 'my-bucket', 'Key': 'remote.txt'},
    ExpiresIn=3600
)
```

### 6.2 DynamoDB

```python
import boto3

dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('Users')

# Put an item
table.put_item(Item={'id': '1', 'name': 'Alice', 'age': 30})

# Get an item
response = table.get_item(Key={'id': '1'})
print(response['Item'])

# Query by partition key
response = table.query(
    KeyConditionExpression='id = :id',
    ExpressionAttributeValues={':id': '1'}
)
for item in response['Items']:
    print(item)

# Scan the entire table (slow on large tables)
response = table.scan()
for item in response['Items']:
    print(item)
```

### 6.3 Lambda Invocation

```python
import boto3
import json

lambda_client = boto3.client('lambda')

response = lambda_client.invoke(
    FunctionName='my-function',
    InvocationType='RequestResponse',  # synchronous
    Payload=json.dumps({'key': 'value'})
)
result = json.loads(response['Payload'].read())
print(result)
```

### 6.4 SNS Publish

```python
import boto3

sns = boto3.client('sns')

sns.publish(
    TopicArn='arn:aws:sns:us-east-1:123456789012:my-topic',
    Subject='Alert',
    Message='Something happened!'
)
```

### 6.5 SQS Send and Receive

```python
import boto3

sqs = boto3.client('sqs')

# Send
sqs.send_message(
    QueueUrl='https://sqs.us-east-1.amazonaws.com/123456789012/my-queue',
    MessageBody='Hello, world!'
)

# Receive (long polling)
response = sqs.receive_message(
    QueueUrl='https://sqs.us-east-1.amazonaws.com/123456789012/my-queue',
    MaxNumberOfMessages=10,
    WaitTimeSeconds=20
)
for message in response.get('Messages', []):
    print(message['Body'])
    sqs.delete_message(
        QueueUrl='https://sqs.us-east-1.amazonaws.com/123456789012/my-queue',
        ReceiptHandle=message['ReceiptHandle']
    )
```

## 7. Pagination

APIs that return lists paginate. Boto3 provides a `Paginator` for each:

```python
import boto3

s3 = boto3.client('s3')

# Manual (you handle pagination)
response = s3.list_objects_v2(Bucket='my-bucket')
while True:
    for obj in response.get('Contents', []):
        print(obj['Key'])
    if response.get('IsTruncated'):
        response = s3.list_objects_v2(
            Bucket='my-bucket',
            ContinuationToken=response['NextContinuationToken']
        )
    else:
        break

# Automatic (with Paginator)
paginator = s3.get_paginator('list_objects_v2')
for page in paginator.paginate(Bucket='my-bucket'):
    for obj in page.get('Contents', []):
        print(obj['Key'])
```

Always use Paginators for list operations.

## 8. Waiters

Some operations are asynchronous (e.g., creating an EC2 instance, an RDS cluster). Waiters poll until the resource reaches the desired state:

```python
import boto3

ec2 = boto3.client('ec2')

# Launch an instance
response = ec2.run_instances(
    ImageId='ami-0c7217cdde317cfec',
    InstanceType='t2.micro',
    MinCount=1,
    MaxCount=1
)
instance_id = response['Instances'][0]['InstanceId']

# Wait for it to be running
waiter = ec2.get_waiter('instance_running')
waiter.wait(InstanceIds=[instance_id])
print(f'{instance_id} is running')
```

## 9. Error Handling

```python
import boto3
from botocore.exceptions import ClientError, NoCredentialsError

s3 = boto3.client('s3')

try:
    s3.head_bucket(Bucket='my-bucket')
except ClientError as e:
    error_code = e.response['Error']['Code']
    if error_code == '404':
        print('Bucket does not exist')
    elif error_code == '403':
        print('Access denied')
    else:
        raise
except NoCredentialsError:
    print('No AWS credentials found')
```

Common error patterns:

- `ClientError` — The AWS API returned an error. Inspect `e.response['Error']['Code']`.
- `NoCredentialsError` — No credentials found.
- `EndpointConnectionError` — Cannot reach AWS (or LocalStack).

## 10. Retries and Exponential Backoff

Boto3 retries failed requests automatically (default: 3 retries with exponential backoff). For more control:

```python
from botocore.config import Config

config = Config(
    retries={'max_attempts': 10, 'mode': 'adaptive'}
)
s3 = boto3.client('s3', config=config)
```

For throttled requests (429) or service errors (5xx), implement your own backoff:

```python
import boto3
import time
from botocore.exceptions import ClientError

def call_with_retry(operation, **kwargs):
    for attempt in range(5):
        try:
            return operation(**kwargs)
        except ClientError as e:
            if e.response['Error']['Code'] in ['Throttling', 'ServiceUnavailable']:
                time.sleep(2 ** attempt)
            else:
                raise
    raise Exception('Max retries exceeded')
```

## 11. Common Student Mistakes

> [!warning] Mistake 1 — Hardcoding Credentials
> Use IAM roles on EC2/ECS/Lambda. On your laptop, use the shared credentials file. Never hardcode.

> [!warning] Mistake 2 — Not Using Paginators
> `list_objects_v2` returns up to 1000 objects. For more, you must paginate. Use the Paginator API.

> [!warning] Mistake 3 — Forgetting to Handle Errors
> Network errors, throttling, and permission errors happen. Wrap AWS calls in try/except and handle them.

> [!warning] Mistake 4 — Using the Wrong Region
> A DynamoDB table in `us-east-1` is not visible from `eu-west-1`. Always set the region.

> [!warning] Mistake 5 — Forgetting to Set the Endpoint for LocalStack
> If using Boto3 with LocalStack, set `endpoint_url='http://localhost:4566'`. Without it, Boto3 calls real AWS.

> [!warning] Mistake 6 — Calling AWS APIs in a Hot Loop
> Each AWS API call takes 50-500 ms. Calling S3 in a tight loop is slow. Batch operations where possible.
