# 1. Users and Groups

> [!info] Chapter Context
> Linux is a multi-user system. Every file and process belongs to a **user** and a **group**. This note covers how users and groups work, how to create and modify them, and the important files (`/etc/passwd`, `/etc/shadow`, `/etc/group`).

Related: [[02 - File System and Permissions/2. Permissions and Ownership]] | [[05 - Users and Security/2. sudo and Privilege Escalation]] | [[05 - Users and Security/3. SSH and Key Management]]

## 1. Users

A **user** is an account that can log in and run processes. Every user has:

- A **username** (e.g., `alice`).
- A **UID** (User ID, an integer). The kernel uses UIDs, not usernames.
- A **primary group** (GID).
- A **home directory** (e.g., `/home/alice`).
- A **login shell** (e.g., `/bin/bash`).
- Optionally, a password (stored hashed in `/etc/shadow`).

### 1.1 Special Users

| UID | User | Purpose |
| :--- | :--- | :--- |
| 0 | root | The superuser. Can do anything. |
| 1-99 | system | System services (daemon, bin, mail, etc.). |
| 100-999 | system | More system services (varies by distro). |
| 1000+ | regular | Regular user accounts. |

### 1.2 The `/etc/passwd` File

Every user has a line in `/etc/passwd`:

```
alice:x:1000:1000:Alice Smith:/home/alice:/bin/bash
```

Fields (colon-separated):

1. `alice` — Username.
2. `x` — Password placeholder (the actual hash is in `/etc/shadow`).
3. `1000` — UID.
4. `1000` — GID of the primary group.
5. `Alice Smith` — GECOS field (full name, comments).
6. `/home/alice` — Home directory.
7. `/bin/bash` — Login shell.

> [!note] `/etc/passwd` Is World-Readable
> Despite the name, `/etc/passwd` does not contain passwords — only an `x` placeholder. The actual hashes are in `/etc/shadow`, which is only readable by root. `/etc/passwd` must be world-readable because many tools (`ls -l`, `ps`) need to map UIDs to usernames.

### 1.3 The `/etc/shadow` File

```
alice:$6$xyz...:19000:0:99999:7:::
```

Fields:

1. `alice` — Username.
2. `$6$xyz...` — Password hash (`$6$` = SHA-512). `*` or `!` means no password (login disabled).
3. `19000` — Days since epoch (1970-01-01) when the password was last changed.
4. `0` — Minimum password age (days).
5. `99999` — Maximum password age (days).
6. `7` — Password warning period (days before expiry).
7. (empty) — Inactivity period, expiration date, reserved.

Only root can read `/etc/shadow`. This is why an attacker who gets root can extract all password hashes (for offline cracking).

## 2. Groups

A **group** is a collection of users. Groups make it easy to share files among multiple users. Every user has one **primary group** (listed in `/etc/passwd`) and can be a member of additional **supplementary groups** (listed in `/etc/group`).

### 2.1 The `/etc/group` File

```
developers:x:1001:alice,bob,carol
```

Fields:

1. `developers` — Group name.
2. `x` — Group password placeholder (rarely used).
3. `1001` — GID.
4. `alice,bob,carol` — Members (supplementary group membership).

### 2.2 Primary vs. Supplementary Groups

```bash
id                              # show your UID, primary GID, and supplementary groups
# uid=1000(alice) gid=1000(alice) groups=1000(alice),27(sudo),1001(developers)
```

- **Primary group (`gid`)** — Assigned to every file you create (unless setgid is set on the directory).
- **Supplementary groups (`groups`)** — Used only for permission checks. You can access files owned by any of your supplementary groups.

When you create a file, its group is your **primary** group, not all of your supplementary groups. To make files inherit a specific group, use the setgid bit on the directory (see [[02 - File System and Permissions/3. Special Permission Bits]]).

## 3. Managing Users

### 3.1 Creating Users

```bash
sudo useradd -m -s /bin/bash alice               # create alice with home dir and bash shell
sudo useradd -m -s /bin/bash -c "Alice Smith" alice    # with full name
sudo useradd -m -s /bin/bash -G sudo,developers alice   # with supplementary groups
sudo passwd alice                                 # set alice's password
```

`useradd` is the low-level command. `adduser` (Debian/Ubuntu only) is a friendlier interactive wrapper.

```bash
sudo adduser alice                                # interactive: prompts for full name, password, etc.
```

### 3.2 Modifying Users

```bash
sudo usermod -aG developers alice                 # add alice to developers group (-a = append)
sudo usermod -aG sudo alice                       # give alice sudo privileges
sudo usermod -s /bin/zsh alice                    # change alice's shell
sudo usermod -c "Alice Smith-Jones" alice         # change full name
sudo usermod -L alice                             # lock alice's account (disable login)
sudo usermod -U alice                             # unlock alice's account
sudo usermod -l newname oldname                   # rename user (does not rename home dir)
```

> [!warning] Never Forget `-a` with `-G`
> `usermod -G developers alice` REMOVES alice from all other supplementary groups and adds her only to `developers`. Always use `-aG` (append) to add to a group without removing others.

### 3.3 Deleting Users

```bash
sudo userdel alice                                # remove user, keep home directory
sudo userdel -r alice                             # remove user AND home directory
```

### 3.4 Changing Passwords

```bash
passwd                                            # change your own password
sudo passwd alice                                 # change alice's password (as root)
sudo passwd -l alice                              # lock alice's password
sudo passwd -u alice                              # unlock alice's password
sudo chage -l alice                               # show alice's password aging info
sudo chage -M 90 alice                            # set alice's password to expire in 90 days
```

## 4. Managing Groups

```bash
sudo groupadd developers                          # create a group
sudo groupadd -g 2000 developers                  # with a specific GID
sudo groupmod -n devs developers                  # rename group
sudo groupdel developers                          # delete group (must have no members)
gpasswd -a alice developers                       # add alice to developers (alternative)
gpasswd -d alice developers                       # remove alice from developers
```

## 5. The `newgrp` Command

`newgrp` switches your primary group for the current shell:

```bash
id
# gid=1000(alice)

newgrp developers
id
# gid=1001(developers)

# Files you create now have group "developers"
touch file.txt
ls -l file.txt
# -rw-r--r-- 1 alice developers 0 ... file.txt

exit         # exit the newgrp shell, return to normal
```

This is rare in practice — setgid on directories is usually a better solution for shared group work.

## 6. PAM — Pluggable Authentication Modules

Linux authentication is handled by **PAM** (Pluggable Authentication Modules). PAM lets you configure:

- How users authenticate (passwords, SSH keys, biometrics, OTP).
- Password policies (length, complexity).
- Session setup (resource limits, environment).
- Account policies (expiration, lockout).

PAM configuration is in `/etc/pam.d/`. Each service (login, sshd, sudo) has its own file. Modifying PAM is advanced — a mistake can lock you out of the system.

Common PAM-related files:

- `/etc/pam.d/common-auth` — Authentication rules (Debian/Ubuntu).
- `/etc/pam.d/common-password` — Password policies.
- `/etc/security/limits.conf` — Resource limits per user/group.
- `/etc/login.defs` — Default values for new users (UID range, password aging).

## 7. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting `-a` with `usermod -G`
> `usermod -G group user` removes the user from all other supplementary groups. Always use `usermod -aG group user` to append.

> [!warning] Mistake 2 — Forgetting to Set a Password After `useradd`
> `useradd` creates the account but does not set a password. The account is locked until you run `sudo passwd username`.

> [!warning] Mistake 3 — Editing `/etc/passwd` Directly
> Use `useradd`, `usermod`, `userdel`. Directly editing `/etc/passwd` can corrupt it. Use `vipw` if you must edit it manually (it locks the file to prevent concurrent edits).

> [!warning] Mistake 4 — Forgetting to Create the Home Directory
> `useradd alice` (without `-m`) creates the user but does not create `/home/alice`. Use `useradd -m alice`.

> [!warning] Mistake 5 — Deleting a User Without Checking Running Processes
> `userdel alice` does not kill alice's running processes. Check with `ps -u alice` and kill them first.

> [!warning] Mistake 6 — Confusing Primary and Supplementary Groups
> Your primary group is the one in `/etc/passwd` (and shown as `gid` by `id`). Supplementary groups are the ones in `groups=`. Files you create use your primary group, not supplementary groups (unless setgid is set on the directory).

## 8. Summary Checklist

- [ ] Every user has a UID; every group has a GID. The kernel uses IDs, not names.
- [ ] `/etc/passwd` has user info (world-readable); `/etc/shadow` has password hashes (root-only).
- [ ] `/etc/group` has group membership.
- [ ] `useradd -m -s /bin/bash username` creates a user; `passwd username` sets the password.
- [ ] `usermod -aG group username` adds to a supplementary group (always use `-a`!).
- [ ] `userdel -r username` removes user and home directory.
- [ ] Primary group: assigned to new files. Supplementary groups: used for permission checks.
- [ ] `id` shows your UID, GID, and supplementary groups.
- [ ] `newgrp group` switches your primary group for the current shell.
- [ ] PAM handles authentication; configured in `/etc/pam.d/`.
