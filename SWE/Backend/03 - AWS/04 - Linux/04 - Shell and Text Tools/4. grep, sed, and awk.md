# 4. grep, sed, and awk

> [!info] Chapter Context
> These three tools form the holy trinity of Unix text processing. `grep` searches for patterns; `sed` edits text streamingly; `awk` is a small programming language for column-based data. Mastering these lets you manipulate text in ways that would take hundreds of lines of Python.

## 1. grep — Search for Patterns

`grep` prints lines that match a pattern (literal string or regex).

### 1.1 Basic Usage

```bash
grep "error" /var/log/syslog          # lines containing "error"
grep -i "error" file.txt              # case-insensitive
grep -v "debug" file.txt              # lines NOT containing "debug" (invert)
grep -c "error" /var/log/syslog       # count matching lines
grep -n "error" file.txt              # show line numbers
grep -w "error" file.txt              # whole word only (not "errors" or "errorlog")
grep -r "error" /var/log/             # recursive (search all files)
grep -rn "error" /var/log/            # recursive with line numbers
grep -l "error" *.log                 # list filenames that contain "error"
grep -L "error" *.log                 # list filenames that do NOT contain "error"
```

### 1.2 Multiple Patterns

```bash
grep -e "error" -e "warning" file.txt          # match either
grep -E "error|warning" file.txt               # extended regex (same as egrep)
grep -f patterns.txt file.txt                  # patterns from a file (one per line)
```

### 1.3 Regular Expressions

`grep` uses Basic Regular Expressions (BRE) by default. Use `-E` for Extended Regular Expressions (ERE), which support `+`, `?`, `|`, `()` without escaping.

```bash
grep "^error" file.txt                # lines starting with "error"
grep "error$" file.txt                # lines ending with "error"
grep "err.r" file.txt                 # "error", "errar", etc. (. matches any char)
grep -E "error(s|ing)?" file.txt      # "error", "errors", "erroring"
grep -E "[0-9]{3}-[0-9]{4}" file.txt  # phone-number-like pattern
grep -E "^[A-Z][a-z]+" file.txt       # lines starting with a capitalized word
```

### 1.4 Context Lines

```bash
grep -B 2 "error" file.txt            # 2 lines Before
grep -A 2 "error" file.txt            # 2 lines After
grep -C 2 "error" file.txt            # 2 lines before AND after
```

### 1.5 Binary Files

```bash
grep "pattern" binary-file           # by default, says "binary file matches"
grep -a "pattern" binary-file        # treat as text
grep -I "pattern" .                  # ignore binary files in recursive search
```

### 1.6 Performance with Large Files

```bash
grep --mmap "pattern" hugefile.txt   # use mmap (faster for some files)
LC_ALL=C grep "pattern" hugefile.txt # use C locale (faster byte-by-byte comparison)
```

For very large files, `LC_ALL=C` can make grep 10x faster because it skips UTF-8 decoding.

## 2. sed — Stream Editor

`sed` applies editing commands to each line of input. It is most commonly used for find-and-replace, but it can do much more.

### 2.1 The Substitution Command

```bash
sed 's/old/new/' file.txt             # replace first "old" with "new" on each line
sed 's/old/new/g' file.txt            # replace all (global)
sed 's/old/new/gi' file.txt           # global + case-insensitive
sed 's/old/new/2' file.txt            # replace only the 2nd occurrence on each line
sed 's/old/new/3g' file.txt           # replace from the 3rd occurrence onward
```

### 2.2 In-Place Editing

```bash
sed -i 's/old/new/g' file.txt         # edit the file in-place (no backup)
sed -i.bak 's/old/new/g' file.txt     # in-place, save original as file.txt.bak
```

> [!warning] `sed -i` Modifies Files
> `sed -i` changes the file on disk with no undo. Always test your `sed` command without `-i` first to verify the output, then add `-i`.

### 2.3 Address Ranges

You can specify which lines to apply a command to:

```bash
sed '5d' file.txt                     # delete line 5
sed '5,10d' file.txt                  # delete lines 5-10
sed '1d;$d' file.txt                  # delete first and last line
sed '/^#/d' file.txt                  # delete lines starting with #
sed '/pattern/!d' file.txt            # delete lines NOT matching pattern (keep matches)
sed -n '20,30p' file.txt              # print lines 20-30 (like head/tail)
```

### 2.4 Multiple Commands

```bash
sed -e 's/foo/bar/g' -e 's/baz/qux/g' file.txt
sed 's/foo/bar/g; s/baz/qux/g' file.txt     # same with semicolons
sed -f script.sed file.txt                   # commands from a file
```

### 2.5 Using Different Delimiters

If your pattern contains `/`, use a different delimiter:

```bash
sed 's|/usr/local/bin|/opt/bin|g' file.txt    # use | as delimiter
sed 's#/usr/local/bin#/opt/bin#g' file.txt    # use # as delimiter
```

### 2.6 Capture Groups

In extended regex mode (`-E`), use `()` for groups and `\1`, `\2` for backreferences:

```bash
echo "John Smith" | sed -E 's/(\w+) (\w+)/\2, \1/'
# Smith, John
```

### 2.7 Common sed Recipes

```bash
# Remove leading whitespace
sed 's/^[ \t]*//' file.txt

# Remove trailing whitespace
sed 's/[ \t]*$//' file.txt

# Remove empty lines
sed '/^$/d' file.txt

# Replace tabs with spaces
sed 's/\t/  /g' file.txt

# Comment out a line (add # at start)
sed 's/^/## /' file.txt

# Lowercase the entire file
sed 's/.*/\L&/' file.txt
```

## 3. awk — Column-Based Text Processing

`awk` is a small programming language optimized for processing text that comes in columns (CSVs, log files, `ps` output, etc.).

### 3.1 Basic Structure

```bash
awk 'PATTERN { ACTION }' file.txt
```

For each line, awk checks the PATTERN. If it matches, the ACTION is executed. If PATTERN is omitted, the action runs for every line. If ACTION is omitted, the line is printed.

### 3.2 Field Splitting

By default, awk splits each line on whitespace into fields `$1`, `$2`, ..., `$NF` (last field). `$0` is the whole line.

```bash
echo "alice 30 engineer" | awk '{print $1}'        # alice
echo "alice 30 engineer" | awk '{print $3}'        # engineer
echo "alice 30 engineer" | awk '{print $1, $3}'    # alice engineer
echo "alice 30 engineer" | awk '{print $NF}'       # engineer (last field)
echo "alice 30 engineer" | awk '{print $(NF-1)}'   # 30 (second-to-last)
```

### 3.3 Custom Field Separator

```bash
awk -F, '{print $1}' data.csv           # comma-separated
awk -F: '{print $1}' /etc/passwd        # colon-separated
awk -F'\t' '{print $1}' data.tsv        # tab-separated
awk 'BEGIN {FS=":"} {print $1}' /etc/passwd   # alternative syntax
```

### 3.4 Patterns

```bash
# Lines matching a regex
awk '/error/' /var/log/syslog                   # like grep
awk '/error/ {print $1, $2}' /var/log/syslog

# Numeric comparisons
awk '$3 > 100' file.txt                         # lines where field 3 > 100
awk '$3 > 100 {print $1, $3}' file.txt

# Line numbers
awk 'NR == 5' file.txt                          # line 5 (NR = Number of Records)
awk 'NR >= 10 && NR <= 20' file.txt             # lines 10-20
awk 'NR % 2 == 0' file.txt                      # even-numbered lines

# Field count
awk 'NF == 3' file.txt                          # lines with exactly 3 fields
awk 'NF > 5 {print NR, $0}' file.txt            # lines with more than 5 fields
```

### 3.5 BEGIN and END Blocks

`BEGIN` runs before any lines are read; `END` runs after all lines are processed. Useful for setup and summary.

```bash
# Sum the 2nd column
awk 'BEGIN {sum=0} {sum += $2} END {print sum}' data.txt

# Count lines
awk 'END {print NR}' file.txt                   # like wc -l

# Average of column 2
awk '{sum += $2; count++} END {print sum/count}' data.txt
```

### 3.6 Variables

```bash
awk '{total = $2 + $3; print $1, total}' file.txt
awk -F, '{names[NR] = $1} END {for (i=1; i<=NR; i++) print names[i]}' data.csv
```

awk supports arrays, arithmetic, string operations, conditionals (`if`), and loops (`for`, `while`). It is a real programming language.

### 3.7 Common awk Recipes

```bash
# Sum the sizes of all files listed by ls -l
ls -l | awk '{sum += $5} END {print sum}'

# Print the longest line
awk '{if (length > max) {max = length; line = $0}} END {print line}' file.txt

# Count occurrences of each value in column 1
awk '{count[$1]++} END {for (k in count) print count[k], k}' file.txt | sort -rn

# Print columns in reverse order
awk '{for (i=NF; i>=1; i--) printf "%s ", $i; print ""}' file.txt
```

## 4. When to Use What

| Tool | Use for |
| :--- | :--- |
| `grep` | Searching for lines that match a pattern. |
| `sed` | Simple substitutions, line deletions, in-place edits. |
| `awk` | Column-based processing, arithmetic, summaries, multi-condition logic. |
| `cut` | Simple column extraction (when you do not need awk's full power). |
| `sort` | Sorting lines. |
| `uniq` | Removing or counting duplicate adjacent lines. |
| `head`/`tail` | First/last N lines. |

For anything more complex (real programming logic, structured data), use Python.

## 5. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting `-E` for Extended Regex
> `grep "a|b"` does not work (BRE); use `grep -E "a|b"` or `egrep "a|b"`. Same for sed (`sed -E`).

> [!warning] Mistake 2 — `sed -i` Without Testing
> `sed -i` modifies files in place. Always test without `-i` first.

> [!warning] Mistake 3 — Forgetting `g` Flag in sed
> `sed 's/old/new/'` replaces only the first occurrence per line. Use `sed 's/old/new/g'` for all occurrences.

> [!warning] Mistake 4 — Confusing awk's `$1` with Shell's `$1`
> In awk, `$1` is the first field of the current line. In a shell script, `$1` is the first argument. They look the same but mean different things.

> [!warning] Mistake 5 — Forgetting That `awk` Splits on Whitespace by Default
> To process CSVs, you need `-F,`. Without it, awk treats consecutive spaces/tabs as one separator, which mangles CSVs.

> [!warning] Mistake 6 — Using `grep` Where `awk` Is Better
> If you need to filter based on a column value, `awk '$3 > 100'` is cleaner than `grep` with a complex regex.
