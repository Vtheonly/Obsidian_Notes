# 5. find and locate

> [!info] Chapter Context
> `find` searches for files by name, type, size, modification time, permissions, and many other criteria. `locate` is a faster alternative that uses a prebuilt database. Both are essential for navigating the filesystem efficiently.

## 1. The `find` Command

`find` walks a directory tree and prints files that match given criteria. It is the most flexible file-search tool on Linux.

### 1.1 Basic Syntax

```bash
find <path> <criteria> <action>
```

- `<path>` — Where to start searching (e.g., `.`, `/etc`, `/`).
- `<criteria>` — What to match (name, type, size, etc.). Multiple criteria are AND-ed together.
- `<action>` — What to do with matches (default: print).

### 1.2 Searching by Name

```bash
find . -name "*.txt"                    # files ending in .txt (case-sensitive)
find . -iname "*.TXT"                   # case-insensitive
find /etc -name "*.conf"                # all .conf files in /etc
find . -name "node_modules" -type d     # directories named node_modules
```

### 1.3 Searching by Type

```bash
find . -type f -name "*.py"             # regular files
find . -type d -name "test*"            # directories
find . -type l                          # symbolic links
find . -type b                          # block devices
find . -type c                          # character devices
find . -type s                          # sockets
find . -type p                          # named pipes
```

### 1.4 Searching by Size

```bash
find . -size +100M                      # larger than 100 MB
find . -size -1k                        # smaller than 1 KB
find . -size 10M                        # exactly 10 MB
find /var/log -size +1G -type f         # log files larger than 1 GB
```

Size units: `c` (bytes), `k` (KB), `M` (MB), `G` (GB). `+` means greater than; `-` means less than; no prefix means exactly.

### 1.5 Searching by Time

```bash
find . -mtime -7                        # modified in the last 7 days
find . -mtime +30                       # modified more than 30 days ago
find . -mmin -60                        # modified in the last 60 minutes
find . -ctime -1                        # changed (metadata) in the last day
find . -atime +365                      # accessed more than 365 days ago
find . -newer reference.txt             # modified more recently than reference.txt
```

- `mtime` — Modification time (content changed).
- `ctime` — Change time (metadata or content changed).
- `atime` — Access time (last read).

### 1.6 Searching by Permission and Owner

```bash
find . -user alice                      # owned by alice
find . -group developers                # group is developers
find . -perm 644                        # exactly 644 permissions
find . -perm -644                       # at least 644 (all bits set)
find . -perm /644                       # any of the bits set
find / -perm -4000 -type f 2>/dev/null  # setuid binaries (security audit)
find / -perm -2000 -type f 2>/dev/null  # setgid binaries
find . -empty                           # empty files and directories
```

### 1.7 Combining Criteria

```bash
# AND (implicit)
find . -name "*.py" -size +10k

# OR (-o)
find . -name "*.py" -o -name "*.txt"

# NOT (!)
find . ! -name "*.py"

# Parentheses (must be escaped or quoted)
find . \( -name "*.py" -o -name "*.txt" \) -size +10k
```

### 1.8 Acting on Results

```bash
# Print (default)
find . -name "*.py"

# Print with details (like ls -l)
find . -name "*.py" -ls

# Execute a command for each match
find . -name "*.py" -exec wc -l {} \;

# Execute once with all matches (more efficient)
find . -name "*.py" -exec wc -l {} +

# Delete (DANGEROUS — confirm first by running without -delete)
find . -name "*.pyc" -delete
find /tmp -type f -mtime +7 -delete

# Prompt before each action
find . -name "*.pyc" -ok rm {} \;
```

`{}` is a placeholder for the matched file. `\;` ends the command (one invocation per file). `+` ends the command (one invocation for all files, more efficient).

### 1.9 Practical Examples

```bash
# Find all .log files older than 30 days and compress them
find /var/log -name "*.log" -mtime +30 -exec gzip {} \;

# Find all files owned by alice in /home and change ownership to bob
find /home -user alice -exec chown bob {} \;

# Find all world-writable files (security audit)
find / -perm -2 -type f 2>/dev/null

# Find the 10 largest files in /var
find /var -type f -exec du -h {} + 2>/dev/null | sort -rh | head -10

# Find and delete all .DS_Store files (Mac OS metadata)
find . -name ".DS_Store" -delete
```

## 2. The `locate` Command

`locate` searches a prebuilt database of file paths. It is much faster than `find` but only as fresh as the last database update.

### 2.1 Installing

```bash
sudo apt install plocate            # Debian/Ubuntu (modern)
sudo dnf install mlocate            # Fedora/RHEL
```

### 2.2 Usage

```bash
locate nginx.conf                   # find files matching "nginx.conf"
locate -i nginx                     # case-insensitive
locate -r "\.py$"                   # regex match
locate -c nginx                     # count matches
locate -S                           # show database stats
```

### 2.3 Updating the Database

The database is updated daily by a cron job (`/etc/cron.daily/mlocate`). To update it manually:

```bash
sudo updatedb
```

### 2.4 locate vs find

| Feature | `find` | `locate` |
| :--- | :--- | :--- |
| Speed | Slow (walks the filesystem) | Fast (searches a database) |
| Freshness | Always current | As of last `updatedb` |
| Criteria | Many (size, time, perms, etc.) | Filename only |
| Newly created files | Found immediately | Not found until `updatedb` runs |

Use `locate` for quick "where is X" lookups. Use `find` when you need to filter by anything other than name, or when you need to find files created in the last few minutes.

## 3. Other Useful Search Commands

### 3.1 `which` and `whereis`

```bash
which python3                       # first match in PATH
whereis python3                     # binary, source, man page
type python3                        # how the shell would interpret the command
```

### 3.2 `dpkg -S` and `rpm -qf`

```bash
dpkg -S /usr/bin/curl               # which package installed this file? (Debian/Ubuntu)
rpm -qf /usr/bin/curl               # which package installed this file? (Fedora/RHEL)
```

### 3.3 `command -v`

```bash
command -v docker                   # check if docker is in PATH (returns 0 if found)
if command -v docker > /dev/null; then
  echo "Docker is installed"
fi
```

## 4. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting the Starting Path
> `find -name "*.py"` works (defaults to `.`), but explicit is better: `find . -name "*.py"`.

> [!warning] Mistake 2 — Using `find` Without `-type`
> `find . -name "node_modules"` matches both files and directories named `node_modules`. Usually you want directories only: `find . -name "node_modules" -type d`.

> [!warning] Mistake 3 — Forgetting to Quote Patterns
> `find . -name *.py` is expanded by the shell if any `.py` files exist in the current directory. Always quote: `find . -name "*.py"`.

> [!warning] Mistake 4 — Forgetting `2>/dev/null` for Permission Errors
> `find / -name "*.conf"` produces many "Permission denied" errors. Discard them: `find / -name "*.conf" 2>/dev/null`.

> [!warning] Mistake 5 — Using `-exec` Inefficiently
> `find . -name "*.py" -exec grep "import" {} \;` runs `grep` once per file (slow). Use `+` instead: `find . -name "*.py" -exec grep "import" {} +` (runs grep with all files at once).

> [!warning] Mistake 6 — Forgetting `-delete` Is Dangerous
> `find . -name "*.pyc" -delete` deletes without confirmation. Always run without `-delete` first to see what would be deleted.

> [!warning] Mistake 7 — Using `locate` for Recently Created Files
> `locate` searches a database that is updated daily. Files created in the last few hours will not be found. Use `find` for fresh files.

- [ ] `find <path> <criteria> <action>` walks a directory tree.
- [ ] Common criteria: `-name`, `-iname`, `-type`, `-size`, `-mtime`, `-mmin`, `-user`, `-group`, `-perm`, `-empty`.

- [ ] Add `2>/dev/null` to suppress permission errors when searching `/`.
- [ ] `-delete` is dangerous; test without it first.
- [ ] `locate` is faster but only as fresh as the last `updatedb`.
- [ ] Use `locate` for quick lookups; `find` for criteria-based searches.
