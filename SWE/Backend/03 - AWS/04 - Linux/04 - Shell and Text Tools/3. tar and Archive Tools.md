# 3. tar and Archive Tools

> [!info] Chapter Context
> `tar` (tape archive) is the standard tool for creating and extracting archives on Linux. This note covers `tar` thoroughly, plus related tools: `gzip`, `bzip2`, `xz`, `zip`, and `unzip`.

Related: [[04 - Shell and Text Tools/1. The Shell and Bash Basics]] | [[04 - Shell and Text Tools/2. Pipelines and Redirection]] | [[01 - Installing Apps/7. .tar.gz Files and Manual Installation]]

## 1. The `tar` Command

`tar` combines multiple files into a single archive (a `.tar` file). It does not compress by default — compression is a separate step, applied with a flag.

### 1.1 The Three Modes

`tar` has three primary modes:

| Flag | Mode | Purpose |
| :--- | :--- | :--- |
| `c` | **C**reate | Make a new archive. |
| `x` | E**x**tract | Extract files from an archive. |
| `t` | Lis**t** | List contents without extracting. |

### 1.2 Common Flags

| Flag | Meaning |
| :--- | :--- |
| `f` | **F**ile — the next argument is the archive file name. (Without `f`, tar writes to default tape device.) |
| `v` | **V**erbose — show files as they are processed. |
| `z` | g**Z**ip — compress/decompress with gzip (`.tar.gz` or `.tgz`). |
| `j` | bz**i**p2 — compress with bzip2 (`.tar.bz2`). |
| `J` | **X**Z — compress with xz (`.tar.xz`). |
| `C` | **C**hange to directory before operating. |
| `--strip-components=N` | Strip N leading path components when extracting. |
| `-T file` | Read file list from `file`. |
| `--exclude=PATTERN` | Exclude files matching PATTERN. |

## 2. Creating Archives

### 2.1 Basic Create

```bash
# Create an uncompressed .tar
tar cf archive.tar file1 file2 dir/

# Create a gzipped .tar.gz
tar czf archive.tar.gz dir/
tar czf archive.tgz dir/          # .tgz is a common shorthand

# Create a bzip2-compressed .tar.bz2 (smaller, slower)
tar cjf archive.tar.bz2 dir/

# Create an xz-compressed .tar.xz (smallest, slowest)
tar cJf archive.tar.xz dir/
```

### 2.2 Verbose Output

```bash
tar czvf archive.tar.gz dir/
# Shows each file as it is added:
# dir/
# dir/file1.txt
# dir/subdir/
# dir/subdir/file2.txt
```

### 2.3 Excluding Files

```bash
tar czf archive.tar.gz --exclude='*.log' dir/
tar czf archive.tar.gz --exclude='node_modules' --exclude='.git' project/
```

`--exclude` can be specified multiple times. Patterns are globs; quote them to prevent the shell from expanding them first.

### 2.4 Reading File List from a File

```bash
find . -name "*.py" > python-files.txt
tar czf python.tar.gz -T python-files.txt
```

This is useful when you have a complex file selection that is easier to express with `find` than with `tar`'s built-in options.

## 3. Extracting Archives

### 3.1 Basic Extract

```bash
tar xf archive.tar                    # extract (auto-detects compression)
tar xzf archive.tar.gz                # explicitly gzip
tar xjf archive.tar.bz2               # explicitly bzip2
tar xJf archive.tar.xz                # explicitly xz
```

Modern `tar` auto-detects compression, so `tar xf` works for any format.

### 3.2 Verbose Extract

```bash
tar xvf archive.tar.gz                # show files as they are extracted
```

### 3.3 Extract to a Specific Directory

```bash
tar xzf archive.tar.gz -C /opt/       # extract into /opt/
```

The target directory must exist. `tar` does not create it.

### 3.4 Extract Specific Files

```bash
tar xzf archive.tar.gz path/to/file.txt          # extract just this file
tar xzf archive.tar.gz "file 1.txt" "file 2.txt"  # extract multiple
tar xzf archive.tar.gz --wildcards '*.log'        # extract all .log files
```

### 3.5 Strip Leading Components

```bash
# Archive has: node-v20.10.0-linux-x64/bin/node
# We want: /usr/local/bin/node (strip the top directory)
tar -C /usr/local --strip-components=1 -xJf node-v20.10.0-linux-x64.tar.xz
```

`--strip-components=1` removes the first path component (`node-v20.10.0-linux-x64/`), so files go directly under `/usr/local/`.

## 4. Listing Archive Contents

```bash
tar tf archive.tar.gz                 # list files
tar tvf archive.tar.gz                # verbose (with sizes, permissions, timestamps)
tar tzf archive.tar.gz                # explicitly gzip
```

Always list an archive before extracting, especially from untrusted sources, to check for path traversal (e.g., `../../../etc/passwd`).

## 5. Compression Comparison

| Format | Flag | Extension | Compression ratio | Speed |
| :--- | :--- | :--- | :--- | :--- |
| None | (none) | `.tar` | 1x | Fastest |
| gzip | `z` | `.tar.gz` / `.tgz` | ~3-4x | Fast |
| bzip2 | `j` | `.tar.bz2` | ~4-5x | Slower |
| xz | `J` | `.tar.xz` | ~5-6x | Slowest (but best ratio) |

For most use cases, gzip is a good balance. For maximum compression (e.g., distributing large binaries), use xz.

## 6. zip and unzip

`.zip` is a Windows-compatible archive format. It combines archiving and compression into one step.

### 6.1 Creating zip Archives

```bash
zip archive.zip file1 file2
zip -r archive.zip dir/               # recursively include a directory
zip -r archive.zip . -x "*.log"       # exclude .log files
zip -e archive.zip file.txt           # encrypt with a password
```

### 6.2 Extracting zip Archives

```bash
unzip archive.zip
unzip archive.zip -d /tmp/            # extract to /tmp/
unzip -l archive.zip                  # list contents
unzip archive.zip file.txt            # extract one file
```

### 6.3 zip vs tar

| Feature | tar | zip |
| :--- | :--- | :--- |
| Native to | Unix/Linux | Windows |
| Compression | Separate (gzip, bzip2, xz) | Built-in (DEFLATE) |
| Permissions | Preserves Unix permissions | Limited (no setuid, no symlinks on Windows) |
| Single-file extract | Yes | Yes |
| Random access | No (must read the whole archive) | Yes (can extract one file without reading the rest) |

For Linux-to-Linux distribution, use `.tar.gz`. For Linux-to-Windows distribution, use `.zip`.

## 7. gzip, bzip2, xz — Single-File Compression

These tools compress a **single file** (not an archive of multiple files). They are typically used together with `tar`:

```bash
tar cf - dir/ | gzip > archive.tar.gz       # equivalent to tar czf archive.tar.gz dir/
```

But they can also be used standalone:

```bash
gzip file.txt                # creates file.txt.gz, removes file.txt
gzip -k file.txt             # creates file.txt.gz, keeps file.txt
gunzip file.txt.gz           # decompresses to file.txt

bzip2 file.txt               # creates file.txt.bz2
bunzip2 file.txt.bz2

xz file.txt                  # creates file.txt.xz
unxz file.txt.xz
```

The `-k` (keep) flag prevents the original file from being deleted.

## 8. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting the `f` Flag
> `tar cz archive.tar.gz dir/` writes the archive to the default tape device (which usually does not exist on modern systems). Always include `f`: `tar czf archive.tar.gz dir/`.

> [!warning] Mistake 2 — Extracting Without Checking Contents First
> `tar xf archive.tar.gz` extracts into your current directory, which can create a mess. Always `tar tf archive.tar.gz` first to see what is inside.

> [!warning] Mistake 3 — Forgetting `-C` to Extract to a Directory
> `tar xzf archive.tar.gz` extracts into the current directory. Use `tar xzf archive.tar.gz -C /opt/` to extract into `/opt/`.

> [!warning] Mistake 4 — Using bzip2 or xz with the Wrong Flag
> - `tar czf` for gzip (`z`).
> - `tar cjf` for bzip2 (`j`).
> - `tar cJf` for xz (`J` — capital).
> Mixing them up produces "this does not look like a tar archive" errors.

> [!warning] Mistake 5 — Forgetting `--exclude` Patterns Are Globs
> `--exclude=*.log` may be expanded by the shell if any `.log` files exist in the current directory. Quote it: `--exclude='*.log'`.

> [!warning] Mistake 6 — Forgetting That `gzip` Removes the Original
> `gzip file.txt` produces `file.txt.gz` and deletes `file.txt`. Use `gzip -k file.txt` to keep the original.
