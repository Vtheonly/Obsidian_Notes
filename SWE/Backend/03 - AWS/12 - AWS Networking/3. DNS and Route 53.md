# 3. DNS and Route 53

> [!info] Chapter Context
> Amazon Route 53 is AWS's DNS service. It is also a domain registrar and supports health checks, routing policies, and DNS-based failover. This note covers Route 53 concepts and common configurations.

Related: [[2. Subnets, Route Tables, and NAT]] | [[4. Load Balancers and CloudFront]] | [[00 - Foundations/4. DNS and TLS]]

---

## 1. What Route 53 Is

Route 53 is a highly available and scalable DNS web service. The name comes from TCP/UDP port 53 (DNS). Route 53:

- Registers domains (registrar service).
- Hosts DNS zones (the records for your domain).
- Performs health checks on endpoints.
- Routes traffic based on policies (latency, geolocation, weighted, failover).

---

## 2. Hosted Zones

A **hosted zone** is a collection of records for a domain. Two types:

- **Public hosted zone** — Records are reachable from the internet.
- **Private hosted zone** — Records are reachable only from within a specified VPC (useful for internal service discovery).

```bash
# Create a public hosted zone
aws route53 create-hosted-zone --name example.com --caller-reference $(date +%s)
# Returns ZoneId

# Create a private hosted zone (for VPC)
aws route53 create-hosted-zone --name internal.example.com \
  --caller-reference $(date +%s) \
  --vpc VPCRegion=us-east-1,VPCId=vpc-12345
```

---

## 3. Record Types

Route 53 supports all standard DNS record types. The most common:

- **A** — IPv4 address.
- **AAAA** — IPv6 address.
- **CNAME** — Alias to another name (cannot be at the zone apex).
- **Alias** — Route 53-specific; points to an AWS resource (ALB, CloudFront, S3 bucket, another Route 53 record). Free queries; supports the zone apex.
- **MX** — Mail server.
- **TXT** — Arbitrary text (SPF, DKIM, verification).
- **NS** — Nameservers for the zone.
- **SOA** — Start of authority.

### 3.1 Alias vs. CNAME

| Feature | CNAME | Alias |
| :--- | :--- | :--- |
| Points to | Any DNS name | AWS resource (ALB, CloudFront, S3, etc.) |
| Zone apex? | No (cannot use for `example.com`) | Yes |
| Cost | Charged per query | Free queries |
| Changes | Follows DNS TTL | Updates automatically (when the target's IP changes) |

Use **Alias** for AWS resources. Use CNAME only for non-AWS targets.

## 4. Creating Records

```bash
# Create an A record pointing to an ALB (Alias)
aws route53 change-resource-record-sets \
  --hosted-zone-id Z123ABC \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "api.example.com",
        "Type": "A",
        "AliasTarget": {
          "HostedZoneId": "Z35SXDOTRQ7X7K",
          "DNSName": "dualstack.my-alb-123.us-east-1.elb.amazonaws.com",
          "EvaluateTargetHealth": true
        }
      }
    }]
  }'
```

> [!note] The Alias HostedZoneId for ALBs
> For Alias records to ALBs, CloudFront, S3, etc., the `HostedZoneId` field is a fixed value (not your hosted zone ID). Look up the value in the AWS docs (e.g., `Z35SXDOTRQ7X7K` for ALBs in us-east-1).

## 5. Routing Policies

### 5.1 Simple

Default. One record, one target (or multiple targets returned in random order).

### 5.2 Weighted

Distribute traffic across multiple targets by weight. Use for canary deployments, A/B testing.

```json
{
  "Name": "api.example.com",
  "Type": "A",
  "SetIdentifier": "v1",
  "Weight": 90,
  "AliasTarget": {"DNSName": "api-v1.example.com", ...}
}
```

Create multiple records with the same name and type, different `SetIdentifier` and `Weight`.

### 5.3 Latency-Based

Route users to the region with the lowest latency.

```json
{
  "Name": "api.example.com",
  "Type": "A",
  "SetIdentifier": "us-east",
  "Region": "us-east-1",
  "AliasTarget": {"DNSName": "api-us-east.example.com", ...}
}
```

Use for multi-region deployments.

### 5.4 Geolocation

Route users based on their geographic location (country, continent, state).

```json
{
  "Name": "api.example.com",
  "Type": "A",
  "SetIdentifier": "europe",
  "GeoLocation": {"ContinentCode": "EU"},
  "AliasTarget": {"DNSName": "api-eu.example.com", ...}
}
```

Use for data sovereignty (EU users to EU region) or localization.

### 5.5 Failover

Route to a primary; if its health check fails, route to a secondary.

```json
{
  "Name": "api.example.com",
  "Type": "A",
  "SetIdentifier": "primary",
  "Failover": "PRIMARY",
  "HealthCheckId": "abc123",
  "AliasTarget": {"DNSName": "api-primary.example.com", ...}
}
```

Use for disaster recovery.

### 5.6 Geoproximity (with Route 53 Traffic Flow)

Route based on geographic distance, with bias to shift traffic to specific regions.

### 5.7 Multivalue Answer

Return multiple IP addresses (up to 8) randomly. Like simple routing, but the client can fail over to another IP if one fails.

## 6. Health Checks

Route 53 can monitor endpoints and use the result for routing decisions.

```bash
aws route53 create-health-check --caller-reference $(date +%s) \
  --health-check-config '{
    "Type": "HTTPS",
    "FullyQualifiedDomainName": "api.example.com",
    "ResourcePath": "/health",
    "Port": 443,
    "RequestInterval": 30,
    "FailureThreshold": 3
  }'
```

- **RequestInterval** — 10 or 30 seconds.
- **FailureThreshold** — Number of consecutive failures before marking unhealthy (1-10).
- **Types** — HTTP, HTTPS, HTTP_STR_MATCH, HTTPS_STR_MATCH, TCP, CALCULATED (combine other health checks), CLOUDWATCH_METRIC (alarm-based).

Attach health checks to records with `HealthCheckId`. Route 53 stops routing to unhealthy records.

---

## 7. Private Hosted Zones

For internal DNS (e.g., `db.internal.example.com` resolves to a private RDS endpoint):

```bash
aws route53 create-hosted-zone \
  --name internal.example.com \
  --caller-reference $(date +%s) \
  --vpc VPCRegion=us-east-1,VPCId=vpc-12345
```

Records in a private hosted zone are only resolvable from the associated VPC(s). Use for service discovery within a VPC.

To associate additional VPCs:

```bash
aws route53 associate-vpc-with-hosted-zone \
  --hosted-zone-id Z123ABC \
  --vpc VPCRegion=us-west-2,VPCId=vpc-67890
```

## 8. Registering a Domain

Route 53 is a domain registrar. You can register new domains or transfer existing ones.

```bash
aws route53domains register-domain \
  --domain-name example.com \
  --duration-in-years 1 \
  --admin-contact '{"ContactType":"PERSON","FirstName":"Alice","LastName":"Smith","Email":"alice@example.com",...}' \
  --tech-contact '...' \
  --registrant-contact '...'
```

For most users, the Console is easier for domain registration (lots of contact fields).

## 9. Common Student Mistakes

> [!warning] Mistake 1 — Using CNAME for AWS Resources
> Use Alias records for AWS resources (ALB, CloudFront, S3). Aliases are free, support the zone apex, and update automatically.

> [!warning] Mistake 2 — Forgetting to Update NS Records at the Registrar
> After creating a hosted zone, you must update your registrar's NS records to point to Route 53's nameservers. Otherwise, the internet cannot find your zone.

> [!warning] Mistake 3 — Forgetting the Alias HostedZoneId
> Alias records to AWS resources need a specific `HostedZoneId` (not yours). Look up the value in the AWS docs.

> [!warning] Mistake 4 — Not Using Health Checks for Failover
> Without health checks, Route 53 routes to a dead endpoint. Always attach health checks for failover and weighted routing.

> [!warning] Mistake 5 — Forgetting TTL
> TTL determines how long resolvers cache your record. Long TTL (1 hour+) reduces query cost but slows DNS changes. Short TTL (60s) is faster for changes but more expensive.

> [!warning] Mistake 6 — Forgetting Private Hosted Zones for Internal DNS
> For internal services (databases, internal APIs), use a private hosted zone. Public records expose your internal naming to the internet.

## 10. Summary Checklist

- [ ] Route 53 is a DNS service + domain registrar.
- [ ] Hosted zones: public (internet-reachable) and private (VPC-only).
- [ ] Record types: A, AAAA, CNAME, Alias (Route 53-specific, for AWS resources), MX, TXT, NS, SOA.
- [ ] Use Alias (not CNAME) for AWS resources — free, supports zone apex.
- [ ] Routing policies: simple, weighted, latency, geolocation, failover, multivalue.
- [ ] Health checks monitor endpoints; used for routing decisions.
- [ ] Private hosted zones for internal service discovery.
- [ ] Update NS records at the registrar after creating a hosted zone.
