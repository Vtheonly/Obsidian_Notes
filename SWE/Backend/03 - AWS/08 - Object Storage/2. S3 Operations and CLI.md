## 1. High-Level vs. Low-Level CLI

The AWS CLI has two S3 interfaces:

- **`aws s3`** — High-level, file-system-like commands (`cp`, `mv`, `ls`, `sync`). Simpler, fewer options.
- **`aws s3api`** — Low-level, API-matching commands (`put-object`, `get-object`, `list-objects-v2`). More options, more verbose.

```bash
# High-level (simple)
aws s3 cp file.txt s3://my-bucket/

# Low-level (equivalent, more options)
aws s3api put-object --bucket my-bucket --key file.txt --body file.txt
```

Use `aws s3` for everyday tasks. Use `aws s3api` when you need a specific option not exposed by `aws s3`.

## 2. Multipart Uploads

For files larger than 100 MB, use multipart upload. S3 splits the file into parts (5 MB to 5 GB each), uploads them in parallel, and reassembles.

`aws s3 cp` automatically uses multipart upload for files > 8 MB. For more control:

```bash
# Manually initiate, upload parts, complete
aws s3api create-multipart-upload --bucket my-bucket --key large-file.bin
# Returns UploadId

aws s3api upload-part --bucket my-bucket --key large-file.bin \
  --part-number 1 --upload-id ABCD... --body part1.bin
# Returns ETag

aws s3api complete-multipart-upload --bucket my-bucket --key large-file.bin \
  --upload-id ABCD... --multipart-upload file://parts.json
```

For most use cases, let `aws s3 cp` handle it:

```bash
aws s3 cp large-file.bin s3://my-bucket/ \
  --expected-size 5000000000
```

### 2.1 Aborted Multipart Uploads

If a multipart upload is interrupted, the parts remain in S3 (and you pay for them). Use a lifecycle rule to delete incomplete uploads:

```json
{
  "Rules": [{
    "ID": "Delete incomplete uploads",
    "Status": "Enabled",
    "Filter": {},
    "AbortIncompleteMultipartUpload": {"DaysAfterInitiation": 7}
  }]
}
```

## 3. Sync and Differential Uploads

`aws s3 sync` uploads only changed files (by size and timestamp):

```bash
aws s3 sync ./local-dir s3://my-bucket/prefix/

# Delete files in S3 that no longer exist locally
aws s3 sync ./local-dir s3://my-bucket/prefix/ --delete

# Exclude patterns
aws s3 sync ./local-dir s3://my-bucket/prefix/ --exclude "*.tmp" --exclude ".git/*"

# Dry run (see what would happen)
aws s3 sync ./local-dir s3://my-bucket/prefix/ --dryrun
```

## 4. Presigned URLs

A presigned URL lets anyone with the URL download (or upload) an object for a limited time, without AWS credentials.

```bash
# Generate a presigned download URL (valid for 1 hour)
aws s3 presign s3://my-bucket/file.pdf --expires-in 3600

# Generate via s3api (more options)
aws s3api get-object --bucket my-bucket --key file.pdf file.pdf --cli-input-json file://presign.json
```

With Boto3:

```python
import boto3

s3 = boto3.client('s3')

# Presigned GET URL
url = s3.generate_presigned_url(
    'get_object',
    Params={'Bucket': 'my-bucket', 'Key': 'file.pdf'},
    ExpiresIn=3600
)

# Presigned PUT URL (let a client upload directly to S3)
url = s3.generate_presigned_url(
    'put_object',
    Params={'Bucket': 'my-bucket', 'Key': 'upload.pdf', 'ContentType': 'application/pdf'},
    ExpiresIn=3600
)
```

Use cases:

- A web app lets users download private files without proxying through the server.
- A mobile app uploads photos directly to S3 (bypassing the server).
- Share a file with a third party for a limited time.

> [!warning] Presigned URLs Bypass IAM
> Anyone with the URL can access the object. The URL is the credential. Keep it private; use short expirations.

## 5. S3 Select and Glacier Select

Retrieve a subset of an object (filter rows from a CSV/JSON) without downloading the whole thing:

```bash
aws s3api select-object-content \
  --bucket my-bucket \
  --key data.csv \
  --expression "SELECT * FROM s3object s WHERE s.age > 30" \
  --expression-type SQL \
  --input-serialization '{"CSV": {"FileHeaderInfo": "USE"}, "CompressionType": "NONE"}' \
  --output-serialization '{"CSV": {}}' \
  output.csv
```

Useful for querying large CSV/JSON files without downloading them.

## 6. Cross-Region Replication (CRR)

Automatically replicate new objects to a bucket in another region:

```bash
# Enable versioning on both buckets (required)
aws s3api put-bucket-versioning --bucket source-bucket --versioning-configuration Status=Enabled
aws s3api put-bucket-versioning --bucket dest-bucket --versioning-configuration Status=Enabled

# Set up replication
aws s3api put-bucket-replication --bucket source-bucket --replication-configuration file://replication.json
```

```json
// replication.json
{
  "Role": "arn:aws:iam::123456789012:role/s3-replication-role",
  "Rules": [{
    "Status": "Enabled",
    "Filter": {},
    "Destination": {
      "Bucket": "arn:aws:s3:::dest-bucket"
    }
  }]
}
```

The role must have permissions to read from the source and write to the destination.

## 7. Performance Optimization

### 7.1 Use Multipart Upload

For files > 100 MB, multipart upload is faster (parallel uploads) and more resilient (retry only failed parts).

### 7.2 Use S3 Transfer Acceleration

For uploads from distant locations, use S3 Transfer Acceleration (routes through CloudFront edge locations):

```bash
aws s3 cp file.bin s3://my-bucket/ --endpoint-url https://bucket-name.s3-accelerate.amazonaws.com
```

### 7.3 Use Byte-Range Fetches

For large files where you only need parts, use byte-range GETs:

```bash
aws s3api get-object --bucket my-bucket --key large.bin --range bytes=0-1023 part.bin
```

### 7.4 Use S3 Batch Operations

For operations on millions of objects (copy, replace tags, invoke Lambda), use S3 Batch Operations:

```bash
aws s3control create-job --account-id 123456789012 \
  --operation '{"LambdaInvoke": {"FunctionArn": "arn:aws:lambda:us-east-1:123456789012:function:process"}}' \
  --manifest '{"Spec": {"Format": "S3BatchOperations_CSV_20180820", "Fields": ["Bucket", "Key"]}, "Location": {"Bucket": "arn:aws:s3:::manifest-bucket", "Key": "manifest.csv"}}' \
  --report "{\"Bucket\":\"arn:aws:s3:::report-bucket\",\"Format\":\"Report_CSV_20180820\",\"Enabled\":true}" \
  --role-arn arn:aws:iam::123456789012:role/batch-role
```

## 8. Common Student Mistakes

> [!warning] Mistake 1 — Forgetting to Clean Up Aborted Multipart Uploads
> Aborted uploads leave parts in S3 (billed). Use a lifecycle rule to delete them.

> [!warning] Mistake 2 — Long-Lived Presigned URLs
> Use short expirations (1 hour or less). Anyone with the URL can access the object.

> [!warning] Mistake 3 — Syncing Without `--delete`
> `aws s3 sync` adds and updates files but does not delete files in S3 that no longer exist locally. Use `--delete` to mirror.

> [!warning] Mistake 4 — Forgetting Versioning Before Replication
> Cross-region replication requires versioning on both source and destination buckets.

> [!warning] Mistake 5 — Using `aws s3 cp` for Many Small Files
> Each `cp` is an API call. For many files, use `aws s3 sync` (which parallelizes) or `s3api` with batch operations.
