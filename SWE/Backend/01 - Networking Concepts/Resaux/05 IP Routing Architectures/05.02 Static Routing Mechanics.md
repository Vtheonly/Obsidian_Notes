### 5.2 Static Routing Mechanics

Static routes are manually configured by a network administrator to define explicit forwarding paths.

```ios
-- Command Syntax:
Router(config)# ip route [destination_network] [subnet_mask] [next_hop_ip_or_exit_interface]
```

#### 1. Next-Hop vs. Exit Interface Routes
* **Next-Hop Route:** Uses the IP address of the next-hop router interface.
  ```ios
  R1(config)# ip route 192.168.2.0 255.255.255.0 10.10.10.2
  ```
* **Directly Attached Route:** Uses the local exit interface of the routing device.
  ```ios
  R1(config)# ip route 192.168.2.0 255.255.255.0 serial 0/0/0
  ```

#### 2. Default Static Routes
The gateway of last resort. Matches all destinations that do not have a specific route entry in the routing table.
```ios
R1(config)# ip route 0.0.0.0 0.0.0.0 209.165.202.129
```
