### 2.2 Longest Prefix Match (LPM) Algorithmic Routing

#### 1. The Longest Prefix Match Rule
When a router parses a destination IP address, it may find multiple matching prefixes in its routing table. For example, a single IP might match a broad classless route, a smaller subnet route, and a default route simultaneously.

To resolve this, routers use the **Longest Prefix Match (LPM)** rule:

$$\text{Selected Route} = \max_{j} \left( \text{Prefix Length}_j \right) \quad \text{where} \quad \text{Destination IP} \text{ matches } \text{Route}_j$$

The router selects the matching route with the longest prefix length (the largest number of masked bits), as it represents the most specific path to the destination.

---

#### 2. Step-by-Step Binary Lookup Simulation

##### Scenario
A gateway router receives an IP packet destined for the host address **`192.168.1.57`**. Its internal routing table contains the following candidate routes:
* **Route A:** `192.168.0.0/16`
* **Route B:** `192.168.1.0/24`
* **Route C:** `192.168.1.48/28`

##### Step 1: Binary Conversion of Destination IP
Convert the target IP address `192.168.1.57` into its 32-bit binary representation:

$$\begin{aligned}
192_{10} & \implies \mathbf{11000000}_2 \\
168_{10} & \implies \mathbf{10101000}_2 \\
1_{10} & \implies \mathbf{00000001}_2 \\
57_{10} & \implies \mathbf{00111001}_2
\end{aligned}$$

$$\text{Destination IP Binary: } \mathbf{11000000.10101000.00000001.00111001}$$

---

##### Step 2: Binary Comparison with Candidate Routes

###### Evaluating Route A (`192.168.0.0/16`)
* **Mask:** First 16 bits must match.
* **Route A Prefix Binary:** $\mathbf{11000000.10101000}.00000000.00000000$
* **First 16 Bits of Destination IP:** $\mathbf{11000000.10101000}$
* **Result:** **Match!** (Route A is a valid candidate).

###### Evaluating Route B (`192.168.1.0/24`)
* **Mask:** First 24 bits must match.
* **Route B Prefix Binary:** $\mathbf{11000000.10101000.00000001}.00000000$
* **First 24 Bits of Destination IP:** $\mathbf{11000000.10101000.00000001}$
* **Result:** **Match!** (Route B is a valid candidate; prefix length 24 is more specific than 16).

###### Evaluating Route C (`192.168.1.48/28`)
* **Mask:** First 28 bits must match.
* **Route C Prefix Binary:** $\mathbf{11000000.10101000.00000001.0011}0000$
* **First 28 Bits of Destination IP:** $\mathbf{11000000.10101000.00000001.0011}$
* **Result:** **Match!** (Route C is a valid candidate; prefix length 28 is more specific than 24).

---

##### Step 3: Selection
Comparing the prefix lengths of all matching candidates:

$$\text{Prefix Lengths: } \{\text{Route A: } 16, \text{ Route B: } 24, \text{ Route C: } \mathbf{28} \}$$

$$\max(16, 24, 28) = 28$$

The router selects **Route C (`192.168.1.48/28`)** as the forwarding path because it has the longest prefix match (28 matching bits), providing the most specific route to the destination.
