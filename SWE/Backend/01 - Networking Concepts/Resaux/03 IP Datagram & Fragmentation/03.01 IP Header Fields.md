### 3.1 IP Header Fields

```mermaid

    subgraph IPv4_Header["IPv4 Header (20+ bytes)"]
        direction TB

        Row3["TTL (8) | Protocol (8) | Header Checksum (16)"]

        Row5["Destination IP Address (32)"]
    end
```

* **Version (4 bits):** Set to `4` for IPv4.
* **Internet Header Length - IHL (4 bits):** The length of the header in 32-bit (4-byte) words.
  $$\text{Header Length in Bytes} = \text{IHL Value} \times 4$$
  * *Example:* If the IHL value is `5`, the header length is 20 bytes.
* **Total Length (16 bits):** The size of the entire IP packet (Header + Payload) in bytes.
  $$\text{Payload Data Size} = \text{Total Length} - (\text{IHL Value} \times 4)$$
* **Identification (16 bits):** A unique identifier shared by all fragments of an exploded datagram, used by the receiving host to reassemble the packet.
* **Flags (3 bits):**
  * **Bit 0 (Reserved):** Must be `0`.
  * **Bit 1 - Don't Fragment (DF):** If set to `1`, the router cannot fragment the packet. If the packet exceeds the path MTU, it is dropped and an ICMP Destination Unreachable message is returned.
  * **Bit 2 - More Fragments (MF):** Set to `1` for all intermediate fragments; set to `0` for the final fragment.
* **Fragment Offset (13 bits):** Specifies the position of the fragmented payload relative to the start of the original payload, measured in **8-byte blocks (64-bit blocks)**.
