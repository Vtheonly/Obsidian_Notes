# 1. Advanced Relational Algebra Functions

Relational Algebra is not limited to set theory and joins. It also requires the ability to manipulate scalar values (strings, numbers) and calculate statistics (aggregations) to answer complex business queries.

## 2. String Manipulation and Pattern Matching (LIKE)

While standard operators ($=, \neq$) are used for exact matches, the `LIKE` operator provides pattern matching using wildcards.

*   `_` (Underscore): Represents exactly **one** character of any type.
    *   *Example:* $\sigma_{Nom \ LIKE \ 'T\_\_\_\_\_'} (Employé)$ returns names starting with 'T' that are exactly 6 characters long (T + 5 underscores).
    *   *Example:* $\sigma_{Nom \ LIKE \ '\_a\%'} (Employé)$ returns names where the **second** letter is 'a'.
*   `%` (Percent sign): Represents a string of **zero or more** characters.
    *   *Example:* $\sigma_{Nom \ LIKE \ 'T\%'} (Employé)$ returns all names starting with 'T', regardless of length.
    *   *Example:* $\sigma_{Nom \ LIKE \ '\%y'} (Employé)$ returns all names ending with 'y'.

**Transformation Functions:**
You can also format strings before evaluating them using functions like `LOWER()`, `UPPER()`, `LENGTH()`, `CONCAT()`, `LEFT()`, `RIGHT()`, and `MID()`.
*   *Usage Example:* $\sigma_{UPPER(Nom) \ LIKE \ 'TREMBLAY'} (Employé)$ ensures case-insensitive matching.

## 3. Arithmetic Operations

Arithmetic functions can be embedded directly within Selections ($\sigma$) or Projections ($\pi$).

*   **Standard Math:** `+`, `-`, `*`, `/`
*   **Advanced Math:** 
    *   `^` (Exponent)
    *   `div` (Integer division, e.g., $5 \ div \ 2 = 2$)
    *   `mod` (Modulo/Remainder, e.g., $5 \ mod \ 2 = 1$)
*   **Built-in Functions:** `ABS()` (Absolute value), trigonometric functions (`COS`, `SIN`), logarithms (`LOG`), and rounding (`ROUND`, `FLOOR`, `CEIL`).

*Usage Example (Applying a 5% raise to filter):*
$\sigma_{Salaire \ * \ 1.05 \ < \ 25000} (Employé)$

## 4. Aggregate Operators ($\lambda$)

Aggregate operators perform statistical calculations over a set of tuples and return a **single scalar value** (or a single value per group).

**Available Functions:**
*   `COUNT`: Number of non-null values.
*   `MIN` / `MAX`: Smallest / Largest value.
*   `SUM`: Total sum of a numeric column.
*   `AVG`: Average of a numeric column.

**Notation and Grouping:**
The notation is $\lambda_{f1, f2, ...}(R)$, where $f$ is the function.
However, if you want to perform a **GROUP BY**, you prefix the $\lambda$ with the grouping attributes.

*Notation Formula:* $a1, a2 \ \lambda \ f1(b1), f2(b2) \ (R)$
*   `a1, a2`: The attributes to group by.
*   `f1(b1)`: The aggregate function applied to column b1.

**Example 1: Global Count**
If no grouping attribute is specified, it aggregates the whole table.
$R' = \lambda_{COUNT(IdProduit)} (Producteur)$ -> Returns a single number representing all products.

**Example 2: Group By**
Calculate the number of products produced by EACH producer.
$R' = Nom, Prénom \ \lambda_{COUNT(IdProduit)} (Producteur)$
*   This groups the table by `Nom` and `Prénom`. 
*   For each unique person, it counts their `IdProduit`.
*   The resulting table automatically names the new column `COUNT_IdProduit`.

## 5. Renaming Attributes during Operations

A very useful shortcut in relational algebra is renaming the output attributes directly during an operation (like a Cartesian product or a Join) to avoid naming collisions or to clarify the output.

**Syntax:** 
$NewRelationName(Col1, Col2, Col3) = R1 \times R2$

*   The number of declared columns must exactly match the number of output columns generated by the operation.
*   The names are applied sequentially from left to right.
*   *Why use this?* If $R1$ has an `Id` column and $R2$ has an `Id` column, doing $R1 \times R2$ creates two columns both named `Id`. By explicitly renaming them (e.g., `Prod_Id`, `Client_Id`) at the moment of creation, you prevent ambiguity for subsequent operations.
