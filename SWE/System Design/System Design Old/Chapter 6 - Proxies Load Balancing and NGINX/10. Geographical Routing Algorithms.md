# Geographical Routing Algorithms

## Overview
Geographical routing (also called geo-routing or geographic load balancing) directs client requests to the nearest or most appropriate backend server based on the client's geographic location. In globally distributed systems with data centers in multiple regions, geo-routing minimizes latency by serving users from the closest location, improves availability by failing over to alternate regions during outages, and enables compliance with data residency regulations by routing users to servers in the appropriate jurisdiction. Geo-routing is typically implemented at the DNS level (Global Server Load Balancing, or GSLB) or at the CDN edge, using the client's IP address to estimate their location and selecting the best region accordingly. Understanding geo-routing is essential for designing systems that serve a global user base with low latency and high availability.

## Key Concepts

### IP-to-Location Mapping
Geo-routing relies on mapping the client's IP address to a geographic location using a GeoIP database. These databases (such as MaxMind GeoIP2, IP2Location, and DB-IP) map IP address ranges to countries, regions, cities, and approximate latitude/longitude coordinates. The mapping is not perfectly accurate -- it can be off by city or even region, and it cannot determine location for some IP ranges (such as VPN exit nodes or anonymizing proxies). However, it is accurate enough for country-level and continent-level routing, which is sufficient for most geo-routing use cases. The databases must be updated regularly because IP address allocations change over time. Some implementations also use the EDNS Client Subnet (ECS) extension, which passes a truncated client IP address through the DNS resolution chain, allowing authoritative DNS servers to make location-aware decisions even when the query comes through a recursive resolver in a different location.

### DNS-Based Geo-Routing (GSLB)
Global Server Load Balancing (GSLB) is the most common mechanism for geo-routing, operating at the DNS level. When a client resolves a domain name, the DNS query reaches the GSLB-enabled authoritative DNS server. The server examines the source IP of the DNS query (or the ECS extension), determines the client's approximate location, and returns the IP address of the nearest data center. For example, a user in Europe resolving api.example.com might receive the IP address of the European data center, while a user in Asia receives the Asian data center's IP. GSLB also supports health checking: if the primary data center for a region is down, the GSLB can return the IP of a backup data center. DNS TTL (Time to Live) values affect how quickly clients are redirected after a failover -- low TTLs enable faster failover but increase DNS query volume.

### Anycast Routing
Anycast is a network-level geo-routing technique where the same IP address is advertised from multiple data centers. Routers on the internet automatically direct traffic to the nearest instance of that IP address based on BGP routing metrics, which generally correspond to network proximity. Anycast provides automatic geo-routing without DNS-level intelligence -- the routing decision is made by the internet's routing infrastructure. It is commonly used by CDNs and DNS providers (Cloudflare, Google DNS, and Akamai all use anycast) to serve users from the nearest edge location. Anycast provides fast failover because if a data center becomes unavailable, its routes are withdrawn and traffic automatically shifts to the next-nearest location. However, anycast requires control over BGP routing, which is typically available only to organizations with their own autonomous system numbers and direct internet transit relationships.

### Latency-Based Routing
Latency-based routing improves on geographic routing by selecting the data center with the lowest actual network latency to the client, rather than the one that is geographically closest. Geographic proximity does not always correlate with network latency because internet routing, submarine cable paths, and peering arrangements can create counter-intuitive latency patterns. For example, a user in South America might have lower latency to a US data center than to a data center in their own country due to better network infrastructure. Latency-based routing requires the load balancer or DNS service to maintain a latency matrix that tracks the observed or measured latency between client network ranges and each data center. AWS Route 53 offers latency-based routing as a first-class feature, continuously measuring latency between its edge locations and the configured endpoints.

### Data Residency and Compliance Routing
Geo-routing is also used to enforce data residency regulations, which require certain types of data to be stored and processed within specific jurisdictions. For example, GDPR requires that personal data of EU residents be processed within the EU (or in jurisdictions with adequate data protection), and some countries require all citizen data to remain within national borders. Geo-routing can enforce these requirements by directing users from regulated regions to data centers within those regions. This requires accurate IP-to-country mapping and careful configuration to ensure that no cross-region data leakage occurs. Compliance routing must also handle edge cases, such as users who appear to be in one country but are actually in another (due to VPN usage), and must provide mechanisms for manual override when the automated routing is incorrect.

## Practical Applications
CDNs like Cloudflare, Akamai, and AWS CloudFront use geo-routing to serve static content from the nearest edge location to the user. Global SaaS platforms like Slack and Notion use geo-routing to direct users to the nearest regional deployment for low-latency access. AWS Route 53 provides geo-routing and latency-based routing as configurable policies. Google's global load balancer uses a single anycast IP address that routes traffic to the nearest Google Cloud region. Streaming services like Netflix use geo-routing to direct users to the nearest content cache, reducing bandwidth costs and improving streaming quality. Financial institutions use compliance routing to ensure that customer data is processed within the appropriate regulatory jurisdiction.
