# Storage Types: Disk Drives and Solid State Drives

## Overview

Storage is where data persists beyond the lifetime of a running process. The two dominant storage technologies are Hard Disk Drives (HDDs) and Solid State Drives (SSDs). Each has fundamentally different physical characteristics that affect latency, throughput, cost, and durability. Understanding these differences is critical for system designers because storage choice directly impacts database performance, application latency, and infrastructure cost. At extreme scales, such as handling hundreds of thousands of requests per second, the choice between HDD and SSD can determine whether a system meets its service level objectives or fails under load.

## Key Concepts

### Hard Disk Drives (HDDs)

HDDs store data on spinning magnetic platters. Each platter is divided into concentric tracks, and each track is further divided into sectors. A read/write head positioned on an actuator arm floats just above the platter surface and reads or writes data as the platter rotates. The rotational speed, measured in revolutions per minute (RPM), directly affects access latency. Common RPM values are 5,400, 7,200, 10,000, and 15,000. The time to access data on an HDD consists of three components: seek time (the time for the read head to move to the correct track), rotational latency (the time for the platter to rotate to the correct sector), and transfer time (the time to read or write the data). A typical 7,200 RPM drive has an average seek time of about 8-9 milliseconds, making HDDs orders of magnitude slower than RAM for random access patterns.

### Solid State Drives (SSDs)

SSDs store data in NAND flash memory cells, which have no moving parts. Data is stored by trapping electrical charge in floating-gate transistors. NAND flash comes in several types: SLC (Single-Level Cell) stores one bit per cell and offers the highest endurance and speed, MLC (Multi-Level Cell) stores two bits per cell with lower cost but reduced endurance, TLC (Triple-Level Cell) stores three bits per cell with even lower cost and endurance, and QLC (Quad-Level Cell) stores four bits per cell at the lowest cost but with the lowest endurance and slowest write speeds. SSDs offer dramatically lower latency than HDDs for random reads, typically in the range of 0.1 milliseconds (100 microseconds) compared to 10 milliseconds for HDDs.

### IOPS Comparison

Input/Output Operations Per Second (IOPS) is a key metric for comparing storage performance. A typical 7,200 RPM HDD delivers about 75-100 random read IOPS, while a 15,000 RPM HDD delivers about 180-210 IOPS. In contrast, a consumer-grade SSD can deliver 50,000-100,000 random read IOPS, and enterprise NVMe SSDs can exceed 500,000 IOPS. This difference of three to four orders of magnitude means that for random access workloads such as database queries, SSDs are fundamentally superior. However, for sequential access patterns like log file writes or large file transfers, HDDs can perform reasonably well because sequential reads and writes minimize seek time and rotational latency.

### Sequential vs Random Access

The performance gap between HDDs and SSDs is most dramatic for random access patterns. On an HDD, each random read requires a seek and rotational latency, making random IOPS very low. On an SSD, there is no mechanical movement, so random access latency is nearly as fast as sequential access. This distinction is crucial for database design: systems that perform many small random reads (such as key-value lookups) benefit enormously from SSDs, while systems that primarily write sequential logs may find HDDs cost-effective. System designers must understand the access patterns of their applications to choose the right storage tier.

### Wear Leveling and Endurance

NAND flash cells degrade with each write cycle. SLC cells can endure approximately 100,000 write cycles, MLC about 3,000-10,000, and TLC about 1,000-3,000. SSD controllers implement wear leveling algorithms that distribute writes evenly across all cells to prevent any single cell from wearing out prematurely. They also reserve extra capacity (over-provisioning) to replace cells that have worn out. Despite these concerns, modern enterprise SSDs have sufficient endurance for most workloads, and the performance benefits far outweigh the endurance limitations in typical server applications.

## Practical Applications

When designing a database tier for a high-traffic application, SSDs are the standard choice for primary storage because database workloads are dominated by random access patterns. HDDs still have a role in archival storage and cold data tiers where cost per gigabyte matters more than access speed. Many cloud providers offer tiered storage: NVMe SSD for the hottest data, standard SSD for warm data, and HDD-backed storage for cold data. Understanding IOPS and throughput characteristics helps engineers provision the right storage type and avoid performance bottlenecks that would be extremely expensive to fix at scale.
