# Binary Data and Bits

## Overview

At the deepest level, computers understand only binary: zeros and ones. Every piece of data that flows through a computer system, whether it is a text character, an image pixel, a database record, or a network packet, is ultimately represented as a sequence of binary digits. Understanding binary representation is foundational to system design because it governs how data is stored, transferred, and processed across every component in a distributed system. Without a solid grasp of binary data, engineers cannot reason about memory limits, network throughput, storage capacity, or data serialization formats.

## Key Concepts

### Bits and Bytes

A bit (short for binary digit) is the smallest unit of data in computing. It can hold exactly one of two values: 0 or 1. Bits correspond physically to the presence or absence of an electrical charge, a magnetic polarity, or a photon state in optical systems. Eight bits grouped together form one byte. A single byte can represent 256 distinct values (2 to the 8th power), which is sufficient to encode a single ASCII character such as the letter "A" or the digit "7." This relationship between bits and bytes is critical when calculating data sizes: 1 gigabyte (GB) equals 8 gigabits (Gb). Network speeds are typically measured in bits per second, while storage is measured in bytes, and confusing the two leads to order-of-magnitude errors.

### Binary Number System

The binary number system uses base 2, meaning each digit position represents a power of 2 rather than a power of 10 as in the decimal system. For example, the binary number 1011 equals (1 x 2^3) + (0 x 2^2) + (1 x 2^1) + (1 x 2^0) = 8 + 0 + 2 + 1 = 11 in decimal. Converting between binary and decimal is a fundamental skill for anyone working close to hardware or network protocols. Hexadecimal (base 16) is commonly used as a shorthand for binary because each hex digit maps directly to 4 bits, making it easier for humans to read large binary values.

### Byte Addressing and Word Size

Memory in modern computers is byte-addressable, meaning each byte in memory has a unique address. The word size of a processor refers to the number of bits it can process in a single operation. A 64-bit processor has a word size of 64 bits (8 bytes), meaning it can read or write 8 bytes at a time from memory. The word size also determines the maximum amount of memory a processor can address: a 32-bit processor can address 2^32 bytes (4 GB), while a 64-bit processor can address 2^64 bytes (16 exabytes). This is why modern systems moved to 64-bit architectures as memory requirements grew beyond 4 GB.

### Endianness

Endianness describes the order in which bytes are stored in memory for multi-byte values. Big-endian systems store the most significant byte at the lowest memory address, while little-endian systems store the least significant byte first. The x86 architecture uses little-endian, while network protocols (TCP/IP) use big-endian, also called network byte order. When data crosses a network boundary, byte order conversion may be necessary, and failing to handle endianness correctly is a common source of bugs in distributed systems.

### Bit Manipulation

Bit manipulation involves using bitwise operators (AND, OR, XOR, NOT, left shift, right shift) to operate directly on the binary representation of data. These operations are extremely fast because they map directly to CPU instructions. Common applications include setting or clearing flags in a bitfield, computing hash values, implementing bloom filters, and optimizing memory usage by packing multiple values into a single integer. Understanding bit manipulation is essential for performance-critical code and for understanding how data structures work at a low level.

## Practical Applications

Binary data representation directly impacts system design in several ways. Network throughput calculations must account for the difference between bits and bytes: a 10 Gbps network link can transfer 1.25 GB per second, not 10 GB per second. Data serialization formats like Protocol Buffers and MessagePack use variable-length binary encoding to minimize the number of bytes transferred over the network. Database storage engines pack rows into binary formats that optimize for both space and access speed. Understanding binary also helps engineers reason about data alignment, padding, and cache line utilization, all of which affect system performance at scale.
