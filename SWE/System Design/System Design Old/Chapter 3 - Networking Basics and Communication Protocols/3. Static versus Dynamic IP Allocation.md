# Static versus Dynamic IP Allocation

## Overview
IP address allocation determines how devices receive their network addresses. Static IP allocation assigns a fixed, permanent address to a device, while dynamic allocation leverages protocols like DHCP to assign addresses automatically and temporarily. The choice between static and dynamic allocation has significant implications for service availability, network management overhead, and the overall reliability of distributed systems.

## Key Concepts

### Static IP Allocation
Static IP addresses are manually configured on a device and remain constant across reboots and network changes. Servers hosting websites, databases, DNS resolvers, and other critical infrastructure typically use static IPs because their addresses must be predictable and stable. If a DNS A record points to a specific IP address, that address must not change unexpectedly, or the service becomes unreachable. Static allocation requires careful documentation and coordination to prevent address conflicts, and it does not scale well in large networks without automated IP address management tools.

### Dynamic IP Allocation with DHCP
The Dynamic Host Configuration Protocol (DHCP) automates the process of assigning IP addresses to devices on a network. When a device connects, it broadcasts a DHCP discovery request, and a DHCP server responds with an offered IP address from a configurable pool, along with subnet mask, default gateway, and DNS server information. DHCP leases have a finite time-to-live, after which the device must renew the lease or receive a different address. This approach is ideal for workstations, mobile devices, and IoT sensors that do not need a consistent address.

### Lease Management
DHCP leases include a time-to-live value that determines how long a device can retain its assigned address. Short lease times enable rapid address reuse but generate more DHCP traffic, while long lease times reduce overhead but can lead to address exhaustion if devices leave the network without releasing their leases. Administrators must tune lease durations based on network size, device turnover, and available address pool capacity.

### Elastic IPs in Cloud Environments
Cloud providers offer elastic IP addresses, which are static public IPv4 addresses designed for dynamic cloud computing. Unlike traditional static IPs, elastic IPs can be quickly remapped from one instance to another, enabling rapid failover and instance replacement without DNS changes. However, cloud providers often charge for elastic IPs that are allocated but not associated with a running instance, incentivizing efficient use of the limited IPv4 address pool.

### When to Use Each Approach
Static allocation is appropriate for servers, network infrastructure devices, and any service that must be reliably located at a known address. Dynamic allocation is preferred for client devices, development environments, and auto-scaling groups where instances are frequently created and destroyed. In many production environments, a hybrid approach is used: critical services have static IPs while ephemeral resources use DHCP, with DNS records updated dynamically to reflect the current mapping.

## Practical Applications
A production web application might use an elastic IP address on its load balancer so that DNS records remain stable, while application server instances in an auto-scaling group receive dynamic private IPs through DHCP. Service discovery tools like Consul or etcd can register dynamic IPs with an internal DNS service, allowing other services to locate them without manual configuration. This combination of static and dynamic allocation balances stability with flexibility.
