

The histogram is the most fundamental tool for analyzing the tonal distribution of an image. It provides a global description of the appearance of an image without any spatial information (it tells you *how many* pixels are bright, but not *where* they are).

## 1. Histogram Basics

For a grayscale image with a bit-depth of 8 bits (values 0 to 255):
*   **X-Axis:** Represents the gray levels (0 = Pure Black, 255 = Pure White).
*   **Y-Axis:** Represents the count (frequency) of pixels at that specific gray level.

### Visual Analysis (How to "Read" an Image)
*   **Underexposed (Too Dark):** The curve is huddled on the **left**.
*   **Overexposed (Too Bright):** The curve is huddled on the **right**.
*   **Low Contrast (Washed Out):** The curve is narrow and clustered in the **middle**. It doesn't use the full range of black or white.
*   **High Contrast:** The curve spans the entire width (0 to 255). A very high contrast image might have peaks only at the far left and far right (binary-like).

## 2. The Cumulative Histogram ($H_c$)

While the standard histogram counts pixels, the cumulative histogram counts probabilities up to a certain point. It is essential for algorithms like Equalization.

*   **Definition:** For a gray level $k$, $H_c(k)$ is the sum of all pixel counts for levels $0, 1, \dots, k$.
*   **Mathematical Property:** It is a monotonically increasing function. $H_c(255)$ equals the total number of pixels in the image.

## 3. Dynamic Expansion (Linear Stretching)

This is a point-operation technique used to fix **Low Contrast** images.
**Goal:** Stretch the narrow histogram of an image so it fills the entire available dynamic range (usually [0, 255]).

### The Algorithm
1.  Identify the lowest gray level currently in the image ($X_{min}$) and the highest ($X_{max}$).
2.  Define the target range (usually $Y_{min}=0$ and $Y_{max}=255$).
3.  Apply a linear transformation to every pixel $(i,j)$ in the image.

### The Formula (Slide 158)
The transformation is a linear equation $Y = \alpha + \beta X$:

$$ Y(i,j) = \frac{Y_{max} - Y_{min}}{X_{max} - X_{min}} (X(i,j) - X_{min}) + Y_{min} $$

**Simplified for [0, 255] output:**
$$ Y(i,j) = 255 \times \frac{X(i,j) - X_{min}}{X_{max} - X_{min}} $$

*   **Intuition:** Pixels that were originally $X_{min}$ become 0. Pixels that were $X_{max}$ become 255. Everything in between is stretched proportionally.
*   **Limitation:** It is linear. It cannot fix non-linear lighting issues (like a dark foreground against a bright sky).
