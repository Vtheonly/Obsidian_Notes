### 3.3 Practical Walkthrough: Fragmenting a 2000-byte Datagram

*Given Scenario:*
* Total Datagram Size ($D_{\text{original}}$) = 2000 bytes
* Ident Identifier = 10
* Header Length ($H$) = 20 bytes (minimum size)
* Outbound Interface MTU = 512 bytes

#### Walkthrough Calculations

1. **Calculate Payload:**
   $$D_{\text{data}} = 2000 - 20 = 1980 \text{ bytes}$$

2. **Calculate Maximum Payload Capacity per Fragment ($P_{\text{max}}$):**
   $$\text{Maximum raw capacity} = 512 - 20 = 492 \text{ bytes}$$
   Check if 492 is a multiple of 8:
   $$\frac{492}{8} = 61.5 \implies \text{Not a multiple of 8}$$
   Apply the 8-byte alignment rule:
   $$P_{\text{max}} = \lfloor 61.5 \rfloor \times 8 = 61 \times 8 = 488 \text{ bytes}$$

3. **Calculate Total Number of Fragments:**
   $$N_{\text{frag}} = \left\lceil \frac{1980}{488} \right\rceil = \lceil 4.057 \rceil = 5 \text{ fragments}$$
