# The "Shipping Company" Analogy

Imagine an IP packet is a giant package being shipped.

The IPv4 header is simply the shipping label attached to the box.

Every field answers one question:

|Field|Question|
|---|---|
|Version|Which shipping system are we using?|
|IHL|How big is the shipping label?|
|Total Length|How big is the entire package?|
|Identification|Which package family does this piece belong to?|
|Flags|Are we allowed to cut it apart?|
|Fragment Offset|Where does this piece belong?|
|TTL|How many cities can it pass through before being destroyed?|
|Protocol|Which department receives it?|
|Checksum|Is the label damaged?|
|Source IP|Who sent it?|
|Destination IP|Who receives it?|

# The Fragmentation Formula Chain

The entire exercise is always the same 4-step recipe.

Memorize:

> Data → Max → Number → Table

or

> D → M → N → T

Think:

**"Don't Make Networking Terrifying."**

---

## Step 1

Find Data.

```text
Original Packet
      -
Header
      =
Payload
```

Formula:

[  
D_{data}=D_{original}-H  
]

Memory:

> Remove the wrapper first.

---

## Step 2

Find Max Payload.

[  
P_{max}=\left\lfloor\frac{MTU-H}{8}\right\rfloor\times8  
]

Memory:

> MTU minus header, then round DOWN to a multiple of 8.

This is the rule most people forget.

Whenever you see fragmentation, immediately think:

```text
MULTIPLE OF 8
MULTIPLE OF 8
MULTIPLE OF 8
```

because offsets count in 8-byte blocks.

---

## Step 3

Find Number of Fragments.

[  
N=\left\lceil\frac{Data}{MaxPayload}\right\rceil  
]

Memory:

```text
Data ÷ Capacity
Round UP
```

Because even 1 remaining byte needs another fragment.

---

## Step 4

Build Table.

For every fragment:

```text
Payload
Length
Offset
MF
```

Always in that order.

Think:

> PLOM

Pronounce it:

**"Plum"**

Whenever fragmentation appears:

```text
Payload
Length
Offset
MF
```

# The Offset Trick

Students constantly forget offsets.

Never calculate them individually.

Just count how many payload chunks came before.

```text
Offset = Bytes Before Me ÷ 8
```

That's it.

---

Example:

```text
Payload size = 488
```

Fragment 1:

```text
0 bytes before me

0 / 8 = 0
```

Fragment 2:

```text
488 bytes before me

488 / 8 = 61
```

Fragment 3:

```text
976 bytes before me

976 / 8 = 122
```

Fragment 4:

```text
1464 bytes before me

1464 / 8 = 183
```

Fragment 5:

```text
1952 bytes before me

1952 / 8 = 244
```

Notice:

```text
0
61
122
183
244
```

You are just adding 61 each time.

Because:

```text
488 / 8 = 61
```

This lets you compute offsets mentally.

---

# The MF Flag Trick

Imagine a movie series.

```text
To be continued...
```

MF = 1

There is another fragment coming.

---

Final episode:

```text
The End
```

MF = 0

No more fragments.

---

So remember:

```text
MF = 1 → More coming
MF = 0 → Last fragment
```

# Ultra-Short Exam Cheat Memory

If you're in an exam and panic, remember:

```text
1. Data = Original - Header

2. MaxPayload = (MTU - Header)
                 rounded down to multiple of 8

3. Fragments = Data / MaxPayload
               rounded up

4. Offset = PreviousBytes / 8

5. MF = 1 except last fragment
```

That's genuinely about 90% of IPv4 fragmentation questions.

The entire chapter can be compressed into one sentence:

> "Remove the header, fit payload into MTU-sized chunks rounded to multiples of 8, count fragments, divide previous bytes by 8 for offsets, and set MF=1 until the last piece."
