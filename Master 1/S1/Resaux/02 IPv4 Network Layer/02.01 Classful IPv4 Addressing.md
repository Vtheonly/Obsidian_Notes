### 2.1 Classful IPv4 Addressing

Classful addressing partitions the IPv4 32-bit space into distinct classes based on the high-order bits of the first octet.

| Class | First Octet Range (Dec) | High-Order Bits | Default Mask | Network/Host Split | Purpose |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Class A** | $1 - 126$ | `0` | `/8` ($255.0.0.0$) | $8 \text{ Net} / 24 \text{ Host}$ | Large Networks |
| **Class B** | $128 - 191$ | `10` | `/16` ($255.255.0.0$) | $16 \text{ Net} / 16 \text{ Host}$ | Medium Networks |
| **Class C** | $192 - 223$ | `110` | `/24` ($255.255.255.0$) | $24 \text{ Net} / 8 \text{ Host}$ | Small Networks |
| **Class D** | $224 - 239$ | `1110` | N/A | N/A | Multicast |
| **Class E** | $240 - 255$ | `1111` | N/A | N/A | Experimental |

#### Reserved & Special-Purpose Spaces
* **Loopback/Diagnostic:** `127.0.0.0/8` (e.g., `127.1.1.1` is reserved and cannot be assigned to hosts).
* **Limited Broadcast:** `255.255.255.255` (floods the local network segment but is blocked by routers).
* **Network Identifier:** An IP address where all host bits are set to binary `0`.
* **Directed Broadcast:** An IP address where all host bits are set to binary `1` (e.g., on a class C network `192.168.1.0/24`, the directed broadcast is `192.168.1.255`).
