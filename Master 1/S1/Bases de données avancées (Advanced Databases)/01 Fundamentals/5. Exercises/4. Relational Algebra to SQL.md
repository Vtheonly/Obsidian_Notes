# 4. Relational Algebra to SQL

This note maps mathematical relational algebra concepts to actual SQL code.

## Key Symbols

- $\pi$ (Pi) = **Projection** (Select specific columns).
- $\sigma$ (Sigma) = **Selection** (Filter rows / WHERE).
- $\bowtie$ (Bowtie) = **Join**.
- $\cup$ (Union), $\cap$ (Intersection), $-$ (Difference).

## The 20 Query Examples

### Basic Operations

**1. Projection** (Only specific columns)

- Algebra: $\pi_{Name, Major}(Students)$
- SQL: `SELECT Name, Major FROM Students;`

**2. Selection** (Filtering Rows)

- Algebra: $\sigma_{Age > 21}(Students)$
- SQL: `SELECT * FROM Students WHERE Age > 21;`

**3. Selection + Projection** (Filter then pick columns)

- Algebra: $\pi_{Name}(\sigma_{Major='CS'}(Students))$
- SQL: `SELECT Name FROM Students WHERE Major = 'CS';`

### Set Operations

**4. Union** (Combine unique lists)

- Algebra: $\pi_{Name}(Students) \cup \pi_{Name}(Instructors)$
- SQL: `SELECT Name FROM Students UNION SELECT Name FROM Instructors;`

**5. Intersection** (Common values)

- Algebra: $\pi_{Name}(Students) \cap \pi_{Name}(Instructors)$
- SQL: `SELECT Name FROM Students INTERSECT SELECT Name FROM Instructors;`
- _MySQL:_ `... WHERE Name IN (SELECT Name ...)`

**6. Difference** (In A but not B)

- Algebra: $\pi_{CID}(Courses) - \pi_{CID}(Enrollments)$
- SQL: `SELECT CID FROM Courses EXCEPT SELECT CID FROM Enrollments;`
- _MySQL:_ `... WHERE CID NOT IN (SELECT CID ...)`

### Joins and Products

**7. Cartesian Product** (All combinations)

- Algebra: $Students \times Courses$
- SQL: `SELECT * FROM Students, Courses;`

**8. Natural Join** (Auto-match columns)

- Algebra: $Students \bowtie Enrollments$
- SQL: `SELECT * FROM Students NATURAL JOIN Enrollments;`

**9. Theta Join / Inner Join** (Standard linking)

- Algebra: $Students \bowtie_{S.SID=E.SID} Enrollments$
- SQL: `SELECT * FROM Students S JOIN Enrollments E ON S.SID = E.SID;`

**10. Left Outer Join** (Keep all students)

- Algebra: $Students \leftouterjoin Enrollments$
- SQL: `SELECT * FROM Students LEFT JOIN Enrollments ON ...`

### Aggregation and Advanced Logic

**11. Rename**

- Algebra: $\rho_{CourseName/Title}(\pi_{Title}(Courses))$
- SQL: `SELECT Title AS CourseName FROM Courses;`

**12. Count**

- Algebra: $\Im_{COUNT(SID)}(Students)$
- SQL: `SELECT COUNT(SID) FROM Students;`

**13. Average**

- Algebra: $\Im_{AVG(Age)}(Students)$
- SQL: `SELECT AVG(Age) FROM Students;`

**14. Group By** (Count per Major)

- Algebra: $_{Major}\Im_{COUNT(SID)}(Students)$
- SQL: `SELECT Major, COUNT(SID) FROM Students GROUP BY Major;`

**15. Division** (The "Has done ALL" query)

- _Scenario:_ Find students who took ALL courses.
- Algebra: $Students \div \pi_{CID}(Courses)$
- SQL: Uses `NOT EXISTS` twice (See detailed breakdown in Note 6).

**16. Logical AND**

- Algebra: $\sigma_{Major='CS' \land Age>20}(Students)$
- SQL: `SELECT * FROM Students WHERE Major='CS' AND Age > 20;`

**17. NULL Handling**

- Algebra: $\sigma_{Grade=null}(Enrollments)$
- SQL: `SELECT * FROM Enrollments WHERE Grade IS NULL;` (Crucial: Use `IS`, not `=`)

**18. Pattern Matching**

- Algebra: $\sigma_{Name \text{ like } 'J\%'}(Instructors)$
- SQL: `SELECT * FROM Instructors WHERE Name LIKE 'J%';`

**19. Distinct**

- Algebra: Implicit in set theory.
- SQL: `SELECT DISTINCT Department FROM Instructors;`

**20. Conditional Join** (Non-equi join)

- _Scenario:_ Students older than Instructors.
- SQL: `SELECT * FROM Students S JOIN Instructors I ON S.Major = I.Dept WHERE S.Age > I.Age;`
