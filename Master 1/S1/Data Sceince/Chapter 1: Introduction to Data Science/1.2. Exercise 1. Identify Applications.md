

## 1.2. Exercise 1. Identify Applications

### Problem Statement
Identify which field (**Data Science, ML, AI, Big Data, BI**) primarily applies to each of the following scenarios, providing thorough reasoning:
1. *A company wants to predict customer churn using historical data.*
2. *An app recommends products based on user behavior.*
3. *Storing and processing terabytes of sensor data from IoT devices.*
4. *A dashboard shows last year's sales trends.*
5. *An AI agent plays chess and learns strategies over time.*

---

### Solution and Analytical Reasoning

#### Scenario 1: Predicting customer churn using historical data
* **Primary Domain**: **Data Science** (and **Machine Learning**)
* **Reasoning**: The objective is to use historical customer attributes (demographics, transaction history, usage patterns) to predict a future event (whether a customer will leave). This task follows the classic Data Science lifecycle:
  * Ingesting the customer dataset from multiple databases.
  * Cleaning missing values and engineering relevant features (e.g., "days since last active").
  * Applying a classification model (e.g., Logistic Regression, XGBoost).
  * Evaluating model calibration and generating risk scores.

#### Scenario 2: Recommending products based on user behavior
* **Primary Domain**: **Machine Learning**
* **Reasoning**: Recommendation engines (collaborative filtering, content-based filtering, matrix factorization) rely on automated mathematical optimization. The system continuously maps user-item interactions to latent vector spaces to learn affinity patterns without explicit human-written rules, dynamically updating weights as new behavioral interactions occur.

#### Scenario 3: Storing and processing terabytes of sensor data from IoT devices
* **Primary Domain**: **Big Data**
* **Reasoning**: This scenario exhibits the classic characteristics of Big Data, defined by the "V"s:
  * **Volume**: Terabytes of ongoing storage requirements.
  * **Velocity**: Real-time high-speed data stream generation from IoT devices.
  * **Variety**: Raw, semi-structured telemetry data.
  The processing demands distributed filesystems (HDFS, S3) and processing frameworks (Apache Spark, Kafka) rather than local memory environments.

#### Scenario 4: A dashboard showing last year's sales trends
* **Primary Domain**: **Business Intelligence (BI)**
* **Reasoning**: This scenario is focused entirely on historical data analysis (descriptive analytics). It evaluates what occurred in the past (last year's trends) to report performance metrics and KPIs, presenting them through an interactive visualization layer (dashboard) to assist business leaders. No future-focused predictive modeling or autonomous learning is described.

#### Scenario 5: An AI agent plays chess and learns strategies over time
* **Primary Domain**: **Artificial Intelligence (AI)** (specifically **Reinforcement Learning**)
* **Reasoning**: This task represents an autonomous agent interacting with a dynamic environment (the chessboard) to maximize a long-term reward (winning the game). It uses a feedback loop of actions, states, and rewards, adapting its strategies over time to demonstrate cognitive reasoning and decision-making capabilities.
