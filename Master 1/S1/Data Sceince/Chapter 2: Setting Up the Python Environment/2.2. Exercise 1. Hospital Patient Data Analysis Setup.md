
## 2.2. Exercise 1. Hospital Patient Data Analysis Setup

### Problem Statement
Establish a clean, robust data science directory structure for a project called `HospitalDataAnalysis`. Set up environment isolation, install core libraries, generate a sample medical dataset, perform descriptive statistics, visualize data using `matplotlib`, configure a `.gitignore` file, and export the environment specification.

---

### Step-by-Step Implementation Guide

#### Step 1: Directory Setup and Project Structure
Execute the following commands in your shell to construct a standardized data science directory tree:

```bash
mkdir HospitalDataAnalysis
cd HospitalDataAnalysis
mkdir data notebooks scripts
touch .gitignore README.md
```

The resulting directory tree should match this configuration:
```
HospitalDataAnalysis/
├── env/                    # Virtual environment folder (ignored by Git)
├── data/                   # Folder containing raw and processed CSV assets
│   └── patients.csv
├── notebooks/              # Jupyter Notebook files
│   └── patients_analysis.ipynb
├── scripts/                # Utility production Python scripts
├── .gitignore              # Files and directories ignored by Git version control
└── README.md               # Markdown documentation describing the project
```

#### Step 2: Environment Isolation Setup
Initialize and activate your virtual environment:

```bash
# Windows
python -m venv env
.\env\Scripts\activate

# macOS / Linux
python -m venv env
source env/bin/activate
```

#### Step 3: Install Required Dependencies
Upgrade `pip` and install the analytical stack:

```bash
pip install --upgrade pip
pip install numpy pandas matplotlib jupyter
```

#### Step 5: Data Analysis Notebook Implementation
Create the notebook:
```bash
jupyter notebook notebooks/patients_analysis.ipynb
```

Add the following code blocks to the notebook cells:

##### Cell 1: Ingesting Data and Preliminary Inspection
```python
import pandas as pd
import matplotlib.pyplot as plt

# Load dataset using a relative file path
df = pd.read_csv("../data/patients.csv")

# Print the shape of the DataFrame (rows, columns)
print(f"Dataset Dimensions: {df.shape}")

# Preview the top 5 records
df.head()
```

##### Cell 2: Descriptive Statistical Calculations
```python
# Compute descriptive statistics for numerical columns
print("--- Numerical Column Statistics ---")
print(df.describe())

# Compute class distributions for categorical columns
print("\n--- Sex Categorical Distribution ---")
print(df['Sex'].value_counts())

print("\n--- Diagnosis Categorical Distribution ---")
print(df['Diagnosis'].value_counts())
```

##### Cell 3: Data Visualization using Matplotlib
```python
# Set the plot style and figure size
plt.figure(figsize=(8, 5))

# Generate scatter plot: Age vs Cholesterol, color-coded
plt.scatter(df['Age'], df['Cholesterol'], color='darkred', marker='o', s=80, alpha=0.8, edgecolor='black', label='Patients')

# Apply labels, title, grid, and legend
plt.xlabel("Age (Years)", fontsize=12)
plt.ylabel("Cholesterol Level (mg/dL)", fontsize=12)
plt.title("Patient Cholesterol Levels vs. Patient Age", fontsize=14, fontweight='bold')
plt.grid(True, linestyle='--', alpha=0.6)
plt.legend()

# Render plot cleanly in output window
plt.show()
```

#### Step 6: Git Version Control Setup
Initialize Git, construct a proper `.gitignore` file, link to a remote repository, and commit your changes:

```bash
git init
```

Open the `.gitignore` file and add the following lines:
```text
# Virtual environment folder
env/
.venv/
venv/

# Python cache files
__pycache__/
*.pyc
*.pyo
*.pyd

# Jupyter Notebook checkpoints
.ipynb_checkpoints/

# Local configuration files
.env
```

Link your local project to a remote repository, stage your changes, and push your initial commit:
```bash
# Link to remote (replace with your repository URL)
git remote add origin https://github.com/username/HospitalDataAnalysis.git

# Stage all tracked files
git add .

# Commit changes
git commit -m "Initial commit: Set up repository, data directory, patients CSV and descriptive notebook"

# Push to primary branch
git branch -M main
git push -u origin main
```

#### Step 7: Exporting Environment Configurations
To make your environment reproducible on other machines, export your current package specifications:

```bash
# If using a standard pip virtual environment
pip freeze > requirements.txt

# If using a Conda environment
conda env export > environment.yml
```

When checking out this repository on a different machine, dependencies can be re-installed using:
```bash
# For pip users
pip install -r requirements.txt

# For Conda users
conda env create -f environment.yml
```
