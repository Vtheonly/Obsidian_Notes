## 3.4. Advanced Virtualization Paradigms and Security

Hypervisors use different techniques to manage guest operating system instructions:

### 3.4.1. Full Virtualization
The guest operating system runs without modifications, unaware that it is virtualized. It attempts to execute sensitive kernel instructions directly on the CPU.
*   **Instruction Handling:** The hypervisor intercepts sensitive guest OS instructions (such as CPU ring 0 commands) and translates them on the fly into safe instructions.
*   **Advantages:** Complete compatibility with unmodified guest operating systems like Windows and Linux.
*   **Disadvantages:** Translation overhead reduces CPU performance compared to other virtualization models.

### 3.4.2. Hardware-Assisted Virtualization
This model leverages virtualization instructions built directly into physical processors (such as Intel VT-x and AMD-V).
*   **Instruction Handling:** The CPU provides a new, highly privileged execution mode (Root Mode) for the hypervisor, while guest virtual machines run in a separate mode (Non-Root Mode). Sensitive guest instructions are automatically handled by the hardware, eliminating the need for software translation.
*   **Advantages:** Higher execution speeds and lower CPU overhead compared to full virtualization.
*   **Disadvantages:** Requires hardware compatibility with modern Intel or AMD processors.

### 3.4.3. Paravirtualization
The guest operating system's kernel is modified before installation, making it aware of the hypervisor.
*   **Instruction Handling:** Instead of trying to execute sensitive ring 0 instructions directly, the modified guest kernel sends requests directly to the hypervisor using specialized calls called **hypercalls**.
*   **Advantages:** High performance, as the system eliminates translation overhead.
*   **Disadvantages:** Cannot run unmodified operating systems (like Windows) where the kernel code cannot be modified.

### 3.4.4. Virtualization Security Risks
*   **VM Escape:** An exploit where an attacker gains unauthorized access to the host hypervisor from within a virtual machine, allowing them to compromise other VMs on the same physical host.
*   **VM Sprawl:** Occurs when administrators deploy too many virtual machines without central tracking, leading to wasted hardware resources and unpatched security vulnerabilities.
*   **VM Hopping:** A security exploit where an attacker moves laterally from one compromised VM to another virtual machine running on the same hypervisor.
*   **Hyperjacking:** A highly advanced attack where malware installs itself below the host hypervisor, giving the attacker control over the entire system while remaining invisible to standard security tools.
