## 8.3. TP 4 and 5 - Private Cloud Deployment with Nextcloud and Docker

This lab guides you through deploying a private cloud storage solution using **Nextcloud** and **MariaDB** inside Docker containers.

### 8.3.1. Complete `docker-compose.yml` Configuration

```yaml
version: '3.9'

services:
  db:
    image: mariadb:11
    container_name: nextcloud-db
    restart: always
    command: --transaction-isolation=READ-COMMITTED --binlog-format=ROW
    volumes:
      - ./db:/var/lib/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=rootpassword
      - MYSQL_PASSWORD=nextcloud
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
    networks:
      - nextcloud-net

  app:
    image: nextcloud:latest
    container_name: nextcloud-app
    restart: always
    ports:
      - "8080:80"
    depends_on:
      - db
    volumes:
      - ./data:/var/www/html
    environment:
      - MYSQL_PASSWORD=nextcloud
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_HOST=db
    networks:
      - nextcloud-net

networks:
  nextcloud-net:
    driver: bridge
```

---

### 8.3.2. Detailed Analysis of Configuration Parameters
*   **`--transaction-isolation=READ-COMMITTED`:** Configures MariaDB's transaction isolation level. Nextcloud requires this setting to prevent database lock conflicts and ensure data consistency during concurrent file operations.
*   **`--binlog-format=ROW`:** Configures the binary log format to record changes at the row level instead of the statement level. This prevents replication errors and ensures reliable database recovery.
*   **`restart: always`:** Configures the Docker engine to restart containers automatically if they crash or if the host system reboots.
*   **`./db:/var/lib/mysql` & `./data:/var/www/html`:** Mounts host directories to the containers, ensuring that the database and uploaded user files are saved on the host and survive container updates or deletions.

### 8.3.3. Solutions to the Lab Exercises

#### Exercise 1: Volume Persistence Verification
To verify that data persists when containers are restarted:
1.  Access Nextcloud at `http://localhost:8080` and upload a test file named `persistence_test.txt`.
2.  Stop and delete the running containers:
    ```bash
    docker-compose down
    ```
3.  Restart the containers:
    ```bash
    docker-compose up -d
    ```
4.  Refresh the Nextcloud page in your browser. The file `persistence_test.txt` remains accessible because it was saved directly to the host's `./data` directory.

#### Exercise 3: Database Backup and Restore

*   **Backup Command:**
    ```bash
    docker exec nextcloud-db mysqldump -u root -prootpassword nextcloud > backup.sql
    ```
    *   **Explanation:** Runs the `mysqldump` utility inside the running database container, exporting the structure and data of the `nextcloud` database. The output is redirected to a `backup.sql` file on your host machine.

*   **Restore Command:**
    ```bash
    docker exec -i nextcloud-db mysql -u root -prootpassword nextcloud < backup.sql
    ```
    *   **Explanation:** Reads the `backup.sql` file from your host machine and pipes it into the `mysql` command line inside the running container, restoring the database state.
