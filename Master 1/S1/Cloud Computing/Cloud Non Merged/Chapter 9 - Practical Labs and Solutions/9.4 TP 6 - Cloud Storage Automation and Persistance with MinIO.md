## 8.4. TP 6 - Cloud Storage Automation and Persistance with MinIO

This lab covers deploying **MinIO** (an S3-compatible object storage server) inside Docker, automating file uploads, and managing object metadata.

### 8.4.1. MinIO Deployment via Docker
Run the following command to start a MinIO container:
```bash
docker run -d -p 9000:9000 -p 9001:9001 --name minio \
  -e "MINIO_ROOT_USER=admin" \
  -e "MINIO_ROOT_PASSWORD=supersecretpassword" \
  -v ./minio_data:/data \
  minio/minio server /data --console-address ":9001"
```
*   **`-p 9000:9000`:** Binds the S3 API port, allowing applications to read and write objects.
*   **`-p 9001:9001`:** Binds the MinIO web console port, providing an administration dashboard.
*   **`-v ./minio_data:/data`:** Persists uploaded objects to the host system.

### 8.4.2. Automation Script (Python)
This script automatically monitors a local directory, uploads new files to MinIO, and records object metadata in a database (such as PostgreSQL or SQLite).

```python
import os
import hashlib
import mimetypes
from datetime import datetime
import minio
import psycopg2

# Configuration
MINIO_URL = "localhost:9000"
ACCESS_KEY = "admin"
SECRET_KEY = "supersecretpassword"
BUCKET_NAME = "documents"
FOLDER_TO_WATCH = "./incoming_files"

# Initialize MinIO Client
mc = minio.Minio(MINIO_URL, access_key=ACCESS_KEY, secret_key=SECRET_KEY, secure=False)
if not mc.bucket_exists(BUCKET_NAME):
    mc.make_bucket(BUCKET_NAME)

# Initialize Database Connection
db_conn = psycopg2.connect("dbname=storage_metadata user=postgres password=dbpassword host=localhost")
cursor = db_conn.cursor()

def calculate_sha256(filepath):
    sha256 = hashlib.sha256()
    with open(filepath, 'rb') as f:
        while chunk := f.read(8192):
            sha256.update(chunk)
    return sha256.hexdigest()

def upload_and_log():
    for filename in os.listdir(FOLDER_TO_WATCH):
        filepath = os.path.join(FOLDER_TO_WATCH, filename)
        if os.path.isfile(filepath):
            # Compute Metadata
            file_hash = calculate_sha256(filepath)
            file_size = os.path.getsize(filepath)
            mime_type, _ = mimetypes.guess_type(filepath)
            mime_type = mime_type or "application/octet-stream"
            upload_date = datetime.now()

            # Upload to MinIO
            mc.fput_object(BUCKET_NAME, filename, filepath)
            print(f"Uploaded {filename} to object storage.")

            # Record Metadata in Database
            cursor.execute("""
                INSERT INTO file_metadata (filename, bucket, file_size, mime_type, file_hash, upload_date)
                VALUES (%s, %s, %s, %s, %s, %s)
                ON CONFLICT (file_hash) DO NOTHING;
            """, (filename, BUCKET_NAME, file_size, mime_type, file_hash, upload_date))
            db_conn.commit()

            # Remove local file after successful upload and log
            os.remove(filepath)

if __name__ == "__main__":
    upload_and_log()
```

### 8.4.3. Versioning Strategy
To implement a "conserve the last 3 versions" policy for your files:
1.  **Enable Versioning in MinIO:**
    ```python
    mc.set_bucket_versioning(BUCKET_NAME, minio.common.VersioningConfig("Enabled"))
    ```
2.  **Versioning Management Script:**
    When uploading a new file with an existing name, query MinIO to retrieve the version list. If the number of versions exceeds 3, identify the oldest version and delete it from both MinIO and your metadata database.

```python
def enforce_version_limit(object_name):
    # Retrieve all versions of the object
    versions = list(mc.list_objects(BUCKET_NAME, prefix=object_name, include_version=True))
    # Sort versions by modification time (oldest first)
    versions.sort(key=lambda obj: obj.last_modified)

    while len(versions) > 3:
        oldest_version = versions.pop(0)
        mc.remove_object(BUCKET_NAME, object_name, version_id=oldest_version.version_id)
        print(f"Removed old version {oldest_version.version_id} of {object_name}")
```
