# 1.4 Exam-Style Optimization Exercises & Python Speed Kit

This note covers **10 optimization problems that actually show up on university exams** — the kind where you're given a piece of code, told it's slow, and asked to fix it. No clever interview tricks, no obscure algorithms. Just the straightforward stuff your professor puts on the test: loop counting, simple formula replacements, picking the right data structure, and basic Big-O analysis. After the exercises, there's a **Python Speed Kit** with practical tricks for writing faster code under exam conditions.

> [!info] What Makes These "Exam-Style"?
> Exam questions test whether you **recognize the pattern**, not whether you can invent something new. Each exercise below is exactly the type you'd see on a 2-hour algorithms or HPC final: "Here's slow code → What's the complexity? → Rewrite it faster → What's the new complexity?"

---

## Exercise 1: Loop Counting — How Many Times Does This Run?

### Problem (The most common exam question of all)

You're given this code and asked: **"How many times does the inner statement execute? Give the exact count and the Big-O."**

### Pattern A: Square Loop
```python
for i in range(N):
    for j in range(N):
        count += 1
```
**Answer:** $N \times N = N^2$ iterations. Big-O: $O(N^2)$.

### Pattern B: Triangular Loop
```python
for i in range(N):
    for j in range(i, N):
        count += 1
```
**Answer:** When $i=0$, inner loop runs $N$ times. When $i=1$, runs $N-1$ times. ... When $i=N-1$, runs 1 time. Total = $N + (N-1) + \ldots + 1 = \frac{N(N+1)}{2}$. Big-O: $O(N^2)$. This is exactly Gauss's formula from [[1.3 Algorithmic Optimization Exercises|1.3]]!

### Pattern C: Dependent Inner Loop
```python
for i in range(N):
    for j in range(i):
        count += 1
```
**Answer:** When $i=0$, inner runs 0 times. When $i=1$, runs 1 time. ... When $i=N-1$, runs $N-1$ times. Total = $0 + 1 + \ldots + (N-1) = \frac{N(N-1)}{2}$. Big-O: $O(N^2)$.

### Pattern D: Halving Loop
```python
i = N
while i > 0:
    count += 1
    i = i // 2
```
**Answer:** $i$ takes values $N, N/2, N/4, \ldots, 1$. That's $\lfloor \log_2 N \rfloor + 1$ iterations. Big-O: $O(\log N)$.

### Pattern E: Nested — Outer Linear, Inner Halving
```python
for i in range(N):
    j = N
    while j > 0:
        count += 1
        j //= 2
```
**Answer:** Outer loop runs $N$ times. Each inner loop runs $O(\log N)$ times. Total: $O(N \log N)$.

> [!exam] Exam Survival Rule
> **Loop variable increments by +1 → count it as $N$. Loop variable multiplies/divides by 2 → count it as $\log N$.** Then multiply across nested loops. This one rule solves 80% of complexity questions on exams.

## Exercise 2: Replace the Loop with a Math Formula

### Problem
The professor gives you a loop that computes a mathematical sum and asks you to eliminate the loop entirely.

### Example A: Sum of 1 to N (from Lab 1)
```python
# NAIVE: O(N)
total = 0
for i in range(1, N + 1):
    total += i

# EXAM ANSWER: O(1)
total = N * (N + 1) // 2
```

### Example B: Sum of Squares
```python
# NAIVE: O(N)
total = 0
for i in range(1, N + 1):
    total += i * i

# EXAM ANSWER: O(1) — use the formula
total = N * (N + 1) * (2 * N + 1) // 6
```

### Example C: Sum of Even Numbers from 2 to 2N
```python
# NAIVE: O(N)
total = 0
for i in range(2, 2 * N + 1, 2):
    total += i

# EXAM ANSWER: O(1) — this is 2 + 4 + 6 + ... + 2N = 2(1 + 2 + ... + N) = 2 * N(N+1)/2
total = N * (N + 1)
```

### Example D: Geometric Sum $1 + 2 + 4 + \ldots + 2^N$
```python
# NAIVE: O(N)
total = 0
for i in range(N + 1):
    total += 2 ** i

# EXAM ANSWER: O(1) — geometric series formula
total = 2 ** (N + 1) - 1
```

> [!exam] Exam Tip
> The exam will always give you a loop that computes a recognizable sum. The four formulas you must memorize:
> - **Arithmetic sum:** $\sum_{i=1}^{N} i = \frac{N(N+1)}{2}$
> - **Sum of squares:** $\sum_{i=1}^{N} i^2 = \frac{N(N+1)(2N+1)}{6}$
> - **Geometric sum:** $\sum_{i=0}^{N} r^i = \frac{r^{N+1} - 1}{r - 1}$ (for $r \neq 1$)
> - **Harmonic sum:** $\sum_{i=1}^{N} \frac{1}{i} \approx \ln N$ (no closed form, but $O(\log N)$)

## Exercise 3: Fast Exponentiation ($x^n$)

### Problem
Compute $x^n$ where $n$ is a non-negative integer. This shows up on almost every algorithms exam.

### Naive — $O(N)$
```python
def power_naive(x, n):
    result = 1
    for _ in range(n):
        result *= x
    return result
```
$n$ multiplications. For $n = 10^9$, this does a billion operations.

### Improved — $O(\log N)$ Exponentiation by Squaring
```python
def power_fast(x, n):
    result = 1
    while n > 0:
        if n % 2 == 1:       # n is odd
            result *= x
        x *= x               # square the base
        n //= 2              # halve the exponent
    return result
```

**Why it works:**
- If $n$ is even: $x^n = (x^2)^{n/2}$ — square $x$, halve $n$.
- If $n$ is odd: $x^n = x \cdot x^{n-1} = x \cdot (x^2)^{(n-1)/2}$ — pull out one $x$, then square and halve.

Each step cuts $n$ in half. After $k$ steps, $n$ becomes $n/2^k$. When $2^k > n$, we're done. So $k = O(\log N)$ steps total.

**Trace for $3^{13}$:**
| Step | n | n odd? | result | x |
|:---:|:---:|:---:|:---:|:---:|
| Start | 13 | yes | 1×3 = 3 | 3→9 |
| | 6 | no | 3 | 9→81 |
| | 3 | yes | 3×81 = 243 | 81→6561 |
| | 1 | yes | 243×6561 = 1594323 | 6561→... |
| | 0 | stop | **1594323** ✓ | |

4 steps instead of 13. For $n = 10^9$: ~30 steps instead of a billion.

> [!exam] Exam Tip
> The question is usually: "Compute $7^{15} \mod 11$ using fast exponentiation. Show each step." Just add `result %= mod` after each multiplication to keep numbers small.

## Exercise 4: GCD — Euclidean Algorithm

### Problem
Find the Greatest Common Divisor of $a$ and $b$. A standard exam question.

### Naive — $O(\min(a,b))$
```python
def gcd_naive(a, b):
    for i in range(min(a, b), 0, -1):
        if a % i == 0 and b % i == 0:
            return i
```
Scans backwards from $\min(a,b)$ checking both divisions. For large numbers, this is extremely slow.

### Improved — $O(\log(\min(a,b)))$ Euclidean Algorithm
```python
def gcd(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

**Why it works:** $\gcd(a, b) = \gcd(b, a \mod b)$. The remainder $a \mod b$ is always strictly less than $b$, so the numbers shrink every step. In fact, after every two steps, the larger number is at least halved. So it terminates in at most $O(\log(\min(a,b)))$ steps.

**Trace for $\gcd(48, 18)$:**
| Step | a | b | a % b |
|:---:|:---:|:---:|:---:|
| 1 | 48 | 18 | 12 |
| 2 | 18 | 12 | 6 |
| 3 | 12 | 6 | 0 |
| 4 | 6 | 0 | stop → **6** |

3 steps instead of scanning 18 numbers.

> [!exam] Exam Tip
> You WILL be asked to trace through this by hand. Practice it. The extended Euclidean algorithm (which also finds $x, y$ such that $ax + by = \gcd(a,b)$) is also commonly tested.

## Exercise 5: Frequency Counting — Dictionary vs Nested Loop

### Problem
Count how many times each element appears in a list. This is the most basic "use a hash table" exam question.

### Naive — $O(N^2)$
```python
def frequency_naive(lst):
    result = {}
    for x in lst:
        count = 0
        for y in lst:          # Scan entire list for each element
            if y == x:
                count += 1
        result[x] = count
    return result
```
For every element, scan the whole list again. Quadratic.

### Improved — $O(N)$ with Dictionary
```python
def frequency_fast(lst):
    result = {}
    for x in lst:
        result[x] = result.get(x, 0) + 1   # O(1) dict update per element
    return result
```

### Python Shortcut — $O(N)$ with `Counter`
```python
from collections import Counter
result = Counter(lst)           # One line, C-optimized
```

**Key Insight:** Instead of searching the list for each element, we build a running tally. Dictionary lookups are $O(1)$ average, so one pass through the list gives us all counts. This is the same hash-table principle from [[1.3 Algorithmic Optimization Exercises|1.3 Exercise 3]].

> [!exam] Exam Tip
> Frequency counting is the building block for many exam questions: "find the most frequent element," "check if all elements are unique," "find elements appearing more than $N/3$ times." All start with a frequency dictionary.

## Exercise 6: Bubble Sort Early Exit

### Problem
The exam gives you bubble sort and asks: "Can you optimize this without changing the algorithm?"

### Standard Bubble Sort — Always $O(N^2)$
```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr
```

### Optimized — Best Case $O(N)$
```python
def bubble_sort_optimized(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:         # No swaps = already sorted!
            break               # Exit early
    return arr
```

**Key Insight:** If a full pass completes without any swaps, the array is already sorted — we can stop immediately. This doesn't change the worst case (still $O(N^2)$ for a reverse-sorted array), but the **best case becomes $O(N)$** for an already-sorted or nearly-sorted array.

> [!exam] Exam Tip
> The question is usually: "What is the best-case time complexity of optimized bubble sort?" Answer: **$O(N)$** — one pass with no swaps, then we exit. The naive version is always $O(N^2)$ regardless of input.

## Exercise 7: Prefix Sum for Range Queries

### Problem
Given an array, answer $Q$ queries of the form: "What is the sum from index $i$ to $j$?"

### Naive — $O(N)$ per query
```python
def range_sum_naive(arr, i, j):
    total = 0
    for k in range(i, j + 1):
        total += arr[k]
    return total
```
Each query scans the range. For $Q = 10^6$ queries on $N = 10^6$ elements: $10^{12}$ operations.

### Improved — $O(1)$ per query with Prefix Sum
```python
# Build prefix array ONCE: O(N)
prefix = [0] * (len(arr) + 1)
for i in range(len(arr)):
    prefix[i + 1] = prefix[i] + arr[i]

# Answer each query in O(1)
def range_sum(prefix, i, j):
    return prefix[j + 1] - prefix[i]
```

**Why it works:** `prefix[k]` = sum of `arr[0]` through `arr[k-1]`. So `sum(arr[i..j]) = prefix[j+1] - prefix[i]`. The sum of everything up to $j$, minus the sum of everything before $i$.

**Example:** `arr = [3, 1, 4, 1, 5]` → `prefix = [0, 3, 4, 8, 9, 14]`
- Sum from index 1 to 3: `prefix[4] - prefix[1] = 9 - 3 = 6` ✓ (1 + 4 + 1)

> [!exam] Exam Tip
> Prefix sum is the most commonly tested precomputation technique. If the exam says "answer $Q$ range queries," always precompute a prefix array first. The 2D version (for matrix sub-region queries) is also popular on exams.

## Exercise 8: Duplicate Detection — Set vs Double Loop

### Problem
Check if a list contains any duplicate elements.

### Naive — $O(N^2)$
```python
def has_duplicate_naive(lst):
    for i in range(len(lst)):
        for j in range(i + 1, len(lst)):
            if lst[i] == lst[j]:
                return True
    return False
```
Compare every pair. For $N = 100{,}000$: ~5 billion comparisons.

### Improved — $O(N)$ with Set
```python
def has_duplicate_fast(lst):
    seen = set()
    for x in lst:
        if x in seen:        # O(1) lookup
            return True
        seen.add(x)
    return False
```

### Even Shorter
```python
def has_duplicate_one_liner(lst):
    return len(lst) != len(set(lst))   # If sizes differ, duplicates exist
```

**Key Insight:** Converting a list to a set removes duplicates. If the set is smaller than the list, there were duplicates. The one-liner is $O(N)$ but creates the full set upfront. The loop version can **exit early** as soon as a duplicate is found.

> [!exam] Exam Tip
> This is the simplest exam question that tests "do you know hash tables exist?" Variations: "find the first repeating element," "count unique elements," "find the missing number in 1..N" (use set difference or sum formula).

## Exercise 9: Finding Min and Max — One Pass vs Two Passes

### Problem
Find both the minimum and maximum values in an array.

### Naive — Two Passes, $2N$ Comparisons
```python
minimum = min(arr)    # N comparisons
maximum = max(arr)    # N comparisons
# Total: 2N comparisons
```

### Improved — One Pass, $N$ Comparisons
```python
def min_max(arr):
    minimum = maximum = arr[0]
    for x in arr[1:]:
        if x < minimum:
            minimum = x
        elif x > maximum:       # elif! If it's a new min, it can't also be a new max
            maximum = x
    return minimum, maximum
```

**Key Insight:** You don't need to scan the array twice. Track both values simultaneously in one pass. The `elif` saves comparisons — if `x` is less than the minimum, it can't possibly be greater than the maximum, so we skip that check.

> [!exam] Exam Tip
> The exam asks: "How many comparisons does your code make?" The naive approach makes $2(N-1)$. The one-pass approach makes between $N-1$ and $2(N-1)$ depending on the data (average: ~1.5N). There's also an advanced "tournament method" that uses exactly $\lceil 3N/2 \rceil - 2$ comparisons — this is the theoretical minimum and sometimes asked as a bonus question.

## Exercise 10: Factorial — Iterative vs Recursive

### Problem
Compute $N! = 1 \times 2 \times 3 \times \ldots \times N$.

### Version 1: Recursive — $O(N)$ Time, $O(N)$ Space
```python
def fact_recursive(n):
    if n <= 1: return 1
    return n * fact_recursive(n - 1)
```
Each recursive call adds a frame to the call stack. For $N = 1000$: 1000 stack frames. Python's default recursion limit is ~1000, so this crashes for large $N$.

### Version 2: Iterative — $O(N)$ Time, $O(1)$ Space
```python
def fact_iterative(n):
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result
```
No stack growth. Handles arbitrarily large $N$ (limited only by memory for the result itself).

**Key Insight:** Any simple recursion (where the function calls itself once at the end) can be converted to a loop. This is called **tail-call elimination**. The loop version is always better in Python because:
1. No stack overflow risk.
2. No function call overhead per iteration.
3. $O(1)$ space instead of $O(N)$ space.

> [!exam] Exam Tip
> The exam will ask: "What is the space complexity of the recursive version?" Answer: **$O(N)$** for the call stack. "Convert it to an iterative version with $O(1)$ space." This pattern applies to ANY single-branch recursion: factorial, sum of digits, reversing a string, etc.

# Python Speed Kit for Exams

These are practical Python tricks that make your code faster — things you can write on an exam without changing the algorithm. No imports needed (mostly), no clever tricks — just "this is the faster way to write the same thing."

---

## Trick 1: List Comprehension Instead of Loop + Append

```python
# SLOW
result = []
for x in data:
    result.append(x ** 2)

# FAST — 30-50% faster because Python optimizes it as a single C call
result = [x ** 2 for x in data]
```

---

## Trick 2: Built-in Functions Are C-Optimized

```python
# Don't write your own loops for these:
total = sum(data)               # Not: for x in data: total += x
biggest = max(data)             # Not: for x in data: if x > max...
smallest = min(data)            # Same idea
sorted_data = sorted(data)      # Timsort — O(N log N), heavily optimized
```

These are implemented in C. They will always beat your Python loop.

---

## Trick 3: Set for Membership — The Single Most Important Trick

```python
# SLOW: O(N) per check
if target in my_list: ...

# FAST: O(1) per check
my_set = set(my_list)
if target in my_set: ...
```

Any `if x in list` inside a loop → convert the list to a set first. This is worth more points on an exam than almost anything else.

---

## Trick 4: `join()` for Strings, Not `+=`

```python
# SLOW: O(N^2) — each += creates a new string object
result = ""
for word in words:
    result += word + " "

# FAST: O(N) — allocate once, copy into place
result = " ".join(words)
```

Strings are immutable in Python. Every `+=` creates a new string and copies everything. `join()` allocates the final string once.

---

## Trick 5: Dictionary `.get()` Instead of If-Else

```python
# SLOW: Two lookups
if key in my_dict:
    my_dict[key] += 1
else:
    my_dict[key] = 1

# FAST: One lookup
my_dict[key] = my_dict.get(key, 0) + 1
```

`.get(key, default)` returns the default if the key doesn't exist — no `if` statement needed. Faster and cleaner.

---

## Trick 6: Generator Expressions Save Memory

```python
# SLOW: Creates full list in memory (~80 MB for 10M elements)
total = sum([x ** 2 for x in range(10_000_000)])

# FAST: No list created — values generated lazily (~0 extra MB)
total = sum(x ** 2 for x in range(10_000_000))
```

Use `()` instead of `[]` when feeding into `sum()`, `max()`, `any()`, etc. Same speed, fraction of the memory.

---

## Trick 7: Early Termination — Stop As Soon As You Can

```python
# SLOW: Always checks every element
def has_negative(arr):
    found = False
    for x in arr:
        if x < 0:
            found = True
    return found

# FAST: Returns immediately when found
def has_negative_fast(arr):
    for x in arr:
        if x < 0:
            return True
    return False

# SHORTEST: any() short-circuits automatically
def has_negative_best(arr):
    return any(x < 0 for x in arr)
```

Same worst case, but average case can be dramatically better. If the first element is negative, you finish in 1 step instead of $N$.

---

## Trick 8: `enumerate` and `zip` Replace Index Math

```python
# SLOW: Manual indexing
for i in range(len(names)):
    print(names[i], scores[i])

# FAST: zip iterates both in C
for name, score in zip(names, scores):
    print(name, score)

# enumerate gives index + value
for i, val in enumerate(data):
    print(f"Index {i}: {val}")
```

No index arithmetic, no bounds checking. Cleaner and slightly faster.

---

## Trick 9: NumPy for Any Numerical Loop

```python
# SLOW: Python loop
result = [x ** 2 + 2 * x + 1 for x in data]

# FAST: NumPy (~100x faster)
import numpy as np
arr = np.array(data)
result = arr ** 2 + 2 * arr + 1
```

As covered in [[1.2 Profiling and Timing Tools]], NumPy replaces Python's per-element overhead with compiled C operations. If the exam allows NumPy, always use it for math on arrays.

---

## Trick 10: Avoid Repeated Work Inside Loops

```python
# SLOW: radius**2 recomputed every iteration
for x in data:
    result = x * (3.14159 * radius ** 2)

# FAST: compute the constant once
area = 3.14159 * radius ** 2
for x in data:
    result = x * area
```

This is called **loop-invariant code motion**. In C, the compiler does this for you. In Python, you must do it yourself. On an exam, this shows you understand that constant work inside a loop is wasteful.

## Master Checklist (In Order of Impact)

When your code is too slow, go through this list **in this exact order**:

1. **Can you reduce the Big-O?** → Better algorithm (see 10 exercises)
2. **Are you using `set`/`dict` for lookups?** → $O(1)$ instead of $O(N)$
3. **Are you using built-in functions?** → `sum()`, `max()`, `sorted()`
4. **Can you precompute?** → Prefix sums, lookup tables
5. **List comprehension instead of loop+append?**
6. **Can you exit early?** → `return`, `break`, `any()`, `all()`
7. **NumPy for math on arrays?**
8. **`join()` for string building?**
9. **Generator expressions for memory?**
10. **Hoist constants out of loops?**

> [!abstract] The Rule
> **Algorithm > Data Structure > Built-in > Micro-optimization**
> A better algorithm always wins. But on an exam, using a `set` instead of a nested loop is often the entire answer.
