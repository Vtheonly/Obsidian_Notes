## 1. Semantic SEO URLs using Slugs
A **Slug** is a URL-friendly string representation of a model's name or title. Slugs make URLs more human-readable and improve Search Engine Optimization (SEO) by replacing arbitrary ID numbers with descriptive text.

```
Non-SEO Friendly ID:  http://mysite.com/blog/24/
SEO-Friendly Slug:    http://mysite.com/blog/introduction-to-django/
```

### Model Declaration for Slugs
To use slugs, define a `SlugField` in your model. You can generate slug values automatically on save using the `slugify` utility:
```python
from django.db import models
from django.utils.text import slugify

class MedicalArticle(models.Model):
    title = models.CharField(max_length=200)
    # db_index improves database lookup speeds for slug strings
    slug = models.SlugField(max_length=250, unique=True, db_index=True)
    content = models.TextField()

    def save(self, *args, **kwargs):
        # Generate slug automatically if it doesn't exist
        if not self.slug:
            self.slug = slugify(self.title)
        super().save(*args, **kwargs)
```

### URL Conf Configuration
```python
from django.urls import path
from . import views

urlpatterns = [
    path('articles/<slug:slug>/', views.article_detail, name='article-detail'),
]
```

## 2. Handling File Directories Using the `path` Converter
The `path` converter matches complete, non-empty paths including folder subdirectory slashes (`/`). This is useful for building download endpoints, file managers, or routing dynamic static paths.

### Configuration
```python
urlpatterns = [
    # Matches paths like 'files/documents/reports/2026/file.pdf'
    path('files/<path:filepath>', views.serve_secure_file, name='secure-file'),
]
```

### View Implementation
```python
from django.http import HttpResponse, Http404
import os

def serve_secure_file(request, filepath):
    # The 'filepath' variable captures directories and slashes (e.g., 'reports/2026/file.pdf')
    file_path_on_disk = os.path.join('/var/protected_files/', filepath)
    
    if os.path.exists(file_path_on_disk):
        with open(file_path_on_disk, 'rb') as fh:
            response = HttpResponse(fh.read(), content_type="application/octet-stream")
            response['Content-Disposition'] = f'attachment; filename={os.path.basename(file_path_on_disk)}'
            return response
    raise Http404("Requested file not found.")
```
