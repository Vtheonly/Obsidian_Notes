
# 4.2. Routing URL Pattern Matching and Dynamic Parameters

## 1. Routing Basics
Django manages routing using path definitions located in your project's `urls.py` file. The router matches incoming request paths against defined patterns, extracts variables, and directs the request to the matching view.

### Standard Path Structure
```python
from django.urls import path
from . import views

urlpatterns = [
    # Static Route matching '/contact/' exactly
    path('contact/', views.contact_view, name='contact'),
]
```

## 2. Dynamic Routing Parameters
You can capture variables from the URL path using path converters: `<type:variable_name>`.

```python
urlpatterns = [
    # Captures an integer argument and passes it to the view as 'patient_id'
    path('patient/<int:patient_id>/', views.patient_detail, name='patient-detail'),
]
```

### Basic Path Converters Reference
* **`str`**: Matches any non-empty string except for the path separator `/`. This is the default converter if none is specified.
* **`int`**: Matches zero or any positive integer and passes it to the view as an integer type.
* **`slug`**: Matches any slug string (alphanumeric characters, hyphens, and underscores).
* **`uuid`**: Matches a formatted UUID (e.g., `075194d3-6885-417e-a8a8-6c931e272f00`).
* **`path`**: Matches any non-empty string, including path separators `/`. This allows you to match complete file paths or directories.

## 3. Namespacing and Reverse Resolution
Using absolute, hardcoded URL paths in your views and templates can lead to broken links if your routing patterns change. Instead, use Django's reverse resolution system to generate URLs dynamically using their `name` attribute.

### Target `urls.py` Configuration
```python
# To avoid route name collisions across apps, define an app_name namespace
app_name = 'clinical'

urlpatterns = [
    path('patient/<int:patient_id>/', views.patient_detail, name='patient-detail'),
]
```

### Generating URLs Dynamically

#### Inside Python Views (Reverse Resolution)
```python
from django.urls import reverse
from django.shortcuts import redirect

def redirect_to_patient(request, patient_id):
    # Generates '/clinical/patient/1/' dynamically
    target_url = reverse('clinical:patient-detail', args=[patient_id])
    return redirect(target_url)
```

#### Inside Django HTML Templates
```html
<!-- Generates '/clinical/patient/5/' dynamically during rendering -->
<a href="{% url 'clinical:patient-detail' patient_id=5 %}">View Patient Record</a>
```
