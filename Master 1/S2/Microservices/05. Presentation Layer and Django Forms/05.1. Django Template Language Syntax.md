
# 5.1. Django Template Language Syntax

## 1. Core Syntax of the Django Template Language (DTL)
The Django Template Language (DTL) is an application-level template engine used to generate HTML dynamically. DTL separates presentation layouts from your Python controller views.

DTL uses three primary tag structures:

| Syntax Type | Representation | Purpose / Usage | Example |
| :--- | :--- | :--- | :--- |
| **Variables** | `{{ variable }}` | Outputs values from the view's context dictionary. | `{{ patient.nom }}` |
| **Tags** | `{% tag %}` | Handles control flow logic like loops, conditions, and template inheritance. | `{% if patient.is_active %}` |
| **Filters** | `{{ var\|filter }}` | Modifies how variables are formatted and displayed. | `{{ patient.nom\|upper }}` |

## 2. Practical Syntax Examples

### Conditional Logic
```html
{% if patient.capacity >= 100 %}
    <span class="badge danger">Capacity Overlimit Warning</span>
{% elif patient.capacity > 50 %}
    <span class="badge warning">Moderate Capacity Load</span>
{% else %}
    <span class="badge success">Normal Operating Levels</span>
{% endif %}
```

### Looping Constructs
```html
<ul>
    {% for item in medical_procedures %}
        <!-- loop.counter outputs the current iteration count (1-indexed) -->
        <li>{{ forloop.counter }}. {{ item.name|title }}</li>
    {% empty %}
        <!-- Executed if the list is empty or evaluates to None -->
        <li>No diagnostic procedures registered today.</li>
    {% endfor %}
</ul>
```

### Formatting Filters
```html
<!-- Formats date outputs -->
<p>Admitted: {{ patient.admission_date|date:"Y-m-d H:i" }}</p>

<!-- Fallback default if value is empty or null -->
<p>Contact: {{ patient.phone|default:"No phone number provided" }}</p>

<!-- Limits characters displayed -->
<p>Notes summary: {{ patient.medical_notes|truncatechars:50 }}</p>
```

## 3. Security: Automatic HTML Escaping
To prevent **Cross-Site Scripting (XSS)** attacks, Django automatically escapes dynamic content rendered in HTML templates.

If a malicious user submits input containing script tags:
```python
# Context passed to template:
bad_input = "<script>alert('malicious')</script>"
```
Django renders this as literal text:
```html
&lt;script&gt;alert('malicious')&lt;/script&gt;
```

### Overriding Auto-Escaping (Use with Caution)
If you need to render pre-formatted safe HTML generated by trusted backend editors, you can bypass escaping using the `safe` filter:
```html
<!-- Renders raw HTML markup directly in the page layout -->
<div>{{ trusted_html_content|safe }}</div>
```
* **Security Warning**: Only use the `safe` filter on verified, internally generated data. Never use it on user-submitted input. Doing so can expose your application to XSS security vulnerabilities.
