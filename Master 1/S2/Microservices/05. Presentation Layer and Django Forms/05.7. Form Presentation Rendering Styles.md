
# 5.7. Form Presentation Rendering Styles

## 1. Quick Presentation Layout Rendering Options
Django forms provide three default layouts for rendering form fields in your templates:

```html
<!-- Renders fields wrapped in paragraph elements (<p>) -->
{{ form.as_p }}

<!-- Renders fields as table rows (<tr>) (Requires a parent <table> container) -->
{{ form.as_table }}

<!-- Renders fields as list elements (<li>) (Requires a parent <ul> container) -->
{{ form.as_ul }}
```

While these shortcuts are useful for rapid development, they generate basic, unstyled HTML.

## 2. Complete Manual Rendering (For Custom CSS/Tailwind Frameworks)
To apply custom styles from frameworks like Tailwind CSS or Bootstrap, you should loop through and render each form field manually. This gives you complete control over your form layouts.

### Manual Layout Implementation
Below is a custom form template layout styled with Tailwind CSS utility classes:

```html
<form method="POST" action="." class="space-y-6">
    <!-- 1. Enforce CSRF Security Protection -->
    {% csrf_token %}

    <!-- 2. Display Global Form-Level Validation Errors -->
    {% if form.non_field_errors %}
        <div class="bg-red-50 border-l-4 border-red-500 p-4 mb-4" role="alert">
            <p class="text-sm text-red-700">Please correct the following errors:</p>
            <ul class="list-disc pl-5 mt-2 text-xs text-red-600">
                {% for error in form.non_field_errors %}
                    <li>{{ error }}</li>
                {% endfor %}
            </ul>
        </div>
    {% endif %}

    <!-- 3. Loop Through and Render Individual Fields -->
    {% for field in form %}
        <div class="flex flex-col space-y-2">
            <!-- Label Element -->
            <label for="{{ field.id_for_label }}" class="text-sm font-semibold text-gray-700">
                {{ field.label }}
                {% if field.field.required %}
                    <span class="text-red-500" title="This field is required">*</span>
                {% endif %}
            </label>

            <!-- Render the Form Widget -->
            <div class="relative-input-wrapper">
                {{ field }}
            </div>

            <!-- Display Help/Guidance Text -->
            {% if field.help_text %}
                <p class="text-xs text-gray-400">{{ field.help_text }}</p>
            {% endif %}

            <!-- Display Field-Specific Validation Errors -->
            {% if field.errors %}
                <p class="text-xs text-red-500 font-medium">
                    {{ field.errors|striptags }}
                </p>
            {% endif %}
        </div>
    {% endfor %}

    <!-- Submit Action Button -->
    <button type="submit" class="w-full bg-blue-600 hover:bg-blue-700 text-white font-semibold py-2 px-4 rounded-md transition duration-200">
        Register Patient
    </button>
</form>
```

## 3. Customizing Input Widgets in Python code
Rather than hardcoding styling classes directly inside your templates, you can inject Tailwind or Bootstrap classes directly into your form field widgets in Python:
```python
class StyledForm(forms.Form):
    name = forms.CharField(
        widget=forms.TextInput(attrs={
            'class': 'w-full px-3 py-2 border border-gray-300 rounded-md focus:ring-2 focus:ring-blue-500 focus:outline-none',
            'placeholder': 'Enter full legal name'
        })
    )
```
