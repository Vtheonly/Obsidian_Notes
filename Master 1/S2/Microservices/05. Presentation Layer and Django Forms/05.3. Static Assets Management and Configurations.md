
# 5.3. Static Assets Management and Configurations

## 1. Setting Up Static File Directories
Static assets are non-dynamic files served directly to users, such as CSS stylesheets, JavaScript files, images, and fonts. Django separates static files from dynamic database views and templates.

To manage static files, add these configurations to your project's **`settings.py`**:

```python
import os
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

# 1. The public URL path used to access static files in the browser
STATIC_URL = '/static/'

# 2. Local directories where Django looks for raw static assets during development
STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'static'),
]

# 3. The absolute system folder path where Django collects all static files for production deployment
STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
```

## 2. Loading Static Files in HTML Templates
To reference static assets in your HTML templates, load the static template engine using **`{% load static %}`**:

```html
{% load static %}
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Clinical Database Interface</title>
    <!-- Resolves to '/static/css/styles.css' dynamically -->
    <link rel="stylesheet" href="{% static 'css/styles.css' %}">
</head>
<body>
    <!-- Resolves to '/static/images/logo.png' dynamically -->
    <img src="{% static 'images/logo.png' %}" alt="Hospital Logo">

    <script src="{% static 'js/app.js' %}"></script>
</body>
</html>
```

## 3. Deploying Static Files to Production
During development, Django's local test server (`runserver`) serves static files automatically. However, this test server is slow and insecure for production use.

When deploying to production:
1. Run the **`collectstatic`** command. This command scans your project and copies all static files from your development directories (and installed apps) into the single directory defined by `STATIC_ROOT`.
   ```bash
   python manage.py collectstatic --noinput
   ```
2. Configure a dedicated, high-performance web server like **Nginx** or a specialized service like **WhiteNoise** to serve the files inside your `STATIC_ROOT` folder directly. This avoids routing static asset requests through your slower Python application server (e.g., Gunicorn).
