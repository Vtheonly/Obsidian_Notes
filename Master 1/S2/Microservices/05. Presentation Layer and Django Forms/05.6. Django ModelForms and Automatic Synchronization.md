
# 5.6. Django ModelForms and Automatic Synchronization

## 1. Background of ModelForms
Standard forms require you to write validation logic and update database fields manually. This can lead to redundant code if your form fields map directly to a database model.

To solve this, Django provides **`ModelForms`**. A `ModelForm` maps directly to a Django database model. It automatically:
1. Generates the appropriate HTML form fields based on the model's fields.
2. Applies the validation rules defined on the model (e.g., maximum string lengths, unique constraints).
3. Saves form submissions directly to database records in a single step using the `.save()` method.

## 2. Python Implementation Example
To create a `ModelForm`, inherit from **`forms.ModelForm`** and define a nested class named **`Meta`**:

```python
from django import forms
from .models import Patient

class PatientRegistrationForm(forms.ModelForm):
    # You can add additional fields that are not part of the database model
    terms_agreement = forms.BooleanField(required=True, label="I agree to clinical terms.")

    class Meta:
        # 1. Associate the form with your Model
        model = Patient
        
        # 2. Specify which fields to include on the form
        fields = ['nom', 'email', 'date_naissance', 'terms_agreement']
        
        # 3. Customize form labels
        labels = {
            'nom': 'Full Name',
            'date_naissance': 'Date of Birth',
        }
        
        # 4. Customize CSS classes or HTML widgets
        widgets = {
            'date_naissance': forms.DateInput(attrs={'type': 'date', 'class': 'form-control'}),
            'nom': forms.TextInput(attrs={'placeholder': 'E.g., John Doe'}),
        }
```

## 3. Saving Form Submissions to the Database
The `save()` method on a `ModelForm` validates the submitted data and writes the database record in one step:

```python
# Create a new record from form submission
form = PatientRegistrationForm(request.POST)
if form.is_valid():
    new_patient_instance = form.save()
```

### Delaying Database Writes with `commit=False`
If you need to perform additional modifications on the model instance (like assigning an editor user ID) before saving it to the database, set **`commit=False`**:

```python
if form.is_valid():
    # Instantiates the model in memory, but does not save it to the database yet
    patient_instance = form.save(commit=False)
    
    # Modify attributes manually
    patient_instance.created_by = request.user
    
    # Save the modified instance to the database
    patient_instance.save()
    
    # Important: If your ModelForm contains many-to-many relationship fields,
    # you must call save_m2m() manually after saving the model with commit=False.
    form.save_m2m()
```
