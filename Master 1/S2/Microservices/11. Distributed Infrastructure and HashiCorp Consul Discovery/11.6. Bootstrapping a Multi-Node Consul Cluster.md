
# 11.6. Bootstrapping a Multi-Node Consul Cluster

## 1. Local Testing Environment Details
This guide covers how to set up a 3-node Consul cluster (1 Leader Server, 1 Follower Server, and 1 Client Node) on a single machine for development and testing.

## 2. Command Line Bootstrap Pipeline

### Step 1: Start the Server Leader
Start the first server node in bootstrap mode. This node will automatically elect itself as the cluster Leader.
```powershell
# Run in Terminal 1
consul agent `
  -server `
  -bootstrap-expect=1 `
  -node=leader1 `
  -bind=127.0.0.1 `
  -data-dir=C:\consul-data\leader1 `
  -client=0.0.0.0 `
  -http-port=8500 `
  -dns-port=8600
```

### Step 2: Start the Server Follower
Start the second server node on custom ports, pointing it to the Leader's LAN gossip address (`127.0.0.1:8301`) using the `-retry-join` flag.
```powershell
# Run in Terminal 2
consul agent `
  -server `
  -node=follower1 `
  -bind=127.0.0.1 `
  -data-dir=C:\consul-data\follower1 `
  -client=0.0.0.0 `
  -retry-join=127.0.0.1:8301 `
  -server-port=8305 `
  -serf-lan-port=8304 `
  -serf-wan-port=8306 `
  -http-port=8501 `
  -dns-port=8601
```

### Step 3: Start the Client Node
Start the client node, pointing it to the Leader's gossip pool. This client node will act as a proxy for your local applications.
```powershell
# Run in Terminal 3
consul agent `
  -node=client-agent-1 `
  -bind=127.0.0.1 `
  -data-dir=C:\consul-data\client1 `
  -client=0.0.0.0 `
  -retry-join=127.0.0.1:8301 `
  -serf-lan-port=8307 `
  -serf-wan-port=8308 `
  -http-port=8502 `
  -dns-port=8602
```
