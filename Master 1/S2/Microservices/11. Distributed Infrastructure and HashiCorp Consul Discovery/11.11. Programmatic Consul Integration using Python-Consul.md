
# 11.11. Programmatic Consul Integration using Python-Consul

## 1. What is the `python-consul` Library?
Rather than managing service registration manually using shell scripts or cURL, your application should handle registering and discovering services programmatically on startup.

In Python, we can integrate with Consul using the **`python-consul2`** library (the active, maintained fork of `python-consul`).

First, install the package in your active virtual environment:
```bash
pip install python-consul2
```

## 2. Programmatic Service Registration
Below is a utility module that automates registering your Django application service with Consul:

```python
# clinical/consul_service.py
import consul
import socket

def register_django_service(service_name="django-clinical-service", port=8000):
    """Programmatically register this service instance with the local Consul agent."""
    # 1. Initialize the Consul client pointing to the local client agent's HTTP port
    c = consul.Consul(host="127.0.0.1", port=8502)

    # 2. Retrieve the local machine's IP address dynamically
    hostname = socket.gethostname()
    ip_address = socket.gethostbyname(hostname)
    
    # Define a unique ID for this specific service instance
    service_id = f"{service_name}-{port}"

    # 3. Define the service registration configuration schema
    registration_payload = {
        "name": service_name,
        "service_id": service_id,
        "address": ip_address,
        "port": port,
        "check": {
            "http": f"http://{ip_address}:{port}/api/health/",
            "interval": "10s",
            "timeout": "3s"
        }
    }

    # 4. Register the service with the local Consul client agent
    try:
        c.agent.service.register(
            name=registration_payload["name"],
            service_id=registration_payload["service_id"],
            address=registration_payload["address"],
            port=registration_payload["port"],
            check=registration_payload["check"]
        )
        print(f"[Consul API] Successfully registered service: {service_id} at {ip_address}:{port}")
    except Exception as e:
        print(f"[Consul API Error] Failed to register service with Consul: {str(e)}")
```

## 3. Programmatic Service Discovery
Below is a utility function showing how to look up healthy instances of a target microservice:

```python
def discover_service_instance(service_name):
    """Retrieve a healthy instance network address for a target service from Consul."""
    c = consul.Consul(host="127.0.0.1", port=8502)

    try:
        # Retrieve a list of healthy instances from the catalog
        # The passing=True flag filters out offline or unhealthy instances automatically
        index, instances = c.health.service(service_name, passing=True)
        
        if not instances:
            raise Exception(f"No active, healthy instances of '{service_name}' found in registry.")

        # Standard load balancing: Select the first available healthy instance
        target_node = instances[0]["Service"]
        target_address = target_node["Address"]
        target_port = target_node["Port"]
        
        resolved_url = f"http://{target_address}:{target_port}"
        print(f"[Consul Discovery] Resolved '{service_name}' to address: {resolved_url}")
        return resolved_url

    except Exception as e:
        print(f"[Consul Discovery Error] Unable to resolve service address: {str(e)}")
        return None
```
