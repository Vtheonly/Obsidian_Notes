# 11.8. Consul Agent Command Line Tools

## 1. Core Agent Command Arguments Reference
Below is a reference guide for the command-line flags used to configure Consul agents:

| Command Flag Option | Configuration Parameters | Operational Function |
| :--- | :--- | :--- |
| **`-server`** | None (Binary flag) | Switches the agent to Server mode. If omitted, the agent runs in Client mode. |
| **`-bootstrap-expect`** | Integer value (e.g., `-bootstrap-expect=3`) | Specifies the number of server nodes required to elect a Leader and start the cluster. |
| **`-node`** | String identifier (e.g., `-node=prod-db-1`) | Assigns a unique name to the node in the cluster. |
| **`-bind`** | IP network address (e.g., `-bind=192.168.1.10`) | Specifies the network interface the agent uses to listen for Gossip and RPC traffic. |
| **`-client`** | IP network address (e.g., `-client=0.0.0.0`) | Specifies which IP addresses can access the agent's HTTP, DNS, and gRPC APIs. |
| **`-data-dir`** | File system path (e.g., `-data-dir=/var/consul/`) | Specifies where the agent stores write logs, certificates, and metadata. |
| **`-retry-join`** | Network address (e.g., `-retry-join=10.0.0.5:8301`) | Specifies the address of an active gossip node to join when starting up. |

## 2. Using Developer Mode for Local Testing
To start a lightweight, single-node Consul agent locally without writing configuration files or running migrations, use the **`-dev`** flag:
```bash
consul agent -dev
```
* **Note**: Developer mode is designed for testing only. It runs entirely in memory and deletes all data and configurations when stopped.
