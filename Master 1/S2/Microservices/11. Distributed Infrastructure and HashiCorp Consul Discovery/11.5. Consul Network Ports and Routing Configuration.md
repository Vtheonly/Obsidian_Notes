
# 11.5. Consul Network Ports and Routing Configuration

## 1. Port Architecture Reference
Consul uses several ports to manage its control plane, WAN federation, and DNS lookups. Opening and configuring these ports correctly is essential for cluster stability and network security.

| Default Port | Network Protocol | Function Name | Purpose / Description |
| :---: | :---: | :--- | :--- |
| **`8300`** | TCP | Server RPC | Used by Server nodes to replicate database logs and handle client requests. |
| **`8301`** | TCP & UDP | Serf LAN | Used by LAN Gossip pools to discover nodes and monitor member health. |
| **`8302`** | TCP & UDP | Serf WAN | Used by WAN Gossip pools to discover and connect clusters across datacenters. |
| **`8500`** | TCP | HTTP API / Web UI | Used by application services to register endpoints and access the web dashboard. |
| **`8600`** | TCP & UDP | DNS Resolver | Used for Service Discovery via DNS. |

## 2. Preventing Conflicts on a Single-Machine Simulation
When running multiple Consul nodes on a single developer machine for testing, you must assign unique ports to each node to prevent socket collisions.

Here is a recommended port mapping for a 3-node local cluster simulation:

```
                  Local Machine Loopback (127.0.0.1)
   +--------------------------------------------------------------+
   |                                                              |
   |   [Server Leader]         [Server Follower]      [Client Node]
   |   HTTP API: 8500          HTTP API: 8501         HTTP API: 8502
   |   Serf LAN: 8301          Serf LAN: 8304         Serf LAN: 8307
   |   Server RPC: 8300        Server RPC: 8305       DNS Server: 8602
   |                                                              |
   +--------------------------------------------------------------+
```
