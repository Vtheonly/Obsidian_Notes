
# 10.2. Custom User Models with AbstractBaseUser

## 1. Implementing an Email-Based Custom User Model
Below is an implementation of a custom user model (`Compte`) that uses the **email address** as the primary login credential and adds a `role` field for Role-Based Access Control:

```python
from django.db import models
from django.contrib.auth.models import AbstractBaseUser, PermissionsMixin
from .managers import CompteManager

class Compte(AbstractBaseUser, PermissionsMixin):
    ROLE_CHOICES = [
        ('ADMIN', 'Administrator'),
        ('EDITOR', 'Editor'),
        ('VIEWER', 'Viewer'),
    ]

    # 1. Custom Identity Fields
    email = models.EmailField(unique=True, db_index=True)
    nom = models.CharField(max_length=100)
    prenom = models.CharField(max_length=100)
    
    # 2. Custom Role-Based Access Field
    role = models.CharField(max_length=20, choices=ROLE_CHOICES, default='VIEWER')
    
    # 3. Core Django Operational Flags
    is_active = models.BooleanField(default=True)
    is_staff = models.BooleanField(default=False)
    date_creation = models.DateTimeField(auto_now_add=True)

    # 4. Associate custom manager
    objects = CompteManager()

    # 5. Define identity and configuration fields
    # USERNAME_FIELD defines the field used as the login credential
    USERNAME_FIELD = 'email'
    
    # REQUIRED_FIELDS lists the fields required when running 'createsuperuser'
    REQUIRED_FIELDS = ['nom', 'prenom', 'role']

    def __str__(self):
        return f"{self.email} ({self.role})"
```

## 2. Configuration Settings Reference
* **`USERNAME_FIELD`**: Specifies the database field used to identify users during login. This field must be unique (`unique=True`).
* **`REQUIRED_FIELDS`**: A list of fields that the `createsuperuser` command-line tool will prompt you to enter. It should not contain the `USERNAME_FIELD` or `password` fields, as these are always required.
