
# 10.6. Method-Level Custom View Permissions

## 1. Custom Class-Based Views Permissions
By default, Django's `PermissionRequiredMixin` applies a single, static set of permissions across **all** HTTP request methods in a Class-Based View. 

However, real-world views often require different permissions for different actions (such as allowing any logged-in user to read records via `GET`, but restricting creation and updates via `POST` or `PUT` to editors).

You can handle this by overriding the **`get_permission_required()`** and **`dispatch()`** methods in your Class-Based Views.

## 2. Code Implementation Example
Below is an implementation of a Class-Based View that enforces method-level permissions:

```python
from django.contrib.auth.mixins import PermissionRequiredMixin
from django.views import View
from django.shortcuts import render, redirect, get_object_or_404
from django.http import HttpResponseForbidden, HttpResponse
from clinical.models import Patient

class PatientManagementController(PermissionRequiredMixin, View):
    template_name = 'clinical/patient_detail.html'
    
    # Override raise_exception to raise an HTTP 403 Forbidden response instead of redirecting
    raise_exception = True

    def get_permission_required(self):
        """Return the required permissions dynamically based on the HTTP method."""
        method = self.request.method.upper()
        
        if method == 'GET':
            # Require 'view' permission for read requests
            return ('clinical.view_patient',)
        elif method == 'POST':
            # Require 'add' permission for create requests
            return ('clinical.add_patient',)
        elif method in ['PUT', 'PATCH']:
            # Require 'change' permission for update requests
            return ('clinical.change_patient',)
        elif method == 'DELETE':
            # Require 'delete' permission for delete requests
            return ('clinical.delete_patient',)
            
        # Default fallback
        return ('clinical.view_patient',)

    def dispatch(self, request, *args, **kwargs):
        """Override dispatch to evaluate permissions before executing the view handler."""
        # Check permissions using get_permission_required()
        if not self.has_permission():
            return self.handle_no_permission()
        return super().dispatch(request, *args, **kwargs)

    def get(self, request, pk):
        patient = get_object_or_404(Patient, pk=pk)
        return render(request, self.template_name, {'patient': patient})

    def post(self, request, pk):
        patient = get_object_or_404(Patient, pk=pk)
        new_name = request.POST.get('name')
        if new_name:
            patient.nom = new_name
            patient.save()
            return redirect('clinical:dashboard')
        return HttpResponse("Missing name parameter", status=400)
```
