
## 1. Critical Warning: Changing the User Model Mid-Project
```
       +-------------------------------------------------------------+
       |                           WARNING                           |
       |  Define your custom user model BEFORE running your initial  |
       |  database migrations. Changing AUTH_USER_MODEL in settings  |
       |  mid-project can break relational integrity in external     |
       |  packages and core system tables, often requiring you to    |
       |  re-create your database.                                   |
       +-------------------------------------------------------------+
```

## 2. Setting Up the Custom User Model in settings.py
To tell Django to use your custom user model instead of the default model, add the **`AUTH_USER_MODEL`** setting to your project's **`settings.py`**:

```python
# settings.py

# Format: '<app_name>.<model_class_name>'
AUTH_USER_MODEL = 'clinical.Compte'
```

## 3. Database Migration and Integration Steps
If you are starting a fresh project:
1. Register your custom model in `settings.py` first.
2. Run migrations to generate the appropriate database tables:
   ```bash
   python manage.py makemigrations clinical
   python manage.py migrate
   ```
3. Create your system administrator account:
   ```bash
   python manage.py createsuperuser
   ```
   Django will automatically use your custom model and prompt you for the email and custom fields you defined as `REQUIRED_FIELDS`.
