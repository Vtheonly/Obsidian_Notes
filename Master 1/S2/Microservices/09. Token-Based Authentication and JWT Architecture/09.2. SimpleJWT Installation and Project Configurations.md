
# 9.2. SimpleJWT Installation and Project Configurations

## 1. Background and Package Selection
While Django REST Framework includes built-in TokenAuthentication, it uses static database-backed tokens that do not expire automatically. For modern stateless architectures, we use **SimpleJWT** (`djangorestframework-simplejwt`), a highly configurable, community-supported package built specifically for managing JSON Web Tokens in DRF.

## 2. Installation and Registration Steps
First, install the library inside your active Python virtual environment:
```bash
pip install djangorestframework-simplejwt
```

Next, add the package to your project's `INSTALLED_APPS` and update the default authentication settings in **`settings.py`**:

```python
# settings.py

INSTALLED_APPS = [
    # ... Standard Django Apps
    'rest_framework',
    # Registrating the SimpleJWT Blacklist app is optional but recommended 
    # to support token revocation during user logouts.
    'rest_framework_simplejwt',
    'rest_framework_simplejwt.token_blacklist',
    'clinical',
]

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        # Sets JWT as the primary authentication mechanism across the entire API
        'rest_framework_simplejwt.authentication.JWTAuthentication',
        'rest_framework.authentication.SessionAuthentication',
    ),
    'DEFAULT_PERMISSION_CLASSES': (
        'rest_framework.permissions.IsAuthenticated',
    ),
}
```

## 3. Verifying the Configuration
After updating your settings, run database migrations to create the database tables required by SimpleJWT's token blacklist module:
```bash
python manage.py migrate
```
This migration creates tables like `outstandingtoken` and `blacklistedtoken` to keep track of revoked refresh tokens.
