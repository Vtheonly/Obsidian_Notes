
# 9.4. SimpleJWT Root Routing Views and Endpoints

## 1. Setting Up Token Endpoints
SimpleJWT includes pre-configured views to handle retrieving, refreshing, and revoking tokens. 

Register these views in your application's primary **`urls.py`** routing file:

```python
# project_core/urls.py
from django.contrib import admin
from django.urls import path, include
from rest_framework_simplejwt.views import (
    TokenObtainPairView,
    TokenRefreshView,
    TokenBlacklistView,
)

urlpatterns = [
    path('admin/', admin.site.urls),
    
    # 1. Endpoint to obtain Access and Refresh tokens (Login)
    path('api/token/', TokenObtainPairView.as_view(), name='token_obtain_pair'),
    
    # 2. Endpoint to exchange a Refresh Token for a new Access Token
    path('api/token/refresh/', TokenRefreshView.as_view(), name='token_refresh'),
    
    # 3. Endpoint to revoke a Refresh Token (Logout)
    path('api/token/blacklist/', TokenBlacklistView.as_view(), name='token_blacklist'),
    
    path('api/', include('clinical.urls')),
]
```

## 2. How These Endpoints Work

### I. `TokenObtainPairView` (Authentication / Login)
Clients send a POST request containing user credentials to retrieve a token pair.
* **Endpoint**: `POST /api/token/`
* **Request Payload**:
  ```json
  {
    "email": "doctor@hospital.org",
    "password": "SecurePassword123"
  }
  ```
* **Response Payload (200 OK)**:
  ```json
  {
    "refresh": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ0b2tlbl90eXBlIjoicmVmcmVzaCIsImV4cCI6MTc4MjY1MjgwMC...",
    "access": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ0b2tlbl90eXBlIjoiYWNjZXNzIiwiZXhwIjoxNzgyMDQ4OTAw..."
  }
  ```

### II. `TokenRefreshView` (Token Renewal)
When the short-lived access token expires, the client sends their refresh token to obtain a new, valid access token.
* **Endpoint**: `POST /api/token/refresh/`
* **Request Payload**:
  ```json
  {
    "refresh": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ0b2tlbl90eXBlIjoicmVmcmVzaCIsImV4cCI6MTc4MjY1MjgwMC..."
  }
  ```
* **Response Payload (200 OK)**:
  ```json
  {
    "access": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ0b2tlbl90eXBlIjoiYWNjZXNzIiwiZXhwIjoxNzgyMDQ5ODAw..."
  }
  ```

### III. `TokenBlacklistView` (Logout / Revocation)
When a user logs out, the client app sends the user's refresh token to this endpoint to permanently revoke it.
* **Endpoint**: `POST /api/token/blacklist/`
* **Request Payload**:
  ```json
  {
    "refresh": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ0b2tlbl90eXBlIjoicmVmcmVzaCIsImV4cCI6MTc4MjY1MjgwMC..."
  }
  ```
* **Response Payload (200 OK)**:
  ```json
  {
    "detail": "Token successfully blacklisted."
  }
  ```
