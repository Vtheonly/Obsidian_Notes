
# 6.2. Installation and REST Configuration Core Settings

## 1. Setup and Environment Configuration
To build RESTful APIs with Django, install the **Django REST Framework (DRF)** package within your active virtual environment:

```bash
pip install djangorestframework
```

Next, add the framework to your project's **`settings.py`** under `INSTALLED_APPS`:

```python
INSTALLED_APPS = [
    # Core Django apps
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    
    # Third-party applications
    'rest_framework',
    
    # Local application modules
    'clinical',
]
```

## 2. Managing the Global `REST_FRAMEWORK` Dictionary
You can configure DRF's global behaviors in `settings.py` using the `REST_FRAMEWORK` configuration dictionary. This centralizes settings like authorization rules, rendering engines, and pagination across your entire API.

```python
REST_FRAMEWORK = {
    # 1. Global Authentication classes (determines who is sending the request)
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework.authentication.SessionAuthentication',
        'rest_framework.authentication.BasicAuthentication',
    ],
    
    # 2. Global Permission classes (determines what the request user can do)
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',
    ],
    
    # 3. Standard Request Pagination Configurations
    'DEFAULT_PAGINATION_CLASS': 'rest_framework.pagination.PageNumberPagination',
    'PAGE_SIZE': 25,
    
    # 4. Payload Parser Handlers (processing incoming formats)
    'DEFAULT_PARSER_CLASSES': [
        'rest_framework.parsers.JSONParser',
        'rest_framework.parsers.FormParser',
        'rest_framework.parsers.MultiPartParser',
    ],
    
    # 5. Payload Rendering Handlers (processing output response formats)
    'DEFAULT_RENDERER_CLASSES': [
        'rest_framework.renderers.JSONRenderer',
        # BrowsableAPIRenderer provides a user-friendly web interface for testing your API endpoints
        'rest_framework.renderers.BrowsableAPIRenderer',
    ],
}
```

## 3. Important Reminders & Student Traps
* **Production Security Warning**: The default `BrowsableAPIRenderer` should be disabled in production environments. Leaving it active exposes details about your API structure, models, and endpoints to anyone who visits the URL in a browser. 
  
  Disable it in production by limiting your renderers to the JSON renderer:
  ```python
  if not DEBUG:
      REST_FRAMEWORK['DEFAULT_RENDERER_CLASSES'] = [
          'rest_framework.renderers.JSONRenderer',
      ]
  ```
