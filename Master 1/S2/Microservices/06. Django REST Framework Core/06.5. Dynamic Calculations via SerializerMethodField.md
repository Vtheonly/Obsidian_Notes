
# 6.5. Dynamic Calculations via SerializerMethodField

## 1. When to Use Calculated Fields
Sometimes, your API needs to return calculated or derived data that is not stored directly in your database models (for example, formatting a user's full name, calculating their age from a birthdate, or counting related child records).

In DRF, you can add read-only, calculated fields to your serializers using **`SerializerMethodField`**.

## 2. Python Implementation Example
```python
from rest_framework import serializers
from clinical.models import Patient, Consultation
from django.utils import timezone

class ComplexPatientSerializer(serializers.ModelSerializer):
    # 1. Define the SerializerMethodField
    age = serializers.SerializerMethodField()
    consultations_count = serializers.SerializerMethodField()

    class Meta:
        model = Patient
        fields = ['id', 'nom', 'date_naissance', 'age', 'consultations_count']

    # 2. Implement the getter method for 'age'
    # By default, DRF looks for a method named 'get_<field_name>'
    def get_age(self, obj):
        # 'obj' is the model instance being serialized
        if obj.date_naissance:
            return timezone.now().year - obj.date_naissance.year
        return None

    # 3. Implement the getter method for 'consultations_count'
    def get_consultations_count(self, obj):
        # Counts the related consultations for this patient
        return obj.consultations.count()
```

## 3. Important Performance Considerations
While `SerializerMethodField` is useful, it can introduce performance issues if not handled carefully.

Calling database queries inside a `get_<field_name>` method (such as `obj.consultations.count()`) will execute a separate query for **every** record in a list view. If you are retrieving a list of 100 patients, this will trigger 100 separate database queries. This is known as the **N+1 Query Problem**.

### How to Prevent the N+1 Problem
If your calculated fields rely on related database records, optimize your views by pre-calculating or pre-fetching the data using Django's `annotate` or `prefetch_related` methods:
```python
# Optimize the view's queryset using Django's annotate method:
from django.db.models import Count
queryset = Patient.objects.annotate(total_consultations=Count('consultations'))
```

```python
# Then, retrieve the annotated value directly in your serializer:
def get_consultations_count(self, obj):
    # Returns the pre-calculated database value instead of running a new query
    return getattr(obj, 'total_consultations', 0)
```
