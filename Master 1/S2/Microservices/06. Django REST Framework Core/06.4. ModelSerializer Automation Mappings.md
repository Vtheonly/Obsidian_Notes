
# 6.4. ModelSerializer Automation Mappings

## 1. What is a ModelSerializer?
Writing manual serializers for every model in a large application can result in highly repetitive code.

To simplify this, DRF provides the **`ModelSerializer`** class. A `ModelSerializer` inspects your Django models and automatically:
1. Generates serializer fields that match your model's database columns.
2. Configures default validators based on your model constraints (such as `unique=True`).
3. Implements standard `create()` and `update()` methods.

## 2. Python Implementation Example
```python
from rest_framework import serializers
from clinical.models import Patient

class PatientModelSerializer(serializers.ModelSerializer):
    class Meta:
        # 1. Specify the target database model
        model = Patient
        
        # 2. Define which fields to include in the serialized output
        fields = ['id', 'nom', 'email', 'date_naissance', 'is_active']
        
        # Alternatives:
        # fields = '__all__'  # Includes all fields on the model
        # exclude = ['is_active']  # Includes all fields EXCEPT specified ones
        
        # 3. Define write-once / read-only fields
        read_only_fields = ['id', 'is_active']
        
        # 4. Configure extra validation constraints or overrides
        extra_kwargs = {
            'email': {'required': True, 'allow_blank': False},
        }
```

## 3. Under the Hood: Meta Options Reference

* **`fields`**:
  * We recommend listing fields explicitly (e.g., `['id', 'nom']`).
  * Avoid using `fields = '__all__'` in production. If you add sensitive fields to your database model later (such as a password hash or internal status field), they will be automatically exposed on your API endpoints.
* **`read_only_fields`**: Marks fields as read-only. These fields are included in GET responses, but are ignored when processing write requests (like POST, PUT, or PATCH).
* **`extra_kwargs`**: Allows you to customize or override field parameters on the auto-generated serializer fields without declaring them explicitly on the class.
