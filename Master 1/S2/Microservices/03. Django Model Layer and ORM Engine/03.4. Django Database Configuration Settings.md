
# 3.4. Django Database Configuration Settings

## 1. Deep-Dive: Configuring `settings.py`
Django configures its database connections using the `DATABASES` dictionary located in the central configuration file (`settings.py`). This dictionary allows you to manage multiple engine connections.

### Default SQLite Configuration (Development)
By default, Django is configured to use SQLite, which is a file-based database ideal for development but not recommended for production environments.
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}
```

### Production PostgreSQL Configuration
Production applications often use more robust database engines like PostgreSQL. This requires setting up hostnames, credentials, and ports:
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'hospital_db',
        'USER': 'db_admin',
        'PASSWORD': 'SecureProductionPassword123',
        'HOST': 'postgresql-cluster.internal',
        'PORT': '5432',
    }
}
```

## 2. Best Practice: Environment Variables and Security
**Never** hardcode credentials like database names, usernames, or passwords in your codebase. Doing so is a major security risk, especially if the repository is pushed to a public platform like GitHub.

### Security Implementation Using `python-dotenv`
To secure your database settings, use environment variables. Install the library first:
```bash
pip install python-dotenv
```

Create a `.env` file in your root project directory:
```env
DB_NAME=hospital_db
DB_USER=db_admin
DB_PASSWORD=SecureProductionPassword123
DB_HOST=127.0.0.1
DB_PORT=5432
```

Configure `settings.py` to load these variables:
```python
import os
from pathlib import Path
from dotenv import load_dotenv

BASE_DIR = Path(__file__).resolve().parent.parent
load_dotenv(BASE_DIR / '.env')

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.getenv('DB_NAME'),
        'USER': os.getenv('DB_USER'),
        'PASSWORD': os.getenv('DB_PASSWORD'),
        'HOST': os.getenv('DB_HOST'),
        'PORT': os.getenv('DB_PORT', '5432'),
    }
}
```

## 3. Database Drivers
To connect to databases other than SQLite, you must install the appropriate database driver in your virtual environment:
* **PostgreSQL**: Requires `psycopg2` or `psycopg2-binary`.
  ```bash
  pip install psycopg2-binary
  ```
* **MySQL / MariaDB**: Requires `mysqlclient`.
  ```bash
  pip install mysqlclient
  ```
