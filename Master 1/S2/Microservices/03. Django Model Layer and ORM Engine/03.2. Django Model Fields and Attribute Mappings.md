
# 3.2. Django Model Fields and Attribute Mappings

## 1. Core Model Fields Reference
Django uses Python attributes to define model fields. These classes translate directly to database data types. Here is a mapping of common Django field types to their PostgreSQL and SQLite counterparts:

| Django Model Field Class | PostgreSQL Data Type | SQLite Data Type | Primary Validation / Usage |
| :--- | :--- | :--- | :--- |
| `CharField(max_length=...)` | `VARCHAR(max_length)` | `TEXT` | Short to medium text strings. |
| `TextField()` | `TEXT` | `TEXT` | Long text or unstructured content. |
| `IntegerField()` | `INTEGER` | `INTEGER` | Standard integer values. |
| `FloatField()` | `DOUBLE PRECISION` | `REAL` | Floating-point numbers. |
| `DecimalField(max_digits, decimal_places)` | `NUMERIC(max, dec)` | `NUMERIC` | Fixed-point numbers for currency/precision. |
| `BooleanField()` | `BOOLEAN` | `INTEGER` | Binary `True` or `False`. |
| `DateField()` | `DATE` | `TEXT` | Standard date calendar formats (YYYY-MM-DD). |
| `DateTimeField()` | `TIMESTAMP WITH TZ` | `TEXT` | Exact point in time with timezones. |
| `EmailField()` | `VARCHAR(254)` | `TEXT` | Validates email syntax on save. |
| `UUIDField()` | `UUID` | `TEXT` | Universally Unique Identifier for primary keys. |

## 2. Code Implementation Example
Below is a model demonstrating how to implement these fields:

```python
from django.db import models
import uuid

class MedicalFacility(models.Model):
    # UUID as primary key prevents object enumeration attacks
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    name = models.CharField(max_length=200)
    description = models.TextField()
    capacity = models.IntegerField()
    efficiency_rating = models.FloatField()
    daily_operational_cost = models.DecimalField(max_digits=10, decimal_places=2)
    is_active = models.BooleanField(default=True)
    established_date = models.DateField()
    last_audit_time = models.DateTimeField()
    contact_email = models.EmailField()

    def __str__(self):
        return self.name
```

## 3. Important Reminders & Student Traps

### Floating Point vs. Decimal for Financial Records
* **Float (`FloatField`)** uses binary floating-point representation, which can introduce rounding errors:
  $$\text{e.g., } 0.1 + 0.2 \neq 0.3 \text{ due to floating-point representation limits.}$$
* **Decimal (`DecimalField`)** uses exact decimal arithmetic, making it the correct choice for currency, financial values, or high-precision measurements.
* **Trap**: Attempting to use a `DecimalField` without specifying both `max_digits` and `decimal_places` will raise a configuration error.

### EmailField Validation Limits
* **Trap**: The database engine treats `EmailField` as a standard `VARCHAR` column. While Django applies built-in form-level email validation, a database-level INSERT statement bypassed through raw queries or external tools can write invalid email strings to this field. Ensure that data pipeline validations are enforced at multiple levels.
