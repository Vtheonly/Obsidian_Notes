
# 3.9. Django ORM CRUD Operations API

## 1. Complete ORM CRUD Operations Reference

This guide covers basic Create, Read, Update, and Delete operations using Django's ORM API.

### CREATE Operations
You can write records to your database in two ways:

Using the `.save()` method:
```python
from myapp.models import Patient

# Instantiates the object in memory
new_patient = Patient(nom="Alice", email="alice@email.com")
# Writes the record to the database
new_patient.save()
```

Using the `.create()` shortcut method, which instantiates and saves the object in a single operation:
```python
# Creates and saves the record in one step
new_patient = Patient.objects.create(nom="Alice", email="alice@email.com")
```

---

### READ Operations

#### Retrieving a Single Record (`get`)
Retrieves a single record matching the query lookup parameters.
```python
patient = Patient.objects.get(id=1)
```
* **Warning**: `.get()` expects exactly one matching record.
  * If no records match, it raises `Patient.DoesNotExist`.
  * If multiple records match, it raises `Patient.MultipleObjectsReturned`.
  * Always wrap `.get()` in a `try-except` block for safety:
    ```python
    try:
        patient = Patient.objects.get(email="unique@email.com")
    except Patient.DoesNotExist:
        patient = None
    ```

#### Retrieving Lists of Records (`filter` and `exclude`)
Returns a `QuerySet` containing records matching the query parameters:
```python
# Returns all patients named Alice
alices = Patient.objects.filter(nom="Alice")

# Returns all patients except those named Alice
non_alices = Patient.objects.exclude(nom="Alice")
```

#### Querying All Records (`all`)
Returns a `QuerySet` containing all records in the table:
```python
all_patients = Patient.objects.all()
```

---

### UPDATE Operations

#### Updating a Single Instance
Retrieve the object, modify its attributes, and call `.save()`:
```python
patient = Patient.objects.get(id=1)
patient.nom = "Alice Smith"
patient.save()  # Triggers an SQL UPDATE statement
```

#### Batch Updating Multiple Records
Update multiple records at once using a filtered QuerySet:
```python
# Updates all matching records in a single database query
Patient.objects.filter(nom="Alice").update(nom="Alice Verified")
```

---

### DELETE Operations

#### Deleting a Single Instance
```python
patient = Patient.objects.get(id=1)
patient.delete()
```

#### Batch Deleting Multiple Records
```python
# Deletes all matching records from the database
Patient.objects.filter(nom="Alice Verified").delete()
```

## 2. SQL Equivalents Comparison

| Django ORM Statement | Raw SQL Translation |
| :--- | :--- |
| `Patient.objects.create(nom="Bob")` | `INSERT INTO app_patient (nom) VALUES ('Bob');` |
| `Patient.objects.get(id=5)` | `SELECT * FROM app_patient WHERE id = 5 LIMIT 2;` |
| `Patient.objects.filter(nom="Bob")` | `SELECT * FROM app_patient WHERE nom = 'Bob';` |
| `Patient.objects.filter(id=1).update(nom="B")` | `UPDATE app_patient SET nom = 'B' WHERE id = 1;` |
| `patient.delete()` | `DELETE FROM app_patient WHERE id = patient_id;` |

## 3. Important Performance Optimization Tips
* **Avoid evaluating QuerySets inside loops**:
  ```python
  # BAD: Evaluates the database query on each iteration
  for patient in Patient.objects.all():
      print(patient.nom)
  ```
* **Use `.exists()` to check for records**:
  If you only need to check if a record exists, use `.exists()` instead of retrieving the entire record. This avoids loading the database payload into memory:
  ```python
  # Efficient database check
  if Patient.objects.filter(email="check@email.com").exists():
      pass
  ```
