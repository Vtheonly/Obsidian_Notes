
# 3.7. Model Relationships One-to-One Relationships

## 1. Declaring One-to-One Relationships
A **One-to-One** relationship specifies that a single record in one table can be linked to at most one record in another table.

In Django, this is represented using the `models.OneToOneField`. Conceptually, it behaves like a `ForeignKey` with a unique constraint (`unique=True`) enforced on the foreign key column.

```mermaid

    PATIENT {
        int id PK

```

### Python Implementation
```python
from django.db import models

class Patient(models.Model):
    nom = models.CharField(max_length=100)
    email = models.EmailField(unique=True)

    def __str__(self):
        return self.nom

class DossierMedical(models.Model):
    # 'primary_key=True' makes the foreign key the primary key of this table as well
    patient = models.OneToOneField(
        Patient, 
        on_delete=models.CASCADE, 
        primary_key=True,
        related_name='dossier'
    )
    groupe_sanguin = models.CharField(max_length=3)
    allergies = models.TextField(blank=True)

    def __str__(self):
        return f"Medical Record for {self.patient.nom}"
```

## 2. Key Differences: `OneToOneField` vs. `ForeignKey`

### Query Set Behavior on Reverse lookup
* **`ForeignKey`**: When querying a relationship in reverse, Django returns a manager that evaluates to a list of matching records (`QuerySet`):
  ```python
  # Returns a manager/QuerySet
  patient.consultations.all() 
  ```
* **`OneToOneField`**: When querying a relationship in reverse, Django returns a single model instance rather than a QuerySet. If no related object exists, it raises a `DoesNotExist` exception:
  ```python
  # Returns a single DossierMedical instance
  patient.dossier 
  ```

## 3. Common Student Traps
* **Failing to Handle `DoesNotExist` on Reverse lookups**:
  If a `Patient` does not have a `DossierMedical` on file, calling `patient.dossier` will throw a `DossierMedical.DoesNotExist` exception and crash the application if not handled.
  
  *How to verify safely:*
  ```python
  # Safe attribute checking
  if hasattr(patient, 'dossier'):
      blood_type = patient.dossier.groupe_sanguin
  else:
      blood_type = "Unknown"
  ```
