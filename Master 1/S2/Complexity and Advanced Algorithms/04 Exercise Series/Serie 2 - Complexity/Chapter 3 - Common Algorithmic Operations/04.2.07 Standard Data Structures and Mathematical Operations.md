# 1. Standard Data Structures and Mathematical Operations

Exercise 3 in TD2 asks us to classify standard algorithms into their Big-O time complexities. Memorizing these is crucial for analyzing larger algorithms built from these primitive operations.

| Algorithme associé au problème | O(?) | Justification / Reason |
| :--- | :--- | :--- |
| **Somme de deux vecteurs de n éléments** <br>*(Sum of two 1D arrays)* | **$O(n)$** | You must iterate through the array once `for(i=0 to n)` to add corresponding elements `C[i] = A[i] + B[i]`. |
| **Somme de deux matrices carrées d'ordre n** <br>*(Sum of 2D matrices)* | **$O(n^2)$** | A matrix of order $n$ has $n \times n = n^2$ elements. You need two nested loops to visit and sum every cell `C[i][j] = A[i][j] + B[i][j]`. |
| **Produit de deux matrices carrées d'ordre n** <br>*(Matrix Multiplication)* | **$O(n^3)$** | Matrix multiplication requires three nested loops: iterate rows of A ($n$), columns of B ($n$), and compute the dot product of length $n$. ($n \times n \times n = n^3$). |
| **Calcul itératif de la factorielle de n** <br>*(Iterative Factorial)* | **$O(n)$** | A single `for` or `while` loop running from $1$ to $n$ computing `fact = fact * i`. |
| **Calcul itératif de la somme 1 + 2 + ... + n** | **$O(n)$** | A single loop summing up the numbers. Note: Doing this mathematically via formula $\frac{n(n+1)}{2}$ is $O(1)$, but the *iterative* computation is $O(n)$. |
| **Insertion en tête d'une liste chaînée simple** <br>*(Insert at Head of Linked List)* | **$O(1)$** | Just pointer manipulation: `newNode->next = head; head = newNode;`. No traversal needed. |
| **Insertion en tête d'un tableau** <br>*(Insert at Head of Array)* | **$O(n)$** | An array is contiguous memory. To insert at index 0, you must shift *all* existing $n$ elements one position to the right. |
| **Insertion à la fin d'une liste chaînée simple** <br>*(Insert at Tail of Linked List)* | **$O(n)$** | Unless you have a specific "tail" pointer saved in memory, you must start at the head and traverse all $n$ nodes to find the end before inserting. |
| **Insertion à la fin d'un tableau** <br>*(Insert at Tail of Array)* | **$O(1)$** | Assuming the array is not full (no dynamic resizing needed), you just place the item at `array[n]` directly. |
| **Tester si un nombre n est pair ou impair** <br>*(Check if even/odd)* | **$O(1)$** | A single modulo arithmetic operation: `n % 2 == 0`. |
