# 1. Matrix Search Complexity and Operation Counting

> [!info] Background Knowledge: Complexity Analysis
> When analyzing algorithms, we evaluate them based on three scenarios:
> - **Best Case ($\Omega$)**: The minimum number of operations required. Everything aligns perfectly.
> - **Worst Case ($O$)**: The absolute maximum number of operations. The algorithm is forced to do the maximum amount of work.
> - **Average Case ($\Theta$)**: The expected number of operations based on probabilities.
> 
> To find exact complexity, we must count the primitive operations (assignments, comparisons, arithmetic, logical evaluations) executed by the algorithm.

## The Algorithm: Searching `x` in an $n \times n$ Matrix

The provided algorithm is a brute-force search over a 2D matrix $M$ of size $n \times n$. It uses two `repeat...until` loops to traverse rows (`i`) and columns (`j`).

```mermaid
flowchart TD
    Start([Start]) --> Init1["found &larr; Faux<br>i &larr; 1"]
    Init1 --> OuterLoop["Outer Loop: repeat"]
    OuterLoop --> InitJ["j &larr; 1"]
    InitJ --> InnerLoop["Inner Loop: repeat"]
    
    InnerLoop --> CondX{"M["i"][j] == x ?"}
    CondX -- Yes --> SetFound["found &larr; Vrai"]
    CondX -- No --> IncJ["j &larr; j + 1"]
    SetFound --> IncJ
    
    IncJ --> CondInner{"j > n OR found == Vrai"}
    CondInner -- False --> InnerLoop
    CondInner -- True --> IncI["i &larr; i + 1"]
    
    IncI --> CondOuter{"i > n OR found == Vrai"}
    CondOuter -- False --> OuterLoop
    CondOuter -- True --> Print["afficher(found)"]
    Print --> End([End])
```

---

## 1. Best Case Scenario
**Situation**: The element $x$ is found on the very first try, at the very first cell $M[1][1]$.
**Slide Answer**: $16 \in O(1)$

### Where does "16" come from? (Gap Fill)
The slide assumes the answer is 16 operations but skips the counting. Let's break down exactly what the computer executes if the element is at $M[1][1]$:

1. `found <- Faux` *(1 op: assignment)*
2. `i <- 1` *(1 op: assignment)*
3. `j <- 1` *(1 op: assignment)*
4. `si M[i][j] = x` *(1 op: comparison)* 
5. `found <- Vrai` *(1 op: assignment)*
6. `j <- j + 1` *(2 ops: 1 addition, 1 assignment)*
7. `jusqua (j > n ou found = Vrai)` *(3 ops: 1 comparison `j>n`, 1 comparison `found=Vrai`, 1 logical `ou`)*
   *(Condition is met because found=Vrai, inner loop breaks)*
8. `i <- i + 1` *(2 ops: 1 addition, 1 assignment)*
9. `jusqua (i > n ou found = Vrai)` *(3 ops: 1 comparison `i>n`, 1 comparison `found=Vrai`, 1 logical `ou`)*
   *(Condition is met because found=Vrai, outer loop breaks)*
10. `afficher(found)` *(1 op: function call)*

**Total Operations**: $1 + 1 + 1 + 1 + 1 + 2 + 3 + 2 + 3 + 1 = 16 \text{ operations.}$
Because 16 is a constant number that never changes regardless of how big $n$ is, the Best Case complexity is strictly **$O(1)$**.

---

## 2. Worst Case Scenario
The worst case has two sub-possibilities:
1. The element $x$ **does not exist** in the matrix at all.
2. The element $x$ is located in the **very last cell** $M[n][n]$.

### Possibility 1: Element does not exist
If $x$ is not found, the algorithm must check every single cell. 

**Step A: Cost of the Inner Loop (`j`) per iteration**
Inside the inner loop, for every single step, we do:
- `si M[i][j] = x` *(1 op)* (Always false here)
- `j <- j + 1` *(2 ops)*
- `jusqua (j > n ou found = Vrai)` *(3 ops)*
**Inner loop total per step**: $1 + 2 + 3 = 6 \text{ operations.}$
Because it runs $n$ times (the width of the matrix), the total inner loop cost is **$6 \times n = 6n$**.

**Step B: Cost of the Outer Loop (`i`) per iteration**
For each row `i`, we do:
- `j <- 1` *(1 op)*
- Execute the entire inner loop *(6n ops)*
- `i <- i + 1` *(2 ops)*
- `jusqua (i > n ou found = Vrai)` *(3 ops)*
**Outer loop total per step**: $1 + 6n + 2 + 3 = 6n + 6$.
Because it runs $n$ times (the height of the matrix), the total outer loop cost is $n \times (6n + 6) = \mathbf{6n^2 + 6n}$.

**Step C: The Global Cost**
Add the operations outside the loops:
- `found <- Faux` (1 op)
- `i <- 1` (1 op)
- `afficher(found)` (1 op)
Total outer ops = 3.

**Final Formula ($Nbr \ d'op1$)**: $6n^2 + 6n + 3$

### Possibility 2: Element is in the very last cell
If the element is at $M[n][n]$, it does the exact same work as Possibility 1, *except* on the very last iteration of the very last cell, `found <- Vrai` is executed.
- `found <- Vrai` adds exactly 1 operation.

**Final Formula ($Nbr \ d'op2$)**: $6n^2 + 6n + 4$

> [!tip] Big-O Conclusion
> In Big-O analysis, we drop constants and lower-order terms. Therefore, $6n^2 + 6n + 4$ simplifies to $O(n^2)$. 
> This makes logical sense: verifying every cell of an $n \times n$ matrix requires checking $n^2$ items.
