# 3. Complexity Lookup Table for Standard Operations

> [!info] How to Read This Note
> The material provided simply listed answers in a table. In exams and real-world development, you must understand *why* these complexities are true. Below is the expanded table from Exercice 03, fully explained.

| Algorithmic Problem | Complexity | Detailed Reasoning & Explanation |
| :--- | :--- | :--- |
| **Sum of two vectors of $n$ elements** | $O(n)$ | A vector is a 1D array. To sum two vectors, you must iterate through them once from index $1$ to $n$. A single loop over $n$ items results in $O(n)$ linear time. |
| **Sum of two square matrices of order $n$** | $O(n^2)$ | A matrix is a 2D array ($n \times n$). To sum two matrices, you must add the corresponding cell from Matrix A to Matrix B. This requires visiting every single cell. Number of cells is $n \times n = n^2$. Hence, $O(n^2)$. |
| **Product of two square matrices of order $n$** | $O(n^3)$ | Matrix multiplication is more complex than addition. For each cell in the resulting $n \times n$ matrix, you must calculate the dot product of an entire row of Matrix A and a column of Matrix B (which takes $n$ steps). $n^2$ cells $\times n$ steps per cell = $O(n^3)$. |
| **Iterative calculation of $n!$ (Factorial)** | $O(n)$ | An iterative factorial ($n! = 1 \times 2 \times 3 \dots \times n$) requires a single loop that multiplies a running total $n$ times. 1 loop = $O(n)$. |
| **Iterative calculation of sum $1 + 2 + \dots + n$** | $O(n)$ | Using a loop to add numbers from $1$ to $n$ iteratively means $n$ additions in a row. *(Note: If done using the mathematical formula $n(n+1)/2$, it would be $O(1)$, but the prompt strictly specifies "iteratif" / iterative).* |
| **Insertion at the head of a singly linked list** | $O(1)$ | Linked lists hold pointers to the "head". To insert at the front, you simply create a new node, point it to the current head, and update the head pointer. This takes 2 operations, independent of list size $n$. |
| **Insertion at the head of an array of size $n$** | $O(n)$ | Arrays are contiguous in memory. If you insert an item at index 0, you must shift every single existing item $1$ space to the right. Moving $n$ items takes $O(n)$ time. |
| **Insertion at the end of a singly linked list** | $O(n)$ | In a *standard* singly linked list (without a "tail" pointer), you only know where the head is. To add to the end, you must traverse the entire list node-by-node to find the last item before attaching the new one. Traversal takes $O(n)$. |
| **Insertion at the end of an array of size $n$** | $O(1)$ | You just place the element at index `n` (the next available space). No shifting is required. Constant time. *(Assuming the array is not completely full).* |
| **Test if a number $n$ is even or odd** | $O(n)$ | ⚠️ **READ TIP BELOW** |

> [!warning] Exception / Trick Question Alert: Even or Odd
> In standard modern programming, checking if a number is even or odd is done using the modulo operator (`n % 2 == 0`) or bitwise AND (`n & 1 == 0`). Both of these operations evaluate in **$O(1)$** constant time.
> 
> However, **the provided slide strictly states $O(n)$**. 
> Why? Because in theoretical algorithmic exercises, if you do not have hardware modulo operations, you must calculate even/odd using iterative subtraction:
> ```python
> while n >= 2:
>     n = n - 2
> if n == 0: return "Even"
> ```
> This loop subtracts $2$ repeatedly, running $\frac{n}{2}$ times. Since constants are dropped in Big-O, $\frac{n}{2}$ becomes **$O(n)$**. Always clarify with your professor if they expect the mathematical hardware $O(1)$ or the iterative subtractive $O(n)$. Based strictly on TD 2, the answer is $O(n)$.
