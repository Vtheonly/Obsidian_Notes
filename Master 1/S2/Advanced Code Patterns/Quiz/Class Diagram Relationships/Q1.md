# UML Fundamentals Quiz

### True or False

> [!question] Attributes written inside the middle compartment of a class box are the only attributes that class will have in the final code.
>> [!success]- Answer
>> False

> [!question] A unidirectional association (arrow) means that both classes will contain reference variables pointing to each other.
>> [!success]- Answer
>> False

> [!question] In a UML diagram, a solid line between two classes represents an implicit attribute in memory.
>> [!success]- Answer
>> True

> [!question] An Association Class is used when the relationship between two classes carries its own specific data.
>> [!success]- Answer
>> True

> [!question] Relational databases (SQL) can easily implement Many-to-Many relationships without using a middleman or join table.
>> [!success]- Answer
>> False

> [!question] A child class inherits both the explicit attributes and the implicit reference variables of its parent class.
>> [!success]- Answer
>> True

> [!question] Aggregation (hollow diamond) implies that the child object cannot exist if the parent object is destroyed.
>> [!success]- Answer
>> False

> [!question] Composition (solid diamond) represents a strict "Has-A" relationship where the parent manages the lifecycle of the child.
>> [!success]- Answer
>> True

> [!question] If no Role name is written on an association line, the convention is to use the target Class name (starting with a lowercase letter) as the variable name.
>> [!success]- Answer
>> True

> [!question] The "multiplicity" of an association determines whether the attribute is a single object or a collection like a List.
>> [!success]- Answer
>> True

### Multiple Choice

> [!question] Which compartment of a standard UML class box contains the Methods (Functions)?
> a) Top Compartment
> b) Middle Compartment
> c) Bottom Compartment
> d) Outside the box
>> [!success]- Answer
>> c) Bottom Compartment

> [!question] What does a solid line with a hollow triangle pointing to a class represent?
> a) Association
> b) Composition
> c) Inheritance
> d) Aggregation
>> [!success]- Answer
>> c) Inheritance

> [!question] In the context of an association, what determines the *name* of the hidden parameter in the code?
> a) Multiplicity
> b) Navigability
> c) The Role
> d) The Class Name only
>> [!success]- Answer
>> c) The Role

> [!question] If a multiplicity is marked as `0..*`, how should the attribute be implemented in Java?
> a) `private Object obj;`
> b) `private List<Object> objs;`
> c) `private int count;`
> d) `private String name;`
>> [!success]- Answer
>> b) `private List<Object> objs;`

> [!question] When translating an Association Class to a Database, it becomes a:
> a) Primary Key
> b) Column in the parent table
> c) Join Table (Table de jointure)
> d) View
>> [!success]- Answer
>> c) Join Table (Table de jointure)

> [!question] Which symbol represents "Composition" in UML?
> a) A hollow diamond
> b) A solid black diamond
> c) A hollow triangle
> d) A dashed arrow
>> [!success]- Answer
>> b) A solid black diamond

> [!question] In a `Vol` (Flight) class with two lines pointing to `Aeroport` labeled `depart` and `arrivee`, how many variables of type `Aeroport` will the `Vol` class have?
> a) One
> b) Two
> c) Zero
> d) Four
>> [!success]- Answer
>> b) Two

> [!question] An "Is-A" relationship is best described by:
> a) Association
> b) Aggregation
> c) Inheritance
> d) Composition
>> [!success]- Answer
>> c) Inheritance

> [!question] What is a common "Primitive" data type found in the middle compartment?
> a) List
> b) Integer (int)
> c) Association Class
> d) Interface
>> [!success]- Answer
>> b) Integer (int)

> [!question] In Composition, if the parent object is set to `null` and garbage collected:
> a) The child objects remain in memory.
> b) The child objects are also destroyed.
> c) The child objects are moved to a different class.
> d) Nothing happens to the child objects.
>> [!success]- Answer
>> b) The child objects are also destroyed.
