### Dependency Injection

To make this pattern truly flexible, we shouldn't hardcode the behavior inside the Duck. We should "inject" the behavior when we create the Duck. The most common method is **Constructor Injection**.

### Code Example (Java/C# Style)

#### 1. The Interfaces

First, define the abstract behaviors.

```java
public interface IFlyBehavior {
    void fly();
}

public interface IQuackBehavior {
    void quack();
}
```

#### 2. The Concrete Strategies (Algorithms)

Implement the specific behaviors.

```java
// Strategy A: Standard wings
public class SimpleFlying implements IFlyBehavior {
    public void fly() {
        System.out.println("I'm flying with wings!");
    }
}

// Strategy B: Rocket power
public class JetFlying implements IFlyBehavior {
    public void fly() {
        System.out.println("ZOOM! I'm flying with a jetpack!");
    }
}

// Strategy C: Cannot fly
public class NoFlying implements IFlyBehavior {
    public void fly() {
        System.out.println("... I can't fly.");
    }
}
```

#### 3. The Context (The Duck Class)

The Duck class focuses on **delegation**.

```java
public class Duck {
    // Composition: Duck HAS-A behavior
    private IFlyBehavior flyBehavior;
    private IQuackBehavior quackBehavior;

    // Constructor Injection
    public Duck(IFlyBehavior flyBehavior, IQuackBehavior quackBehavior) {
        this.flyBehavior = flyBehavior;
        this.quackBehavior = quackBehavior;
    }

    // The Delegate Method
    public void performFly() {
        // Delegate the action to the behavior class
        this.flyBehavior.fly();
    }

    public void performQuack() {
        this.quackBehavior.quack();
    }
}
```

#### 4. Usage (The "Main" Method)

Here we assemble the ducks. This is where the flexibility shines.

```java
// Create a Wild Duck (Standard flying, Loud quack)
Duck wildDuck = new Duck(new SimpleFlying(), new LoudQuack());
wildDuck.performFly(); // Output: I'm flying with wings!

// Create a Rubber Duck (No flying, Squeak)
Duck rubberDuck = new Duck(new NoFlying(), new Squeak());
rubberDuck.performFly(); // Output: ... I can't fly.

// Create a Robo-Duck (Jet flying, Robo-quack)
Duck roboDuck = new Duck(new JetFlying(), new RoboQuack());
roboDuck.performFly(); // Output: ZOOM! I'm flying with a jetpack!
```

### Critical Concept: Runtime Flexibility

Because `flyBehavior` is a variable, we can add a "Setter" method to the Duck class:

```java
public void setFlyBehavior(IFlyBehavior fb) {
    this.flyBehavior = fb;
}
```

This allows us to **change the behavior dynamically at runtime**. A duck could start with `SimpleFlying`, break its wing, and be switched to `NoFlying` without creating a new object. Inheritance cannot do this.
