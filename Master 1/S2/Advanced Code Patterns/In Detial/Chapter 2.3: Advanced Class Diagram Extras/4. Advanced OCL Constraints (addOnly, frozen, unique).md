
# 4. Advanced OCL Constraints (addOnly, frozen, unique)

Standard UML sometimes isn't strict enough. OCL (Object Constraint Language) allows you to add textual rules inside curly braces `{}` to lock down the system's logic. (Covered in Slide 39).

### 1. `{unique}`
Used on an attribute or an association to indicate that there can be no duplicate values.
* *Example:* A bank might have a list of client IDs. If the association is marked `{unique}`, it means `[101, 102, 103]` is valid, but `[101, 101, 102]` is rejected. (In Java, this implies a `Set` rather than a `List`).

### 2. `{frozen}` or `{readOnly}`
Once the value is set (usually during instantiation), it **cannot ever be modified**. 
* *Example:* The `dateNaissance` of a Personne, or the `numeroSecuriteSociale`.
* *Exam Code Translation:* If you see `{frozen}` on an attribute, you must **NOT** write a Setter method for it in the Java code! You only set it in the Constructor, and you only provide a Getter.

### 3. `{addOnly}`
Applied to collections (associations with `*` multiplicity). It means you can insert new objects into the collection, but you **cannot remove or replace** existing ones.
* *Example:* A log or an `Historique` of transactions. You can add a new log entry, but deleting a past log entry is strictly forbidden.

### 4. Derived Attributes Re-visited (The `/` symbol)
If an attribute has a `/` in front of it (e.g., `/prixDuPlein`), it is a derived attribute. 
**The Hidden Rule:** A derived attribute usually requires an underlying method to calculate it.
If you have `-/age : int`, you must also have a `+getAge() : int` method in the operations compartment that actually performs `CurrentDate - dateNaissance`.
