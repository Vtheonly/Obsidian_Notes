# 5. Version Control with Git and GitHub

**Git** is a Distributed Version Control System (DVCS). It tracks changes in source code during software development.

## 5.1 Why Version Control?
1.  **History:** You can undo mistakes by reverting to a previous state.
2.  **Collaboration:** Multiple people can work on the same file without overwriting each other.
3.  **Backup:** Code is stored remotely (cloud).

## 5.2 Git vs. GitHub
*   **Git:** The software installed on your computer (Local).
*   **GitHub:** A website that hosts Git repositories (Remote). Other examples include GitLab and Bitbucket.

## 5.3 Basic Git Workflow

### The "Happy Path" Commands:

1.  **Init:** Turn a folder into a Git repository.
    ```bash
    git init
    ```
2.  **Status:** Check which files have changed.
    ```bash
    git status
    ```
3.  **Add:** Move changes to the "Staging Area" (prepare them to be saved).
    ```bash
    git add filename.py    # Add specific file
    git add .              # Add ALL changed files
    ```
4.  **Commit:** Save the snapshot with a message.
    ```bash
    git commit -m "Fixed bug in data cleaning function"
    ```
5.  **Push:** Upload the commit to GitHub.
    ```bash
    git push origin main
    ```

## 5.4 The .gitignore File

**Critical Concept:** You should **never** commit certain files to Git.
1.  **Sensitive Data:** Passwords, API keys, credentials.
2.  **Large Data:** CSVs larger than 100MB (Git is for code, not data storage).
3.  **System Files:** `.DS_Store`, `__pycache__`.
4.  **Virtual Environments:** The `env/` folder (it contains thousands of files; you should re-install them using requirements.txt).

You create a file named `.gitignore` and list these items inside. Git will then completely ignore them.

## 5.5 Requirements Files

Since we ignore the virtual environment, how do we share the setup?
We generate a **requirements.txt** file.

*   **Export:** `pip freeze > requirements.txt` (Saves list of installed libs).
*   **Install:** `pip install -r requirements.txt` (Installs libs from the list).
