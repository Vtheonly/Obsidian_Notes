# 8. Error Handling and Exception Management

Robust code does not crash when it encounters unexpected input. It catches errors and handles them gracefully.

## 8.1 The `try...except` Block

```python
try:
    # Code that might crash
    numerator = 10
    denominator = 0
    result = numerator / denominator
except ZeroDivisionError:
    # Code that runs if the specific error occurs
    print("Error: You cannot divide by zero.")
except TypeError:
    print("Error: Inputs must be numbers.")
else:
    # Runs ONLY if no exception occurred
    print(f"Success! Result is {result}")
finally:
    # Runs ALWAYS (good for cleanup, closing files)
    print("Execution finished.")
```

## 8.2 Common Python Exceptions

1.  **SyntaxError:** You wrote invalid Python code (e.g., missing `:`).
2.  **NameError:** Variable not defined.
3.  **TypeError:** Operation applied to object of inappropriate type (e.g., `"5" + 5`).
4.  **ValueError:** Function receives argument of correct type but inappropriate value (e.g., `int("hello")`).
5.  **KeyError:** Dictionary key not found.
6.  **IndexError:** List index out of range.
