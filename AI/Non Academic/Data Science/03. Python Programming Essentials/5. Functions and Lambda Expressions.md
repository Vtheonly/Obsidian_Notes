# 5. Functions and Lambda Expressions

## 5.1 Defining Functions

Functions are reusable blocks of code. They help adhere to the **DRY Principle** (Don't Repeat Yourself).

```python
def calculate_mean(numbers):
    """
    Docstring: Calculates the average of a list of numbers.
    Args: numbers (list)
    Returns: float
    """
    if not numbers:
        return 0
    return sum(numbers) / len(numbers)
```

### Parameters vs. Arguments
*   **Parameter:** The variable listed inside the parentheses in the function definition (`numbers`).
*   **Argument:** The value sent to the function when it is called (`[1, 2, 3]`).

### Return Values
Python functions can return multiple values using tuples.
```python
def get_stats(nums):
    return min(nums), max(nums)

low, high = get_stats([10, 20, 30]) # Unpacking result
```

## 5.2 Default Arguments and Keywords

```python
def greet(name, message="Hello"):
    print(f"{message}, {name}")

greet("Alice")           # Uses default: "Hello, Alice"
greet("Bob", "Goodbye")  # Overrides default: "Goodbye, Bob"
```

> [!WARNING] **Mutable Defaults**
> Never use a mutable object (like a list) as a default argument. It retains values between calls. Use `None` instead.

## 5.3 Lambda Expressions

A **Lambda** is a small, anonymous function defined with the `lambda` keyword. It can take any number of arguments, but can only have **one expression**.

**Syntax:** `lambda arguments : expression`

### Comparison

*   **Standard Function:**
    ```python
    def square(x):
        return x ** 2
    ```
*   **Lambda:**
    ```python
    square = lambda x: x ** 2
    ```

### Practical Use Case: `map` and `filter`
Lambdas are best used as arguments to higher-order functions.

```python
nums = [1, 2, 3, 4]

# Map: Apply function to all items
squared = list(map(lambda x: x**2, nums))  # [1, 4, 9, 16]

# Filter: Keep items where function returns True
evens = list(filter(lambda x: x%2 == 0, nums)) # [2, 4]
