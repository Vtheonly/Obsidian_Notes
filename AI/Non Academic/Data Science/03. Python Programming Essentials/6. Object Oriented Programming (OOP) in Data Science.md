# 6. Object Oriented Programming (OOP) in Data Science

OOP is a programming paradigm based on the concept of **Objects**, which contain data (attributes) and code (methods).

## 6.1 Core Concepts

1.  **Class:** The blueprint or template (e.g., `CarBlueprint`).
2.  **Object:** A specific instance of the class (e.g., `MyRedToyota`).

## 6.2 Basic Syntax

```python
class Student:
    # The Constructor
    def __init__(self, name, age):
        self.name = name  # Attribute
        self.age = age    # Attribute

    # Method
    def introduce(self):
        return f"Hi, I am {self.name} and I am {self.age} years old."

# Creating Instances
s1 = Student("Alice", 20)
s2 = Student("Bob", 22)

print(s1.introduce())
```

### The `self` parameter
`self` represents the specific instance of the class. It allows each object to maintain its own data.

## 6.3 Inheritance

Inheritance allows a class (Child) to derive attributes and methods from another class (Parent). This promotes code reuse.

```python
class DataScientist(Student): # Inherits from Student
    def __init__(self, name, age, tool):
        super().__init__(name, age) # Initialize parent
        self.tool = tool

    def code(self):
        return f"{self.name} is coding in {self.tool}."
```

## 6.4 Encapsulation

In Python, we denote private variables (internal use only) with an underscore `_`.
*   `self.name`: Public.
*   `self._password`: Protected (convention).
*   `self.__secret`: Private (Python name mangling).

## 6.5 Magic Methods (Dunder Methods)

Special methods surrounded by double underscores. They allow objects to behave like built-in types.

*   `__init__`: Constructor.
*   `__str__`: Defines the string representation (for `print()`).
*   `__len__`: Defines behavior for `len()`.
