# 7. Iterators, Generators, and Memory Efficiency

In Data Science, we often handle datasets larger than our computer's RAM. Loading a 100GB file into a standard List will crash the system. **Generators** solve this.

## 7.1 Iterables vs. Iterators

*   **Iterable:** An object capable of returning its members one at a time (e.g., list, string).
*   **Iterator:** An object representing a stream of data. It returns data one element at a time using `__next__()`.

## 7.2 Generators and `yield`

A Generator is a function that returns an iterator. Instead of returning a single value and exiting, it **yields** a series of values, pausing execution after each one.

### Standard Function (Eager Evaluation)
Builds the entire list in memory.
```python
def get_squares(n):
    result = []
    for i in range(n):
        result.append(i**2)
    return result
```

### Generator (Lazy Evaluation)
Generates one item at a time. Zero memory overhead for storage.
```python
def get_squares_gen(n):
    for i in range(n):
        yield i**2
```

## 7.3 Memory Comparison

If `n = 1,000,000`:
*   **List:** Stores 1 million integers in RAM (~40MB).
*   **Generator:** Stores only the current integer and the formula (~100 Bytes).

> [!RECOMMENDATION]
> Always use generators when processing large files (e.g., reading a CSV line-by-line) or streaming data.
