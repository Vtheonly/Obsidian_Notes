# 2. Duplicate Detection and Record Linkage

Duplicate records inflate dataset size, bias statistical results, and lead to data leakage in ML models (e.g., same record in both Train and Test sets).

## 2.1 Exact Duplicates

Rows where every single column is identical.

```python
# 1. Check for duplicates (Returns Boolean Series)
print(df.duplicated().sum())

# 2. View duplicate rows
duplicates = df[df.duplicated()]

# 3. Drop duplicates
df.drop_duplicates(inplace=True)
```

## 2.2 Partial Duplicates (Subsets)

Rows that are identical *except* for a unique ID or timestamp.

```python
# Check duplicates based on specific columns
subset = ['Name', 'Email', 'Phone']
print(df.duplicated(subset=subset).sum())

# Drop, keeping the first (or last) occurrence
df.drop_duplicates(subset=subset, keep='first', inplace=True)
```

## 2.3 Fuzzy Matching (Record Linkage)

Handling **typos** and variations (e.g., "John Smith" vs. "Jon Smith" vs. "J. Smith").
We use algorithms like **Levenshtein Distance** to calculate string similarity.

*   **Library:** `fuzzywuzzy` or `textdistance`.

```python
from fuzzywuzzy import process

# List of correct names
choices = ["Apple", "Banana", "Orange"]

# Messy input
process.extractOne("Aplle", choices)
# Output: ('Apple', 91) -> 91% match confidence
```

> [!TIP] **Soundex**
> Phonetic algorithms (like Soundex or Metaphone) index words by how they *sound*. Useful for matching names with different spellings (e.g., "Smith" vs "Smyth").
