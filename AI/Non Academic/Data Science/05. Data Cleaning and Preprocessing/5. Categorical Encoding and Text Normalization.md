# 5. Categorical Encoding and Text Normalization

Models require numbers, not strings. We must convert categories (e.g., "Red", "Blue") into numerical format.

## 5.1 Text Normalization

Before encoding, clean the text to ensure consistency.
1.  **Lowercasing:** "Apple" -> "apple".
2.  **Stripping Whitespace:** " Apple " -> "apple".
3.  **Removing Special Chars:** "Apple!" -> "apple".

```python
df['City'] = df['City'].str.lower().str.strip()
```

## 5.2 Label Encoding

Assigns a unique integer to each category.
*   "Low" -> 0, "Medium" -> 1, "High" -> 2.
*   **Pros:** Compact (1 column).
*   **Cons:** Implies **Order** (Ordinality). The model thinks $2 > 0$. Only use for Ordinal data (Rankings).

```python
from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
df['Encoded_Rank'] = le.fit_transform(df['Rank'])
```

## 5.3 One-Hot Encoding (OHE)

Creates a new binary column (dummy variable) for *each* category.
*   "Red" -> [1, 0, 0]
*   "Green" -> [0, 1, 0]
*   "Blue" -> [0, 0, 1]
*   **Pros:** No implied order. Good for Nominal data.
*   **Cons:** **Curse of Dimensionality.** If a category has 10,000 unique values (e.g., Zip Codes), you create 10,000 columns.

```python
# Using Pandas
df_ohe = pd.get_dummies(df, columns=['Color'], drop_first=True)
# drop_first=True removes one column to prevent multicollinearity (Dummy Variable Trap).
```

## 5.4 Target Encoding (Mean Encoding)

Replaces category with the **mean of the target variable** for that category.
*   **Use Case:** High cardinality categorical features (e.g., City names).
*   **Risk:** Overfitting. Must be calculated on Training data only.
