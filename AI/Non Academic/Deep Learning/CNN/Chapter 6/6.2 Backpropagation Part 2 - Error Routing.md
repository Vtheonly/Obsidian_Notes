# 6.2 Backpropagation Part 2: Gradients of the Input (Error Routing)

After updating the weights of an arbitrary Layer $L$, the network must pass the localized error backwards physically *through* Layer $L$ and directly into Layer $L-1$, so Layer $L-1$ can continue learning securely. We must calculate the derivative of the Loss with respect to the original spatial Input Map ($\frac{\partial L}{\partial X}$).

Once again, **passing the error backward is achieved elegantly using a core Convolution.** But the specific convolution used for error routing is different from the one used for weight gradients — it requires a critical 180-degree flip of the filter.

### Why Error Routing Is Necessary

Updating the weights of Layer $L$ is only half of the backpropagation story. The layers *before* Layer $L$ also need to receive error signals so they can update their own weights. If we stop the gradient flow at Layer $L$, then Layers $1$ through $L-1$ will receive zero gradient and will never learn. Error routing is the mechanism that passes the error signal backward through each layer, ensuring that every layer in the network receives the gradient information it needs to update its weights.

The challenge is: how do we take a small error gradient map (which has the same spatial dimensions as the output of the convolution) and map it back to the larger input feature map? The output is smaller than the input (because convolution reduces spatial dimensions), so we need to "expand" the gradient back to the input size.

### The Mathematics of Part 2

How do we take a generated error gradient matrix and accurately map it back to the exact physical pixels of the larger incoming input image?

1. **Take the Incoming Error Gradient.** This is a 2D map with the same spatial dimensions as the output of the forward convolution. Each value represents how much the loss changes with respect to that output pixel.

2. **Apply Full Padding to it.** If the spatial kernel is size $K$, you must add $K-1$ external rings of zeros around the error gradient matrix. This math fundamentally ensures the output tensor size will perfectly match the original input size. For example, if the kernel is $3 \times 3$, you add $3-1 = 2$ rings of zeros around the gradient map. This "full padding" is what expands the small gradient map back to the larger input size.

3. **Take the active filter parameters, and physically Flip the matrix 180 Degrees** (both horizontally and vertically). This means reversing the rows and then reversing the columns. For a $3 \times 3$ filter:
   - Original: $\begin{bmatrix} w_1 & w_2 & w_3 \\ w_4 & w_5 & w_6 \\ w_7 & w_8 & w_9 \end{bmatrix}$
   - Flipped: $\begin{bmatrix} w_9 & w_8 & w_7 \\ w_6 & w_5 & w_4 \\ w_3 & w_2 & w_1 \end{bmatrix}$

4. **Perform a standard Convolution:** **Padded Error Gradient $\circledast$ Flipped Filter = Input Gradient**.

### Why the 180-Degree Flip?

During the forward sequence, a standard CNN technically executes "Cross-Correlation," not a strict mathematical convolution. In strict calculus, to correctly compute the accurate mathematical derivative of a direct cross-correlation mapping with respect exclusively to its basic input, the spatial rule specifically dictates the kernel must be physically flipped. This correctly mechanically routes the error back perfectly to exactly the individual pixels that organically visually contributed it.

To understand this intuitively, consider what happens during the forward pass: when the filter is at position $(i, j)$, weight $W_{0,0}$ (top-left of the filter) is multiplied by input pixel $(i, j)$ (also top-left of the current window). But weight $W_{2,2}$ (bottom-right of the filter) is multiplied by input pixel $(i+2, j+2)$ (bottom-right of the current window). During backpropagation, we need to route the gradient from the output back to the correct input pixels. The 180-degree flip reverses this mapping: the gradient that flowed through $W_{0,0}$ is routed back to input position $(i+2, j+2)$, and the gradient that flowed through $W_{2,2}$ is routed back to input position $(i, j)$. The flip compensates for the reversed spatial relationship that exists in the forward pass.

### A Concrete Example

Consider a $4 \times 4$ input with a $3 \times 3$ filter producing a $2 \times 2$ output:

**Forward Pass (Cross-Correlation):**
- Output pixel $(0,0)$ depends on input pixels $(0:2, 0:2)$ and all 9 filter weights
- Output pixel $(0,1)$ depends on input pixels $(0:2, 1:3)$ and all 9 filter weights

**Backward Pass (Error Routing):**
- Input gradient at pixel $(0,0)$ receives contribution only from output gradient $(0,0)$ multiplied by $W_{0,0}$
- Input gradient at pixel $(1,1)$ receives contributions from all four output gradients, multiplied by different weights
- The full convolution with the flipped filter correctly accumulates these contributions

The 180-degree flip ensures that each input pixel receives gradient contributions from exactly the output pixels that it influenced during the forward pass, weighted by the correct filter weight.

> [!info] Cross-Correlation vs. Convolution
> In most deep learning frameworks, what is called `Conv2D` is actually **cross-correlation**, not strict mathematical convolution. In a true mathematical convolution, the kernel is flipped 180 degrees before the sliding operation. Deep learning frameworks skip this flip for convenience, which is why the 180-degree flip appears in the backward pass — it compensates for the flip that was omitted in the forward pass. This is a mathematical necessity, not an arbitrary design choice.
