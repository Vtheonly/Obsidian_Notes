# 1.4 Color Spaces and Models

A color model is a mathematical way to describe colors as tuples of numbers (e.g., triplets or quadruplets). Different models exist because they solve different problems (displaying light, printing ink, or human perception).

## 1. RGB (Red, Green, Blue)
**Type:** Additive Color Model.
**Used In:** Monitors, Cameras, Screens.

*   **Concept:** It models the mixing of light.
    *   **Black:** No light $(0,0,0)$.
    *   **White:** All light at max intensity $(255,255,255)$.
    *   **Colors:** Mixed by adding wavelengths. Red + Green = Yellow.
*   **Geometry:** It is represented as a **Cartesian Unit Cube**.
    *   Axes: R, G, B.
    *   The "Main Diagonal" from $(0,0,0)$ to $(255,255,255)$ represents the **Grayscale** line. Points on this line have equal amounts of R, G, and B ($r=g=b$).

## 2. CMYK (Cyan, Magenta, Yellow, Key/Black)
**Type:** Subtractive Color Model.
**Used In:** Printing.

*   **Concept:** It models the mixing of pigments/ink.
    *   Pigments absorb (subtract) light. White paper reflects all light.
    *   **Cyan** absorbs Red.
    *   **Magenta** absorbs Green.
    *   **Yellow** absorbs Blue.
*   **Why K?** Theoretically, C+M+Y = Black. In practice, it creates a muddy brown. Pure Black ink (Key) is added for sharpness and cost savings.

## 3. HSL / HSV (Hue, Saturation, Lightness/Value)
**Type:** Perceptual Model.
**Used In:** Color Pickers, Image Editing, Human-Computer Interaction.

RGB is not intuitive. If I ask you to "make this pixel a bit more pink," calculating the change in R, G, and B is hard. HSL solves this.

*   **Hue (H):** The "type" of color (Red, Orange, Blue). Measured in degrees ($0^\circ$ to $360^\circ$) around a color wheel.
*   **Saturation (S):** The "purity" of the color.
    *   100% = Vivid color.
    *   0% = Gray.
*   **Lightness (L) / Value (V):** The brightness.
    *   0% = Black.
    *   100% = White.

**Use Case in Processing:** If you want to detect a "Red Car" in an image with shadows, RGB is terrible because the shadow changes the RGB values entirely. In HSV, the shadow only changes the Value (V), but the Hue (H) remains "Red." This makes object detection much easier.

## 4. Grayscale Conversion (Luminance)
Converting a color image to grayscale is not just `(R+G+B)/3`.
The human eye is more sensitive to Green light than Red or Blue. Therefore, we use a weighted average to calculate Luminance ($Y$):

$$ Y = 0.299R + 0.587G + 0.114B $$

*   **Note:** Notice that the weight for Green (0.587) is much higher than Blue (0.114).
