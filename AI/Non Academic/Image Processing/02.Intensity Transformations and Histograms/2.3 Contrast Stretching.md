# 2.3 Contrast Stretching

Contrast stretching (often called Normalization) is a linear scaling function designed to expand the range of intensity levels in an image so that it spans the full intensity range of the recording medium or display device.

## 1. The Problem: Low Contrast
Images often look "washed out" because of poor lighting, lack of dynamic range in the sensor, or wrong aperture settings.
*   *Example:* An image where the darkest pixel is 100 and the brightest is 150. The range [0, 99] and [151, 255] is wasted.

## 2. Linear Stretching Formula
We want to map the current range $[r_{min}, r_{max}]$ to a target range $[s_{min}, s_{max}]$ (usually $[0, 255]$).

$$ s = \left( \frac{s_{max} - s_{min}}{r_{max} - r_{min}} \right) (r - r_{min}) + s_{min} $$

### Simplified Case (Full Expansion)
To stretch to the full 8-bit range $[0, 255]$:
$$ s = 255 \cdot \frac{r - r_{min}}{r_{max} - r_{min}} $$

*   **Logic:**
    1.  Subtract the minimum ($r - r_{min}$): This shifts the histogram to start at 0.
    2.  Divide by range ($r_{max} - r_{min}$): This normalizes values to $[0, 1]$.
    3.  Multiply by 255: This scales values to $[0, 255]$.

## 3. Thresholding (Extreme Contrast)
Thresholding is a special case of contrast stretching where the function becomes a step function.
$$ s = \begin{cases} 0 & \text{if } r < \text{threshold} \\ 255 & \text{if } r \ge \text{threshold} \end{cases} $$
*   **Result:** A binary image (Black and White only).
*   **Use Case:** Segmentation (separating object from background).

## 4. Clipping
Sometimes, strict linear stretching is affected by outliers (e.g., one dead white pixel makes $r_{max}=255$, preventing any stretching).
*   **Solution:** Pick the 5th percentile and 95th percentile as $r_{min}$ and $r_{max}$.
*   Any pixel $< 5th$ percentile is set to 0 (Clipped to Black).
*   Any pixel $> 95th$ percentile is set to 255 (Clipped to White).
*   The middle data is stretched. This produces a much punchier, higher-contrast image.
