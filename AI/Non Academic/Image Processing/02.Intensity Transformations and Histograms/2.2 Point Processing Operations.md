# 2.2 Point Processing Operations

## 1. The Transformation Function
Point processing techniques are the simplest form of image processing. They transform a pixel's value based *solely* on its current value.

$$ s = T(r) $$
Where:
*   $r$ is the intensity of the input pixel.
*   $s$ is the intensity of the output pixel.
*   $T$ is the transformation function.

## 2. Image Negatives (Inversion)
The equivalent of a photographic negative.
$$ s = (L - 1) - r $$
For 8-bit images ($L=256$): $s = 255 - r$.
*   **Use Case:** Enhancing white or gray detail embedded in dark regions of an image, especially in medical imaging (e.g., analyzing breast tissue in mammograms).
*   **Visual:** Black becomes White, White becomes Black.

## 3. Log Transformations
$$ s = c \cdot \log(1 + r) $$
*   **Mechanism:** This function maps a narrow range of low gray-level values in the input image into a wider range of output levels.
*   **Visual Effect:** Expands the values of dark pixels while compressing the values of bright pixels.
*   **Use Case:** Essential for displaying the **Fourier Spectrum**. Fourier spectra often have pixel values ranging from 0 to $10^6$. On a standard monitor (0-255), everything would look black except the center dot. The Log transform brings these massive values down to a visible range.

## 4. Power-Law (Gamma) Transformations
$$ s = c \cdot r^{\gamma} $$
Where $c$ and $\gamma$ are positive constants.

### Gamma Correction ($\gamma$)
This is the most common form of point processing in consumer electronics.
*   **$\gamma < 1$ (e.g., 0.5):** Maps a narrow range of dark inputs into a wider range of outputs. Similar to Log transform. **Brightens the image.**
*   **$\gamma > 1$ (e.g., 2.5):** Maps a wide range of dark inputs into a narrow range of dark outputs. **Darkens the image.**

**Why is this critical?**
Cathode Ray Tube (CRT) monitors and modern LCDs have an intensity response that is a power function with exponents varying from roughly 1.8 to 2.5. If you send a linear signal to a monitor, the image will look dark.
*   **Correction:** We apply the inverse gamma ($\gamma \approx 0.45$) to the image file *before* displaying it, so that the monitor's darkening effect cancels out, resulting in correct color.
