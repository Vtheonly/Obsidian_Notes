Why Swin? The Problem with ViT

Before understanding what the Swin Transformer does, you must understand why it was built. The original Vision Transformer (ViT) was a breakthrough — it applied the raw Transformer architecture to images. But it had critical limitations for computer vision tasks beyond classification.

⚠️ ViT Problems

  • Quadratic complexity: Every patch attends to every other patch. For a 224×224 image with 16×16 patches → 196 tokens → 196² = 38,416 attention pairs.
  • Fixed scale: ViT only produces one feature map resolution — no hierarchy like CNNs. Dense prediction tasks (detection, segmentation) need multi-scale features.
  • Needs huge data: No inductive biases means ViT needs ImageNet-21k or JFT-300M to work well.

✅ Swin Solutions

  • Linear complexity: Attention only within small local windows → complexity is O(N) not O(N²).
  • Hierarchical features: Like ResNet, Swin produces multi-scale feature maps (1/4, 1/8, 1/16, 1/32 of input) — perfect for FPN, dense tasks.
  • Shifted windows: Windows shift between layers so patches CAN communicate across window boundaries — solving the isolation problem.

🎯 The Core Idea in One Sentence

Swin Transformer = local windowed self-attention (for efficiency) + shifted windows (for cross-window communication) + patch merging (for hierarchy) — giving you a general-purpose backbone that works like ResNet but is powered by Transformers.

Patch Partitioning & Linear Embedding

The very first thing the Swin Encoder does is convert the raw image into a sequence of tokens. This happens in two sub-steps:

① Patch Partition

The input image H×W×3 is split into non-overlapping patches of size 4×4. Each patch is a 4×4×3 = 48-dimensional vector of raw pixel values.

Result: (H/4) × (W/4) patches, each with 48 features.

② Linear Embedding

A linear layer projects each 48-dim patch vector to C dimensions (the model's "embedding dimension" — 96 for Swin-T).

This is exactly like a token embedding in NLP. Each patch becomes a C-dim token.

\[ \text{Input: } H \times W \times 3 \xrightarrow{\text{Patch Partition}} \frac{H}{4} \times \frac{W}{4} \times 48 \xrightarrow{\text{Linear}} \frac{H}{4} \times \frac{W}{4} \times C \]
💡
Why 4×4 patches? ViT uses 16×16. Swin uses smaller 4×4 patches to preserve more spatial detail — critical for dense prediction tasks like segmentation where pixel-level accuracy matters.

Window-based Multi-Head Self-Attention (W-MSA)

This is the heart of efficiency in Swin. Instead of computing self-attention across ALL tokens (like ViT), Swin partitions tokens into non-overlapping windows of M×M tokens and computes attention within each window independently.

Standard MSA (ViT-style)

Every token attends to every other token globally.

\[ \text{Complexity} = O\left(N^2 \cdot d\right) \]

N = total tokens, d = head dim. Grows quadratically!

W-MSA (Swin-style)

Each window of M×M tokens attends within itself only.

\[ \text{Complexity} = O\left(N \cdot M^2 \cdot d\right) \]

Linear in N! M is fixed (e.g., 7×7=49). 🚀

📐 How Windows are Created

Given a feature map of (H/4) × (W/4) tokens and window size M=7:

  1. Reshape the 2D token grid into windows: num_windows = (H/4/M) × (W/4/M)
  2. Each window contains M×M = 49 tokens
  3. Run multi-head attention independently within each window
  4. Reshape back to original spatial layout
\[ \text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}} + B\right)V \]

where B is the relative position bias (explained in Section 4)

Shifted Window Attention (SW-MSA)

The ingenious solution to the isolation problem. In alternating Transformer blocks, the window partition is shifted by (M/2, M/2) tokens (i.e., half the window size in each direction). This creates new windows that span boundaries of the previous windows — enabling cross-window information flow.

🔀 The Shift Operation

If M=7, the shift is (3, 3). This means:

  • New windows straddle old boundaries
  • Tokens from 4 different original windows are now in the same new window
  • They attend to each other → cross-window communication achieved!

⚡ Problem: Shifted Windows Create Irregular Sizes

After shifting, windows near the edges and corners are smaller than M×M — they get "cropped" by the image boundary. You'd need to pad them back to M×M and then mask out the padded positions during attention.

The paper proposes an efficient solution: Cyclic Shift — rolling the entire feature map so that boundary pieces wrap around, forming complete M×M windows. An attention mask ensures only valid positions within each logical window attend to each other.

Relative Position Bias

In standard Transformers, positional information is added as absolute embeddings to the input. Swin uses relative position bias — added to the attention scores, not the inputs. This encodes the relative spatial relationship between query and key tokens inside a window.

❌ Absolute Position (ViT)

Add fixed or learned vectors to each token embedding. Position info is encoded in the token itself, but gets mixed with content through layers. Doesn't transfer well across resolutions.

✅ Relative Position Bias (Swin)

For each query-key pair (i,j) inside a window, compute their relative position (Δrow, Δcol) and look up a learned bias B[Δrow, Δcol]. Added to attention logits before softmax.

\[ \text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d}} + B\right)V \]

B is indexed by relative position. For M×M windows: relative positions range from -(M-1) to (M-1) in each axis. Total unique biases = (2M-1)² = 169 for M=7.

The Complete Swin Transformer Block

A Swin Transformer Block is a standard Transformer block with windowed self-attention instead of global. Crucially, blocks come in pairs: one W-MSA block followed by one SW-MSA block, to enable cross-window communication.

🔢 Block Equations

\[ \hat{z}^l = \text{W-MSA}(\text{LN}(z^{l-1})) + z^{l-1} \] \[ z^l = \text{MLP}(\text{LN}(\hat{z}^l)) + \hat{z}^l \] \[ \hat{z}^{l+1} = \text{SW-MSA}(\text{LN}(z^l)) + z^l \] \[ z^{l+1} = \text{MLP}(\text{LN}(\hat{z}^{l+1})) + \hat{z}^{l+1} \]

LN = LayerNorm applied before the sub-layer (Pre-LN, more stable than Post-LN). The residual connections (+) ensure gradient flow.

MLP Sub-block

A 2-layer MLP with GELU activation. Hidden dimension = 4×C.

\[\text{MLP}: C \to 4C \xrightarrow{\text{GELU}} 4C \to C\]

LayerNorm Placement

Pre-LN (before attention/MLP) is used. This normalizes token features to zero mean & unit variance before each operation.

\[\text{LN}(x) = \frac{x-\mu}{\sigma} \cdot \gamma + \beta\]

Patch Merging (Hierarchical Downsampling)

At the end of each stage (except the last), Patch Merging reduces spatial resolution by 2× while doubling the feature dimension. This is Swin's equivalent of stride-2 convolution in CNNs — it builds the hierarchy.

🔀 How Patch Merging Works

  1. Take the H×W×C feature map
  2. Sample every 2nd patch in each spatial dimension → creates 4 grids of (H/2)×(W/2)×C
  3. Concatenate the 4 grids along feature dim → (H/2)×(W/2)×4C
  4. Apply LayerNorm → Linear projection to 2C → output: (H/2)×(W/2)×2C
\[(H \times W \times C) \xrightarrow{\text{Patch Merge}} \left(\frac{H}{2} \times \frac{W}{2} \times 2C\right)\]

Complexity Analysis & Interactive Demo

W-MSA Complexity

\[\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2hwC\]

h,w = height/width in patches. Linear in hw (the image size). M is the window size (constant).

MSA Complexity (global)

\[\Omega(\text{MSA}) = 4hwC^2 + 2(hw)^2C\]

The (hw)² term is the killer — quadratic in image area.

PyTorch Code Walkthrough

Here's a clean, annotated implementation of the key components. Each piece maps directly to what you've learned above.

① Window Partition Function

def window_partition(x, window_size):
    """
    x: (B, H, W, C)  — batch of 2D token grids
    window_size: M    — e.g., 7
    Returns: (num_windows*B, M, M, C)
    """
    B, H, W, C = x.shape
    # Reshape into windows
    x = x.view(B, H // window_size, window_size,
                   W // window_size, window_size, C)
    # Permute: (B, nH, nW, M, M, C) -> contiguous
    windows = x.permute(0,1,3,2,4,5).contiguous()
    # Merge batch and window dims
    windows = windows.view(-1, window_size, window_size, C)
    return windows  # (num_windows*B, M, M, C)

def window_reverse(windows, window_size, H, W):
    """Inverse: reconstruct spatial layout from windows"""
    B = int(windows.shape[0] / (H * W / window_size ** 2))
    x = windows.view(B, H // window_size, W // window_size,
                     window_size, window_size, -1)
    x = x.permute(0,1,3,2,4,5).contiguous()
    x = x.view(B, H, W, -1)
    return x

② Window Attention

class WindowAttention(nn.Module):
    def __init__(self, dim, window_size, num_heads):
        super().__init__()
        self.num_heads = num_heads
        head_dim = dim // num_heads
        self.scale = head_dim ** -0.5

        # Relative position bias TABLE: (2M-1, 2M-1, num_heads)
        self.relative_position_bias_table = nn.Parameter(
            torch.zeros((2*window_size[0]-1) * (2*window_size[1]-1),
                        num_heads))

        # Precompute relative position INDEX for each token pair
        coords_h = torch.arange(self.window_size[0])
        coords_w = torch.arange(self.window_size[1])
        coords = torch.stack(torch.meshgrid(coords_h, coords_w))  # 2,M,M
        coords_flatten = torch.flatten(coords, 1)              # 2,M*M
        relative_coords = (coords_flatten[:, :, None]
                         - coords_flatten[:, None, :])          # 2,M²,M²
        relative_coords = relative_coords.permute(1,2,0).contiguous()
        relative_coords[:,:,0] += self.window_size[0] - 1   # shift to >=0
        relative_coords[:,:,1] += self.window_size[1] - 1
        relative_coords[:,:,0] *= 2 * self.window_size[1] - 1 # flatten 2D to 1D
        relative_position_index = relative_coords.sum(-1)        # M²,M²
        self.register_buffer('relative_position_index', relative_position_index)

        self.qkv = nn.Linear(dim, dim * 3, bias=False)
        self.proj = nn.Linear(dim, dim)

    def forward(self, x, mask=None):
        B_, N, C = x.shape   # B_ = num_windows*batch
        qkv = self.qkv(x).reshape(B_, N, 3, self.num_heads, C//self.num_heads)
        qkv = qkv.permute(2,0,3,1,4)
        q, k, v = qkv.unbind(0)

        attn = (q @ k.transpose(-2,-1)) * self.scale  # QK^T / sqrt(d)

        # Add relative position bias
        relative_position_bias = self.relative_position_bias_table[
            self.relative_position_index.view(-1)].view(N, N, -1)
        relative_position_bias = relative_position_bias.permute(2,0,1)
        attn = attn + relative_position_bias.unsqueeze(0)

        # Apply mask for SW-MSA (cyclic shift boundaries)
        if mask is not None:
            attn = attn + mask.unsqueeze(0).unsqueeze(0) * -100.0

        attn = attn.softmax(dim=-1)
        x = (attn @ v).transpose(1,2).reshape(B_, N, C)
        x = self.proj(x)
        return x

③ Patch Merging Layer

class PatchMerging(nn.Module):
    def __init__(self, input_resolution, dim):
        super().__init__()
        self.norm = nn.LayerNorm(4 * dim)
        self.reduction = nn.Linear(4 * dim, 2 * dim, bias=False)

    def forward(self, x):
        # x: (B, H*W, C)
        H, W = self.input_resolution
        B, L, C = x.shape
        x = x.view(B, H, W, C)

        # Sample every other patch — 4 sub-grids
        x0 = x[:, 0::2, 0::2, :]  # top-left
        x1 = x[:, 1::2, 0::2, :]  # bottom-left
        x2 = x[:, 0::2, 1::2, :]  # top-right
        x3 = x[:, 1::2, 1::2, :]  # bottom-right

        x = torch.cat([x0, x1, x2, x3], dim=-1)  # (B, H/2, W/2, 4C)
        x = x.view(B, -1, 4 * C)                 # (B, H/2*W/2, 4C)
        x = self.norm(x)
        x = self.reduction(x)                     # (B, H/2*W/2, 2C)
        return x

④ Cyclic Shift & Mask (SW-MSA)

# Cyclic shift the feature map by (-shift_size, -shift_size)
shifted_x = torch.roll(x, shifts=(-shift_size, -shift_size), dims=(1, 2))

# Build attention mask to prevent cross-region attention
img_mask = torch.zeros((B, H, W, 1), device=x.device)
h_slices = (slice(0, -window_size),
            slice(-window_size, -shift_size),
            slice(-shift_size, None))
w_slices = (slice(0, -window_size),
            slice(-window_size, -shift_size),
            slice(-shift_size, None))

cnt = 0
for h in h_slices:
    for w in w_slices:
        img_mask[:, h, w, :] = cnt
        cnt += 1

# Each pixel in img_mask has an ID (0-8)
# Tokens with same ID = same logical region = can attend to each other
# Different IDs = different logical regions = mask out
mask_windows = window_partition(img_mask, window_size)   # (nW, M, M, 1)
mask_windows = mask_windows.view(-1, window_size * window_size)
attn_mask = mask_windows.unsqueeze(1) - mask_windows.unsqueeze(2)
# 0 where same region, non-zero where different → added as -100 to logits
attn_mask = attn_mask.masked_fill(attn_mask != 0, -100.0)
attn_mask = attn_mask.masked_fill(attn_mask == 0, 0.0)

🎓 Complete Summary: The Swin Encoder in 7 Points

  1. Patch Partition (4×4): Image → non-overlapping 4×4 pixel patches → linear embed to C-dim tokens. Produces (H/4)×(W/4) token grid.
  2. W-MSA: Partition tokens into M×M windows, run self-attention inside each window. O(N) complexity.
  3. SW-MSA: In alternating blocks, shift window partition by (M/2,M/2). Enables cross-window communication.
  4. Cyclic Shift + Masking: Efficient SW-MSA — roll the map, compute once, mask invalid cross-region pairs.
  5. Relative Position Bias: Learned bias added to QK^T, indexed by relative token position within window. (2M-1)² parameters.
  6. Swin Block: LN→W-MSA→residual→LN→MLP→residual, paired with LN→SW-MSA→residual→LN→MLP→residual.
  7. Patch Merging: Subsample 4 sub-grids, cat, linear project → 2× smaller, 2× deeper. Creates hierarchy across 4 stages.
