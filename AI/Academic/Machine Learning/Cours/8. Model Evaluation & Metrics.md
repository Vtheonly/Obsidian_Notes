# 8. Model Evaluation & Metrics

## 8.1. Data Splitting Strategies

Before training a model, the dataset must be divided to ensure the model can "generalize" (perform well on new, unseen data) rather than just memorizing the answers ("overfitting").

### A. The Standard Split (Hold-out Method)
As shown on **Page 10 and 11**, the dataset is typically divided into three parts:

1.  **Training Set (~70%):** Used to teach the model (calculate weights, find neighbors).
2.  **Validation Set (~15%):** Used *during* training to tune hyperparameters (like choosing $K$ in KNN or the learning rate in Neural Networks). It acts as a check to stop training if the model starts overfitting.
3.  **Test Set (~15%):** A completely "locked" dataset used only **once** at the very end to evaluate the final performance.

### B. Cross-Validation Techniques
When you don't have enough data to "waste" 30% on testing, or to ensure the model isn't just lucky with a specific split, we use Cross-Validation.

#### 1. LOOCV (Leave-One-Out Cross-Validation)
*   **Method:** Train on all data points except **one**. Test on that single left-out point. Repeat this process $N$ times (for every single data point).
*   **Pros:** Uses almost all data for training.
*   **Cons:** Extremely computationally expensive (slow) for large datasets.

#### 2. K-Fold Cross-Validation (The Standard)
*   **Method:**
    1.  Split the data into $K$ equal parts (folds). (Commonly $K=5$ or $K=10$).
    2.  **Iteration 1:** Train on Folds 1-4, Test on Fold 5.
    3.  **Iteration 2:** Train on Folds 1,2,3,5, Test on Fold 4.
    4.  Repeat until every fold has been used as the test set exactly once.
*   **Result:** The final score is the **average** of the scores from all $K$ iterations.
*   **Stratified K-Fold:** Ensures that each fold has the same percentage of samples for each class as the complete set (prevents one fold from having only "Cats" and no "Dogs").

## 8.2. The Confusion Matrix

The Confusion Matrix (Page 11) is the foundation for calculating accuracy in classification problems. It compares the **Predicted Class** against the **Actual Class**.

| | **Predicted: Positive (1)** | **Predicted: Negative (0)** |
| :--- | :---: | :---: |
| **Actual: Positive (1)** | **TP** (True Positive) | **FN** (False Negative) |
| **Actual: Negative (0)** | **FP** (False Positive) | **TN** (True Negative) |

*   **TP (True Positive):** Correctly identified (e.g., Sick person predicted as Sick).
*   **TN (True Negative):** Correctly rejected (e.g., Healthy person predicted as Healthy).
*   **FP (False Positive):** "Type 1 Error" or False Alarm (e.g., Healthy person predicted as Sick).
*   **FN (False Negative):** "Type 2 Error" or Missed Detection (e.g., Sick person predicted as Healthy).

> [!DANGER] **Type 1 vs Type 2 Errors**
> *   **Type 1 Error (FP):** The model cries "Wolf!" when there is no wolf.
> *   **Type 2 Error (FN):** The model sleeps while the wolf attacks. In medical diagnosis, Type 2 is often more dangerous.

## 8.3. Performance Metrics Formulas

Using the values from the Confusion Matrix, we calculate the following metrics (Page 12).

### 1. Accuracy
The percentage of total predictions that were correct.
$$Accuracy = \frac{TP + TN}{TP + TN + FP + FN} = \frac{\text{Total Correct}}{\text{Total Data}}$$
*   *Limitation:* If 95% of data is Class A, a model that *always* guesses "A" has 95% accuracy but is useless.

### 2. Precision
"Of all the instances we **predicted** as positive, how many were actually positive?"
$$Precision = \frac{TP}{TP + FP}$$
*   *Use case:* Important in Spam Filters (You don't want to classify a real email as Spam/FP).

### 3. Recall (Sensitivity)
"Of all the **actual** positive instances, how many did we manage to find?"
$$Recall = \frac{TP}{TP + FN}$$
*   *Use case:* Important in Cancer detection (You don't want to miss a sick patient/FN).

### 4. F1-Score
The Harmonic Mean of Precision and Recall. It combines them into a single number.
$$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$
*   *Why use it?* If you have high Precision but low Recall (or vice versa), the F1 score will be low. It forces the model to be good at both.

### 5. Specificity
The ability of the test to correctly identify negative cases.
$$Specificity = \frac{TN}{TN + FP}$$

## 8.4. Multi-Class Classification Example

Page 12 shows a matrix for 3 classes: **Dog, Cat, Horse**.

To calculate metrics for a specific class (e.g., **Dog**), you treat it as a "One vs All" problem:
*   **TP:** Actual Dog predicted as Dog.
*   **FP:** Cat or Horse predicted as Dog.
*   **FN:** Actual Dog predicted as Cat or Horse.
*   **TN:** Cat or Horse predicted as Cat or Horse.

**Example Calculation from Page 12:**
*   Total Instances = 10
*   Correctly Classified = 8
*   $$Accuracy = \frac{8}{10} = 0.8 \text{ or } 80\%$$
