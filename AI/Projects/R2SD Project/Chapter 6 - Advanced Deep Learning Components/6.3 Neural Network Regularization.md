## 6.3 Neural Network Regularization (Dropout & BatchNorm)
Deep learning models are prone to **Overfitting**—memorizing the exact training data (including the noise) instead of learning the underlying physical rules. To ensure your model generalizes to future, unseen weather, we applied specific regularization techniques.

1.  **Dropout (0.2 - 0.3):** During each training step, we randomly "turn off" (set to zero) 20% to 30% of the neurons in a layer. This forces the network to develop redundant pathways for information, ensuring it never relies too heavily on any single feature (e.g., a slightly inaccurate temperature sensor). It acts like training an ensemble of different smaller networks within the same architecture.
2.  **Batch Normalization (`BatchNormalization()`):** Neural network activations (the output of each layer) can shift wildly during training, causing "internal covariate shift." Batch Normalization mathematically re-centers and re-scales the data flowing between layers. This stabilizes training, allows for higher learning rates, and improves overall convergence speed.
3.  **Early Stopping (`EarlyStopping(patience=3)`):** We monitor the error on a separate "Validation" dataset (a small portion of data unseen during core training). If the training error improves but the validation error stops improving for a specified number of epochs (e.g., `patience=3`), training halts immediately, saving the optimal weights before the model starts memorizing noise.
4.  **TerminateOnNaN (`TerminateOnNaN()`):** This callback explicitly stops training if the loss ever becomes `NaN`. This is crucial for noisy energy datasets, preventing the model from continuing to train with corrupted weights if an unexpected outlier or bug causes a numerical instability.
