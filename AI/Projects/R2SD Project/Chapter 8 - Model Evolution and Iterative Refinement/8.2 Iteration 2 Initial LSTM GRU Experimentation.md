## 7.2 Iteration 2: Initial LSTM/GRU Experimentation (Wind)
### Goal
To introduce temporal memory using deep learning, specifically GRUs, and to address the data corruption issues discovered in Iteration 1.

### Model Choice & Architecture
*   **Model:** `GRUPredictor` (from `src/modeling/predictor.py`).
*   **Architecture:** A `Sequential` Keras model with `GRU` layers.
    *   `Input`: Shape `(24, 6)` for a 24-step sequence (4 hours at 10-min intervals) with 6 features.
    *   `GRU(64)`: The core recurrent layer to capture temporal dependencies.
    *   `Dropout(0.2)`: Regularization to prevent overfitting.
    *   `Dense(1)`: Output layer for a single power prediction.
*   **Parameters:** `epochs=10`, `batch_size=128`.
*   **Loss Function:** `mae` (Mean Absolute Error).

### Data Processing Focus
*   **NREL Reshaping Refinement:** The `NRELAdapter` was significantly rewritten to correctly handle the repeating 11-column blocks in the wide-format NREL file.
*   **Physics-Guided Target Generation:** For NREL data, where `power` was `N/A`, we started calculating `power = (wind_speed_ms**3) * 0.2` to provide a target for the model to learn from.
*   **Robust Scaling (`RobustScaler`):** Introduced to handle outliers in noisy wind data.
*   **Imputation over Dropping:** The `WindPhysicsEngine` shifted from blanket `dropna()` to intelligent `fillna(median())` for missing `TEMPERATURE` and `PRESSURE`.
*   **Clipping:** `np.clip(value, 0, 5000)` was added to both calculated and observed power values to prevent gradient explosions from extreme numbers.
*   **Strict Type Conversion:** `pd.to_numeric(..., errors='coerce')` was used more widely, followed by `dropna(subset=[Schema.TIMESTAMP])`, to ensure only valid numeric and time values remain.

### Results (from logs)
*   **NREL Loaded:** `~61,000 clean rows` (NREL data was finally correctly ingested!).
*   **Turkey Loaded:** `~50,000 clean rows`.
*   **Combined Data:** `~111,000 rows`.
*   **MAE: ~298.49 kW** (Still high, indicates large errors).
*   **R² Score: ~0.8767** (Looks good, but MAE suggests model has significant misses).
*   **Observation:** The model was running on **CPU** not GPU, making training slow.

### Analysis & Problems Encountered
1.  **CPU Bottleneck:** Despite the GRU architecture, the training was slow (`228ms/step`) because Colab's runtime was set to CPU. `failed call to cuInit` confirmed no GPU usage.
2.  **High MAE with High R²:** An $R^2$ of 0.87 is usually excellent, but an MAE of ~300 kW (for a turbine with max 2000 kW) suggests the model has **large, infrequent errors**. This is typical when the model struggles with the "noisy" nature of real wind or the theoretical power curve from NREL.
3.  **Dataset Mixing Conflict:** The model was learning from two distinct sources: theoretical physics (NREL) and noisy reality (Turkey). The GRU didn't know *which "physics"* to apply to a given row.

### Code Decisions & Evolution in this Stage
*   **`src/ingestion/adapters.py`:** Implemented sophisticated NREL reshaping. Introduced power simulation for NREL.
*   **`src/processing/physics.py`:** Switched to median imputation. Added `np.clip()` for safety.
*   **`src/modeling/predictor.py`:** Defined the `GRUPredictor` class. Used `MinMaxScaler` (later found to be less robust) and `mse` loss.
*   **`main.py`:** Orchestrated loading both datasets, combining them, and training the GRU.
