## 7.1 Iteration 1: Initial XGBoost Baseline (Wind)
### Initial Goal
To get a quick baseline for wind power prediction using the most straightforward approach.

### Model Choice & Architecture
*   **Model:** `XGBoostModel` (from `src/modeling/predictor.py`).
*   **Architecture:** A single XGBoost Regressor (`xgb.XGBRegressor`).
*   **Parameters:** `n_estimators=500`, `learning_rate=0.05`, `max_depth=6`.

### Initial Data Processing
*   The `NRELAdapter` simply loaded the NREL data (often just 1 row initially due to errors).
*   The `TurkeyAdapter` loaded the `T1.csv` directly.
*   The `WindPhysicsEngine` applied basic physics (e.g., `wind_cubed`).
*   `dropna()` was heavily used.

### Initial Results (from logs)
*   **MAE: ~71.38 kW**
*   **R² Score: ~0.9356** (This was misleadingly high due to data corruption, as we later found out)

### Analysis & Problems Encountered
1.  **NREL Data Loss (`✅ NREL Loaded: 1 rows`):** The `NRELAdapter` was failing to correctly parse the "Wide Format" NREL data, resulting in only 1 row being loaded from an 8-year dataset. This meant the model was almost exclusively training on the Turkey SCADA data.
2.  **"Ghost Power" / Data Corruption (NREL):** The `NRELAdapter` was misinterpreting pressure values as power, leading to nonsensical `91780.00 kW` readings for `active_power_kw` from NREL. This effectively "poisoned" the dataset.
3.  **The `dropna()` Trap:** The `WindPhysicsEngine`'s blanket `dropna()` command, combined with missing `PRESSURE` and `TEMPERATURE` columns in the Turkey dataset (when merged with NREL), resulted in 0 rows being left for training if NREL data was loaded correctly. This caused `ValueError: With n_samples=0`.
4.  **Misleading $R^2$:** The high $R^2$ of 0.93 was an artifact of the data corruption. The model was learning perfect relationships within a distorted dataset.

### Code Decisions & Evolution in this Stage
*   **`src/ingestion/loaders.py`:** Initial attempts to fix NREL "Wide Format" with manual slicing (e.g., `raw_df.iloc[:, 0:5]`). This was fragile.
*   **`src/processing/physics.py`:** Implemented basic `wind_cubed` and `U/V` vectors. Included a simple `df.dropna()` that later proved destructive.
*   **`main.py`:** Used `train_test_split(shuffle=False)` for chronological integrity.
