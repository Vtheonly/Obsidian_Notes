## 2.3 Data Harmonization and Canonical Schemas
When merging datasets from different sources, we must ensure they "speak the same language." 
For example, the Turkey SCADA dataset might lack a "Pressure" sensor, while the NREL dataset has it.

The **Canonical Schema** (`src/core/schema.py`) acts as a contract.
1.  It defines all expected columns (e.g., `WIND_SPEED`, `TEMPERATURE`, `PRESSURE`, `ACTIVE_POWER`).
2.  If a source dataset (like Turkey SCADA) is missing an essential column (e.g., `PRESSURE`), its Adapter must provide a physically reasonable default (e.g., setting missing pressure to `1013.25 hPa`, the standard sea-level atmospheric pressure) so the AI doesn't crash later.
3.  It adds a `DATASET_SOURCE` flag (`is_real` = 0 for NREL, 1 for Turkey) so the AI can learn the different characteristics of each data source.
