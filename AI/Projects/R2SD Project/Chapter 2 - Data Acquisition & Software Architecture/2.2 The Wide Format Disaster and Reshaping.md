## 2.2 The "Wide Format" Disaster and Reshaping (NREL)
### The Bug
Initially, the system crashed with a `KeyError: 'power'`, and the data statistics showed wind speeds of `2014 m/s`. 

This happened because the NREL data was downloaded in a **Wide Format**. Instead of having one row per timestamp (vertical), the dataset placed entire years side-by-side horizontally across columns:
`[Time Data] | [2007 Weather Block] | [2008 Weather Block] | [2009 Weather Block]`

The standard Pandas `read_csv` function, when used without specifying headers, read the year "2007" as a column name, and subsequent year numbers ("2008", "2009") as wind speed measurements, completely corrupting the training data.

### The Reshaping Logic
To solve this, the `NRELAdapter` (`src/ingestion/loaders.py`) was built with an intelligent slicing mechanism:
1.  It reads the raw NREL file without headers (`header=None`).
2.  It identifies that each year's data is a repeating block of 11 columns (Year, Month, Day, Hour, Minute, plus 6 attributes like speed, direction, temp, pressure, speed_80m, and power).
3.  It loops through the 53 available columns in chunks of 11.
4.  For each 11-column chunk, it reassigns the correct header names.
5.  It then stacks these yearly blocks vertically (`pd.concat`).

This operation transformed the initial corrupted "wide" data into **560,000 "long" and perfectly structured rows** (8 years * ~70,000 rows/year), drastically multiplying the model's learning capacity.
