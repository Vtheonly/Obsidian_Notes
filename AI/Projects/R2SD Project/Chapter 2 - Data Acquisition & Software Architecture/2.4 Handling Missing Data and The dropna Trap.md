## 2.4 Handling Missing Data & The `dropna()` Trap
Machine Learning models (especially neural networks) cannot process `NaN` (Not a Number) or `Inf` (Infinity) values. If a single value in the 3D tensor is `NaN`, the model's mathematical weights are instantly destroyed (`loss: nan`).

### The `dropna()` Trap
Initially, our `WindPhysicsEngine` (`src/processing/physics.py`) used a command `df.dropna()`. Because the Turkey data was genuinely missing "Pressure" (it didn't have a sensor for it), this command detected `NaN` values in the merged dataset and deleted *all 50,000 rows* of real turbine data, leaving the training set empty (`0 rows`).

### Intelligent Imputation and Clipping (The Final Fixes)
Instead of deleting valuable rows, we implemented a robust strategy:
1.  **Median Imputation (`SimpleImputer(strategy='median')`):** For features like `PRESSURE` or `TEMPERATURE` that are missing in one dataset but present in another, we fill the `NaN` values with the `median` value of that column from the entire dataset. The median is more robust to outliers than the mean.
2.  **Physics-Guided Target Generation (for NREL Power):** The NREL dataset did not originally have an active power column or it was filled with `N/A`. Instead of throwing away 560,000 rows of excellent weather data, we *calculated* the theoretical power: `P = V^3 * 0.18`. This allowed the model to learn the fundamental physics of wind before fine-tuning on the real Turkey data.
3.  **Safety Clipping (`np.clip(value, min, max)`):** Wind data is highly chaotic. A sensor glitch might record a wind speed of `100 m/s`. If we cube this ($100^3 = 1,000,000$), the power value becomes enormous. We apply `np.clip(value, 0, 5000)` to limit power to realistic turbine maximums (e.g., 5000 kW), preventing mathematical explosions during training.
4.  **Final Finite Filter:** Before passing data to the AI, `df.replace([np.inf, -np.inf], np.nan).dropna()` is used as a last resort to remove any lingering non-finite values that could still crash the model.
