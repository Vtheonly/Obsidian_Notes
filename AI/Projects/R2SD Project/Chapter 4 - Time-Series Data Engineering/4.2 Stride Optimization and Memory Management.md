## 4.2 Stride Optimization and Memory Management
### The RAM Explosion
When creating sliding windows on 560,000 rows, if we slide the window forward by just 1 step (10 minutes) at a time, we create 560,000 overlapping 2D matrices. This will immediately exhaust the RAM on any standard computer or Google Colab instance, crashing the kernel.

### The Stride Solution
We implemented a **Stride** parameter (e.g., `Stride = 20` or `Stride = 4`).
Instead of creating a window at 12:00, 12:10, 12:20 (overlap of 23 steps), we skip forward. We create a window at 12:00, then the next window starts at 3:20.

*   **Memory Efficiency:** This drastically reduces the number of generated sequences, saving massive amounts of RAM (e.g., reducing the dataset size by a factor of 20).
*   **Data Diversity:** Moving the window by only 10 minutes (a stride of 1) provides the AI with a sequence that is 99% identical to the previous one. By using a larger stride, each batch is highly diverse, forcing the model to generalize better and train faster.
