# 4.2 Floating Scale Surface Reconstruction (FSSR)

## The Core Concept
While [[4.1 Depth Map Fusion and TSDF]] is the global standard for volumetric fusion, it possesses a massive inherent mathematical flaw: **The Hallucination Problem**.

A TSDF grid mathematically hates empty space in its volume. If you have a voxel containing wall data here, and another voxel containing wall data 1 meter away, and absolutely no depth map observed what was between them (perhaps blocked by a plant), the continuous mathematical function gradient between them will heavily *guess* and bridge the gap. 
This results in TSDF heavily "hallucinating" ugly, blobby geometric webbing stretching across doorways, or generating massive ceilings across open skies outdoors.

To counter this, academic pipelines evolved into purely **Point-Based Implicit Functions**, most notably FSSR (Fuhrmann and Goesele, 2014) and Poisson Reconstruction.

---

## 1. How FSSR Abandons the Grid

FSSR completely abandons the rigid mathematical voxel grid. It is an entirely **Unstructured Point-Based** approach that integrates the concept of sensor "Scale" natively.

### The Algorithm Sequence

1.  **Micro-Triangulation:** Every input depth map is individually triangulated locally based *only* on adjacent pixels. This generates 100 separate "micro-meshes".
2.  **Defining Scale ($s$):** The system calculates the physical geometric length of the edges of these triangles. This provides a perfect, native "scale" value for each vertex. 
    *   A triangle from a macro wide-shot has huge physical edges (Coarse Scale). 
    *   A triangle from a zoomed-in detail shot has tiny physical edges (Fine Scale).
3.  **The Giant Point Cloud:** All the vertices from every single micro-mesh are dumped into one massive, unstructured array. Every point holds: Position $(X, Y, Z)$, Normal Vector $(N_x, N_y, N_z)$, and Scale Numeric float $(s)$.

### The Implicit Mathematical Function

Instead of a voxel grid, FSSR constructs a mathematically continuous **implicit function** dynamically over this chaos of points using Basis Functions (Gaussians). 

The $Scale$ value dictates how much a specific point is allowed to "influence" its neighbors. 
*   A point describing a tiny scratch (macro detail) has a very tight, sharp Gaussian sphere of influence. It only affects the exact millimeter adjacent to it.
*   A point describing a blank wall from far away has a broad, massive, smooth mathematical influence across space, acting as low-frequency foundational "glue".

---

## 2. Advantages over TSDF

> **💡 The Invisible Sky Advantage:**
> The greatest strength of FSSR's implicit function is that it leaves unsampled regions completely mathematically empty. 
> FSSR handles open scenes elegantly. It will cleanly reconstruct a castle wall but correctly leave the open windows totally empty because there are no data points there to exert "influence." A standard TSDF grid would desperately attempt to string geometry across the window bars.

## 3. Poisson Surface Reconstruction Alternative
A more widely available fallback to FSSR is **Poisson Surface Reconstruction** (Kazhdan et al., 2006). Poisson acts similarly by mapping an indicator function based on point density and normal vectors.
However, Poisson is mathematically guaranteed to mathematically seal every hole, meaning it *will* hallucinate massive spheres across unobserved areas. To fix this, an engineer must run a post-process script to delete any triangle whose geometric centroid resides in an area of space where the original point cloud density was zero.

### Implementation Status 🛠️
* **Requires Training?** **No.** Pure geometry.
* **Solo Developer Feasibility:** **Highly complex calculus.** An individual cannot reasonably write a Poisson solver or an FSSR implicit evaluation from scratch in Python efficiently. However, invoking `open3d.geometry.TriangleMesh.create_from_point_cloud_poisson()` is a single line of code that executes the exact C++ academic backend dynamically.
