# 3.4 Deep Learning Multi-View Stereo (MVSNet)

## The Core Concept
While classical Multi-View Stereo (See: [[3.1 Classical Multi-View Stereo (MVS)]]) uses strict classical geometry and patch matching for triangulation, Deep Learning MVS attempts to marry deep feature extraction with epipolar physics to create faster, denser, and more structurally informed guesses about scene depth.

The quintessential deep learning MVS architecture is **MVSNet**. The network does not look at all images simultaneously to build a 3D model. Instead, it systematically breaks the problem down view-by-view.

---

## 1. The Reference-Source Workflow

Deep Learning MVS builds 3D models via a systematic pipeline of local depth map creation. 

1. **Selection:** The system selects **one** image to act as the "reference" image.
2. **Context:** It then selects a few overlapping images (usually 2 to 4) to act as "source" images. 
3. **Prediction:** The model evaluates the deep visual features between them and predicts a continuous **depth map** for that specific reference image, effectively calculating the metric distance to the environment for every single pixel.
4. **Iteration and Swapping:** When the system computes the depth map for Image 1, it then moves to the next image. Image 2, which was just a "source" neighbor, now becomes the "reference" image. This iteration repeats until every single image in the dataset has a corresponding high-quality, metric-accurate depth map (e.g., 80 images = 80 depth maps). 
5. **Coverage:** Because every image gets its turn as the "reference", the target object (e.g., a building) is comprehensively covered from every single captured angle. 

These depth maps are then sent to a fusion backend (See: [[4.1 Depth Map Fusion and TSDF]]) to rigorously construct the final 3D point cloud.

---

## 2. Cost Volume Architecture (The Memory Problem)

In its native form, MVSNet generates a 3D Cost Volume. It uses 2D CNNs to extract features, warps them into a view frustum using camera poses from Phase 1 SfM (using mathematically differentiable homography), and predicts depth planes. 

### The GPU Bottleneck
Building a massive 3D cost volume at high resolution consumes catastrophic amounts of GPU memory.
A standard architectural implementation on high-resolution images can effortlessly push past 16+ GB of VRAM. Attempting to fit standard MVSNet on a consumer GPU (like a 14GB Nvidia T4) will immediately crash out of memory. If resolution is scaled down universally (e.g. forced to a tiny 512x640 window to prevent crashing), the final 3D point cloud evaluates the building through a low-resolution lens. The structure will have "stepped" artifacts (looking like it's built out of coarse, blurry layers of bricks). 

---

## 3. High-Efficiency MVSArchitectures

To effectively process scenes on a consumer GPU while retaining spatial sharpness, modern Deep Learning MVS foregoes the single massive cost volume in favor of Cascade structures. 

### Cascade-Attention MVS (CasMVSNet / R-MVSNet variants)
This approach prevents memory overflow while maintaining structural precision for complex architecture. 

*   **Cascade Cost Volume:** Instead of a single massive cost volume, it builds a pyramid.
    *   *Level 1:* Starts at 1/4 resolution. Estimates a coarse depth map covering the entire scene geometry.
    *   *Level 2 & 3:* Upsamples the depth map and builds extremely narrow depth search slices around the predicted hypothesis at much higher resolutions (1/2, then Full res). Memory stays low since the high-resolution sweeps search a tiny sliver of physical space.
*   **Attention Mechanisms (ECA-Net/CBAM):** Adding attention modules (like Efficient Channel Attention) allows the feature extraction network to prioritize thin details (like railings, window frames, and sharp corners) overriding the innate blurring tendency of pure CNN convolution.
*   **Edge-Aware Refinement:** After initial depth, a lightweight Refinement Network compares the coarse depth map back to the original sharp RGB image to logically "snap" predicted depths mathematically flush with visible boundary lines, preventing blurry roof-edges and "surface noise" in urban scenes. 

### The Deep Learning MVS Quality Gap
Despite architecture optimizations, models like CasMVSNet are fundamentally designed for *inference speed*. Deep learning regularization is historically prone to "surface noise" and "tearing" around sharp edges because it is guessing probabilities rather than firing strict geometric light rays. 

If visual perfection on static real estate is prioritized over fast prediction frames, deep learning MVS is generally abandoned in favor of pure iterative optimization methods (See: [[5.2 3D Gaussian Splatting]]).
