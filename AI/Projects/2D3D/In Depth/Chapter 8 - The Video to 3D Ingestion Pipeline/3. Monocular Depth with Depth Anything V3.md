
# 3. Monocular Depth with Depth Anything V3

The `DepthEstimator` class is the heart of the video pipeline, replacing classical stereo matching with a powerful neural network.

## What is Depth Anything?
Depth Anything V3 (DA3) is a state-of-the-art monocular depth estimation model. Unlike older models that were trained on specific indoor datasets (like NYU Depth V2) or outdoor driving datasets (like KITTI), DA3 was trained on a massive, unlabeled dataset of **62 million images**. This gives it unprecedented generalization capabilities, allowing it to accurately predict depth for almost any scene—indoor, outdoor, objects, landscapes, etc.

## Why Resize to 518?
```python
Config.FRAME_RESIZE = 518       # DA3 input resolution
```
This is not an arbitrary number. The Depth Anything V3 model, specifically the "base" version (`da3-base`), was trained on images of a specific resolution. Its internal neural network architecture (a Vision Transformer, or ViT) is hard-coded to process images of this size.

*   **Resizing:** If you feed it an image of a different size, the model's `inference` function will automatically resize it internally. By resizing the frames to `518` pixels on the longest edge *before* passing them to the model, you ensure that the aspect ratio is preserved correctly and that you are providing data in the exact format the model expects, which can lead to slightly more accurate and consistent depth predictions.

## Batch vs. Per-Image Fallback
The code includes a robust error-handling mechanism:
```python
try:
    prediction = self.model.inference(image_paths, ...)
except Exception as e:
    # ...
    return self._estimate_one_by_one(image_paths, output_dir)
```
*   **Batch Inference:** Processing all 100 frames in a single batch (`self.model.inference(image_paths)`) is vastly more efficient. It allows the GPU to perform matrix multiplications in parallel across all images, maximizing hardware utilization.
*   **The Problem:** If even one image in the batch is corrupted or causes a specific numerical error, the entire batch operation can fail.
*   **The Fallback:** The `_estimate_one_by_one` method provides a safety net. If the batch fails, it switches to a slower but more robust loop, processing each image individually. This ensures that a single bad frame doesn't cause the entire reconstruction pipeline to crash.
