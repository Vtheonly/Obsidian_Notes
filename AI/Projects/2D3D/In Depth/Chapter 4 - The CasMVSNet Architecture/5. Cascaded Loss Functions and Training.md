# 5. Cascaded Loss Functions and Training

To train the multi-stage cascade effectively, we must penalize depth prediction errors at every resolution scale simultaneously. 

If we only evaluated the loss at the final high-resolution stage, the coarse stage would never learn to provide an accurate initial guess, and the entire pipeline would collapse.

## The Cascade Loss Logic
In your code, `cascade_loss` dictates the mathematical penalty applied during backpropagation.

```python
def cascade_loss(outputs, gt_depth, weights=[0.5, 1.0, 2.0, 4.0]):
    total_loss = 0.0
    stages = ['depth_stage1', 'depth_stage2', 'depth_stage3', 'depth_final']
    for stage_name, w in zip(stages, weights):
        pred = outputs[stage_name]
        gt_resized = F.interpolate(gt_depth, size=pred.shape[2:], mode='bilinear')
        total_loss += w * F.smooth_l1_loss(pred, gt_resized)
    return total_loss
```

### Key Mathematical Decisions
1.  **The Multipliers (0.5, 1.0, 2.0, 4.0):** The network applies a heavily escalating multiplier. Stage 1 (Coarse) only incurs a 0.5 penalty, forgiving slight blurry mismatches. The Final Refined output incurs a brutal 4.0 multiplier, strictly forcing the Edge-Aware Refinement network to perfectly align with structural boundaries.
2.  **Smooth L1 Loss:** Unlike standard L2 (Mean Squared Error), Smooth L1 calculates an absolute difference for large errors (preventing extreme outliers like sky pixels from causing exploding gradients), while acting quadratically near zero to allow the network to comfortably settle on micro-details.
