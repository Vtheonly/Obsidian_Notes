# 3. Regularization and Soft Argmin

Once the Cost Volume is built via Homography Warping, it contains raw variance noise. The network applies a **3D-UNet** to regularize the volume.

## 3D Regularizer
*   **Structure:** Uses 3D Convolutions (scanning Height, Width, *and* Depth simultaneously). 
*   **GroupNorm + LeakyReLU:** Because the network processes 3D volumes, the batch size is usually `1`. Standard BatchNorm fails catastrophically at `batch_size=1`. GroupNorm is utilized instead to ensure mathematical stability.
*   **Shared Parameters:** For extreme memory efficiency, the exact same 4.2M parameter Regularizer is executed across Stage 1, Stage 2, and Stage 3.

## Differentiable Soft Argmin
Once the volume is smoothed, we need to pick the "Winning Depth" (the depth slice with the highest probability). 
Normally, you would use `Argmax`. However, `Argmax` is a rigid step-function that breaks neural network gradients (you cannot backpropagate through it).

Instead, we use **Soft Argmin**.
1.  Convert the depth dimension into a probability distribution summing to 1 (using Softmax).
2.  Multiply each depth hypothesis value by its probability weight.
3.  Sum them all together to find the expected value.

This outputs a continuous, floating-point depth value that exists *between* the discrete grid planes, enabling infinitely smooth gradients and fully differentiable backpropagation.
