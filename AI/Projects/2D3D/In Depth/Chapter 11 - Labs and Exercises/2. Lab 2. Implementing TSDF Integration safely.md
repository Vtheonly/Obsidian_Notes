# 2. Lab 2. Safe Volumetric Integration

**Objective:** Safely run TSDF fusion without crashing Python memory, using Open3D's C++ bindings.

**Theory Applied:**
Instead of allocating the empty air (which wastes 90% of RAM), sparse TSDF volumes (like Open3D's `ScalableTSDFVolume`) only allocate memory blocks exactly where the depth map hits a surface. This is achieved through a technique called **voxel hashing**.

**Conceptual Python Workflow:**
```python
import open3d as o3d

# Initialize the sparse volume
# voxel_length determines the physical size of the "Minecraft blocks"
# sdf_trunc determines how far from the surface we average data
volume = o3d.pipelines.integration.ScalableTSDFVolume(
    voxel_length=0.01,  # 1 cm resolution
    sdf_trunc=0.04,     # 4 cm truncation band
    color_type=o3d.pipelines.integration.TSDFVolumeColorType.RGB8
)

# Loop over your aligned depth maps and integrate
for image, depth, pose in dataset:
    rgbd = o3d.geometry.RGBDImage.create_from_color_and_depth(...)
    volume.integrate(rgbd, intrinsics, pose)

# Extract perfectly smooth mesh using internal Marching Cubes
mesh = volume.extract_triangle_mesh()
```
This matches the Fuhrmann theoretical requirement of handling overlapping data without destroying detail, and relies on optimized C++ data structures to manage memory efficiently.
