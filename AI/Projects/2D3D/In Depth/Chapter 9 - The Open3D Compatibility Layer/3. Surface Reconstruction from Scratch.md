# 3. Surface Reconstruction from Scratch

The compatibility layer also provides from-scratch implementations of the two main reconstruction algorithms.

## Poisson Reconstruction (`create_from_point_cloud_poisson`)
This is a simplified, grid-based version of the true Poisson algorithm.

1.  **Voxelization:** It defines a grid (e.g., `200x200x200`) over the point cloud's bounding box.
2.  **Density Grid:** It iterates through the points and increments the count of the voxel each point falls into. This creates a 3D histogram, or a density grid.
3.  **Gaussian Blur:** The grid is blurred using a separable 3D Gaussian kernel (`_gauss3d`). This smooths the density field, which is essential for `marching_cubes` to produce a smooth surface rather than a blocky one.
4.  **Marching Cubes:** It uses the `mcubes.marching_cubes` function on the smoothed density grid, extracting a surface at an isosurface level (e.g., 10% of the maximum density). This is a clever trick: instead of solving the complex Poisson equation, it treats the problem as extracting a surface from a density field, which produces a visually similar (though less mathematically rigorous) result.

## TSDF Volume Integration (`_ScalableTSDFVolume`)
This class implements a simplified, non-scalable TSDF volume entirely in NumPy.

1.  **Initialization:** On the first frame, it creates a fixed-size NumPy array (e.g., `200x200x200`) for the TSDF values (`_tsdf`) and the weights (`_w`). The volume is centered around the first camera's position.
2.  **Voxel Projection:** For each frame, instead of iterating over pixels, it iterates over the **voxels** in the grid (`np.meshgrid`).
3.  **SDF Calculation:** Each voxel center is projected into the current camera's view. Its distance from the camera (`z`) is compared to the depth value measured in the depth map at that projected pixel (`dv`). The difference, `sdf = dv - z`, is the signed distance.
4.  **Weighted Averaging:** The new `sdf` value is averaged with the existing value in the `_tsdf` grid using a running average weighted by the values in the `_w` grid.
    ```python
    self._tsdf[gi,gj,gk] = (self._tsdf[gi,gj,gk] * ow + sdf) / nw
    self._w[gi,gj,gk] = nw
    ```
5.  **Extraction:** Finally, `mcubes.marching_cubes` is called on the `_tsdf` grid, extracting the mesh at the zero-crossing (`level=0.0`).
