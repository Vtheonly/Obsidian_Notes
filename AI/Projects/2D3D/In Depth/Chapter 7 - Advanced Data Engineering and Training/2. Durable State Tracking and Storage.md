# 2. Durable State Tracking and Storage

Deploying this architecture on Google Colab requires bypassing severe hardware constraints: a 5-hour runtime limit and a 78GB disk quota. You cannot download all 6 datasets simultaneously.

## The Durable Iteration Protocol
Your deployment utilizes an autonomous Hugging Face checkpointing system to create an infinite, state-aware training loop.

1.  **State Loading (`load_or_create_state`):** Pulls a `training_state.json` tracker from Hugging Face Hub, identifying exactly which dataset and epoch the model was processing before the last Colab timeout.
2.  **Iterative Download:** Only downloads the immediate active dataset subset.
3.  **Auto Backup:** Pushes `.pth` weights and the JSON state file to Hugging Face at the end of every epoch via `HfApi.upload_file`.
4.  **Disk Wipe (`cleanup_dataset`):** Obliterates the dataset from the Colab drive using `shutil.rmtree()` the moment the stage is complete, opening 100% of the drive space for the next dataset.

This guarantees that if the GPU drops, running the exact same script the next day resumes the mathematical optimization flawlessly from the exact epoch interrupted.
