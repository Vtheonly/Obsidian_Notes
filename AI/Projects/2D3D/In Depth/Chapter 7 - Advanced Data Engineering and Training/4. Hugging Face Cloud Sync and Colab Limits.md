# 4. Hugging Face Cloud Sync and Colab Limits

The code explicitly targets Google Colab T4 environments, which inherently carry volatile instances subject to abrupt shutdown commands.

## The state tracking logic
The `training_state.json` stores dictionary arrays:
```json
{
    "completed_stages": ["Stage_1_DTU"],
    "current_stage": "Stage_2_WHU",
    "current_epoch": 4,
    "total_epochs_for_stage": 10,
    "dataset_downloaded": true
}
```

By reading this exact configuration before any dataset download occurs, the **Master Curriculum Runner** guarantees the GPU does not waste 15 minutes unzipping `DTU` if it is already logged in `completed_stages`.

## Aggressive Storage Sweeping
Colab's physical storage limit is easily exhausted when unzipping `WHU-OMVS` directly after `BlendedMVS`.
Your pipeline implements explicit cleanup logic:
```python
if os.path.exists(folder_path):
    shutil.rmtree(folder_path)
```
This is executed sequentially in the main loop right as an epoch loop reaches its `total_epochs_for_stage` target, physically destroying the raw image files of the previous stage *before* initializing `urllib.request.urlretrieve` on the next Zip file payload.
