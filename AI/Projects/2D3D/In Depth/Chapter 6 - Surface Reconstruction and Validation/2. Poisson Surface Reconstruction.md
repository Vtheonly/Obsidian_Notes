# 2. Poisson Surface Reconstruction

If you use the Multi-Scale Point Cloud strategy, you cannot use Marching Cubes. You must reconstruct the surface using **Screened Poisson Surface Reconstruction**.

Poisson views the point cloud as an array of oriented samples from an unknown solid object.
1.  **Indicator Function:** It attempts to compute a 3D function that equals `1` inside the solid object and `0` outside in the empty air.
2.  **Vector Field:** The normal vectors (which must strictly point towards the cameras) define the gradient (the barrier between inside and outside).
3.  **Poisson Equation:** It solves a massive sparse linear system equating the Laplacian of the indicator function to the divergence of the normal vector field: $\Delta \chi = \nabla \cdot \vec{V}$.

> ** Critical Implementation Rule:**
> Poisson reconstruction will explode into bizarre, inside-out bubbles if your Point Cloud Normal Vectors are not explicitly oriented toward the camera center. Your code uses `pcd.orient_normals_towards_camera_location(camera_center)` precisely to satisfy this mathematical requirement.
