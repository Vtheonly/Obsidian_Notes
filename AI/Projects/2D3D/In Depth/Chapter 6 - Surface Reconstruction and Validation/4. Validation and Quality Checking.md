# 4. Validation and Quality Checking

Post-reconstruction meshes can contain hidden mathematical errors that crash renderers or 3D printers downstream. The pipeline uses a `check_mesh_integrity` step to prevent this.

## Degenerate Triangles
A degenerate triangle has an area of exactly `0.0`. This happens when three vertices lie on a perfectly straight line, or when multiple vertices overlap perfectly. 
To validate this, the code computes the triangle area using the cross product:
`Area = 0.5 * norm(cross(v1 - v0, v2 - v0))`
If Area < 1e-10, the triangle is flagged and stripped from the final mesh using `remove_degenerate_triangles()`.

## NaN and Infinity Verification
Occasionally, poor floating-point divisions inside the TSDF weight calculation result in `NaN` (Not a Number) or `Inf` vertices. The validation step leverages `np.isnan(verts)` to catch and log these fatal corruptions.

## Point Cloud Extent Sanity Checks
If the camera extrinsics fail entirely during SfM, all points might collapse into the $0,0,0$ origin coordinate. The `check_pointcloud_coverage` function calculates the Axis-Aligned Bounding Box (AABB) by evaluating `points.max(axis=0) - points.min(axis=0)`. If the extent is absurdly small (e.g., 0.001 units wide), the pipeline raises an immediate safety warning.
