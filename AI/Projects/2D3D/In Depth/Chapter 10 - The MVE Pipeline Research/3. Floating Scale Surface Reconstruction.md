# 3. Floating Scale Surface Reconstruction

## Evolving past the Octree (FSSR - 2014)
The PDF you provided notes that MVE eventually upgraded from the 2011 Octree method to **FSSR (Floating Scale Surface Reconstruction)**.

Why? Volumetric approaches (like TSDF) are forced to define a grid. Even with an Octree, memory consumption limits how big the scene can be. Furthermore, TSDF tends to "hallucinate" geometry in areas it hasn't seen to force the mesh to be watertight.

## How FSSR Works
FSSR abandons the rigid voxel grid. It is a completely **Point-Based** approach.

1.  **Depth to Triangles:** Every depth map is individually triangulated based on adjacent pixels. 
2.  **Scale as Edge Length:** The length of the edges of these triangles provides a perfect, natural "scale" value for each vertex.
3.  **Point Cloud Union:** All vertices from all depth maps are dumped into a giant, unstructured point cloud. Each point now has three properties:
    *   Coordinate $(X, Y, Z)$
    *   Normal Vector $(Nx, Ny, Nz)$ - indicating surface orientation.
    *   Scale $(s)$ - indicating the resolution of the sensor at that specific point.
4.  **Implicit Function:** FSSR constructs a continuous implicit function defined mathematically over the unstructured points. The scale value $s$ is used to determine how much a point should "influence" its neighbors. A point with a tiny scale (macro detail) has a very tight, sharp influence. A point with a large scale (background structure) has a broad, smooth influence.
