# 5.3 Tradeoffs Classical vs Neural

## The Core Concept
If you are designing a pipeline, which technology do you choose? Classical Photogrammetry (COLMAP/OpenMVS), Neural Geometry (Depth Anything + TSDF), or Novel View Synthesis (NeRFs / Gaussian Splats)?

The choice completely depends on the engineering requirements of the project. There is no absolute "best" geometry engine.

| Attribute | Classical Pipeline (COLMAP + MVS) | Deep Learning MVS | Adv. Synthesis (NeRF/Splats) |
| :--- | :--- | :--- | :--- |
| **Method Mechanism** | Matches Pixels via Epipolar Geometry | Predicts Geometry from deep features | Optimizes Geometry to fit pixels |
| **Direct Output Format** | 3D Mesh (Triangles + Textures) | 3D Metric Depth Maps / Point Clouds | Fuzzy Cloud / Mathematical Function |
| **Compute Execution** | Heavy CPU (Triangulation Geometry) | Moderately Heavy GPU (CNN Cost Volumes) | Very Heavy GPU (Millions of SGD descents) |
| **Visual Quality** | Highly Accurate, often missing data | Often noisy, blurry, or "mushy" | **Photorealistic**, razor sharp edges |
| **Ideal Use Case** | Precision Engineering, historical survey | Real-time depth for robotics / drones | Digital twins, Architecture, Visual Media |
| **Textureless Blank Walls** | **Catastrophic Failure (Black Holes)** | **Excellent semantic filling** | Moderate Failure (Fog/Floaters) |
| **Physics and Collision** | Yes, strict boundary | Yes, strict metric boundary | No physical boundary, cannot collide |

---

## 1. Classical Geometry (COLMAP)
*   **What Problem it Solves:** Extracts mathematically rigorous geometry from overlapping textures.
*   **When to Use:** Use this when scanning highly textured, matte physical objects outdoors (like stone statues, drone surveys of rocky terrain, or brick buildings). 
*   **Why Not AI?:** Classical geometry accurately captures sub-millimeter cracks in stone. A Neural depth estimator often "blurs" over tiny cracks because it relies on contextual semantics rather than pixel-perfect geometry, returning a structurally inaccurate model.

## 2. Neural Depth Fusion / DL-MVS
*   **What Problem it Solves:** The total failure of Epipolar geometry indoors, and the need for extremely fast inference.
*   **When to Use:** Use this for scanning pristine, modern interior real estate. An empty room with white walls, a beige rug, and zero clutter provides $0$ texture for SfM matching. Neural Depth understands the $X, Y, Z$ coordinates of the corners of the room semantically and generates a flawless structural mesh without needing texture. Deep learning MVS is also used when you need inference in *milliseconds* per view.
*   **Why Not NeRF?:** A Real Estate mesh must be converted to an `.obj` file so clients can click walls to measure them or import them into WebGL on a phone. Neural Depth generates a strict TSDF Triangle mesh natively. If you need structural accuracy (to measure the gap between two windows), MVS gives metric depth; NeRF/Splats just give pretty colored fog.

## 3. NeRFs and Gaussian Splats
*   **What Problem it Solves:** Capturing Light, View-Dependent geometry (Specular highlights), and eliminating the "Surface Noise" of standard AI.
*   **When to Use:** Use this when generating photorealistic video fly-throughs for VFX or virtual production. If you want **the absolute highest visual fidelity of a building structure**, ignore MVSNet entirely and optimize 3DGS. MVSNet makes a smart *guess* but produces mushy models. 3DGS handles sharp building edges and complex geometry beautifully.
*   **Why Not Classical?:** Classical Photogrammetry will literally project "glare" as a white blob physically painted onto the texture map of the car, which looks terrible from alternate angles. However, because Splats lack strict boundaries, you cannot import a Gaussian Splat into a physics engine, drop a ball on it, and expect it to bounce.

### Development Summary
The ultimate pipeline uses **all three**.
1. It uses Classical SfM (COLMAP) to firmly locate the cameras.
2. It uses Neural Networks to execute Depth/Segmentation or Splatting.
3. It uses Classical Math (Marching Cubes or point-clouds) to extract and clean the final render.

The lines between Classical geometry and Deep Learning in Reconstruction are rapidly dissolving into highly integrated Hybrid pipelines.
