# 6.2 Computational and Dataset Requirements

## The Core Concept
Understanding the absolute hardware and data limits of the 3D pipeline prevents developers from designing software architectures that literally cannot execute on consumer hardware.

The computational load naturally divides into **Inference/Reconstruction** (which takes minutes to hours) and **Deep Learning Training** (which takes days to weeks).

---

## 1. Classical Pipeline Requirements (Reconstruction)

If you are running a standard reconstruction (running COLMAP + taking Neural Depth Inference + Open3D TSDF Fusion), the primary constraints are **RAM** and **CPU Threads**.

### Structure from Motion (SfM)
*   **CPU:** COLMAP's Feature Matching is highly multi-threaded. An 8-core CPU will process 100 images reasonably fast. A 32-core Threadripper will process them instantly.
*   **GPU:** The Feature *Extraction* step (finding SIFT corners) heavily utilizes CUDA cores for math. A standard consumer GPU (e.g., RTX 3060) is flawless here.
*   **RAM Constraint:** Bundle Adjustment loads all matched variables into memory simultaneously. 1,000 images might require 16GB of RAM. 10,000 images will require 128GB+ of physical RAM or it will crash.

### Volumetric TSDF Fusion
*   **RAM Nightmare:** If you attempt to use dense NumPy grids ($N^3$), your RAM requirement scales cubically. A $512 \times 512 \times 512$ grid requires only $500$ MB. A $2048 \times 2048 \times 2048$ grid requires **$32$ Gigabytes** of continuous RAM just to hold empty air.
*   **The Fix:** You must use Sparse Voxel Hash Maps (like Open3D) to restrict RAM usage purely to the physical surface of the mesh.

---

## 2. Deep Learning MVS Constraints (The 14GB VRAM Wall)

Attempting to run standard R-MVSNet or Deep Learning cost-volume analysis is extremely perilous on consumer GPUs (like an Nvidia T4 with 14GB of VRAM).

*   **Resolution and Layers:** If you run a cost volume at high resolution ($1080p$), 14 GB of VRAM will immediately OOM (Out of Memory). To fit on a T4, resolution must actively be downscaled (e.g., $512 \times 640$), which causes the resulting model to look like it is built out of coarse, blurry layers of LEGO bricks.
*   **The Cascade Solution:** To build on hardware constrained to 14GB, an engineer MUST transition to a **Cascade-MVSNet** architecture (See: [[3.4 Deep Learning Multi-View Stereo (MVSNet)]]). By building a pyramid that computes high-resolution sweeps over only a tiny fraction of the depth range at a time, peak memory usage drops safely under 12 GB.
*   **Training Time Limit:** Training these networks from scratch is fast regarding *inference* (seconds per scan), but training the base architecture across 17,000 images on a single 14GB GPU can take days or weeks. 

---

## 3. Neural Representations (NeRFs and Splats)

If you are rendering implicit geometry (See: [[5.1 Neural Radiance Fields (NeRF)]]), the math moves entirely to the GPU VRAM.

*   **NeRF Training:** An original NeRF took 48 hours to optimize on an A100. Modern implementations (Instant-NGP) use hash-grids to train in 10 minutes on an RTX 3090/4090 ($24$GB VRAM). 
*   **Gaussian Splatting:** The training (optimization loop) fits easily on consumer hardware (12GB - 24GB VRAM) and finishes in $5$ to $15$ minutes. 
*   **Storage Space Warning:** A single trained classical MVS mesh is roughly $50$ MB. A single trained Gaussian Splat scene is **1.5 Gigabytes** of `.ply` point cloud data because it stores millions of explicit float variables (Harmonics, Covariance) per point.

## 4. Foundation Model Training (The Extreme Scale)

If you ignore the warnings in [[6.1 Feasibility What a Solo Developer Can Build]] and decide to train a Foundation Monocular Depth Model (like *Depth Anything*) from scratch, these are the actual academic requirements:

### The Dataset
You absolutely must acquire millions of paired **RGB-D** images. You cannot train on normal photos.
1.  **LiDAR Scanners:** A physical depth laser accurately shooting distance arrays, bolted rigidly to a stereoscopic camera rig. 
2.  **Dataset Size:** The *Depth Anything* model was trained on **$1.5$ Million** annotated labeled images, and an additional **$62$ Million** unlabeled images. Downloading the raw datasets alone requires over $10$ Terabytes of hard drive space.

### The Compute
*   **VRAM Demand:** Processing batches of high-resolution images ($1080p$ or $4K$) through a Vision Transformer (ViT) architecture causes activation memory to explode. A single consumer 24GB GPU will run out of memory trying to train on a batch size of 2.
*   **The Solution:** Academic labs use clusters of $8\times$ A100 $80$GB GPUs running 24/7 for several weeks using PyTorch Distributed Data Parallel (DDP) to physically house the massive tensors.
