# 09.2. Pairwise Dataset & Tournament Logic

To train our Neural Ranker, we cannot simply use the list of diseases. We must create a **Pairwise Dataset** that teaches the model how to make "better vs. worse" decisions.

## 1. Creating Training Pairs
For every patient note in our training set, we take the **Top-20 candidates** from Phase 1.
- We then generate **every possible pair** of these candidates: $(A, B), (A, C), (B, C) \dots$
- **Labeling Logic**:
  - **Label = 1**: If the first disease ($A$) is the correct ground truth.
  - **Label = 0**: If the first disease ($A$) is not the ground truth (even if $B$ also isn't).

## 2. Tournament Construction
By comparing every candidate against every other candidate, we "simulate" a tournament.
- **Example Note**: "Patient has White Hair and Shaking eyes."
- **Ground Truth**: Albinism Type 1 ($D_G$).
- **Pairs**: 
  - $(D_G, \text{Marfan Syndrome}) \to \text{Label 1}$
  - $(\text{Marfan Syndrome}, D_G) \to \text{Label 0}$
  - $(\text{Marfan Syndrome}, \text{Cystic Fibrosis}) \to \text{Label 0}$

## 3. Data Augmentation Effect
This logic turns a small dataset into a massive one. 
- If we have 100 patient notes, and we take the Top-20 for each, we generate **thousands of comparisons.**
- This provides the Neural Network with enough "Decision-making examples" to learn the subtle clinical differences between similar diseases.

---

## Technical Summary for the Jury
- **Pairwise Learning-to-Rank (LTR)**: This is the official scientific name for this technique. Mention it to show architectural rigor.
- **Data Balance**: We ensure that the model sees both "Positive Wins" (Label 1) and "Negative Wins" (Label 0) to avoid bias toward always picking the first candidate.
