# 3.4 Tree-Aware Module Implementation

The Tree-Aware Module (TAM) is the central innovation of TAMER. It computes the probability that a character $i$ is the child of a character $j$ by projecting decoder features into a structural semantic space.

##  Step-by-Step Mathematical Process

### Step 1. Feature Extraction via Transformer Encoder
The Transformer Decoder outputs character semantic features $X \in \mathbb{R}^{T \times d}$. TAM applies an additional Transformer Encoder to map these into a space suitable for tree structures:
$$ X' = \text{TransformerEncoder}(X) $$

### Step 2. Child and Parent Projections
A single character can act as a child in one relationship and a parent in another. The model splits the feature representation $X'$ into two distinct vectors using learnable weight matrices $W_c$ and $W_p$:
*   **Child node vector:** $X_c = X' W_c$
*   **Parent node vector:** $X_p = X' W_p$

### Step 3. Constructing the Relationship Feature Matrix ($M$)
To evaluate the link between character $i$ and character $j$, the child representation of $i$ is added to the parent representation of $j$:
$$ M_{i,j} = X^c_i + X^p_j $$
Here, $M_{i,j} \in \mathbb{R}^d$ is the relationship feature vector assuming $i$ is the child and $j$ is the parent.

### Step 4. Generating the Relationship Score Matrix ($S$)
The vector $M$ is passed through a scoring function consisting of a ReLU activation followed by a dot product with a learnable vector $v_s$:
$$ S = \max(0, M) \cdot v_s $$

##  Interpreting the Score Matrix $S$

The output $S \in \mathbb{R}^{T \times T}$ is a 2D heatmap:
*   **Y-axis:** Child Candidates.
*   **X-axis:** Parent Candidates.
*   The model looks at row $i$ and finds the highest scoring column $j$. That column is predicted to be the parent of character $i$.

> [!TIP]
> **Deep Understanding:** Think of $S$ as a rigid skeleton. While the sequence decoder generates the "words," the TAM provides the "bones" that hold those words in a mathematically valid structure.

---
> [!NOTE]
> **Performance Note:** Calculating the full matrix $S$ is computationally expensive ($O(T^2)$), which is why TAMER is slightly slower than pure sequence models like CoMER. However, the gain in structural accuracy justifies the cost.
