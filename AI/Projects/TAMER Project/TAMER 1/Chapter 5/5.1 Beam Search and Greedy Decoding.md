# 5.1 Beam Search and Greedy Decoding

At inference time (when the model is actually making predictions), we need a strategy to select the best output sequence. There are two primary methods: **Greedy Decoding** and **Beam Search**.

## 1. Greedy Decoding

At each time step $t$, the model simply picks the single token with the highest probability.
*   **Pros:** Extremely fast.
*   **Cons:** If the model makes one mistake early on, the error compounds, and the whole sequence collapses. There is no way to "go back" and fix a bad choice.

## 2. Beam Search

Beam search keeps track of the **Top-K** (Beam Width) most likely sequences simultaneously. 

### Step-by-Step Logic
1.  **Expansion:** At step $t$, the model expands all $K$ current sequences by trying every possible next token.
2.  **Scoring:** It calculates the cumulative log-probability for every new path.
3.  **Pruning:** It keeps only the Top-$K$ absolute best paths and discards the rest.
4.  **Final Selection:** Once the `<eos>` token is hit, it selects the path with the highest overall score.

### Why it's critical for Math
Mathematical symbols are highly context-dependent. A symbol that looks like a `v` might actually be a `\sqrt` once the rest of the expression is seen. Beam Search allows the model to keep both "hypotheses" alive until it has enough context to decide.

> [!TIP]
> **Performance Tip:** Increasing the Beam Width ($K$) usually improves accuracy but significantly slows down inference. In HMER, a beam size of 3 to 5 is typical.
