# 3.1 The Mechanics of Spatial Filtering

Spatial filtering is a technique where we pass a small matrix, called a **Kernel**, **Mask**, or **Window**, over every pixel of an image to perform a mathematical operation.

## 1. The Kernel (Filter Mask)
A kernel is typically a small, square matrix (e.g., $3 \times 3$, $5 \times 5$). 
*   **Origin:** Usually the center of the kernel. This is the pixel we are currently "calculating."
*   **Coefficients:** The numbers inside the kernel are weights that determine the effect of the filter (e.g., blurring, sharpening).

## 2. Correlation vs. Convolution
In image processing literature, these terms are often used interchangeably, but there is a mathematical difference that is frequently asked in exams.

### A. Correlation
The process of moving the kernel over the image and calculating the sum of products.
1.  Place the kernel center over pixel $(x,y)$.
2.  Multiply each kernel coefficient by the image pixel it overlaps.
3.  Sum all the results. This is the new value for $(x,y)$.

### B. Convolution
The same as correlation, except the **kernel is rotated by 180 degrees** (flipped horizontally and vertically) before the sum of products is calculated.
*   **Why flip?** Convolution is a fundamental operation in signal processing and satisfies the **associative property** ($f * (g * h) = (f * g) * h$). In many symmetric filters (like Gaussian), correlation and convolution yield identical results because the kernel looks the same flipped.

## 3. Boundary (Padding) Strategies
When the kernel center is at the edge of the image, part of the kernel hangs "outside" the image boundaries. We must decide what values to use for these missing pixels.

| Method | Description | Student Tip |
| :--- | :--- | :--- |
| **Zero Padding** | Assume all pixels outside are 0 (black). | Easiest to implement, but creates dark borders. |
| **Replicate (Clamp)** | Repeat the last known border pixel. | Good for preserving edge intensity. |
| **Reflect (Mirror)** | Mirror the pixels from inside the image. | Best for maintaining texture and preventing artifacts. |
| **Wrap** | Take pixels from the opposite side of the image. | Mostly used in periodic frequency domain operations. |
