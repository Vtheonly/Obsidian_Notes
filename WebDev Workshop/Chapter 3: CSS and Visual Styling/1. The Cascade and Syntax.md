**Concept Overview:**
CSS (Cascading Style Sheets) brings your HTML to life with colors, spacing, and typography. Without CSS, every website would look like a plain text document from the 1990s.

**What "Cascading" Means:**
The term "Cascading" means that CSS rules are applied from top to bottom. If two rules target the same element and change the same property, **the one that appears later wins**.

```css
p {
    color: blue;
}

p {
    color: red;   /* This wins because it comes later */
}
```

**CSS Syntax — The Basic Rule:**
```css
selector {
    property: value;
}
```

**The Three Types of Selectors:**

1.  **Element Selector:** Targets all instances of an HTML tag.
    ```css
    p { color: blue; }        /* ALL paragraphs become blue */
    h1 { font-size: 36px; }   /* ALL h1 headings become 36px */
    ```

2.  **Class Selector (`.`):** Targets any element with a specific class attribute. Classes can be reused on multiple elements.
    ```css
    .btn { background: red; padding: 10px 20px; }
    .highlight { background-color: yellow; }
    ```
    ```html
    <button class="btn">Click Me</button>
    <button class="btn">Another Button</button>  <!-- Both styled -->
    ```

3.  **ID Selector (`#`):** Targets exactly one specific element. IDs must be unique per page — no two elements should share the same ID.
    ```css
    #main-title { font-size: 48px; color: navy; }
    ```
    ```html
    <h1 id="main-title">Welcome</h1>
    ```

**The Specificity (Priority) Rule:**
When multiple selectors conflict, CSS uses **specificity** to decide which wins:
*   **ID selectors** (#) override **Class selectors** (.)
*   **Class selectors** (.) override **Element selectors**
*   **Inline styles** (written directly on the HTML element) override everything
*   `!important` overrides all — but avoid using it as it makes debugging painful.

**Three Ways to Write CSS:**
1.  **Inline:** Written inside the HTML element itself. Avoid this for maintainability.
    ```html
    <p style="color: red;">Red text</p>
    ```
2.  **Internal:** Written inside a `<style>` tag in the `<head>`. Useful for single-page projects.
3.  **External (recommended):** Written in a separate `.css` file and linked via `<link>`. This keeps your code organized and reusable.
