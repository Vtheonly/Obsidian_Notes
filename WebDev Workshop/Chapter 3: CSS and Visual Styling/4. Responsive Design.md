**Concept Breakdown:**
Responsive design ensures your website looks good on every screen size — from large desktop monitors to tablets and small mobile phones. In the modern web, **more than half of all traffic comes from mobile devices**, so responsive design is not optional — it is essential.

**The Core Tool: Media Queries**
Media queries let you apply CSS rules **only when certain conditions are met**, such as the screen width being above or below a specific value.

```css
/* Base styles — these apply to ALL screen sizes (mobile first) */
.container {
    padding: 10px;
    font-size: 16px;
}

/* Tablet: 768px and wider */
@media (min-width: 768px) {
    .container {
        padding: 20px;
        font-size: 18px;
        display: flex;
    }
}

/* Desktop: 1024px and wider */
@media (min-width: 1024px) {
    .container {
        padding: 40px;
        max-width: 1200px;
        margin: 0 auto;
    }
}
```

**Mobile-First Design Philosophy:**
Always write your base CSS for the **smallest screen** (mobile phones). Then use `@media (min-width: ...)` to progressively add complexity as the screen gets larger. This approach has two advantages:
1.  Mobile styles are simpler, so your base code is clean and minimal.
2.  Older devices that cannot process media queries will still get a usable layout.

**Common Breakpoints:**

| Device | Width |
|---|---|
| Small phones | < 576px |
| Large phones | 576px – 767px |
| Tablets | 768px – 1023px |
| Laptops | 1024px – 1199px |
| Desktops | ≥ 1200px |

**The Viewport Meta Tag (Critical):**
Without this tag in your HTML `<head>`, mobile browsers will render your website at desktop width and zoom out, making everything tiny.
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**Responsive Units:**
Instead of using fixed `px` values, use responsive units:
*   `%`: Percentage of the parent element's size.
*   `em`: Relative to the parent's font size.
*   `rem`: Relative to the root (`<html>`) font size. Preferred for most use cases.
*   `vw` / `vh`: Percentage of the **viewport** (visible browser window) width/height.

**Responsive Images:**
```css
img {
    max-width: 100%;
    height: auto;
}
```
This ensures images never overflow their container and maintain their aspect ratio on all screen sizes.
