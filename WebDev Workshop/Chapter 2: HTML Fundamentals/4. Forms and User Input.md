**Concept Overview:**
Forms are the primary way users send data back to a server. Every login page, registration form, search bar, and checkout page is built with HTML forms.

**The Core Form Elements:**

```html
<form action="/submit" method="POST">
    <label for="username">Enter your name:</label>
    <input type="text" id="username" name="username" placeholder="John Doe" required>

    <label for="email">Your Email:</label>
    <input type="email" id="email" name="email" placeholder="john@example.com" required>

    <label for="password">Password:</label>
    <input type="password" id="password" name="password" required>

    <label for="age">Your Age:</label>
    <input type="number" id="age" name="age" min="13" max="120">

    <label for="bio">Tell us about yourself:</label>
    <textarea id="bio" name="bio" rows="4" cols="50"></textarea>

    <label for="country">Select your country:</label>
    <select id="country" name="country">
        <option value="dz">Algeria</option>
        <option value="fr">France</option>
        <option value="us">United States</option>
    </select>

    <button type="submit">Register</button>
</form>
```

**Key Form Attributes:**
*   `action="/submit"`: The URL where the form data will be sent when the user clicks submit.
*   `method="POST"`: The HTTP method used. `POST` is used for sending sensitive data (like passwords). `GET` is used for non-sensitive data (like search queries) and appends data to the URL.
*   `required`: Prevents form submission if the field is empty.
*   `placeholder`: Gray hint text that disappears when the user starts typing.

**Input Types You Should Know:**

| Type | Purpose | Example |
|---|---|---|
| `text` | Standard text box | Name, city |
| `email` | Validates email format | user@example.com |
| `password` | Masks the input | ●●●●●● |
| `number` | Only accepts numbers | Age, quantity |
| `date` | Date picker | Birthday |
| `checkbox` | Yes/No toggle | "I agree to terms" |
| `radio` | Select one from a group | Gender, plan |
| `file` | File upload | Profile picture |

**Common Student Miss — The Label Tag:**
Every `<input>` should have a `<label>` tied to it via the `for` attribute matching the input's `id`. This creates two benefits:
1.  **Usability:** Clicking the label text automatically focuses the input box, making forms easier to use on mobile.
2.  **Accessibility:** Screen readers announce the label when the user focuses on the input, so they know what to type.
