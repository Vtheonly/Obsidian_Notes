**The Boilerplate:**
Every single HTML document must follow a specific structure to be read correctly by modern browsers. This template is called the **boilerplate** and you should memorize it.

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Website</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Welcome to my page</h1>
    <script src="script.js"></script>
</body>
</html>
```

**Line-by-Line Explanation:**

*   `<!DOCTYPE html>`: This declaration tells the browser to use the latest HTML5 standard. Without it, older browsers might render your page incorrectly in a legacy mode called "quirks mode."
*   `<html lang="en">`: The root element wrapping everything. The `lang="en"` attribute tells search engines and screen readers that this page is in English.
*   `<head>`: The **metadata section**. Nothing in here is visible on the web page itself. It contains instructions for the browser.
    *   `<meta charset="UTF-8">`: Sets the character encoding. UTF-8 supports virtually all written languages, including Arabic, Chinese, and emoji characters like 😀.
    *   `<meta name="viewport" ...>`: Essential for responsive design. It tells mobile browsers to set the page width to the device's screen width, preventing the page from appearing zoomed out.
    *   `<title>`: The text that appears in the browser tab. Also used by search engines as the clickable title in search results.
    *   `<link rel="stylesheet" href="style.css">`: Links an external CSS file to style the page.
*   `<body>`: The **visible canvas**. Everything the user sees and interacts with goes inside this tag.
    *   `<script src="script.js"></script>`: Links an external JavaScript file. Placed at the bottom of the body so the HTML loads before the script runs.

**The Head vs The Body — A Simple Rule:**
*   If the user should **see it**, put it in the `<body>`.
*   If the user should **not see it** (settings, metadata, links to files), put it in the `<head>`.
