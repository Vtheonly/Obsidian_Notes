**Concept Breakdown:**
In the past, developers used `<div>` (division) tags for absolutely everything. A `<div>` is a generic, meaningless container. It works, but it tells nothing about what kind of content is inside it. Modern HTML5 introduced **Semantic Tags** — tags that describe their meaning to both the browser and the developer.

**The Key Semantic Tags:**

*   `<header>`: Represents the top section of a page or a section. Typically contains the site logo, navigation bar, and site title.
*   `<nav>`: Wraps the main navigation links. Tells screen readers "this is a navigation menu."
*   `<main>`: The primary, unique content of the page. There should only be one `<main>` per page.
*   `<section>`: A thematic grouping of related content. Each section typically has its own heading.
*   `<article>`: A self-contained piece of content that could stand on its own (e.g., a blog post, a news article, a forum comment).
*   `<aside>`: Content that is tangentially related to the main content (e.g., a sidebar, a pull quote, an advertisement).
*   `<footer>`: The bottom area of a page or section. Usually contains copyright notices, contact info, and secondary links.

**Example of a Semantic Page Layout:**
```html
<body>
    <header>
        <nav>
            <a href="/">Home</a>
            <a href="/about">About</a>
            <a href="/contact">Contact</a>
        </nav>
    </header>

    <main>
        <section>
            <h2>Our Services</h2>
            <p>We offer web design and development.</p>
        </section>

        <article>
            <h2>Latest Blog Post</h2>
            <p>Today we launched our new product...</p>
        </article>
    </main>

    <aside>
        <h3>Related Links</h3>
        <p>Check out our partner sites.</p>
    </aside>

    <footer>
        <p>&copy; 2025 My Company. All rights reserved.</p>
    </footer>
</body>
```

**Why Semantic Tags Matter:**
1.  **Accessibility:** Screen readers use semantic tags to help visually impaired users navigate the page. A screen reader can announce "Navigation section" when it encounters `<nav>`, allowing the user to skip directly to the main content.
2.  **SEO:** Search engines give more weight to content inside `<main>` and `<article>` tags.
3.  **Developer Experience:** Reading code with `<header>`, `<main>`, and `<footer>` is far easier than reading nested `<div>` tags with no clear meaning.
