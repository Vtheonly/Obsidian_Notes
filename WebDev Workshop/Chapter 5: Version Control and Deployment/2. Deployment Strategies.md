**Concept Breakdown:**
Deployment is the act of taking your code from your private laptop and placing it on a **public server** so anyone in the world can access it via a URL.

For beginner frontend projects (pure HTML, CSS, and JavaScript), you do **not** need to rent a server or understand complex infrastructure. Several free platforms handle everything for you.

---

## Free Hosting Platforms for Frontend Projects

### GitHub Pages
*   **Best for:** Simple static sites hosted directly from a GitHub repository.
*   **How it works:** Push your code to a GitHub repo, enable GitHub Pages in the repository settings, and your site is live at `https://yourusername.github.io/repo-name`.
*   **Limitations:** Only serves static files (no server-side code).

### Netlify
*   **Best for:** More advanced static sites, sites using build tools, and form handling.
*   **How it works:** Connect your GitHub repository. Netlify automatically builds and deploys your site every time you push new code.
*   **Bonus features:** Custom domains, HTTPS, form handling, serverless functions.

### Vercel
*   **Best for:** Frontend frameworks like Next.js, React, Vue.
*   **How it works:** Similar to Netlify — connect your repo and it deploys automatically.

---

## Continuous Deployment (CD)

Once you link your GitHub repository to Netlify or Vercel, the deployment becomes **automatic**:

1.  You make changes to your code locally.
2.  You commit and push to GitHub: `git push origin main`.
3.  Netlify/Vercel **detects the push** automatically.
4.  It pulls the new code, builds it (if needed), and updates your live website.
5.  Your site is updated in seconds — no manual work.

This workflow is called **Continuous Deployment** and is how professional teams ship code.

## Custom Domains

By default, these platforms give you a subdomain (e.g., `yoursite.netlify.app`). To use a custom domain like `www.yoursite.com`:

1.  **Buy a domain** from a registrar like Namecheap, Google Domains, or GoDaddy.
2.  In your hosting platform (Netlify, Vercel), add the custom domain in settings.
3.  Update the **DNS records** at your domain registrar to point to the hosting platform's servers.
4.  Wait for DNS propagation (can take up to 48 hours, but usually minutes).

## Deployment Checklist Before Going Live

- [ ] All links work correctly (no broken links).
- [ ] Images load properly (correct file paths).
- [ ] The site is responsive on mobile, tablet, and desktop.
- [ ] Meta tags are set (title, description) for SEO.
- [ ] HTTPS is enabled (most platforms do this automatically).
- [ ] Test in multiple browsers (Chrome, Firefox, Safari, Edge).
- [ ] Validate your HTML at [validator.w3.org](https://validator.w3.org).
