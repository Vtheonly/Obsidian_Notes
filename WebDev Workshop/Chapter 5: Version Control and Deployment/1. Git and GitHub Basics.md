**Concept Overview:**
Imagine you are writing a book. Instead of saving files like `book_final.txt`, `book_final_v2.txt`, `book_final_real_final.txt`, you use a system that tracks exactly **what changed, when it changed, and who changed it**. This is **Version Control**.

*   **Git:** The software running locally on your computer that tracks these changes. Every "save point" is called a **commit**.
*   **GitHub:** A website (cloud platform) where you upload your Git history so it is backed up online and accessible from anywhere. It also allows you to collaborate with other developers on the same project.

---

## Setting Up Git

**Installation:**
*   **Windows:** Download from [git-scm.com](https://git-scm.com) and install.
*   **Mac:** Install via Homebrew: `brew install git`.
*   **Linux:** `sudo apt install git` (Debian/Ubuntu) or `sudo dnf install git` (Fedora).

**First-Time Configuration:**
```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

## The Core Git Workflow

### 1. Initialize a Repository
```bash
git init
```
This creates a hidden `.git` folder in your project directory. Git will now track all changes inside this folder.

### 2. Check the Status
```bash
git status
```
Shows which files have been modified, which are staged (ready to commit), and which are untracked (new files Git does not know about yet).

### 3. Stage Your Changes
```bash
git add .
```
Moves **all** modified and new files to the **staging area** — a preparation zone before committing. You can also stage specific files: `git add index.html`.

### 4. Commit (Save a Snapshot)
```bash
git commit -m "Added the navigation bar and footer"
```
Takes a permanent snapshot of everything in the staging area. The `-m` flag adds a descriptive message. **Write clear, meaningful messages** — your future self will thank you.

### 5. Push to GitHub
```bash
git push origin main
```
Uploads your commits from your local computer to your GitHub repository. `origin` is the default name for your remote repository. `main` is the default branch name.

## Branching — Working on Features Safely

Branches let you work on new features without affecting the main codebase.

```bash
git branch feature-login       # Create a new branch
git checkout feature-login     # Switch to that branch
# ... make changes, commit ...
git checkout main              # Switch back to main
git merge feature-login        # Merge the feature into main
```

**Why Branch?** If your feature breaks something, the `main` branch remains untouched. Only merge when your feature is complete and tested.

## Common Git Problems and Fixes

| Problem | Solution |
|---|---|
| Committed the wrong file | `git reset HEAD~1` (undo last commit, keep files) |
| Need to undo local changes | `git checkout -- filename` |
| Merge conflict | Open the conflicting file, manually choose the correct code, then commit |
| Forgot to pull before pushing | `git pull origin main` then resolve conflicts if any |
