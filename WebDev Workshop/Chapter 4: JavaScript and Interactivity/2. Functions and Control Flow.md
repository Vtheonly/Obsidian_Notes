## Functions — Reusable Blocks of Code

A function is a reusable block of code. Instead of writing the same logic 10 times, you write it once inside a function and **call** it whenever needed.

**Function Declaration:**
```javascript
function greetUser(name) {
    return "Welcome to the site, " + name + "!";
}

const message = greetUser("Ali");
console.log(message);  // "Welcome to the site, Ali!"
```

**Arrow Functions (Modern Syntax):**
```javascript
const greetUser = (name) => {
    return `Welcome to the site, ${name}!`;
};

// Short version for single expressions:
const double = (num) => num * 2;
console.log(double(5));  // 10
```

**Parameters vs Arguments:**
*   **Parameters** are the variable names in the function definition: `function greet(name)` → `name` is a parameter.
*   **Arguments** are the actual values passed when calling the function: `greet("Ali")` → `"Ali"` is an argument.

## Control Flow — Making Decisions

### If / Else Statements
```javascript
const userAge = 17;

if (userAge >= 18) {
    console.log("Access granted. Welcome!");
} else if (userAge >= 13) {
    console.log("Access granted with parental supervision.");
} else {
    console.log("Access denied. You must be at least 13.");
}
```

### The Strict Equality Rule
**Always use `===` (strict equality) instead of `==` (loose equality).**

```javascript
console.log(7 == "7");    // true  — loose equality ignores type
console.log(7 === "7");   // false — strict equality checks BOTH value AND type
```

Using `==` leads to subtle bugs because JavaScript silently converts types. For example, `0 == false` is `true`, and `"" == false` is also `true`. Strict equality prevents these surprises.

## Loops — Repeating Actions

### For Loop
```javascript
for (let i = 0; i < 5; i++) {
    console.log(`Iteration number: ${i}`);
}
// Outputs: 0, 1, 2, 3, 4
```

### For...of Loop (Modern, for Arrays)
```javascript
const colors = ["red", "green", "blue"];
for (const color of colors) {
    console.log(color);
}
```

### While Loop
```javascript
let count = 0;
while (count < 3) {
    console.log(`Count is: ${count}`);
    count++;
}
```

## Common Student Mistakes

1.  **Forgetting `return` in functions:** Without `return`, a function returns `undefined` even if it calculates a result.
2.  **Using `=` instead of `===`:** A single `=` is assignment, not comparison. `if (x = 5)` assigns 5 to x instead of checking if x equals 5.
3.  **Off-by-one errors in loops:** Arrays start at index 0. A 5-element array has indices 0 through 4, not 1 through 5.
