**Concept Breakdown:**
The Document Object Model (DOM) is a live, tree-structured representation of your HTML page that JavaScript can interact with. When the browser loads your HTML file, it creates the DOM — a model where every HTML element becomes a **node** (object) that JavaScript can read, modify, or delete.

Think of the DOM as a puppet version of your HTML page. JavaScript is the puppeteer — it can pull any string to change what the audience sees.

---

## Selecting Elements

Before you can change anything, you need to **select** (grab) the element.

```javascript
// Select by ID (returns a single element)
const title = document.getElementById("main-title");

// Select by CSS selector (returns the FIRST match)
const firstButton = document.querySelector(".btn");

// Select ALL matching elements (returns a NodeList)
const allButtons = document.querySelectorAll(".btn");
```

**Best Practice:** Use `document.querySelector()` and `document.querySelectorAll()` for everything. They accept any CSS selector and are the most flexible.

---

## Modifying Elements

Once you have selected an element, you can change it:

```javascript
const title = document.querySelector("#main-title");

// Change the text
title.textContent = "A Brand New Title";

// Change the HTML inside (including tags)
title.innerHTML = "A <em>Brand New</em> Title";

// Change CSS styles
title.style.color = "red";
title.style.fontSize = "48px";
title.style.backgroundColor = "#f0f0f0";

// Add or remove CSS classes
title.classList.add("highlighted");
title.classList.remove("old-class");
title.classList.toggle("dark-mode");   // Adds if missing, removes if present
```

## Creating and Removing Elements

```javascript
// Create a new element
const newParagraph = document.createElement("p");
newParagraph.textContent = "This paragraph was created by JavaScript!";
newParagraph.classList.add("dynamic-text");

// Add it to the page (append to the body)
document.body.appendChild(newParagraph);

// Remove an element
const oldElement = document.querySelector(".outdated");
oldElement.remove();
```

## Event Listeners — Reacting to User Actions

Events are things that happen on the page: clicks, key presses, mouse movements, form submissions, page loads, scrolling, and more.

To make an element respond to an event, you add an **event listener**:

```javascript
const button = document.querySelector(".btn");

button.addEventListener("click", function() {
    alert("The button was clicked!");
});
```

**Common Events:**

| Event | Trigger |
|---|---|
| `click` | Element is clicked |
| `dblclick` | Element is double-clicked |
| `mouseover` | Mouse enters the element |
| `mouseout` | Mouse leaves the element |
| `keydown` | A key is pressed |
| `keyup` | A key is released |
| `submit` | A form is submitted |
| `input` | An input field value changes |
| `scroll` | The page is scrolled |
| `load` | The page has fully loaded |

**The Event Object:**
Every event listener receives an **event object** containing information about what happened:

```javascript
document.addEventListener("keydown", function(event) {
    console.log(`Key pressed: ${event.key}`);
    
    if (event.key === "Escape") {
        // Close a modal, for example
        console.log("Escape was pressed!");
    }
});
```

**Preventing Default Behavior:**
Some elements have built-in behaviors (e.g., forms reload the page on submit, links navigate away). You can prevent this:

```javascript
const form = document.querySelector("form");
form.addEventListener("submit", function(event) {
    event.preventDefault();  // Stops the page from reloading
    console.log("Form submitted without page reload!");
});
```
